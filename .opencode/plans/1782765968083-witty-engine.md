# Plan: Count-down Timer for Mouse-Leave During Tile Snap

## Problem
When dragging a window header (tile snap in progress) and releasing the mouse button outside the terminal viewport, no `MouseEventKind::Up` event arrives. The `drag_header` and `drag_snap` fields stay set, leaving the snap preview (accent-colored overlay) visible permanently — the UI appears "stuck".

## Solution
Add an idle timeout while snapping: if no mouse events arrive for 2 seconds during a header drag, auto-cancel the snap. Show a countdown timer in the center of the snap preview rectangle.

---

## Files to Create

### 1. `crates/term-wm-core/src/components/center.rs` — CenterComponent
A generic layout wrapper that centers its child component within the given area:

```rust
pub struct CenterComponent<C> {
    content: C,
    content_size: (u16, u16),  // preferred width/height of child
}
impl<C: Component> CenterComponent<C> {
    pub fn new(content: C, width: u16, height: u16) -> Self;
    /// Compute the centered rect for the current frame area
    fn inner_rect(&self, area: Rect) -> Rect;
}
impl<C: Component> Component for CenterComponent<C> {
    fn render(&mut self, frame: &mut UiFrame<'_>, area: Rect, ctx: &ComponentContext) {
        let inner = self.inner_rect(area);
        self.content.render(frame, inner, ctx);
    }
}
```

**Key design decisions:**
- `content_size` is the preferred size of the child (width/height of the countdown text). The center component will create a rect of this size, centered in the available area, clamped to fit.
- `CenterComponent` follows the same pattern as `ScrollViewComponent`: wraps a child, delegates render to it at a computed offset.
- Lives in `term-wm-core` (not `term-wm-ui-components`) because `overlays.rs` in core needs it and cannot depend on the UI components crate.

### 2. `crates/term-wm-core/src/components/countdown_label.rs` — CountdownLabel
A minimal component that renders countdown text:

```rust
pub struct CountdownLabel {
    remaining: Duration,
}
impl Component for CountdownLabel {
    fn render(&mut self, frame: &mut UiFrame<'_>, area: Rect, _ctx: &ComponentContext) {
        let text = self.format_remaining();
        let p = Paragraph::new(text)
            .style(Style::default().fg(crate::theme::accent_alt()))
            .alignment(Alignment::Center);
        frame.render_widget(p, area);
    }
}
impl CountdownLabel {
    pub fn new(remaining: Duration) -> Self;
    fn format_remaining(&self) -> String {
        let secs = self.remaining.as_secs();
        if secs > 0 { format!("Cancelling in {}s", secs) }
        else { "Cancelling...".to_string() }
    }
}
```

### 3. `crates/term-wm-core/src/wm_config.rs`
Add `drag_snap_timeout: Duration` field (default 2000ms) to `WmConfig`, initialized in both `standalone()` and `embedded()`.

### 4. `crates/term-wm-core/src/window/window_manager/mod.rs`
- Add field `drag_last_event: Option<Instant>` to `WindowManager` (default `None`)
- Add method `take_expired_drag_snap() -> bool` — checks if `drag_header.is_some()` and `drag_last_event` is past `config.drag_snap_timeout`. If expired, clears `drag_header` and `drag_snap`, returns `true`.

### 5. `crates/term-wm-core/src/window/window_manager/drag.rs`
- In `handle_header_drag_event()`, update `drag_last_event = Some(Instant::now())` on every `Drag` event (line ~102) and `Down` event (line ~17)

### 6. `crates/term-wm-core/src/window/window_manager/overlays.rs`
- After the existing snap preview render (lines 175-190), if `drag_snap` is set AND `drag_last_event` is set:
  - Calculate `remaining = config.drag_snap_timeout - drag_last_event.elapsed()` (clamped to 0)
  - Construct `CountdownLabel::new(remaining)` — wraps in `CenterComponent::new(.., text_width, 1)`
  - Build a `ComponentContext` via `self.component_context(false)`
  - Call `center.render(frame, snap_preview_rect, &ctx)`
  - The countdown text uses `accent_alt` (amber) fg on the existing accent-colored background

### 7. `crates/term-wm-core/src/runner.rs`
- In the idle path (line ~494, after `take_expired_super_event`), call `app.windows().take_expired_drag_snap()`. If it returns true, the next frame will no longer render the snap preview.

## Rendering Flow

```
render_overlays()
  ├── render snap preview (accent fill) — existing
  ├── if drag_snap.is_some() && drag_last_event.is_some():
  │     remaining = timeout - elapsed
  │     countdown = CountdownLabel::new(remaining)
  │     center = CenterComponent::new(countdown, text_width, 1)
  │     center.render(frame, snap_preview_rect, &ctx)
  │     → renders "Cancelling in 2s" centered in preview rect
  ├── render menu overlay (existing)
  └── render other overlays (existing)
```

The `CountdownLabel` renders a `Paragraph` with `accent_alt` (amber) fg. `CenterComponent` computes the centered inner rect and delegates. Both are created ephemerally each frame — no stored state needed beyond `drag_last_event`.

## Edge Cases
- **Countdown reaches 0**: In the next idle frame, `take_expired_drag_snap()` returns true, clearing drag state → no more preview or countdown
- **Mouse re-enters**: On `Drag` event, `drag_last_event` is updated → countdown resets
- **Mouse button released inside terminal**: Normal `MouseEventKind::Up` → `apply_snap()` is called and state is cleared normally (no change)
- **Very short timeout**: If `remaining` is negative, render "Cancelling..." instead of a number
- **Window closed during drag**: `drag_header` is checked for validity; if the dragged window no longer exists, the snap is simply cancelled
