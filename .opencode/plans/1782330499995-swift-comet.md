# Plan: Reduce Idle CPU via InputDriver-Owned Adaptive Polling

## Root Cause
The event loop at `src/event_loop.rs:63-84` polls crossterm at a fixed 16ms interval and renders every cycle, even idle. This keeps CPU at ~2% on macOS.

## Approach (modular)
Add `poll_interval()` to the `InputDriver` trait. The `ConsoleInputDriver` tracks time since last event and returns 16ms when active, 200ms when idle. `EventLoop::run()` calls `self.driver.poll(self.driver.poll_interval())` instead of a hardcoded field. The timing policy lives entirely in the driver layer.

## Files to modify

### 1. `src/drivers/input_driver.rs`
- Add method to the trait:
  ```rust
  fn poll_interval(&self) -> Duration {
      Duration::from_millis(16)  // default: 60fps
  }
  ```
- Add delegation to the blanket `impl<T: InputDriver + ?Sized> InputDriver for &mut T`:
  ```rust
  fn poll_interval(&self) -> Duration {
      (**self).poll_interval()
  }
  ```
  This is critical — without it, any code using `&mut ConsoleInputDriver` would get the trait default (16ms) instead of the driver's adaptive override.

### 2. `src/drivers/console.rs`
- Add `use std::time::Instant;`
- Add `last_event_at: Option<Instant>` field to `ConsoleInputDriver`
- Initialize to `None` in `new()`
- Update `poll()`: when it detects events (queue non-empty OR crossterm poll returns true), set `self.last_event_at = Some(Instant::now())`
- Update `read()`: when returning an event (from queue or internal read), set `self.last_event_at = Some(Instant::now())`
- Override `poll_interval()`:
  ```rust
  fn poll_interval(&self) -> Duration {
      const ACTIVE: Duration = Duration::from_millis(16);   // 60fps
      const IDLE: Duration = Duration::from_millis(200);    // 5fps
      const IDLE_THRESHOLD: Duration = Duration::from_millis(500);
      match self.last_event_at {
          Some(t) if t.elapsed() < IDLE_THRESHOLD => ACTIVE,
          _ => IDLE,
      }
  }
  ```

### 3. `src/event_loop.rs`
- Remove `poll_interval: Duration` field from `EventLoop<D>` — no longer needed
- Update `EventLoop::new()` to only take `driver` (remove `poll_interval` parameter)
- Update `run()`: use `self.driver.poll(self.driver.poll_interval())` on line 68
- Update `poll()` method (singular, test helper): use `self.driver.poll(self.driver.poll_interval())` on line 37
- Update tests: `EventLoop::new(d)` without the duration parameter

### 4. `src/runner.rs`
- Remove `let poll_interval = Duration::from_millis(16)` on line 328
- Update `run_app()` signature: remove `poll_interval: Duration` parameter
- Update `run_app()` body: `EventLoop::new(driver)` instead of `EventLoop::new(driver, poll_interval)`
- Update `run_window_app()`: remove poll_interval creation and the passing to `run_app()`

## Edge cases
- **Selection auto-scroll**: generates real `Event::Mouse` events → keeps active interval naturally
- **PTY output**: goes through separate reader thread, not the event loop → won't keep active interval (correct)
- **Window resize**: comes through crossterm event stream → keeps active interval during resizes
