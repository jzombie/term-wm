# Plan: Workspace Split into Three Crates

## Goal
Split the monolith into three crates with one-way dependency chain to enforce compile-time separation of concerns.

```
term-wm-core          (io + runner + window + layout + Component trait + pty)
     ↑
term-wm-ui-components (concrete widgets, depends on core for Component trait)
     ↑
term-wm               (this repo, binary: main.rs + tracing_sub.rs)
```

No crate can access anything from a crate "above" it. No circular deps.

---

## Crate 1: `term-wm-core` (library)

**Directory**: `crates/term-wm-core/`

**Dependencies** (`Cargo.toml`):
- `crossterm`, `ratatui`, `arboard` — I/O and rendering backends
- `portable-pty`, `vt100` — PTY abstraction
- `thiserror` — ClipboardError
- `tracing` — driver debug logging

**Files** (everything except concrete components and the binary):
```
src/io/                 # EventSource, RenderTarget, console, clipboard, utils/
src/event_loop.rs       # EventLoop, ControlFlow
src/runner.rs           # WindowManagerHost, WindowProvider, run_app, run_window_app
src/component_context.rs # ComponentContext, ViewportContext, ViewportHandle
src/components/mod.rs   # Component trait only (no concrete impls)
src/window/             # WindowManager, decorators, all sub-modules
src/layout/             # tiling, floating, layout node
src/ui/                 # UiFrame
src/panel/              # Panel
src/pty.rs              # Pty (subprocess + VT parser)
src/wm_config.rs        # WmConfig
src/keybindings.rs      # KeyBindings
src/actions.rs          # Action enum
src/theme.rs            # Theme
src/constants.rs        # Shared constants
src/utils/              # selectable_text, etc.
```

**Changes needed**:
- `ConfirmAction` enum moves from `components/confirm_overlay.rs` into core (e.g., `component_context.rs` or a new small type module). Runner imports it from core.
- Re-export crossterm event types from `io/mod.rs` so consumers don't need crossterm:
  ```rust
  pub use crossterm::event::{Event, KeyCode, KeyEvent, KeyEventKind, KeyModifiers, MouseEvent, MouseEventKind, MouseButton};
  ```

## Crate 2: `term-wm-ui-components` (library)

**Directory**: `crates/term-wm-ui-components/`

**Dependencies** (`Cargo.toml`):
- `term-wm-core` — Component trait, Pty, UiFrame, window types
- `ratatui` — Rect, Style, Buffer, Color (rendering data types)
- Additional: `image`, `resvg`, `usvg`, `tiny-skia`, `base64`, `pulldown-cmark`, `syntect`, `linkify`, `webbrowser`, `unicode-width`

**Files** (all concrete component implementations):
```
src/lib.rs              # module declarations
src/scroll_view.rs
src/terminal.rs         # TermmalComponent (uses Pty from core)
src/list.rs
src/toggle_list.rs
src/markdown_viewer.rs
src/text_renderer.rs
src/dialog_overlay.rs
src/confirm_overlay.rs  # minus ConfirmAction (now in core)
src/ascii_image.rs
src/svg_image.rs
src/sys/debug_log.rs    # DebugLogComponent (note: uses tracing, need to handle global handle)
src/sys/help_overlay.rs
src/sys/selection_preview_overlay.rs
src/sys/mod.rs
```

**Changes needed**:
- All `crate::` imports become `term_wm_core::` imports
- `ConfirmAction` now imported from core instead of local
- `crossterm::event::*` imports become `term_wm_core::io::*` (event types)

---

## Crate 3: `term-wm` (binary, this repo)

**Dependencies** (`Cargo.toml`):
- `term-wm-core`, `term-wm-ui-components` — the framework
- `clap`, `line-ending` — CLI
- `tracing-subscriber` — tracing setup
- `indoc` — build dependency

**Files**: `src/main.rs` + `src/tracing_sub.rs`

**Changes needed**:
- All `term_wm::*` imports become `term_wm_core::*` or `term_wm_ui_components::*`
- `crossterm` removed from deps (event types come through core)
- `ratatui` removed from deps (not needed in main.rs)

## Migration Process

### Phase 1: Move files only (no compilation)

1. Create `crates/term-wm-core/` directory with its `Cargo.toml` and `src/`
2. **`git mv src/io/ crates/term-wm-core/src/io/`**
3. **`git mv src/event_loop.rs crates/term-wm-core/src/`**
4. **`git mv src/runner.rs crates/term-wm-core/src/`**
5. **`git mv src/component_context.rs crates/term-wm-core/src/`**
6. **`git mv src/window/ crates/term-wm-core/src/window/`**
7. **`git mv src/layout/ crates/term-wm-core/src/layout/`**
8. **`git mv src/ui/ crates/term-wm-core/src/ui/`**
9. **`git mv src/panel/ crates/term-wm-core/src/panel/`**
10. **`git mv src/pty.rs crates/term-wm-core/src/`**
11. **`git mv src/wm_config.rs crates/term-wm-core/src/`**
12. **`git mv src/keybindings.rs crates/term-wm-core/src/`**
13. **`git mv src/actions.rs crates/term-wm-core/src/`**
14. **`git mv src/theme.rs crates/term-wm-core/src/`**
15. **`git mv src/constants.rs crates/term-wm-core/src/`**
16. **`git mv src/utils/ crates/term-wm-core/src/utils/`**
17. **`git mv src/components/ crates/term-wm-core/src/components/`** (Component trait only for now — will split concrete components later)
18. Create `crates/term-wm-core/src/lib.rs` with module declarations
19. → STOP and confirm with user. They will commit.

### Phase 2: Split components into separate crate

20. Create `crates/term-wm-ui-components/` directory with `Cargo.toml` and `src/`
21. Move concrete component files from `term-wm-core/src/components/` to `term-wm-ui-components/src/`
22. Keep only `Component` trait in `term-wm-core/src/components/mod.rs`
23. Move `ConfirmAction` into core (a small type module)
24. Re-export crossterm event types from `term-wm-core::io`
25. → STOP and confirm with user. They will commit.

### Phase 3: Fix compilation

26. Update root `Cargo.toml` to workspace with 3 members
27. Update all import paths
28. Fix visibility issues
29. Update `examples/dual_image.rs`
30. `cargo clippy --workspace --all-targets --all-features -- -D warnings`
31. `cargo test --workspace`
