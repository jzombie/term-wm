# Fix: Four remaining issues after MenuComponent integration

## Fix 1: Keyboard navigation not reflected visually
MenuComponent has its own `selected` field. After `set_items`, it's 0.
Sync from palette's selected after setting items.

**File:** `command_palette.rs`, in `render()`:
```rust
self.list_scroll.content.borrow_mut().set_items(menu_items);
self.list_scroll.content.borrow_mut().set_selected(self.selected);
```

## Fix 2: Remove j/k from global keybindings
Delete `MenuNext` (j) and `MenuPrev` (k) lines from default keybindings.

**File:** `crates/term-wm-core/src/keybindings.rs`

## Fix 3: Cannot type in search bar
Char events append to query and return Consumed. Need to verify the runner
doesn't intercept. Add a debug check in the runner's event routing.

## Fix 4: Vertical gap between search bar and top item
MenuComponent's `render_items` reserves row 0 as a header and starts items
at row 1. List area already starts at `bounds.y + 1`, so items render at
`bounds.y + 2` — one row gap. Fix: pass adjusted area with header accounted for.

**File:** `command_palette.rs`, in `render()`:
```rust
let list_area = LayoutRect {
    y: (bounds.y + 1) as i32,
    height: bounds.height.saturating_sub(2),  // was sub(1)
    ...
};
```
