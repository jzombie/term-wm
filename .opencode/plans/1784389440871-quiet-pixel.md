## Issue 2: Scrollbar when items overflow
**File:** `crates/term-wm-ui-components/src/command_palette.rs`

### 2a. Add `scroll_offset: usize` field
### 2b. In `render_content`, after rendering items, call `render_scrollbar` if items
    exceed visible height.
### 2c. Cache visible_height & use it for scroll clamping in update
Add `pub visible_height: usize` field to `CommandPaletteComponent`.
Set it in `render_content` to the exact number of visible list rows for that frame.
In `update`, use `self.visible_height` when computing scroll offset shifts:

```rust
fn update(&mut self, action: TermWmAction, ...) {
    match action {
        TermWmAction::MenuUp => {
            let total = self.filtered_items.len();
            if total > 0 {
                self.selected = self.selected.wrapping_sub(1) % total;
            }
            // Clamp scroll when selection goes above viewport
            if self.selected < self.scroll_offset {
                self.scroll_offset = self.selected;
            }
        }
        TermWmAction::MenuDown => {
            let total = self.filtered_items.len();
            if total > 0 {
                self.selected = (self.selected + 1) % total;
            }
            // Push scroll when selection exits bottom of viewport
            let bottom = self.scroll_offset.saturating_add(self.visible_height);
            if self.selected >= bottom && self.visible_height > 0 {
                self.scroll_offset = self.selected.saturating_sub(self.visible_height).saturating_add(1);
            }
        }
        _ => {}
    }
}
```

### 2e. Enforce iterator slicing in render_content
The item rendering loop must use `.iter().skip(self.scroll_offset).take(visible_count)`
to prevent buffer overflow from rendering items outside the viewport:

```rust

self.visible_height = visible_height;
let visible_count = visible_height.min(self.filtered_items.len().saturating_sub(self.scroll_offset));

for rel_idx in 0..visible_count {
    let abs_idx = rel_idx + self.scroll_offset;
    let y = bounds.y + 1 + rel_idx as u16;
    let item = &self.filtered_items[abs_idx];
    // ... render item ...
}
```
### 2d. **CRITICAL — Fix pointer geometry in mouse handler:**
    The `handle_events` mouse click branch must apply `scroll_offset`:
```rust
let visual_idx = (my.saturating_sub(area.y.max(0) as u16).saturating_sub(1)) as usize;
let actual_idx = visual_idx.saturating_add(self.scroll_offset);
if actual_idx < self.filtered_items.len() {

```

TermWmAction::MenuSelect => {
