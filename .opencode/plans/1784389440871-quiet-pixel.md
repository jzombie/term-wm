**File:** `crates/term-wm-ui-components/src/command_palette.rs`

In `render_content`, when `self.query.is_empty()`, show `"[type to search]"` in the
search bar using a dimmed style. Use `theme.panel_inactive_fg` (no hardcoded colors):

```rust
// In render_content, after the search bar clearing:
if self.query.is_empty() {
    let placeholder = "[type to search]";
    let placeholder_style = Style::default()
        .bg(color_to_ratatui(theme.panel_active_bg))
        .fg(color_to_ratatui(theme.panel_inactive_fg));
    let inner_width = bounds.width.saturating_sub(4) as usize;
    let text: String = placeholder.chars().take(inner_width).collect();

}
```

## Issue 2: Scrollbar when items overflow
**File:** `crates/term-wm-ui-components/src/command_palette.rs`

Add `scroll_offset: usize` field to `CommandPaletteComponent`.

In `render_content`, after rendering items, calculate overflow and call `render_scrollbar`:

```rust
// After item rendering loop:
if visible_count < self.filtered_items.len() {
    let sb_area = Rect::new(
        bounds.x.saturating_add(bounds.width).saturating_sub(1),
        bounds.y + 1,
        1,
        bounds.height.saturating_sub(1),
    );
    render_scrollbar(
        buffer,
        sb_area,
        self.filtered_items.len(),
        visible_count,
        self.scroll_offset,
    );
}
```

In `handle_events`, handle MouseEventKind::ScrollUp/ScrollDown to adjust
`scroll_offset` (clamped to 0..filtered_items.len().saturating_sub(visible_count)).

In `update` for MenuUp/MenuDown, adjust `scroll_offset` when selection goes
outside the visible window.

Import: `use crate::scroll_view::render_scrollbar;`

## Issue 3: Mouse click on item doesn't trigger action
**File:** `crates/term-wm-sys-ui-components/src/wm_command_palette.rs`

In `handle_events`, the `Consumed` branch needs to capture `last_action`:

```rust
EventResult::Consumed => {
    if let Some(action) = self.palette.inner().selected_action().cloned() {
        self.last_action = Some(action);
    }
    EventResult::Consumed
}
```

When the user clicks an item, the inner palette sets `selected` and returns
`Consumed`. This branch now captures the selected item's action into
`last_action`, so `query(SelectedAction)` returns it to the runner.
