# Universal Command Palette — Full Implementation Plan

## Goal
Replace the current simple menu (9 static items, basic up/down nav) with a universal, fuzzy-searchable Command Palette modal overlay — the single entry point for all application actions. Rendered as a centered modal with search bar + filtered list, using `nucleo-matcher` for fuzzy matching.

## New Dependencies (term-wm-core, term-wm-ui-components)
- `nucleo-matcher` — for fuzzy search (Smith-Waterman with affine gaps).
- `bitflags` — for ContextMask bitflags.
- No `typetag`, `erased_serde`, `serde`, `serde_json`, `rusqlite`, or `generational-arena`.

## Architecture (6 Pillars)

### 1. Generational Arena (slotmap, already in tree)
Store menu nodes in `slotmap::SlotMap` in term-wm-core. Each node:
```rust
use slotmap::new_key_type;
new_key_type! { pub struct CommandNodeId; }

/// Declarative command name: supports passive state binding without mutation.
/// During the data_dirty cache rebuild, `Toggle` variants are evaluated against
/// the current ContextMask and formatted into the display name (e.g. "ON"/"OFF").
/// The arena payload itself is never mutated.
pub enum CommandName {
    Static(String),
    Toggle {
        base: String,
        active_flag: ContextMask,
        on_suffix: &'static str,
        off_suffix: &'static str,
    },
}

pub struct CommandNode {
    pub stable_id: String,           // Permanent semantic ID, e.g. "core:split_pane"
                                     // Independent of allocation lifecycle. MRU keys on this.
    pub name: CommandName,           // Declarative name (static or toggle)
    pub description: Option<String>, // Subtitle in palette
    pub action: CommandAction,       // What to execute
    pub icon: Option<&'static str>,  // Optional icon
    pub required_context: ContextMask,
    pub owner_id: Option<ComponentId>,  // For lifecycle cleanup (see pillar 3)
    // NOTE: No usage_count or last_used — those belong exclusively to MruRanker.
    // The node is an immutable payload definition.
}
```

`CommandAction` enum (flat — no submenus, no closures; pure data only):
```rust
pub enum CommandAction {
    AppAction(TermWmAction),
    // NOTE: No Custom(Box<dyn Fn ...>) variant — that would create a borrow-checker
    // deadlock (the registry is owned by AppContext but the closure captures it).
    // Dynamic panes should use TermWmAction::ComponentDispatch{target, payload}
    // and route through the central event loop instead.
}
```

**File:** `crates/term-wm-core/src/command_menu/arena.rs`

### 2. Command Registry
A flat registry mapping string IDs → commands. Also holds the arena.
```rust
pub struct CommandRegistry {
    arena: SlotMap<CommandNodeId, CommandNode>,
    // NOTE: No root_nodes — this is a flat palette, not a hierarchical menu.
    // build_item_list() iterates arena.keys() and filters by context_mask.
}
```
Methods: `register()`, `unregister()`, `get()`, `build_item_list()` (flattens valid nodes into a display vec).

Commands are categorized (Window, System, Session, View) via a `Category` field on the node.

**File:** `crates/term-wm-core/src/command_menu/registry.rs`

### 3. Event Bus (crossbeam-channel, already in tree)
Typed channel bus for dynamic command registration with lifecycle ownership:
```rust
pub type ComponentId = u64;  // Simple incrementing counter, or reuse WindowKey

pub enum CommandMenuEvent {
    RegisterCommands {
        owner: ComponentId,
        nodes: Vec<CommandNode>,
    },
    /// Drop all nodes owned by a terminated component (by ComponentId, not individual NodeIds).
    UnregisterOwner(ComponentId),
    /// Signal that the filtered/scored list is stale and must be rebuilt.
    InvalidateCache,
}
```
- `CommandMenuEventBus` wraps `crossbeam_channel::Sender<CommandMenuEvent>` / `Receiver`.
- Embedded panes send `RegisterCommands` on init with their `ComponentId`. Each node stores `owner_id: Some(ComponentId)`.
- When a pane terminates, send `UnregisterOwner(id)`. The registry drops all nodes where `owner_id == id`. The MruRanker is intentionally NOT pruned — its `HashMap<String, MruEntry>` keys on `stable_id`, which survives node allocation cycles. When the pane reopens, its commands regain their historical ranking immediately.
- `InvalidateCache` signals the palette to re-filter + re-score on the next event poll (not on every render).

**File:** `crates/term-wm-core/src/command_menu/event_bus.rs`

### 4. Context Engine & Bitmask
```rust
bitflags::bitflags! {
    pub struct ContextMask: u64 {
        const NONE          = 0;
        const HAS_FOCUS     = 1 << 0;
        const IS_MOBILE     = 1 << 1;
        const IS_FLOATING   = 1 << 2;
        const CAN_SPLIT     = 1 << 3;
        const HAS_PANES     = 1 << 4;
        const MOUSE_CAPTURE = 1 << 5;
        const CLIPBOARD_ENABLED = 1 << 6;
    }
}
```
- Before each frame, `WindowManager` computes the current `ContextMask` from app state.
- `build_item_list()` filters the arena by `(app_mask & node.required_context) == node.required_context`.
- Passive state binding: toggle items (Mouse Capture, Clipboard) read state lazily from `WindowManager` during `build_item_list()`, not via push.

**File:** `crates/term-wm-core/src/command_menu/context.rs`
**Dep:** Add `bitflags` to workspace and term-wm-core's Cargo.toml.

### 5. Fuzzy Search (nucleo-matcher)
```rust
use nucleo_matcher::{Matcher, Utf32String, Config as NucleoConfig, pattern::Pattern};
```
- Pre-segment command names into `Utf32String` on arena mutation.
- On each keystroke: `Pattern::new(query, ...).score(name_utf32, &mut matcher)` for each visible item.
- Sort results by (nucleo_score * 1000 + mru_score).
- Config: `NucleoConfig::DEFAULT` with case-insensitive matching.

The MRU ranker is integrated here (ephemeral only — Phase 1):
```rust
pub struct MruRanker {
    // Keyed by CommandNode::stable_id (e.g. "core:split_pane", "git_pane:commit").
    // This is a semantic identity that survives node allocation cycles:
    // when a dynamic pane closes and re-opens, its commands get new CommandNodeIds
    // but the same stable_id, so their historical weight is preserved.
    entries: HashMap<String, MruEntry>,
    decay_per_use: f64,            // default 0.95
}

struct MruEntry {
    weight: f64,
    last_used: Instant,
}
```
- On each command execution: `MruRanker::record(&node.stable_id)` — resets weight, applies decay to all others.
- Weight: `w = 1.0;` others `*= decay_per_use` (exponential decay).
- **No prune step:** MRU intentionally outlives node lifetimes. When a dynamic pane closes and reopens, its commands regain their historical ranking immediately. Unbounded growth is prevented by the fixed number of distinct stable_ids in the system.
- **No disk I/O in Phase 1.** MRU is purely in-memory. Zero blocking, zero serialization.

**Files:**
- `crates/term-wm-core/src/command_menu/matcher.rs` — FuzzyMatch + MruRanker (in-memory only)

### 6. Rendering (CommandPaletteComponent)
New component in **term-wm-ui-components**:
```rust
pub struct CommandPaletteComponent {
    query: String,
    cursor: usize,
    filtered_items: Vec<PaletteItem>,  // Pre-filtered & scored cache
    selected: usize,

    // Granular dirty flags — see Event Flow for lifecycle.
    data_dirty: bool,   // registry mutation or context shift → full rebuild
    query_dirty: bool,  // keystroke → re-rank only, zero allocation
    context_mask: ContextMask,

    // Cached state (rebuilt on data_dirty, consumed by query_dirty).
    // Matcher and MruRanker are NOT here — they live in WindowManager and are
    // passed as &mut refs during handle_events (see pillar 5/Integration).
    utf32_items: Vec<(Utf32String, Utf32String)>,  // (name, description), pre-segmented
    active_ids: Vec<CommandNodeId>,                 // 1:1 with utf32_items
}
```
- **Search bar**: top row, pre-filled with blinking cursor. Captures all key events as text input.
- **Filtered list**: rows below, showing icon + name + description.
- **Highlighting**: Use `nucleo` match indices + `Cow::Borrowed` for zero-alloc highlighting.
- **Keybindings**: `/` to focus search, `Esc` to close, `Up`/`Down`/`Ctrl+N`/`Ctrl+P` to navigate, `Enter` to select.
- **Mouse**: Click on item selects it.

**Modal wrapping**: Reuse `PlacementContainerComponent` with `Placement::Centered` for the modal position + `DialogOverlayComponent` for dim backdrop.

**Integration**: Modify `WmCommandPaletteOverlay` (or create new `WmCommandPaletteComponent`) to:
- Hold `PlacementContainerComponent<CommandPaletteComponent>` instead of `PlacementContainerComponent<MenuComponent>`
- Implement `WmComponent` / `Overlay` trait
- Connect to `CommandRegistry` for item list building
- Wire `CommandMenuEventBus` receiver for dynamic commands

## Event Flow (Revised — Event-Driven Scoring, Not Per-Frame)

```text
── Event Handling (not render) ──────────────────────────────────────────
User types text
  → runner.rs routes Event to WmCommandPaletteComponent::handle_events()
    → CommandPaletteComponent captures key as query text
    → Sets query_dirty = true   // re-rank only, no allocation

Pane sends RegisterCommands{owner, nodes} via EventBus
  → receiver drained in handle_events()
  → CommandRegistry::register_nodes(nodes, owner)
  → Sets data_dirty = true     // full rebuild needed

Pane terminates → UnregisterOwner(id)
  → CommandRegistry::drop_owner(id)
  // NOTE: MruRanker is NOT pruned here — entries keyed on stable_id survive
  // node allocation cycles. See pillar 5 for rationale.
  → Sets data_dirty = true     // full rebuild needed

ContextMask changes (focus switch, mouse capture toggle, etc.)
  → Sets data_dirty = true     // active node set changed

── Before Render (granular invalidation) ──────────────────────────────
// Matcher and MruRanker are owned by WindowManager, passed as &mut refs.
// They survive palette open/close cycles — no data loss, no reallocation.

if data_dirty {
    // Full rebuild: re-fetch active nodes, re-allocate utf32 cache.
    // Triggered by registry mutations or context changes — NOT by keystrokes.
    active_ids: Vec<CommandNodeId> = registry.build_item_list(context_mask);
    utf32_items: Vec<(Utf32String, Utf32String)> = active_ids
        .iter()
        .map(|id| {
            let node = &registry.arena()[id];
            // Evaluate Toggle names: check active_flag against context_mask
            let display = node.name.format(context_mask);
            (Utf32String::from(display),
             Utf32String::from(node.description.as_deref().unwrap_or("")))
        })
        .collect();
    query_dirty = true;  // ensure re-rank with new cache
    data_dirty = false;
}

if query_dirty {
    // Re-rank only: reuse pre-existing utf32_items, no allocation.
    // Triggered by keystrokes — O(N) compute but zero heap.
    fuzzy_results = FuzzyMatch::score(query, &utf32_items, matcher);
    filtered_items = reorder(&active_ids, fuzzy_results, mru_ranker);
    query_dirty = false;
}

── Render (reads cache, zero computation) ──────────────────────────────
Render search bar + filtered_items with highlights
  → No nucleo calls, no arena traversal, no allocation
  → Pure buffer writes within 16.6ms

── On Select ───────────────────────────────────────────────────────────
Enter/Mouse click on item:
  → MruRanker::record(selected_id)          // O(n) decay, no I/O
  → Return TermWmAction from selected item
  → runner.rs dispatches action
```
