# Plan: Component-Owned Hitboxes — Eliminate Hitbox/Component Decoupling

## Problem

`dispatch_mouse` (mod.rs:1054-1751) is a ~700-line procedural guessing game because:
1. `HitboxRegistry` stores bare `HitboxId`s — no record of which component registered them.
2. Window content is identified via `w.content_hitbox_id == hitbox_id` strict equality check — **fails for child components** that register their own `HitboxId`.
3. Chrome (resize handles, drag handles, split handles) requires iterating through three separate lists (`resize_handles`, `floating_headers`, `handles`) searching for a matching `hitbox_id`.
4. Scroll/Moved/Press phases each repeat the same window iteration + equality check pattern.

## Solution

Tag every `HitboxEntry` with the component that registered it, so `hit_test()` returns the owner directly — no iteration, no equality checks.

### Step 3: Add `active_owner` tracking to `HitboxRegistry`

**File:** `crates/term-wm-core/src/hitbox_registry.rs`

New fields and methods:
```rust
pub struct HitboxRegistry {
    entries: Vec<HitboxEntry>,
    clip_stack: smallvec::SmallVec<[LayoutRect; CLIP_STACK_INLINE_CAPACITY]>,
    active_owner: Option<ComponentOwner>,
}

impl HitboxRegistry {
    pub fn set_active_owner(&mut self, owner: Option<ComponentOwner>) {
        self.active_owner = owner;
    }

    // Modified register():
    pub fn register(&mut self, id: HitboxId, area: LayoutRect) {
        let owner = self.active_owner.expect("register called without set_active_owner");
        let mut clipped = area;
        for clip in &self.clip_stack { clipped = clipped.clamp(*clip); }
        if clipped.width == 0 || clipped.height == 0 { return; }
        self.entries.push(HitboxEntry { id, owner, area: clipped });
    }

    // Modified clear():
    pub fn clear(&mut self) {
        self.entries.clear();
        self.clip_stack.clear();
        self.active_owner = None;
    }

    // Modified hit_test(): returns owner too
    pub fn hit_test(&self, position: MousePosition) -> Option<(HitboxId, ComponentOwner, LayoutRect)> {
        self.entries.iter().rev()
            .find(|entry| position.is_inside(entry.area))
            .map(|entry| (entry.id, entry.owner, entry.area))
    }
}
```

### Step 5: Set active owner in the render pipeline

**File:** `src/lib.rs` — `render_app()`

For each section of the render pipeline, set the `ComponentOwner` before hitbox registration:

```rust
wm.prepare_draw();    // clears registry + maps

// Panel hitboxes — set Layer owner
wm.hitbox_registry_mut().set_active_owner(Some(ComponentOwner::Layer(panel_layer_id)));
wm.register_panel_hitboxes();

// Split handles — set Layer owner  
if !wm.is_monocle() {
    wm.hitbox_registry_mut().set_active_owner(Some(ComponentOwner::Layer(layout_layer_id)));
    wm.register_layout_handle_hitboxes();
}

// Window loop — set Window owner before each window's block
for each window {
    wm.hitbox_registry_mut().set_active_owner(Some(ComponentOwner::Window(key)));
    // Register content hitbox + chrome hitboxes + component.render()
    // All inherit ComponentOwner::Window(key)
}

// FAB — set Layer owner
wm.hitbox_registry_mut().set_active_owner(Some(ComponentOwner::Layer(fab_layer_id)));
// fab.render() ...

// Notification hitboxes — set Layer owner
wm.hitbox_registry_mut().set_active_owner(Some(ComponentOwner::Layer(notif_layer_id)));
// notification_component.render() ...

// Overlays — set Overlay owner before each overlay
for each overlay {
    wm.hitbox_registry_mut().set_active_owner(Some(ComponentOwner::Overlay(key)));
    overlay.render(...)
}
```

The `LayerId` values for panels, layout, FAB, notifications etc. need to be known by the render pipeline. These can be stored on `WindowManager` or looked up via `semantic_registry`.

### Step 6: Simplify `dispatch_mouse`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

Replace the entire `dispatch_mouse` function with a cleaner flow:

**Phase 1 — Active capture** (unchanged logic, but simplify by extracting handler methods). The `MouseCaptureState::ComponentInteraction` branch no longer needs `screen_area` and `hitbox_id` stored — these come from the event context.

**Phase 2-4 — Hit-test dispatch:**

Instead of the current multi-phase iteration:
```rust
let Some((hitbox_id, owner, hit_rect)) = self.hitbox_registry.hit_test(*position) else {
    return EventResult::Ignored;
};

// Step 1: Check chrome maps (O(1) — no iteration)
if let Some(handle) = self.resize_map.get(&hitbox_id) {
    // Start ResizingWindow capture
}
if let Some(handle) = self.drag_map.get(&hitbox_id) {
    // Start DraggingWindow capture
}
if let Some(key) = self.split_map.get(&hitbox_id) {
    // Start LayoutHandle capture
}

// Step 2: Route by owner — no content_hitbox_id equality check
match owner {
    ComponentOwner::Window(key) => {
        let ctx = self.component_context_for(focused, key)
            .with_screen_area(hit_rect)
            .with_active_hitbox(hitbox_id);
        let result = comp.handle_events(&core_event, &ctx);
        // ... handle result, set capture for Press events
    }
    ComponentOwner::Overlay(key) => {
        // Route to overlay
    }
    ComponentOwner::Layer(id) => {
        // Route through layer_manager
    }
}
```

This eliminates all of the following from `dispatch_mouse`:
- `resize_handles.iter().find(|h| h.hitbox_id == hitbox_id)` 
- `floating_headers.iter().find(|h| h.hitbox_id == hitbox_id)`
- `handles.iter().any(|h| h.hitbox_id == hitbox_id)`
- `self.managed_draw_order.iter().rev()` loop checking `w.content_hitbox_id == hitbox_id`
- Separate `get_semantic_component` lookups for command palette, FAB
- `layer_manager.dispatch_foreground` + `dispatch_background` iteration (replaced by direct owner routing)

The `Moved` and `ScrollUp/Down` event phases consolidate into the same `match owner` dispatch.

### Step 7: Rebuild chrome maps during `prepare_draw` / registration

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

In `prepare_draw()`:
```rust
pub fn prepare_draw(&mut self) {
    self.resize_map.clear();
    self.drag_map.clear(); 
    self.split_map.clear();
    // ... existing clear logic
}
```

In `register_window_chrome_hitboxes`:
```rust
pub fn register_window_chrome_hitboxes(&mut self, key: WindowKey) {
    for handle in self.resize_handles.iter().filter(|h| h.key == key) {
        self.resize_map.insert(handle.hitbox_id, *handle);
        self.hitbox_registry.register(handle.hitbox_id, handle.rect);
    }
    for handle in self.floating_headers.iter().filter(|h| h.key == key) {
        self.drag_map.insert(handle.hitbox_id, *handle);
        self.hitbox_registry.register(handle.hitbox_id, handle.rect);
    }
}
```

In `register_layout_handle_hitboxes`:
```rust
pub fn register_layout_handle_hitboxes(&mut self) {
    for handle in &self.handles {
        self.split_map.insert(handle.hitbox_id, *handle);
        self.hitbox_registry.register(handle.hitbox_id, handle.rect);
    }
}
```
