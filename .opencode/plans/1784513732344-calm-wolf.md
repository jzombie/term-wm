# Plan: Component-Owned Hitboxes — Eliminate Hitbox/Component Decoupling

## Problem

`dispatch_mouse` (mod.rs:1054-1751) is a ~700-line procedural guessing game because:
1. `HitboxRegistry` stores bare `HitboxId`s — no record of which component registered them.
2. Window content is identified via `w.content_hitbox_id == hitbox_id` strict equality check — **fails for child components** that register their own `HitboxId`.
3. Chrome (resize handles, drag handles, split handles) requires iterating through three separate lists (`resize_handles`, `floating_headers`, `handles`) searching for a matching `hitbox_id`.
4. Scroll/Moved/Press phases each repeat the same window iteration + equality check pattern.

## Solution

Tag every `HitboxEntry` with the component that registered it, so `hit_test()` returns the owner directly — no iteration, no equality checks.

**Core invariant:** `HitboxEntry::owner` MUST be a concrete `ComponentOwner` — never `Option`. An un-owned hitbox is an invalid state.  

Key design decisions:
- `ComponentOwner::Test` is **unconditional** (no `#[cfg(test)]`) so `tests/*.rs` integration tests can reference it across crate boundaries.
- Scratch registries (FAB, notifications, offscreen window composites) use `HitboxRegistry::with_owner()` to initialize with a valid owner at construction time.
- `ComponentOwner::System` covers structural chrome (split handles, resize borders) that is intercepted by O(1) maps in `dispatch_mouse` before the owner match — no fake `LayerId` attribution.
- `MouseCaptureState::ComponentInteraction` stores `owner: ComponentOwner` (replacing `key: Option<WindowKey>`) so active capture during drag routes directly to the owning component via `match owner` — no layer array scan.
- All scratch registries (panels, overlays, FAB, notifications, window composites) use `HitboxRegistry::with_owner()` at construction time — both in `src/lib.rs` and `crates/term-wm-console/src/draw_plan_renderer.rs`.

### Step 1: Add `ComponentOwner` enum

**File:** `crates/term-wm-core/src/hitbox_registry.rs`

Add a new public enum (in `hitbox_registry.rs`):

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ComponentOwner {
    Window(WindowKey),
    Overlay(OverlayKey),
    Layer(LayerId),
    /// Structural chrome (split handles, resize borders) intercepted
    /// by O(1) maps before the owner match in dispatch_mouse.
    System,
    /// Test-only owner for standalone component unit tests that
    /// render without a WindowManager context. Never used in
    /// production dispatch paths. Unconditional (no #[cfg(test)])
    /// so integration tests in tests/*.rs can reference it.
    Test,
}
```

### Step 3: Add `active_owner` tracking to `HitboxRegistry`

**File:** `crates/term-wm-core/src/hitbox_registry.rs`

`active_owner` is `Option<ComponentOwner>` internally (no component is "active" between render passes), but `register()` **panics** if `None` — enforcing the invariant that every entry has a concrete owner.

```rust
pub struct HitboxRegistry {
    entries: Vec<HitboxEntry>,
    clip_stack: smallvec::SmallVec<[LayoutRect; CLIP_STACK_INLINE_CAPACITY]>,
    active_owner: Option<ComponentOwner>,  // set per render-unit, cleared between frames
}

impl HitboxRegistry {
    /// Create a registry pre-configured with an active owner.
    /// Use this for scratch/offscreen registries that will be
    /// merged into the main registry after rendering.
    pub fn with_owner(owner: ComponentOwner) -> Self {
        Self {
            entries: Vec::new(),
            clip_stack: smallvec::SmallVec::new(),
            active_owner: Some(owner),
        }
    }

    /// Set the active component owner. All subsequent `register()` calls
    /// will inherit this owner until the next `set_active_owner` call.
    /// Panics if `register()` is called without a prior `set_active_owner`.
    pub fn set_active_owner(&mut self, owner: ComponentOwner) {
        self.active_owner = Some(owner);
    }

    /// Clear the active owner (called after a render unit completes
    /// and before the next one begins, to catch missing-owner bugs).
    pub fn clear_active_owner(&mut self) {
        self.active_owner = None;
    }

    // Modified register(): panics if no active owner
    pub fn register(&mut self, id: HitboxId, area: LayoutRect) {
        let owner = self
            .active_owner
            .expect("HitboxRegistry::register invoked without an active ComponentOwner");
        let mut clipped = area;
        for clip in &self.clip_stack { clipped = clipped.clamp(*clip); }
        if clipped.width == 0 || clipped.height == 0 { return; }
        self.entries.push(HitboxEntry { id, owner, area: clipped });
    }

    // Modified clear():
    pub fn clear(&mut self) {
        self.entries.clear();
        self.clip_stack.clear();
        self.active_owner = None;
    }

    // Modified hit_test(): returns owner too
    pub fn hit_test(&self, position: MousePosition) -> Option<(HitboxId, ComponentOwner, LayoutRect)> {
        self.entries.iter().rev()
            .find(|entry| position.is_inside(entry.area))
            .map(|entry| (entry.id, entry.owner, entry.area))
    }
}
```

### Step 5: Set active owner in the render pipeline

**Files:** `src/lib.rs` — `render_app()` and `crates/term-wm-console/src/draw_plan_renderer.rs` — `render_panels()`, `render_overlays()`, etc.

For each section of the render pipeline, set the `ComponentOwner` before hitbox registration:

```rust
wm.prepare_draw();    // clears registry + maps

// Panel hitboxes — set Layer owner
wm.hitbox_registry_mut().set_active_owner(ComponentOwner::Layer(panel_layer_id));
wm.register_panel_hitboxes();
wm.hitbox_registry_mut().clear_active_owner();

// Split handles — use ComponentOwner::System since these are
// structural chrome intercepted by split_map before the owner match.
// No fake LayerId needed.
if !wm.is_monocle() {
    wm.hitbox_registry_mut().set_active_owner(ComponentOwner::System);
    wm.register_layout_handle_hitboxes();
    wm.hitbox_registry_mut().clear_active_owner();
}

// Window loop — set Window owner before each window's block
for each window {
    wm.hitbox_registry_mut().set_active_owner(ComponentOwner::Window(key));
    // Chrome hitboxes (resize/drag) registered alongside content —
    // they inherit ComponentOwner::Window(key) but are intercepted
    // via resize_map/drag_map before the owner match in dispatch.
    wm.register_window_chrome_hitboxes(key);
    // Register content hitbox
    let content_hitbox_id = wm.window_content_hitbox_id(key);
    wm.hitbox_registry_mut().register(content_hitbox_id, screen_inner);
    // Component render into a scratch registry with_window owner
    let mut local_hb = HitboxRegistry::with_owner(ComponentOwner::Window(key));
    component.render(backend, area, &ctx, &mut local_hb);
    // Merge child hitboxes into the main registry
    wm.hitbox_registry_mut().merge(local_hb);
    wm.hitbox_registry_mut().clear_active_owner();
}

// FAB — set Layer owner
wm.hitbox_registry_mut().set_active_owner(ComponentOwner::Layer(fab_layer_id));
fab.render() ...
wm.hitbox_registry_mut().clear_active_owner();

// Notification hitboxes — set Layer owner
wm.hitbox_registry_mut().set_active_owner(ComponentOwner::Layer(notif_layer_id));
notification_component.render() ...
wm.hitbox_registry_mut().clear_active_owner();

// Overlays — set Overlay owner before each overlay
for each overlay {
    wm.hitbox_registry_mut().set_active_owner(ComponentOwner::Overlay(key));
    overlay.render(...)
    wm.hitbox_registry_mut().clear_active_owner();
}
```

The `LayerId` values for panels, layout, FAB, notifications etc. need to be known by the render pipeline. These can be stored on `WindowManager` or looked up via `semantic_registry`.

### Step 6: Simplify `dispatch_mouse`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

Replace the entire `dispatch_mouse` function with a cleaner flow:

**Phase 1 — Active capture** (unchanged core logic). `MouseCaptureState::ComponentInteraction` replaces `key: Option<WindowKey>` with `owner: ComponentOwner` so active capture during drag/release routes directly via `match owner` instead of falling back to `layer_manager.dispatch_foreground()`:

```rust
MouseCaptureState::ComponentInteraction { owner, screen_area, hitbox_id } => {
    let ctx = self.component_context(false)
        .with_screen_area(*screen_area)
        .with_active_hitbox(*hitbox_id);
    let consumed = match owner {
        ComponentOwner::Window(key) => {
            comp_for_key(key).handle_events(&core_event, &ctx).is_consumed()
        }
        ComponentOwner::Overlay(key) => {
            overlays.get_mut(key).handle_events(&core_event, &ctx).is_consumed()
        }
        ComponentOwner::Layer(id) => {
            layer_manager.get_mut(id).handle_events(&core_event, &ctx).is_consumed()
        }
        ComponentOwner::System | ComponentOwner::Test => false,
    };
    (consumed, !matches!(kind, MouseEventKind::Release(_)))
}
```

This eliminates the `key: None` → `dispatch_foreground()` fallback iteration for layer component interactions (panels, FAB) during active drag sequences.

**Phase 2-4 — Hit-test dispatch (exhaustive match, zero fallback):**

```rust
let Some((hitbox_id, owner, hit_rect)) = self.hitbox_registry.hit_test(*position) else {
    return EventResult::Ignored;
};

// Chrome handle lookup: O(1) maps for resize/drag/split operations
// that require WindowManager-level MouseCaptureState management.
// These are NOT fallback routing — they fire first because chrome
// captures the mouse before any component processes the event.
if let Some(handle) = self.resize_map.get(&hitbox_id) {
    // Start ResizingWindow capture, return Consumed
}
if let Some(handle) = self.drag_map.get(&hitbox_id) {
    // Start DraggingWindow capture, return Consumed
}
if let Some(handle) = self.split_map.get(&hitbox_id) {
    // Start LayoutHandle capture, return Consumed
}

// Exhaustive owner match — every hitbox has exactly one owner.
// NO procedural fallback. NO content_hitbox_id equality checks.
// System hits should not reach here (chrome maps above already
// intercepted them), but we handle defensively.
match owner {
    ComponentOwner::Window(key) => {
        let ctx = self.component_context_for(focused, key)
            .with_screen_area(hit_rect)
            .with_active_hitbox(hitbox_id);
        let result = comp.handle_events(&core_event, &ctx);
        if !result.is_ignored() && matches!(kind, Press(_)) {
            self.mouse_capture = Some(MouseCaptureState::ComponentInteraction {
                owner: ComponentOwner::Window(key),
                screen_area: hit_rect,
                hitbox_id,
            });
        }
    }
    ComponentOwner::Overlay(key) => {
        // Direct dispatch to overlay
        // If consumed, set capture with ComponentOwner::Overlay(key)
    }
    ComponentOwner::Layer(layer_id) => {
        // Single lookup in layer_manager, no iteration
        // If consumed, set capture with ComponentOwner::Layer(layer_id)
    }
    ComponentOwner::System => {
        // Chrome hits (split handles) already intercepted by maps
        // above — this is a defensive fallback.
        EventResult::Ignored
    }
    ComponentOwner::Test => EventResult::Ignored,
}
```

This eliminates all of the following from `dispatch_mouse`:
- `resize_handles.iter().find(|h| h.hitbox_id == hitbox_id)` — replaced by O(1) `resize_map`
- `floating_headers.iter().find(|h| h.hitbox_id == hitbox_id)` — replaced by O(1) `drag_map`
- `handles.iter().any(|h| h.hitbox_id == hitbox_id)` — replaced by O(1) `split_map`
- `self.managed_draw_order.iter().rev()` loop checking `w.content_hitbox_id == hitbox_id` — eliminated entirely
- Separate `get_semantic_component` lookups for command palette, FAB — eliminated (handled via Layer dispatch)
- `layer_manager.dispatch_foreground` + `dispatch_background` iteration — replaced by single layer lookup via `ComponentOwner::Layer`

The `Moved` and `ScrollUp/Down` event phases consolidate into the same `match owner` dispatch.

In `register_window_chrome_hitboxes`:
```rust
pub fn register_window_chrome_hitboxes(&mut self, key: WindowKey) {
    for handle in self.resize_handles.iter().filter(|h| h.key == key) {
        self.resize_map.insert(handle.hitbox_id, *handle);
        self.hitbox_registry.register(handle.hitbox_id, handle.rect);
    }
    for handle in self.floating_headers.iter().filter(|h| h.key == key) {
        self.drag_map.insert(handle.hitbox_id, *handle);
        self.hitbox_registry.register(handle.hitbox_id, handle.rect);
    }
}
```

In `register_layout_handle_hitboxes`:
```rust
pub fn register_layout_handle_hitboxes(&mut self) {
    for handle in &self.handles {
        self.split_map.insert(handle.hitbox_id, *handle);
        self.hitbox_registry.register(handle.hitbox_id, handle.rect);
    }
}
```
