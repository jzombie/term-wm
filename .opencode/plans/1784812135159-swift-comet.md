# Plan: Purge Duplicated Layout Math â€” Migrate Core to Use `term-wm-layout-engine`

## Background

Three SEV-1 violations:
1. **Floating-point math** â€” `LayoutNode::Split` stores `weights: Vec<f32>`; `split_rects_weighted` uses `floor()`/`round()` with f32 arithmetic, causing pixel drift.
2. **Complete boundary failure** â€” 5 private functions duplicate the engine's public API (same algorithm, different types).
3. **Dead constraint abstraction** â€” `constraints: Vec<Constraint>` is stored on every `Split` but the `Layout::split()` solver ignores all constraint values and does equal split only. This is dual-source-of-truth with `weights`.

`Rect` is already `type Rect = LayoutRect` (`core/src/lib.rs:36`), so no geometry conversion needed.
