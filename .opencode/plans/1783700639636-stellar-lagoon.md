# Plan: Lock In Tiling Manager Behavioral Fixes

## Goal
Formalize all tiling manager behavioral guarantees with code fixes and regression tests to prevent regressions. The core architectural rule: **floating windows and tiled windows must never be intermingled in their Z-index hierarchy.**

---

## Phase 1: Fix Critical Bifurcation Bugs

### Bug 1: `apply_snap` void-snap path (line 286)
**File:** `crates/term-wm-core/src/window/window_manager/drag.rs:286`
```rust
self.z_order.push(key);
self.managed_draw_order = self.z_order.clone();  // BUG: No bifurcation!
```

### Bug 2: `apply_snap` normal-snap path (line 345)
**File:** `crates/term-wm-core/src/window/window_manager/drag.rs:345`
```rust
self.z_order.push(key);
self.managed_draw_order = self.z_order.clone();  // BUG: No bifurcation!
```

Both overwrite `managed_draw_order` with a raw clone of `z_order`, breaking the tiled-first/floating-last invariant. This means after any snap-to-tile operation, the draw order is corrupt until the next `register_managed_layout` call.

**Fix:** Replace both `self.managed_draw_order = self.z_order.clone();` with a call to a new `bifurcate_draw_order()` helper.

## Phase 2: Extract `bifurcate_draw_order()` Helper

**File:** `crates/term-wm-core/src/window/window_manager/layout.rs`

### Step 2a: Add the helper method

Insert before `bring_to_front` (line 661):

```rust
/// Rebuild `managed_draw_order` from `z_order` with strict layer separation:
/// all tiled windows first (back), then all floating windows (front).
pub(super) fn bifurcate_draw_order(&mut self) {
    let mut bifurcated = Vec::with_capacity(self.z_order.len());
    for &key in &self.z_order {
        if !self.is_window_floating(key) {
            bifurcated.push(key);
        }
    }
    for &key in &self.z_order {
        if self.is_window_floating(key) {
            bifurcated.push(key);
        }
    }
    self.managed_draw_order = bifurcated;
}
```

### Step 2b: Replace inline bifurcation in `register_managed_layout`

At lines 375-386, replace the 12-line inline block with:
```rust
        self.bifurcate_draw_order();
```

## Phase 4: Add Regression Tests

**File:** `tests/integration_tiling.rs`

Add a new test module `mod floating_tiled_separation` with these tests. Each test uses the existing test helpers (`make_wm`, `make_key`, `render_and_sync`, etc.) and the `TilingLayout` + `WindowManager` APIs.

### Test Helper: `assert_bifurcation_invariant`
Iterates forward: once a floating window is seen, no tiled window may follow.
```rust
fn assert_bifurcation_invariant(wm: &WindowManager) {
    let order = wm.managed_draw_order_all();
    let mut seen_floating = false;
    for &key in order.iter() {
        if wm.is_window_floating(key) {
            seen_floating = true;
        } else if seen_floating {
            panic!(
                "Tiled window {:?} found after floating window in managed_draw_order",
                key
            );
        }
    }
}
```
