# Fix: Coordinate bugs when floating window is dragged off-screen

## Root Cause

`ctx.screen_area()` returns a signed `LayoutRect` (i32 x/y). Throughout the codebase, this is converted to unsigned `ratatui::layout::Rect` (u16 x/y) via `layout_rect_to_rect()`, which does `x: area.x as u16`. When `x` is negative, this wraps (underflow), producing huge values and corrupting all downstream math.

## Fix: Three centralized helpers replace the ad-hoc math

Delete the unsafe `layout_rect_to_rect` and add three explicit, bulletproof helpers in `helpers.rs`:

### 1. `layout_rect_to_clipped_rect` ‚Äî for rendering

Safely absorbs negative coordinates by cropping the invisible portion. Ratatui gets an on-screen bounding box with reduced width/height.

```rust
pub fn layout_rect_to_clipped_rect(area: LayoutRect) -> Rect {
    let x = area.x.max(0) as u16;
    let y = area.y.max(0) as u16;
    let crop_left = area.x.min(0).unsigned_abs() as u16;
    let crop_top = area.y.min(0).unsigned_abs() as u16;
    Rect { x, y, width: area.width.saturating_sub(crop_left), height: area.height.saturating_sub(crop_top) }
}
```

### 2. `localize_coordinate` ‚Äî for event handling (strict)

Translates a global screen coordinate into a component-local coordinate. Returns `None` if outside bounds. Used for click/clickable targets where out-of-bounds = ignored.

```rust
pub fn localize_coordinate(area: LayoutRect, global_col: u16, global_row: u16) -> Option<(u16, u16)> {
    let g_col = i32::from(global_col);
    let g_row = i32::from(global_row);
    let max_x = area.x.saturating_add(i32::from(area.width));
    let max_y = area.y.saturating_add(i32::from(area.height));
    if g_col < area.x || g_col >= max_x || g_row < area.y || g_row >= max_y { return None; }
    Some(((g_col - area.x) as u16, (g_row - area.y) as u16))
}
```

### 3. `localize_coordinate_clamped` ‚Äî for drag selection (clamped)

Clamps the coordinate to the nearest edge if outside bounds. Used for text selection dragging where dragging past the edge = "select to the edge".

```rust
pub fn localize_coordinate_clamped(area: LayoutRect, global_col: u16, global_row: u16) -> Option<(u16, u16)> {
    if area.width == 0 || area.height == 0 { return None; }
    let g_col = i32::from(global_col);
    let g_row = i32::from(global_row);
    let max_x = area.x.saturating_add(i32::from(area.width)).saturating_sub(1);
    let max_y = area.y.saturating_add(i32::from(area.height)).saturating_sub(1);
    let clamped_col = g_col.clamp(area.x, max_x);
    let clamped_row = g_row.clamp(area.y, max_y);
    Some(((clamped_col - area.x) as u16, (clamped_row - area.y) as u16))
}
```

## Changes

### Phase 1: Add helpers + update terminal.rs

**`crates/term-wm-ui-components/src/helpers.rs`**
- Delete `layout_rect_to_rect`
- Add `layout_rect_to_clipped_rect`, `localize_coordinate`, and `localize_coordinate_clamped`
- Update the `pub use` exports in `src/lib.rs` if needed

**`crates/term-wm-ui-components/src/terminal.rs`**
- `handle_events`: Replace `i32::from(mouse.column).saturating_sub(area.x).max(0) as u16` with `localize_coordinate(area, mouse.column, mouse.row)` ‚Äî strict rejection is correct for click targets
- `link_at_position` / `try_handle_link_click`: Replace bounds-check + local coords with `localize_coordinate` ‚Äî strict rejection is correct for link targets
- `SelectionViewport::logical_position_from_point`: Replace i32 clamp/subtract with `localize_coordinate_clamped` ‚Äî **clamped** for drag-outside-window UX
- `RenderDragHost::logical_position_from_point`: Same ‚Äî use `localize_coordinate_clamped` for drag selection
- `rect_contains` calls: These are fine (already take `LayoutRect`)
- Keep parameters as `LayoutRect` (already done)

### Phase 2: Update scroll_view.rs

**`crates/term-wm-ui-components/src/scroll_view.rs`**

- `ScrollbarDrag::handle_mouse`, `scrollbar_offset_from_row`, `scrollbar_offset_from_col`, `rect_contains` ‚Äî already take `LayoutRect` (no change needed)
- Remove the re-added `use crate::helpers::layout_rect_to_rect;` import (will import the new helpers instead)
