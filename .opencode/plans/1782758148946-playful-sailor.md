# Plan: Floating Window Drop Shadow

## Problem

Floating windows in the terminal window manager sit flat against the background with no visual depth cue. A drop shadow behind each floating window would signal stacking order and make the UI feel more polished.

## Approach

Add a drop shadow effect to floating windows using `Modifier::DIM` + a dark background color, rendered as a block offset by (2 columns right, 1 row down) from the window.

Rendering pipeline: Before a floating window is composited onto the main frame, a "shadow" buffer is blitted at an offset position. Since windows are drawn bottom-to-top, each window's shadow naturally goes behind it and lower windows' shadows are overlaid by higher windows.

## Files to Modify

### 1. `crates/term-wm-core/src/constants.rs`
Add shadow offset constants:
```rust
pub const SHADOW_OFFSET_X: i32 = 2;
pub const SHADOW_OFFSET_Y: i32 = 1;
```

### 2. `crates/term-wm-core/src/window/window_manager/mod.rs` (~line 108-113)
Add `is_floating: bool` to `WindowSurface`:
```rust
pub struct WindowSurface {
    pub full: Rect,
    pub inner: Rect,
    pub dest: crate::window::FloatRect,
    pub is_floating: bool,  // NEW
}
```
This is the cleanest integration point — `composite_window` already receives `&WindowSurface` and can decide to render a shadow based on this flag.

### 3. `crates/term-wm-core/src/window/window_manager/layout.rs` (~line 483-493)
In `window_draw_plan`, set `is_floating: self.is_window_floating(id)` when constructing both `SystemWindowDraw` and `WindowDrawContext`:
```rust
surface: super::WindowSurface { full, inner, dest, is_floating: self.is_window_floating(id) },
```

### 4. `crates/term-wm-core/src/runner.rs` (~line 710-742)
Modify `composite_window` to render a shadow before the window content:
- After the offscreen buffer is built (decorator + content rendered, DIM applied if unfocused)
- If `surface.is_floating`, create a temporary shadow buffer (same size as window), fill with `' '` + `Modifier::DIM` + surface bg color
- `blit_from_signed` the shadow buffer at `dest + (SHADOW_OFFSET_X, SHADOW_OFFSET_Y)`
- Then `blit_from_signed` the normal window buffer at `dest`

Pseudocode:
```rust
fn composite_window(...) {
    // ... existing early return, buffer creation, render_content ...
    if !focused { /* dim all cells */ }
    
    // NEW: render drop shadow for floating windows
    if surface.is_floating {
        let shadow_dest = FloatRect {
            x: surface.dest.x.saturating_add(SHADOW_OFFSET_X),
            y: surface.dest.y.saturating_add(SHADOW_OFFSET_Y),
            width: surface.dest.width,
            height: surface.dest.height,
        };
        let mut shadow_buf = Buffer::empty(local_area);
        for cell in shadow_buf.content.iter_mut() {
            cell.set_char(' ');
            cell.modifier.insert(Modifier::DIM);
            cell.bg = crate::theme::surface();
        }
        frame.blit_from_signed(&shadow_buf, shadow_dest);
    }
    
    frame.blit_from_signed(&buffer, surface.dest);  // existing
}
```

### 5. `crates/term-wm-core/src/wm_config.rs` (~line 67-98)
Add `shadow_enabled: bool` field to `WmConfig`:
- Default `true` in `standalone()`
- Default `false` in `embedded()`
This allows users to disable shadows if desired. Check this flag in `composite_window`.

### 6. `crates/term-wm-core/src/theme.rs` (~line 50-85)
Use existing `surface` color for shadow background (no new theme field needed). Verify the `surface` color is accessible via `crate::theme::surface()`.
