# Plan: Window Lifecycle State Machine (Requirement 2b)

Replace ad-hoc imperative lifecycle (`minimized: bool`, scattered z-order/layout/focus mutations) with a canonical state machine embedded directly in `WindowManager`. Add event-driven async teardown via `AppExited(WindowKey)` — no circular crate dependencies, no borrow-checker fights.

---

## Phase 1: `WindowState` Enum + State Machine in `WindowManager`

### 1a. Define `WindowState`

**File:** `crates/term-wm-core/src/window/entry.rs`

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum WindowState {
    Realized,    // Allocated in SlotMap, invisible, not in layout tree
    Mapped,      // Visible, geometry routed to layout tree
    Unmapped,    // Hidden (withdrawn), in memory
    Iconic,      // Mapped but hidden from workspace (minimized)
    Shaded,      // Chrome-only visible (title bar only)
}
```

Replace `pub minimized: bool` on `Window` with:
```rust
pub state: WindowState,
```

Valid transitions:
```
Realized  → Mapped     (tile/set_focus — allocate + show)
Realized  → Unmapped   (hide without ever showing)
Mapped    → Iconic     (minimize)
Iconic    → Mapped     (restore)
Mapped    → Unmapped   (withdraw — hidden, in memory)
Unmapped  → Mapped     (remap — show again)
Mapped    → Shaded     (shade — chrome only)
Shaded    → Mapped     (unshade)
```

### 1b. `transition_window` — borrow-safe, atomic side-effects

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

`WindowManager` owns the `SlotMap` directly. The method must NOT hold a mutable borrow across side-effect calls:

```rust
pub fn transition_window(&mut self, key: WindowKey, new_state: WindowState) {
    // Step 1: Read old state (immutable borrow, immediately dropped)
    let old_state = self.window(key).expect("transition_window: invalid key").state;
    if old_state == new_state {
        return;  // no-op
    }

    // Step 2: Validate transition
    assert!(
        is_valid_transition(old_state, new_state),
        "Illegal window state transition: {:?} -> {:?}", old_state, new_state
    );

    // Step 3: Mutate state (brief mutable borrow, immediately dropped)
    self.window_mut(key).state = new_state;

    // Step 4: Side-effects — full &mut self available, no outstanding borrows
    match (old_state, new_state) {
        (WindowState::Realized, WindowState::Mapped) => {
            self.z_order.push(key);
            self.managed_draw_order.push(key);
            self.focus_add(key);
        }
        (WindowState::Mapped, WindowState::Iconic) => {
            self.z_order.retain(|x| *x != key);
            self.managed_draw_order.retain(|x| *x != key);
            if *self.focus.current() == key {
                self.select_fallback_focus();
            }
            self.detach_from_tiling_layout(key);  // must be idempotent
        }
        (WindowState::Iconic, WindowState::Mapped) => {
            if !self.z_order.contains(&key) { self.z_order.push(key); }
            if !self.managed_draw_order.contains(&key) { self.managed_draw_order.push(key); }
            self.reattach_to_tiling_layout(key);
        }
        (_, WindowState::Unmapped) => {
            self.clear_floating_rect(key);
            self.z_order.retain(|x| *x != key);
            self.managed_draw_order.retain(|x| *x != key);
            self.regions.remove(key);
            self.scroll.remove(&key);
            self.remove_from_focus_ring(key);
            if *self.focus.current() == key {
                self.select_fallback_focus();
            }
            self.detach_from_tiling_layout(key);  // idempotent — safe if already detached
        }
        (WindowState::Unmapped, WindowState::Mapped) => {
            self.z_order.push(key);
            self.managed_draw_order.push(key);
            self.reattach_to_tiling_layout(key);
            self.focus_add(key);
        }
        (WindowState::Mapped, WindowState::Shaded) => {
            if let Some(region) = self.regions.get_mut(key) {
                region.height = 1;  // title bar only
            }
        }
        (WindowState::Shaded, WindowState::Mapped) => {
            self.regions.remove(key);
            self.reattach_to_tiling_layout(key);
        }
        _ => {}
    }
}

fn is_valid_transition(old: WindowState, new: WindowState) -> bool {
    matches!((old, new),
        (WindowState::Realized, WindowState::Mapped)
        | (WindowState::Realized, WindowState::Unmapped)
        | (WindowState::Mapped, WindowState::Iconic)
        | (WindowState::Iconic, WindowState::Mapped)
        | (WindowState::Mapped, WindowState::Unmapped)
        | (WindowState::Unmapped, WindowState::Mapped)
        | (WindowState::Mapped, WindowState::Shaded)
        | (WindowState::Shaded, WindowState::Mapped)
    )
}
```

`detach_from_tiling_layout` must be idempotent — check if the key is in the layout tree before modifying:
```rust
fn detach_from_tiling_layout(&mut self, key: WindowKey) {
    if let Some(ref mut layout) = self.managed_layout {
        if layout.root().subtree_any(|id| id == key) {
            layout.detach_leaf(key);
        }
    }
}
```

### 2g. Handle `AppExited` in runner

**File:** `crates/term-wm-core/src/runner.rs`

In the event handler, match on `UnifiedEvent::AppExited(key)`:
```rust
// In the idle path or event dispatch:
UnifiedEvent::AppExited(key) => {
    app.windows().close_window(key);   // transition to Unmapped + remove from SlotMap
    app.wm_close_window(key)?;         // clean up app state
}
```

**Race condition safety:** `SlotMap` returns `None` for stale keys via generational indexing. If a `PtyWakeup` arrives for a key already removed by `AppExited`, `WindowManager::window(key)` returns `None`. The event loop and all callers must use checked access (`window()`) not panicking access (`window_mut()`) when handling events.

## Phase 3: Runner Boundary (Lightweight)

No `WindowServer` struct. State machine lives directly in `WindowManager`. The `WindowManagerHost` trait exposes only high-level commands, NOT `&mut WindowManager`:

```rust
pub trait WindowManagerHost {
    fn wm_new_window(&mut self) -> io::Result<()>;
    fn wm_close_window(&mut self, key: WindowKey) -> io::Result<()>;
    fn wm_minimize_window(&mut self, key: WindowKey);
    fn wm_close_window_by_key(&mut self, key: WindowKey);
    fn wm_quit_requested(&self) -> bool;
    // No fn windows(&mut self) -> &mut WindowManager
}
```

The runner event loop (`run_app`/`run_window_app`) calls these trait methods. The `App` struct in `main.rs` holds `WindowManager` as a field and delegates.

Read-only query methods (e.g., `focused_window()`, `window_state()`, `enumerate_windows()`) remain accessible via a read-only accessor or as standalone trait methods — but the runner never gets `&mut WindowManager`.
