# Plan: Window Lifecycle State Machine (Requirement 2b)

Replace ad-hoc imperative lifecycle (`minimized: bool`, scattered z-order/layout/focus mutations) with a canonical state machine embedded directly in `WindowManager`. Add event-driven async teardown via `AppExited(WindowKey)` â€” no `Arc<dyn Fn>` closures, no separate `WindowServer` struct.

---

## Phase 1: `WindowState` Enum + State Machine in `WindowManager`

### 1a. Define `WindowState`

**File:** `crates/term-wm-core/src/window/entry.rs`

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum WindowState {
    Realized,    // Allocated in SlotMap, invisible, not in layout tree
    Mapped,      // Visible, geometry routed to layout tree
    Unmapped,    // Hidden (withdrawn), in memory
    Iconic,      // Mapped but hidden from workspace (minimized)
    Shaded,      // Chrome-only visible (title bar only)
}
```

Replace `pub minimized: bool` on `Window` with:
```rust
pub state: WindowState,
```

Valid transitions:
```
Realized  â†’ Mapped     (tile/set_focus â€” allocate + show)
Realized  â†’ Unmapped   (hide without ever showing)
Mapped    â†’ Iconic     (minimize)
Iconic    â†’ Mapped     (restore)
Mapped    â†’ Unmapped   (withdraw â€” hidden, in memory)
Unmapped  â†’ Mapped     (remap â€” show again)
Mapped    â†’ Shaded     (shade â€” chrome only)
Shaded    â†’ Mapped     (unshade)
```

(Removal from SlotMap is separate â€” a window in any state can be closed.)

### 1b. Single `transition_window` method â€” the indivisible owner

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

No separate `WindowServer` struct. `WindowManager` owns the `SlotMap` directly and `transition_window` is the *single, indivisible* method that both mutates the enum AND applies all side-effects:

```rust
pub fn transition_window(&mut self, key: WindowKey, new_state: WindowState) {
    let w = self.window_mut(key);
    let old = w.state;
    validate_transition(old, new_state);  // panic or log on illegal transition
    w.state = new_state;

    // Side-effects â€” applied atomically with the state change:
    match (old, new_state) {
        (WindowState::Realized, WindowState::Mapped) => {
            // Window already allocated; add to z-order, layout, focus
            self.z_order.push(key);
            self.managed_draw_order.push(key);
            self.regions.set(key, Rect::default());  // to be populated by layout
            if !self.focus.order.contains(&key) {
                self.focus.order.push(key);
            }
        }
        (WindowState::Mapped, WindowState::Iconic) => {
            // Remove from z-order and draw order, but keep in layout tree
            // so the window's data is preserved.
            self.z_order.retain(|x| *x != key);
            self.managed_draw_order.retain(|x| *x != key);
            if *self.focus.current() == key {
                self.select_fallback_focus();
            }
            // Layout redistribution: detach window from tiling layout,
            // give its space to siblings. Triggers recalculate_layout.
            self.detach_from_tiling_layout(key);
        }
        (WindowState::Iconic, WindowState::Mapped) => {
            // Re-add to z-order and draw order.
            if !self.z_order.contains(&key) {
                self.z_order.push(key);
            }
            if !self.managed_draw_order.contains(&key) {
                self.managed_draw_order.push(key);
            }
            // Re-insert into tiling layout.
            self.reattach_to_tiling_layout(key);
        }
        (_, WindowState::Unmapped) => {
            // Remove from display, keep in SlotMap.
            self.clear_floating_rect(key);
            self.z_order.retain(|x| *x != key);
            self.managed_draw_order.retain(|x| *x != key);
            self.regions.remove(key);
            self.scroll.remove(&key);
            self.remove_from_focus_ring(key);
            if *self.focus.current() == key {
                self.select_fallback_focus();
            }
            self.detach_from_tiling_layout(key);
        }
        (WindowState::Unmapped, WindowState::Mapped) => {
            // Re-add from Unmapped state.
            // Z-order, draw_order, layout handled by register_managed_layout
            // and window_draw_plan which iterate the SlotMap.
            self.z_order.push(key);
            self.managed_draw_order.push(key);
            self.reattach_to_tiling_layout(key);
        }
        (WindowState::Mapped, WindowState::Shaded) => {
            // Chrome-only: shrink region to just title bar height.
            // Content area becomes 0-height; only decorator renders.
            if let Some(region) = self.regions.get_mut(key) {
                region.height = 1;  // title bar only
            }
        }
        (WindowState::Shaded, WindowState::Mapped) => {
            // Restore full content area via layout recalculation.
            self.regions.remove(key);
            self.reattach_to_tiling_layout(key);
        }
        _ => {}  // no side-effects for same-state or unsupported transitions
    }
}
```

### 2f. Handle `AppExited` in runner

**File:** `crates/term-wm-core/src/runner.rs`

In the event handler, drain `AppExited` alongside `PtyWakeup`:
```rust
// Match on AppExited in event dispatch:
if let Some(UnifiedEvent::AppExited(key)) = event { ... }
```

When `AppExited(key)` is received:
1. `app.windows().close_window(key)` â€” transitions to `Unmapped`, removes from SlotMap
2. `app.wm_close_window(key)?` â€” cleans up app state (removes from `terminals` map, sends to Reaper)

**Race condition safety:** The `SlotMap` returns `None` for stale keys via generational indexing. If a `PtyWakeup` arrives for a key already removed by `AppExited`, `WindowManager::window(key)` returns `None` â€” callers must handle this (e.g., `window_mut` panics, so use checked access). The event loop must NOT unwrap on stale keys.

### 2g. Remove `has_exited()` filtering from `enumerate_windows()`

**File:** `src/main.rs`

Remove the `sv.content.has_exited()` filter. Windows are now closed event-driven; the exited key is already removed from `terminals` by `wm_close_window`.

### 2h. Remove old `set_wakeup` / `WakeupFn` entirely

Delete the `WakeupFn` type alias, `wakeup` field on `Pty`, and `set_wakeup` methods. The `wakeup_tx` sender replaces them completely.

## Phase 3: Runner Boundary (Lightweight)

Skip the separate `WindowServer` struct. Keep the state machine directly in `WindowManager`. The boundary is enforced by convention in `WindowManagerHost`:

```rust
pub trait WindowManagerHost {
    // Read-only access â€” WindowManager exposes query methods.
    // Mutable access is only through high-level commands:
    fn wm_new_window(&mut self) -> io::Result<()>;
    fn wm_close_window(&mut self, key: WindowKey) -> io::Result<()>;
    fn minimize_window(&mut self, key: WindowKey);
    fn close_window(&mut self, key: WindowKey);
    // ... no fn windows(&mut self) -> &mut WindowManager
}
```

The `App` in `main.rs` holds `WindowManager` directly and delegates these calls. The event loop in `runner.rs` calls trait methods, never `&mut WindowManager`.
