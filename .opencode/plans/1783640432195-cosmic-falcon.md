# Notification (Toast) System Implementation Plan

## Overview

Add a self-contained notification/toast system that renders transient messages over the terminal workspace. Notifications are non-interactive, timed display elements that stack from the top-right corner.

## Architecture

Notifications do NOT use the `Component`/`Overlay` trait system — they are simpler, display-only elements managed directly by `WindowManager` and rendered as a pass in the rendering pipeline. Timeout management piggybacks on the existing `SystemTask` + `TaskScheduler` pattern (already used for `DragSnap`, `SuperPassthrough`).

## Files to Modify

### 1. New: `crates/term-wm-ui-components/src/notification.rs`

**`Notification` struct:**
```rust
pub struct Notification {
    pub id: u64,
    pub message: String,
    pub style: NotificationStyle,
    pub created_at: Instant,
}
```

**`NotificationStyle`:**
```rust
pub struct NotificationStyle {
    pub bg: Color,
    pub fg: Color,
    pub border: Color,
}
```

**`NotificationQueue`:**
- Owns `VecDeque<Notification>` + `next_id: u64` + `max_visible: usize` (cap, e.g. 5)
- `push(&mut self, message: String, style: NotificationStyle) -> u64` — creates notification with auto-incrementing ID, returns ID
- `dismiss(&mut self, id: u64) -> bool` — removes by ID, returns whether found
- `iter(&self) -> impl Iterator<Item = &Notification>` — newest-first for rendering
- `render(&mut self, backend: &mut dyn RenderBackend, area: LayoutRect)`:
  1. Downcast to `RatatuiBackend`
  2. Compute corner position (default: top-right, with `NOTIFICATION_MARGIN` from edges)
  3. For each notification (newest first):
     - Calculate height based on text wrapping to `NOTIFICATION_WIDTH` (e.g. 40 cols)
     - Create `Rect` at computed x,y
     - Render `Clear` widget over the rect
     - Render `Block` with borders + `Paragraph` with wrapped text
     - Advance Y by rect.height + `NOTIFICATION_GAP`
     - Stop if next notification would exceed screen height (cap with saturating ops)

## Rendering Logic (detail)

For `NotificationQueue::render`:

```
NOTIFICATION_WIDTH = 40
NOTIFICATION_MARGIN = 2 (x and y from edges)
NOTIFICATION_GAP = 1 (between stacked toasts)

let full_area = layout_rect_to_rect(area);
let mut current_y = NOTIFICATION_MARGIN; // top-right anchor

for notification in self.notifications.iter().rev() {  // newest first
    // Estimate height: every 40 chars wraps, +2 for borders
    let text_width = NOTIFICATION_WIDTH.saturating_sub(2);  // minus borders
    let line_count = estimate_lines(&notification.message, text_width);
    let height = line_count.saturating_add(2);  // +2 for top/bottom borders
    let height = height.min(full_area.height.saturating_sub(current_y + NOTIFICATION_MARGIN));

    if height < 3 { break; }  // too small, stop

    let x = full_area.width.saturating_sub(NOTIFICATION_WIDTH).saturating_sub(NOTIFICATION_MARGIN);
    let rect = Rect { x, y: current_y, width: NOTIFICATION_WIDTH, height };

    // 1. Apply Clear widget to prevent background bleed-through
    Clear.render(rect, buffer);

    // 2. Render styled block
    let block = Block::default()
        .borders(Borders::ALL)
        .border_style(Style::default().fg(notif.style.border))
        .style(Style::default().bg(notif.style.bg).fg(notif.style.fg));
    let paragraph = Paragraph::new(notif.message.as_str())
        .block(block)
        .wrap(Wrap { trim: true });
    paragraph.render(rect, buffer);

    current_y = current_y.saturating_add(height).saturating_add(NOTIFICATION_GAP);
}
```

All coordinate arithmetic uses saturating operations per AGENTS.md requirements.

## Edge Cases Addressed

| Edge Case | Handling |
|-----------|----------|
| Too many notifications | `max_visible` cap (5); oldest dismissed automatically on push |
| Screen too small | Height check before rendering; stop loop if remaining space < 3 rows |
| Saturated coordinates | All `+`/`-` use `saturating_add`/`saturating_sub` on u16 |
| Text overflow | `Paragraph::wrap(Wrap { trim: true })` handles wrapping |
| Background bleed-through | `Clear` widget applied before each toast rect |
| Dismissal after system_task_handle not set | Check `if let Some(handle)` before scheduling; fallback: notification stays until next push |
| Overflow past screen bottom | `height.min(full_area.height - current_y - margin)` clamping |
| Very long messages | Line estimation caps to available height; `Wrap { trim: true }` handles remainder |
| Concurrent pushes | Single-threaded UI loop; no synchronization needed |
