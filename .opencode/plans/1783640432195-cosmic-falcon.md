# Fix: Notification hitbox registration order

## Root Cause

Tiling split handles are registered AFTER notification hitboxes in `render_app`. The hitbox registry uses reverse scan (last registered = highest priority). So tiling handles override notification hitboxes, allowing mouse events to pass through to the tiling layout.

## Fix

Move notification hitbox registration to AFTER the tiling handles section in `src/lib.rs`. Currently it's registered after the window loop but before tiling handles. Move it to after the tiling handles section (after the `render_handles_masked` call).

## File

`src/lib.rs` — move notification hitbox registration block to after tiling handles.
