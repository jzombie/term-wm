# Fix: Render notifications in unified pipeline

## Root Cause

`render_app` skips `RegionType::Notification` at line 90 with `_ => continue`. Notifications are generated but never drawn.

## Fix

### 1. `crates/term-wm-console/src/draw_plan_renderer.rs` — add `render_notification`

Extract notification rendering into a method on `DrawPlanRenderer`:

```rust
pub fn render_notification(
    &self,
    target_buf: &mut Buffer,
    area: Rect,
    msg: &str,
) {
    use ratatui::style::{Color, Style};
    use ratatui::widgets::{Block, Borders, Clear, Paragraph, Widget, Wrap};

    Clear.render(area, target_buf);
    let block = Block::default()
        .borders(Borders::ALL)
        .border_style(Style::default().fg(Color::White))
        .style(Style::default().bg(Color::DarkGray).fg(Color::White));
    Paragraph::new(msg)
        .block(block)
        .wrap(Wrap { trim: true })
        .render(area, target_buf);
}
```

### 2. `src/lib.rs` — unify render loop

Replace `_ => continue` at line 90 with explicit match on all `RegionType` variants:

```rust
for (i, region) in plan_regions.iter().enumerate() {
    let area = layout_rect_to_rect(region.bounds);
    match &region.region_type {
        RegionType::Window(key) => {
            // ... existing window rendering logic (lines 92-153) ...
        }
        RegionType::Notification(msg) => {
            renderer.render_notification(&mut ratatui_backend.buffer, area, msg);
        }
    }
}
```

No ratatui widgets imported in `src/lib.rs` — delegated to console crate.
