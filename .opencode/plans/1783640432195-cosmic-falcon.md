### 1. New: `crates/term-wm-core/src/notification.rs`

Pure data — no Ratatui, no rendering.

```rust
#[derive(Debug, Clone)]
pub struct Notification {
    pub id: u64,
    pub message: String,
}

#[derive(Debug, Default)]
pub struct NotificationQueue {
    notifications: VecDeque<Notification>,
    next_id: u64,
    max_capacity: usize,  // 5
}

impl NotificationQueue {
    pub fn push(&mut self, message: String) -> u64;        // returns id
    pub fn dismiss(&mut self, id: u64) -> bool;
    pub fn renderable(&self) -> impl Iterator<Item = &Notification>;  // newest first
    pub fn len(&self) -> usize;
}
```

`push` evicts oldest when at max capacity.

**Note:** The renderer NEVER accesses `NotificationQueue`. All rendering data is embedded in the `DrawPlan`'s `RenderRegion` during the layout phase.

### 6. Modify: `crates/term-wm-core/src/engine.rs`

In `generate_regions`, after window regions, append self-contained notification regions:

```rust
use textwrap;

const TOAST_W: u16 = 40;
const MARGIN: u16 = 2;
const GAP: u16 = 1;

// Clamp toast width to available terminal space
let managed = wm.managed_area();
let actual_w = TOAST_W.min(managed.width.saturating_sub(MARGIN.saturating_mul(2)));
let inner_w = (actual_w.saturating_sub(2)) as usize;  // minus borders

let mut y_offset = MARGIN;

for notification in wm.notifications().renderable() {
    let lines = textwrap::wrap(&notification.message, inner_w);
    let h = (lines.len() as u16).saturating_add(2);  // +borders
    let h = h.min(managed.height.saturating_sub(y_offset.saturating_add(MARGIN)));
    if h < 3 { break; }

    let x = managed.width.saturating_sub(actual_w).saturating_sub(MARGIN) as i32;

    plan.push(RenderRegion {
        key: WindowKey::default(),
        bounds: LayoutRect { x, y: y_offset as i32, width: actual_w, height: h },
        z_index: 20,                     // above overlays (10), below cursor
        dimmed: false,
        region_type: RegionType::Notification(notification.message.clone()),
    });

    y_offset = y_offset.saturating_add(h).saturating_add(GAP);
}
```

Key details:
- `actual_w` is clamped to `min(40, terminal_width - 4)` — never exceeds buffer bounds
- `y_offset.saturating_add(MARGIN)` — no bare `+` operator (strict saturating arithmetic)
- Message string is **cloned** into the `RegionType` — DrawPlan is fully self-contained

```rust
for region in draw_plan.regions() {
    let area = layout_rect_to_rect(region.bounds);
    match &region.region_type {
        RegionType::Window => { /* existing logic */ }
        RegionType::Notification(msg) => {
            Clear.render(area, target_buf);
            let block = Block::default()
                .borders(Borders::ALL)
                .border_style(Style::default().fg(Color::White))
                .style(Style::default().bg(Color::DarkGray).fg(Color::White));
            Paragraph::new(msg.as_str())
                .block(block)
                .wrap(Wrap { trim: true })
                .render(area, target_buf);
        }
    }
}
```

## Review Compliance

| Flaw | Fix |
|---|---|
| **IR Violation** (renderer queries `wm`) | `RegionType::Notification(String)` embeds the message; renderer uses it directly |
| **Buffer overflow** (TOAST_W > terminal) | `actual_w = min(40, width - 4)` clamps to available space |
| **Bare `+` on y_offset** | `y_offset.saturating_add(MARGIN)` — strict saturating chain |
| **ID erasure** | `RegionType::Notification(String)` carries payload |
| **Y-axis stacking** | Mutable `y_offset` accumulator, not `index * h` |
| **textwrap in wrong crate** | Only in `term-wm-core/Cargo.toml` |
