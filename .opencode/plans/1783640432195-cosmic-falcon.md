# Fix System Panel — Plan

## Issues

1. **Button click broken (two bugs):**
   - **Coordinate bug:** `button_rect` stored in absolute screen coordinates, but `on_mouse_press` receives local (0,0-based) coords. Simple fix: compute button position relative to (0,0).
   - **Action dispatch bug:** `dispatch_mouse` calls `self.process_action(key, action)` which only invokes `comp.update(action, ...)` — a no-op for system-level actions like `SendNotification`. The runner never sees the action.

2. **Panel not scrollable:** Should use `ScrollViewComponent<TextRendererComponent>` like the debug log.

## Fix 1: `dispatch_mouse` action propagation

### `crates/term-wm-core/src/window/window_manager/mod.rs`

Change `dispatch_mouse` return type from `bool` to `EventResult<TermWmAction>`:

```rust
pub fn dispatch_mouse(&mut self, event: &crate::events::WmEvent) -> EventResult<TermWmAction> {
    // ...
    match target {
        HitTarget::Window(key) | HitTarget::Component(key, ..) => {
            // ...
            if let Some(comp) = self.component_for_key_mut(key) {
                let result = comp.handle_events(&core_event, &ctx);
                // Lock capture for subsequent drag/release
                self.mouse_capture = Some(MouseCaptureState::ComponentInteraction { key });
                result  // Return the full EventResult, not just is_consumed()
            } else {
                EventResult::Ignored
            }
        }
        // All other arms: return Consumed or Ignored (no actions from chrome/panels)
        HitTarget::ChromeHeader(..) | HitTarget::ChromeResize(..) => EventResult::Consumed,
        HitTarget::TopPanel | HitTarget::BottomPanel => EventResult::Consumed,
        HitTarget::Overlay(id) => {
            // ...
            EventResult::Consumed  // or Ignored
        }
        HitTarget::LayoutHandle => EventResult::Consumed,
        HitTarget::Notification => EventResult::Consumed,
    }
}
```

### `crates/term-wm-core/src/runner.rs`

Update `handle_focused_app_event` to process mouse actions through `drain_action_queue`:

```rust
if matches!(event, Event::Mouse(_)) {
    if let Some(wm_event) = core_event_to_wm(event) {
        let result = app.wm().dispatch_mouse(&wm_event);
        match result {
            EventResult::Action(action) => {
                let focus_id = app.wm().focused_window();
                let mut queue = VecDeque::from([(focus_id, action)]);
                drain_action_queue(app, &mut queue);
                return true;
            }
            EventResult::Consumed => return true,
            EventResult::Ignored => return false,
        }
    }
    return false;
}
```
