# Fix System Panel Button — Plan

## Root Cause

`handle_events` delegates to `scroll_view.handle_events`, bypassing `on_mouse_press`. Button clicks are never processed.

## Fix

Override `handle_events` to check button clicks first with strict bounds validation:

```rust
fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<TermWmAction> {
    if let Event::Mouse(mouse) = event {
        if let MouseEventKind::Press(MouseButton::Left) = mouse.kind {
            if let Some(screen_area) = ctx.screen_area() {
                // Strict bounds check — no clamping, no phantom clicks
                if mouse.column >= screen_area.x
                    && mouse.column < screen_area.x.saturating_add(screen_area.width)
                    && mouse.row >= screen_area.y
                    && mouse.row < screen_area.y.saturating_add(screen_area.height)
                {
                    // Safe: bounds proven, saturating_sub cannot underflow
                    let local_x = mouse.column.saturating_sub(screen_area.x);
                    let local_y = mouse.row.saturating_sub(screen_area.y);
                    let result = self.on_mouse_press(local_x, local_y, MouseButton::Left, mouse.modifiers, ctx);
                    if !result.is_ignored() {
                        return result;
                    }
                }
            }
        }
    }
    self.scroll_view.handle_events(event, ctx)
}
```
