# Debug: Confirm push_notification is called

## Add one tracing line to `drain_action_queue`

In `crates/term-wm-core/src/runner.rs`, inside the `SendNotification` arm:

```rust
TermWmAction::SendNotification(msg) => {
    tracing::info!("drain_action_queue: SendNotification({})", msg);
    app.wm()
        .push_notification(msg, std::time::Duration::from_secs(3));
}
```

This will confirm whether the action reaches `drain_action_queue` and whether `push_notification` is called.

If this line appears in the debug log, `push_notification` IS being called and the issue is in the rendering pipeline.
If this line does NOT appear, the action is lost before reaching `drain_action_queue`.
