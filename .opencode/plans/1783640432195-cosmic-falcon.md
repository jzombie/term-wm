# Fix: Text selection — return Consumed on Press (gated)

## Fix

`crates/term-wm-core/src/utils/selectable_text.rs`, line 307-316:

```rust
MouseEventKind::Press(MouseButton::Left) => {
    if rect_contains(area, mouse.column, mouse.row)
        && let Some(pos) = host.logical_position_from_point(area, mouse.column, mouse.row)
    {
        let selection = host.selection_controller();
        selection.set_pointer(mouse.column, mouse.row);
        selection.prepare_drag(pos);
        return true;
    }
    false
}
```

Only returns `true` when the click is inside the area AND drag state is prepared. Returns `false` if outside bounds.
