## Fix 2: System panel — button inside scroll view

### `wm_system_panel.rs`

**Button is part of the scrollable content.** The TextRendererComponent renders styled text lines that look like a button. Hit-testing accounts for scroll offset.

**Render flow:**
1. Build text content: info lines + button lines (styled to look like a button) + hint lines
2. Set text on TextRendererComponent
3. Delegate to `self.scroll_view.render(backend, area, ctx, registry)`
4. Cache button virtual position (row in the full content, e.g., row 4)

**Hit-test flow:**
```rust
fn on_mouse_press(&mut self, local_x: u16, local_y: u16, button: MouseButton, ...) {
    if button != MouseButton::Left { return Ignored; }
    let Some(btn) = self.button_rect.get() else { return Ignored; };
    
    // Fetch scroll offset from the scroll view
    let scroll_offset = self.scroll_view.scroll_handle().info().offset_y as u16;
    
    // Translate local viewport Y to virtual content Y
    let virtual_y = local_y.saturating_add(scroll_offset);
    
    // Strict u16 hit-test
    if local_x >= btn.x
        && local_x < btn.x.saturating_add(btn.width)
        && virtual_y >= btn.y
        && virtual_y < btn.y.saturating_add(btn.height)
    {
        return Action(SendNotification("Hello from System Panel!".into()));
    }
    Ignored
}
```

**Button rendered as styled text:**
```
  ╭─────────────────────╮
  │  Send Notification  │
  ╰─────────────────────╯
```

Three lines of styled text (border top, label, border bottom) with cyan fg on dark bg. The `button_rect` stores the virtual position of these 3 lines.

**Key constraint:** All coordinates are `u16`. No `i32` casts in hit-test math. Use `saturating_add` for bounds.
