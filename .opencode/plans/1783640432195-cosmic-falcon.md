# Fix: Notification rendering — unified loop, scoped borrow

## Fix

### 1. `draw_plan_renderer.rs` — change `render_notification` signature

Accept `&mut dyn RenderBackend` and downcast internally:

```rust
pub fn render_notification(
    &self,
    backend: &mut dyn term_wm_render::RenderBackend,
    area: Rect,
    msg: &str,
) {
    use ratatui::style::{Color, Style};
    use ratatui::widgets::{Block, Borders, Clear, Paragraph, Widget, Wrap};

    let Some(rb) = backend.as_any_mut().downcast_mut::<RatatuiBackend>() else {
        return;
    };
    let buf = &mut rb.buffer;
    Clear.render(area, buf);
    let block = Block::default()
        .borders(Borders::ALL)
        .border_style(Style::default().fg(Color::White))
        .style(Style::default().bg(Color::DarkGray).fg(Color::White));
    Paragraph::new(msg)
        .block(block)
        .wrap(Wrap { trim: true })
        .render(area, buf);
}
```

### 2. `src/lib.rs` — unified loop with scoped notification arm

Keep the single `for region in plan_regions.iter()` loop. Replace `_ => continue` with:

```rust
for (i, region) in plan_regions.iter().enumerate() {
    match &region.region_type {
        RegionType::Window(key) => {
            // ... existing window rendering (unchanged) ...
        }
        RegionType::Notification(msg) => {
            let area = layout_rect_to_rect(region.bounds);
            renderer.render_notification(backend, area, msg);
        }
    }
}
```

Borrow of `backend` is scoped to the `render_notification` call — released before next iteration.
