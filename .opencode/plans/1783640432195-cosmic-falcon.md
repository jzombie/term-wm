# System Panel Component — Plan

## Overview

Add a `WmSystemPanelComponent` that can be activated from the command menu. Inside it, a button pushes a test notification via the existing `NotificationQueue`.

Follows the exact same pattern as the debug log: persistent system window, toggled visible/hidden, never destroyed.

## Rendering Detail

The button is rendered as a centered `Block` with `Borders::ALL` and a title. The button area is cached in `self.button_rect` during render so that `on_mouse_press` can check if the click landed on it.

```
┌─────────────────────────────┐
│       System Panel          │
│                             │
│   ┌─────────────────────┐   │
│   │  Send Notification  │   │  ← button_rect cached here
│   └─────────────────────┘   │
│                             │
└─────────────────────────────┘
```

The button rect is computed during render using `LayoutRect` arithmetic:
- Centered horizontally: `x = area.x + (area.width - button_w) / 2`
- Vertically offset: `y = area.y + 4` (below title)
- Fixed size: `width = 20, height = 3`

On `on_mouse_press`, the component checks if `(local_x, local_y)` falls within the button bounds. If so, it returns `EventResult::Action(TermWmAction::SendNotification(...))`.
