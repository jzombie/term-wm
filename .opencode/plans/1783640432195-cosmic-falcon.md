# Fix: Signed-safe local conversion in logical_position_from_point

Replace `logical_position_from_point` in `TextRendererComponent` to do math in `i32` space before downcasting, matching the pattern used by the default `Component::handle_events`.

### `crates/term-wm-ui-components/src/text_renderer.rs`

```rust
fn logical_position_from_point(
    &mut self,
    area: LayoutRect,
    column: u16,
    row: u16,
) -> Option<LogicalPosition> {
    if area.width == 0 || area.height == 0 {
        return None;
    }
    // Signed math: lift to i32, compute offset, clamp negative to 0
    let local_x = (i32::from(column) - area.x).max(0) as u16;
    let local_y = (i32::from(row) - area.y).max(0) as u16;
    let state = self.viewport_cache.get();
    Some(LogicalPosition::new(
        state.offset_y.saturating_add(local_y as usize),
        state.offset_x.saturating_add(local_x as usize),
    ))
}
```

Removes `layout_rect_to_rect` and `logical_position_from_point_impl` indirection.
