# Fix: Route system actions in drain_action_queue

## Root Cause

`drain_action_queue` forwards ALL actions to `comp.update(action, ...)`. System-level actions like `SendNotification` are unknown to components and silently dropped.

## Fix

### `crates/term-wm-core/src/runner.rs`

Modify `drain_action_queue` to intercept system actions before component dispatch:

```rust
fn drain_action_queue<A: WindowManagerHost>(
    app: &mut A,
    queue: &mut VecDeque<(WindowKey, TermWmAction)>,
) {
    while let Some((key, action)) = queue.pop_front() {
        // Route system-level actions to the WM, not to components
        match action {
            TermWmAction::SendNotification(msg) => {
                app.wm().push_notification(msg, std::time::Duration::from_secs(3));
                continue;
            }
            action => {
                let ctx = app.wm().component_context_for(true, key);
                if let Some(comp) = app.wm().component_for_key_mut(key) {
                    comp.update(action, &ctx, queue);
                }
            }
        }
    }
}
```

This works for ALL dispatch paths (mouse, keyboard, programmatic) because they all go through `drain_action_queue`.
