# Plan: Refactor Overlay System from BTreeMap to SlotMap

## Summary

Replace `BTreeMap<OverlayId, Box<dyn Overlay<TermWmAction>>>` with `SlotMap<OverlayKey, Box<dyn Overlay<TermWmAction>>>`, eliminating the `OverlayId` enum entirely and storing typed `Option<OverlayKey>` fields for identity lookup.

## Rationale

Windows already use `SlotMap<WindowKey, Window>` with generational safety. Overlays should follow the same pattern. Using a `SlotMap` with a custom key type provides:
- Generational keys that make stale-key access impossible
- No fixed enum — extensible without enum variant changes
- Consistent with the existing `WindowKey` / slotmap pattern

pub fn handle_help_event(&mut self, event: &Event) -> bool {
    let Some(key) = self.help_key else { return false; };
    if let Event::Mouse(mouse) = event {
        self.hover = Some((mouse.column, mouse.row));
    }
    let ctx = self
        .component_context(true)
        .with_overlay(true)
        .with_screen_area(self.managed_area());
    let Some(boxed) = self.overlays.get_mut(key) else { return false; };

    let was_dragging = boxed.selection_status().dragging;
    let result = boxed.handle_events(event, &ctx);
    let was_handled = !result.is_ignored();

    if let EventResult::Action(action) = result {
        let mut queue = std::collections::VecDeque::new();
        boxed.update(action, &ctx, &mut queue);
        while let Some((_key, _action)) = queue.pop_front() {}
    }

    let status = boxed.selection_status();
    let still_visible = boxed.visible();
    let text = if status.active || status.dragging {
        boxed.selection_text()
    } else {
        None
    };

    self.set_selection_snapshot(status.active, status.dragging, text);
    if was_dragging && !status.dragging && status.active {
        self.copy_selection_to_clipboard();
    }

    if !still_visible {
        self.help_key = None;
        self.overlays.remove(key);
    }
    was_handled
}

pub fn handle_exit_confirm_event(&mut self, event: &Event) -> Option<ConfirmAction> {
    let key = self.exit_confirm_key?;
    if let Event::Mouse(mouse) = event {
        self.hover = Some((mouse.column, mouse.row));
    }
    let comp = self.overlays.get_mut(key)?;
    let overlay: &mut dyn Overlay<TermWmAction> = &mut **comp;
    overlay.handle_confirm_event(event)
}

pub fn handle_command_palette_event(&mut self, event: &Event) -> Option<TermWmAction> {
    if let Event::Mouse(mouse) = event {
        self.hover = Some((mouse.column, mouse.row));
    }
    let ctx = self
        .component_context(false)
        .with_overlay(true)
        .with_screen_area(self.managed_area())
        .with_hover_pos(self.hover);

    let key = self.command_palette_key?;
    let palette = self.overlays.get_mut(key)?;
    match palette.handle_events(event, &ctx) {
        EventResult::Action(action) => {
            self.close_command_palette();
            Some(action)
        }
        _ => None,
    }
}

## Edge Cases & Risks

- **`handle_help_event` borrow pattern** — The `help_key` is copied (OverlayKey is `Copy` due to `new_key_type!`) before `overlays.get_mut(key)` to avoid simultaneous mutable borrow conflicts. The same key value is reused for the removal check at the end.
- **Event dispatch ordering** — The `runner.rs` event dispatch checks `exit_confirm_visible()` before `help_overlay_visible()` before `command_palette_visible()`. This ordering is preserved since the typed helper methods are unchanged.
- **`render_overlays` borrow pattern** — Keys are collected into a `Vec<OverlayKey>` first, then iterated with `get_mut`, avoiding borrow conflicts. Same pattern as the current code which collects `Vec<OverlayId>`.
- **`SelectionPreview` (unused variant)** — The `OverlayId::SelectionPreview` variant is dead code; removing it with the enum is safe. No callers reference it.
- **SlotMap key semantics** — `SlotMap::with_capacity(0)` allocates minimally; slotmap grows dynamically like BTreeMap. Performance is comparable or better.
