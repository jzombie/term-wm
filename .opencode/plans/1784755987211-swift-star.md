# Plan: Fix Remote OSC 52 Clipboard Copy

## Bug

`crates/term-session-client/src/lib.rs:262` calls `extract_osc52_text(&current_content)` where
`current_content` is `parser.screen().contents_formatted()` — the **rendered screen buffer**.
The VT100 parser consumes and discards OSC sequences internally, so they never appear in
`contents_formatted()`. The extraction always returns `None`.

The local PTY path (`pty.rs:539`) already handles this correctly: `Osc52Extractor` intercepts
raw bytes **before** they enter the VT100 parser.

## Changes — Single File: `crates/term-session-client/src/lib.rs`

### 1. Update import (line 26)

```
- use term_wm_pty_engine::clipboard::{Clipboard, extract_osc52_text};
+ use term_wm_pty_engine::clipboard::{Clipboard, Osc52Extractor};
```

### 2. Add clipboard side-channel (after line 153)

After `let (push_tx, push_rx) = ...`, add:

```rust
// Side-channel for intercepted OSC 52 clipboard text
let (clip_tx, mut clip_rx) = mpsc::unbounded_channel::<String>();
```

### 3. Intercept raw bytes in async spawn block (lines 177-186)

Replace the current `rt.spawn` block with one that runs `Osc52Extractor::push()` on each chunk
before forwarding to `push_tx`:

```rust
rt.spawn(async move {
    let mut osc52 = Osc52Extractor::new();
    let mut prev_tail: [u8; 8] = [0; 8];

    while let Some(chunk) = reader.recv().await {
        match chunk {
            Ok(data) => {
                // Extract OSC 52 from the raw byte stream (before VT100 parse)
                if let Some(text) = osc52.push(&data, &prev_tail) {
                    let _ = clip_tx.send(text);
                }

                // Maintain history tail for cross-chunk OSC 52 header detection
                let n = data.len();
                if n >= 8 {
                    prev_tail.copy_from_slice(&data[n - 8..n]);
                } else if n > 0 {
                    prev_tail.rotate_left(n);
                    prev_tail[8 - n..].copy_from_slice(&data[..n]);
                }

                let _ = push_tx.send(data);
            }
            Err(_) => break,
        }
    }
});
```

### 4. Replace OSC 52 extraction in main loop (lines 261-264)

Remove the old `extract_osc52_text(&current_content)` block and drain the side-channel instead:

```rust
// Drain any clipboard texts intercepted by the reader task
while let Ok(text) = clip_rx.try_recv() {
    let _ = clipboard.set(&text);
}
```

The `has_new_data` / `prev_content` diff logic above remains unchanged — it is still needed for
incremental rendering.
