# Plan: Extract Pane + Pty to New Standalone Crate

## Motivation

Extract the terminal pane management layer (Pane trait + Pty implementation) from `term-wm-core` into its own crate. This is a clean extraction with no behavioral changes — the boundary is already well-defined by the `Pane` trait. Enables future session-resume features.

## Crate Name

**`term-wm-pty-engine`** — contains the PTY pane engine (Pane trait + Pty implementation).

## What Moves

From `term-wm-core/src/` to new crate:

| File | Content | LOC |
|------|---------|-----|
| `pane.rs` | `Pane` trait + `impl Pane for Pty` delegation | 87 |
| `pty.rs` | `Pty` struct + `read_loop` + `get_process_name` + tests | 600 |
| `io/clipboard.rs` | `extract_osc52_text`, `try_set`, `format_osc52_bytes`, `Clipboard` struct | 393 |
| `io/title.rs` | `extract_osc_title` | 80 |

Note: `Clipboard` struct (with `arboard`) is used by `WindowManager` — it moves with the rest since it's in clipboard.rs and `extract_osc52_text`/`try_set` are the pieces pty.rs needs. `WindowManager` would import from the new crate.

## What Stays in term-wm-core

- `io/mod.rs` — keeps `console`, `event_source`, `render_target`, `utils`
- `io/console.rs`, `io/event_source.rs`, `io/render_target.rs`, `io/utils/` — not related to PTY
- Everything else (21 modules minus the extracted 4)

## New Crate: `term-wm-pty-engine`

### Dependencies (from root workspace)
```
portable-pty, vt100, arboard, base64, thiserror, tracing, libc
```

### Module Structure
```
pane/
├── Cargo.toml
├── src/
│   ├── lib.rs          # Re-exports Pane, Pty, PtyResult, Clipboard types, helper fns
│   ├── pane.rs         # Pane trait (moved verbatim)
│   ├── pty.rs          # Pty struct (moved verbatim)
│   ├── clipboard.rs    # clipboard helpers (moved from term-wm-core/io/clipboard.rs)
│   └── title.rs        # title extraction (moved from term-wm-core/io/title.rs)
```

`lib.rs` would be minimal:
```rust
pub mod clipboard;
pub mod pane;
pub mod pty;
pub mod title;

pub use pane::Pane;
pub use pty::{Pty, PtyResult, PtySize};
```

## Changes to term-wm-core

1. **Remove modules**: `pane.rs`, `pty.rs`, `io/clipboard.rs`, `io/title.rs`
2. **Update `lib.rs`**: Remove `pub mod pane`, `pub mod pty`, update `io/mod.rs` to remove clipboard/title
3. **Add dependency**: `term-wm-pty-engine = { workspace = true }` in Cargo.toml
4. **Add re-exports** (optional, for backward compat): `pub use term_wm_pty_engine::Pane;` etc.
5. **Update `window/window_manager/mod.rs`**: Change `crate::io::clipboard::Clipboard` → `term_wm_pty_engine::clipboard::Clipboard`

## Execution Order

1. Create `crates/term-wm-pty-engine/` with `Cargo.toml` and `src/lib.rs`
2. Copy `pane.rs`, `pty.rs` into new crate, update imports (`crate::io::clipboard` → `crate::clipboard`, etc.)
3. Copy `io/clipboard.rs`, `io/title.rs` into new crate
4. Update root `Cargo.toml` (workspace members + dep entry)
5. Remove `pane.rs`, `pty.rs`, `io/clipboard.rs`, `io/title.rs` from term-wm-core
6. Update `term-wm-core/src/lib.rs` and `io/mod.rs`
7. Update `WindowManager` clipboard import in `window/window_manager/mod.rs`
8. Add `term-wm-pty-engine` dep to term-wm-ui-components, update import in `terminal.rs`
9. Push `cargo check --workspace` to fix any remaining import errors

## Verification

```bash
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test
```

No behavioral changes expected — this is a pure extraction. If tests pass and clippy is clean, the extraction is complete.

## Future: Session Resume (not part of this plan)

When ready to add session resume, the `term-wm-pty-engine` crate would gain:
- `Session { id, panes: Vec<Box<dyn Pane>>, layout_json: String }`
- `SessionManager { sessions, save_path }`
- Save/load from file
- "Detach" = leave children running (don't call `kill_child`), save session state
- "Attach" = reconnect to orphaned PTYs (needs `portable-pty` capability for reopening master)
