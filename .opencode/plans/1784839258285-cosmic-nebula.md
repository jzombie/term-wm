# Plan: Move System Window Management into `TermWmApp`

## Goal

Encapsulate debug log and system panel window orchestration into `TermWmApp`, eliminating the ~60 lines of repetitive setup currently in `main.rs`'s `App` struct.

## Changes

### 1. `src/term_wm_app.rs` — Extend `TermWmApp`

**a. Add fields** to `TermWmApp` struct:
```rust
pub struct TermWmApp {
    wm: WindowManager<AppRootComponent, LayerComponent, OverlayComponent>,
    debug_key: Option<WindowKey>,
    system_panel_key: Option<WindowKey>,
    should_quit: bool,
    engine: CoreEngine,
    draw_renderer: DrawPlanRenderer,
}
```

**b. Update `from_wm`** to initialize new fields to `None`.

**c. Add new imports:**
- `use term_wm_core::debug_log::set_global_debug_log;`
- `use term_wm_core::window::{ClosePolicy, WindowState};`
- `use term_wm_sys_ui_components::wm_debug_log::{WmDebugLogComponent, install_panic_hook};`
- `use term_wm_sys_ui_components::WmSystemPanelComponent;`
- `use term_wm_ui_facade::core_component::CoreWmComponent;`
- `use crate::logging;`

**d. Add `init_system_windows(&mut self)` method** (no feature gate needed — same as existing unconditional `new` paths):
- **Idempotency guard:** `if self.debug_key.is_some() || self.system_panel_key.is_some() { return; }` at the top.
- Create `WmDebugLogComponent` via `new_default()`, set selection enabled, set global handle
- Create window via `self.wm.create_window(AppRootComponent::Core(CoreWmComponent::DebugLog(...)))` with `ClosePolicy::Unmap`, transition to `Unmapped`, set title
- Call `install_panic_hook()` and `logging::init_default()`
- Create `WmSystemPanelComponent`, create window with `ClosePolicy::Unmap`, transition to `Unmapped`, set title
- Store keys in `self.debug_key` / `self.system_panel_key`

**e. Add `WindowManagerHost` trait methods — use `focus_window_key` (not `set_focus`):**
- `on_panic` — show and focus debug window via `focus_window_key(key)`
- `toggle_debug_window` — toggle Mapped/Unmapped, use `focus_window_key(key)` when showing
- `toggle_system_panel` — toggle Mapped/Unmapped, use `focus_window_key(key)` when showing

> **Note:** `focus_window_key(key)` calls `bring_to_front_key(key)` + `mark_layout_dirty()` internally, ensuring correct Z-order elevation. Do NOT use `set_focus(key)` which only updates the focus ring without elevating the window.

Keep existing trait methods (`wm`, `quit_requested`, `render`, `open_command_palette`, `open_help_overlay`, `open_exit_confirm`) unchanged.

### 2. `src/main.rs` — Simplify `App`

**a. Remove fields** `debug_key` and `system_panel_key` from `App`.

**b. Remove imports** no longer needed:
- `WmDebugLogComponent`, `install_panic_hook`, `set_global_debug_log`
- `WmSystemPanelComponent`
- `CoreWmComponent`
- Possibly `WindowKey` if no longer used directly (it may still be needed for terminal creation)

**c. Simplify `new_with`:**
- Replace the two system window initialization blocks with a single call: `app.inner.init_system_windows();`
- Note: `init_system_windows()` must be called AFTER `from_wm()` since it needs the WM.
- Remove `app.debug_key = None` / `app.system_panel_key = None` from the struct literal.

**d. Simplify `WindowManagerHost` impl for `App`:**
- Replace `on_panic`, `toggle_debug_window`, `toggle_system_panel` with simple delegations:
```rust
fn on_panic(&mut self) { self.inner.on_panic(); }
fn toggle_debug_window(&mut self) { self.inner.toggle_debug_window(); }
fn toggle_system_panel(&mut self) { self.inner.toggle_system_panel(); }
```

Keep `wm_new_window`, `wm`, `open_help_overlay`, `open_exit_confirm`, `open_command_palette`, `set_window_selection_enabled`, `set_clipboard_enabled`, `render`, `spawn_terminal_with_command`, and `pty_wakeup_tx` unchanged.

### 3. `src/components.rs` / `src/lib.rs` (if needed)

No changes expected — `AppRootComponent` and re-exports are fine. But verify after compilation.

## Files Modified

- `src/main.rs` — simplify `App` struct and `WindowManagerHost` impl
- Possibly `src/components.rs` if any re-exports need updating
