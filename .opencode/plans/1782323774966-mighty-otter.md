# term-wm Refactoring Plan

## Goals
1. Make term-wm a **generic, easy-to-use window manager** for terminal apps
2. **Clear naming** — no App/WindowManaged confusion
3. **Modular architecture** — no 3623-line god objects
4. **Declarative config** — feature flags, not mode enums
5. **No if/then hell** — each sub-manager owns its config check, centralized in one place

---

## Core Design: Feature Flags, Not Modes

**Old approach:** `LayoutContract` enum (`AppManaged`/`WindowManaged`) checked in 23 places.

**New approach:** `WmConfig` struct with individual feature flags. `standalone()` / `embedded()` are just preset builder methods, not enum variants. Each sub-manager checks its own flags.

```rust
pub struct WmConfig<Id> {
    // Feature flags (each independently toggleable)
    pub chrome_enabled: bool,
    pub floating_windows_enabled: bool,
    pub panel_enabled: bool,
    pub wm_overlay_enabled: bool,
    pub esc_passthrough: bool,

    // Window control callbacks (only rendered if Some)
    pub on_minimize: Option<Box<dyn FnMut(Id)>>,
    pub on_maximize: Option<Box<dyn FnMut(Id)>>,
    pub on_close: Option<Box<dyn FnMut(Id)>>,

    // Misc options
    pub clipboard_enabled: bool,
    pub mouse_capture_enabled: bool,
    pub scroll_keyboard_enabled_default: bool,
    pub floating_resize_offscreen: bool,
    pub decorator: Option<Arc<dyn WindowDecorator>>,
}

impl<Id: Copy + Eq + Ord> WmConfig<Id> {
    pub fn standalone() -> Self { /* chrome + floating + panel + overlay enabled */ }
    pub fn embedded() -> Self { /* all disabled, esc passthrough */ }
}

impl<Id: Copy + Eq + Ord> Default for WmConfig<Id> {
    fn default() -> Self { Self::standalone() }
}
```

No `WindowManagerMode` enum exists anywhere. The 23 `if layout_contract == ...` checks are replaced by each sub-manager checking its own flag in its own method (one check each, not 23).

## Phase 1: Naming Overhaul (Purely Mechanical)

| Old | New | Reason |
|-----|-----|--------|
| `LayoutContract` | **Deleted** | Replaced by `WmConfig` feature flags |
| `HasWindowManager<W, R>` | `WindowManagerHost<Id>` | "HasWindowManager" is vague |
| `WindowApp<W, R>` | `WindowProvider<Id>` | "WindowApp" conflates app+WM |
| `AppWindowDraw<R>` | `WindowDrawContext<Id>` | Unclear name |
| `WindowDrawTask<R>` | `DrawTask<Id>` | More concise |
| `AppState` | `WmState` | Not app state, it's WM state |

`LayoutContract` is **removed entirely** — no rename, no replacement enum.

---

## Phase 2: Collapse Dual Generics `<W, R>` → Single `<Id>`

**Problem:** `WindowManager<W, R>` requires `WindowManager<PaneId, PaneId>` in all real usage. Redundant.

**Change:** `WindowManager<W: Copy + Eq + Ord, R: Copy + Eq + Ord>` → `WindowManager<Id: Copy + Eq + Ord>`

**Knock-on effects:**
- `FocusRing<W>` → `FocusRing<Id>`, `FocusRing<WindowId<R>>` → `FocusRing<WindowId<Id>>`
- All methods drop one generic
- `WindowManagerHost<Id>`, `WindowProvider<Id>` (not `<W, R>`)
- `run_window_app` drops `FFocus` param (was `Fn(W) -> Option<R>`, always `Some`)
- `run_app` drops `FFocus` generic param

---

## Phase 3: Split WindowManager → Sub-Managers + Remove Mode Checks

**Problem:** `WindowManager` is 3623 lines, 30+ fields, with 23 `LayoutContract` checks.

**Solution:** Extract sub-managers, each taking relevant config from `WmConfig`:

| New File | Extracts | Own Config It Checks |
|----------|----------|----------------------|
| `src/window/focus_ring.rs` | `FocusRing<T>` struct | (no config needed) |
| `src/window/focus_manager.rs` | App/WM focus logic | (no config needed) |
| `src/window/layout_engine.rs` | Tiling layout, regions, draw order | `floating_windows_enabled` |
| `src/window/sys_windows.rs` | Debug log window lifecycle | (no config needed) |
| `src/window/overlays.rs` | Help, exit confirm, selection preview | `wm_overlay_enabled` |
| `src/window/drag.rs` | Header drag + resize drag | `floating_windows_enabled` |
| `src/window/chrome.rs` | Decorator integration, compositing | `chrome_enabled`, `on_minimize/maximize/close` |
| `src/window/panel_manager.rs` | Panel/status bar | `panel_enabled` |
| `src/window/window.rs` | `Window` struct (from mod.rs) | (no config needed) |

**How mode checks are eliminated:**
- `chrome_enabled? ChromeManager::render_window()` renders decorator + calls content callback. If disabled, just calls content callback directly (no decorator). One check.
- `panel_enabled? PanelManager::render()` renders or no-ops. One check.
- `floating_windows_enabled? DragManager` initializes with floating enabled/disabled. Methods early-return if disabled. One check per method.
- `wm_overlay_enabled? OverlayManager` creates overlay or not. One check.
- `esc_passthrough?` Handled in runner — if true, Esc goes to app instead of toggling overlay. One check.

**Sub-managers receive config at construction:**
```rust
fn register_managed_layout(&mut self, area: Rect) {
    self.layout.register(self.managed_area);
    // No mode checks — ChromeManager already knows if chrome is enabled
    // No mode checks — PanelManager already knows if panel is visible
    // No mode checks — DragManager already knows if floating is enabled
}
```

## Phase 4: WmConfig + Builder Pattern

**Replace imperative setup** (`new_managed()` + `set_focus()` + `tile_window()` + `set_window_title()`...):

```rust
// Construction
let config = WmConfig::standalone();  // preset
// or customized:
let config = WmConfig {
    floating_windows_enabled: false,
    on_close: Some(Box::new(|id| { /* custom close */ })),
    ..WmConfig::standalone()
};
let wm = WindowManager::with_config(0, config);
```

Window controls (minimize/maximize/close buttons in titlebar) only render if their callback is `Some`. `standalone()` preset provides default close behavior via `on_close`.

---

## Phase 5: Better Default UX

**Simplify `run_window_app` signature:** 7 params → 3:
```rust
// Before:
run_window_app(output, input, app, focus_regions, map_region, map_focus)
// After:
run_window_app(output, input, app)  // enumerate_windows() provides window list
```

**Simplify traits:** `WindowProvider::window_component()` is the main hook. `enumerate_windows()` can auto-detect from focus order. Fewer required methods.
