# Plan: Unify all overlays under Help Menu architecture

## Scope
Promote `WmCommandPaletteComponent` and `WmMenuOverlay` from WmComponent/ComponentTag system to first-class global overlays in `WindowManager` and `runner.rs`, matching `WmHelpOverlayComponent` exactly.

## Changes

### 1. WindowManager overlay methods (`overlays.rs`)
Add `command_palette_visible()`, `handle_command_palette_event()`, `menu_overlay_visible()`, `handle_menu_overlay_event()` — exact copies of the Help overlay pattern.

### 2. Runner sinks (`runner.rs`)
Replace the entire `if wm_mode && app.wm().command_menu_visible() { ... }` block (lines 368-496) with three unconditional sink blocks:

```rust
if app.wm().command_palette_visible() {
    let _ = app.wm().handle_command_palette_event(&evt);
    update_selection_snapshot(app);
    return flush_state_changes(app, ControlFlow::Continue, false);
}
if app.wm().menu_overlay_visible() {
    let _ = app.wm().handle_menu_overlay_event(&evt);
    update_selection_snapshot(app);
    return flush_state_changes(app, ControlFlow::Continue, false);
}
```

This removes `wm_menu_consumes_event`, `handle_wm_menu_event`, `fold_menu`, and all inline action handling (CloseWindow, NewWindow, etc.). The command palette's action processing moves to `handle_command_palette_event`.

### 3. Component struct parity (`wm_command_palette.rs`, `wm_menu_overlay.rs`)
- Strip all `WmComponent` trait implementations
- Strip `WmComponent` from imports
- Add `area: Cell<LayoutRect>` field
- Add `show()`/`close()`/`visible()` methods matching Help Menu
- In `render()`: add `self.area.set(area);`
- In `handle_events()`: match Help Menu exact pattern — check `dialog.visible()`, use cached area, `handle_click_outside`, delegate to inner content

### 4. Event processing parity (`wm_command_palette.rs`)
Both components implement `handle_events` exactly like `WmHelpOverlayComponent`:

```rust
fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<TermWmAction> {
    if !self.dialog.visible() {
        return EventResult::Ignored;
    }
    match event {
        Event::Key(key) => {
            // Close on Escape (or custom close action)
            self.content.handle_events(event, ctx)
        }
        Event::Mouse(_) => {
            let area = self.area.get();
            let ratatui_area = layout_rect_to_rect(area);
            if self.dialog.handle_click_outside(event, ratatui_area) {
                self.close();
                return EventResult::Consumed;
            }
            let rect = self.dialog.rect_for(ratatui_area);
            let inner = LayoutRect {
                x: rect.x as i32,
                y: rect.y as i32,
                width: rect.width,
                height: rect.height,
            };
            let ctx = ctx.with_screen_area(inner);
            self.content.handle_events(event, &ctx)
        }
        _ => EventResult::Ignored,
    }
}
```

**Command palette override:** The `CloseMenu` action on Escape is handled by the inner `CommandPaletteComponent`. Return `Action(CloseMenu)` instead of `Consumed` when outside click fires (since runner needs the action to disengage command mode).

### 5. Registration migration
Update `term_wm_app.rs`/`main.rs` to register the command palette and menu overlay via `WindowManager::open_overlay(OverlayId::CommandPalette, ...)` instead of `AppBuilder::command_menu(...)` and `ComponentTag::CommandPalette`.

Remove `ComponentAction::SetMenuItems`, `ComponentAction::SetManagedArea` from the LayerManager render path in `draw_plan_renderer.rs`.
