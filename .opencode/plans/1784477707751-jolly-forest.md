# Plan: Unify all overlays under Help Menu architecture

## Design
- `WindowManager` overlay handlers return `EventResult<TermWmAction>` (not bool)
- Runner captures the result, processes actions at the host level, then always consumes with `flush_state_changes`
- The complex `if wm_mode && app.wm().command_menu_visible()` block is replaced by overlay sinks

## Changes

### 1. Runner sinks (`runner.rs`)
Replace the `if wm_mode && app.wm().command_menu_visible() { ... }` block (lines 368-496) with:

```rust
if app.wm().exit_confirm_visible() {
    if let Some(action) = app.wm().handle_exit_confirm_event(&evt) {
        match action {
            ConfirmAction::Confirm => return Ok(ControlFlow::Quit),
            ConfirmAction::Cancel => app.wm().close_exit_confirm(),
        }
    }
    update_selection_snapshot(app);
    return flush_state_changes(app, ControlFlow::Continue, false);
}

if app.wm().help_overlay_visible() {
    let _ = app.wm().handle_help_event(&evt);
    update_selection_snapshot(app);
    return flush_state_changes(app, ControlFlow::Continue, false);
}

if app.wm().command_palette_visible() {
    if let EventResult::Action(action) = app.wm().handle_command_palette_event(&evt) {
        match action {
            TermWmAction::CloseMenu => { app.wm().close_command_palette(); }
            // MenuSelect is unwrapped by handle_command_palette_event — the actual
            // action (NewWindow, CloseWindow, etc.) arrives directly
            TermWmAction::ToggleDebugWindow => { app.toggle_debug_window(); app.wm().close_command_palette(); }
            TermWmAction::ToggleSystemPanel => { app.toggle_system_panel(); app.wm().close_command_palette(); }
            TermWmAction::Help => { app.open_help_overlay(); app.wm().close_command_palette(); }
            TermWmAction::OpenHelp => { app.open_help_overlay(); app.wm().close_command_palette(); }
            TermWmAction::Quit => { app.open_exit_confirm(); }
            TermWmAction::ExitUi => { app.open_exit_confirm(); }
            TermWmAction::CloseWindow => { app.wm().close_window(app.wm().focused_window()); app.wm().close_command_palette(); }
            TermWmAction::NewWindow => { app.wm_new_window()?; app.wm().close_command_palette(); }
            TermWmAction::MinimizeWindow => { app.wm().minimize_window(app.wm().focused_window()); app.wm().close_command_palette(); }
            TermWmAction::MaximizeWindow => { app.wm().toggle_maximize(app.wm().focused_window()); app.wm().close_command_palette(); }
            TermWmAction::ToggleDirectMode => { app.wm().toggle_direct_mode(app.wm().focused_window()); app.wm().close_command_palette(); }
            TermWmAction::ToggleMouseCapture => { app.wm().toggle_mouse_capture(); }
            TermWmAction::ToggleClipboardMode => { app.wm().toggle_clipboard_enabled(); }
            TermWmAction::ToggleWindowSelection => { app.wm().toggle_window_selection(); }
            TermWmAction::SendNotification(msg) => { app.wm().push_notification(msg, std::time::Duration::from_secs(3)); app.wm().close_command_palette(); }
            _ => {}
        }
    }
    update_selection_snapshot(app);
    return flush_state_changes(app, ControlFlow::Continue, false);
}

if app.wm().menu_overlay_visible() {
    if let EventResult::Action(action) = app.wm().handle_menu_overlay_event(&evt) {
        // Same action dispatch as command palette above
        match action {
            TermWmAction::ToggleDebugWindow => { app.toggle_debug_window(); }
            TermWmAction::ToggleSystemPanel => { app.toggle_system_panel(); }
            TermWmAction::CloseWindow => { app.wm().close_window(app.wm().focused_window()); }
            TermWmAction::NewWindow => { app.wm_new_window()?; }
            _ => {}
        }
    }
    update_selection_snapshot(app);
    return flush_state_changes(app, ControlFlow::Continue, false);
}
```

Remove: `wm_menu_consumes_event`, `fold_menu`, `handle_wm_menu_event`, `mapped_action_wm_mode` (WmMode actions are now handled inside the sink).

### 2. WindowManager overlay methods (`overlays.rs`)
Add methods matching the Help overlay pattern but returning `EventResult`:

```rust
pub fn command_palette_visible(&self) -> bool {
    self.overlays.contains_key(&OverlayId::CommandPalette)
}

pub fn close_command_palette(&mut self) {
    self.overlays.remove(&OverlayId::CommandPalette);
}

pub fn handle_command_palette_event(&mut self, event: &Event) -> EventResult<TermWmAction> {
    if let Event::Mouse(mouse) = event {
        self.hover = Some((mouse.column, mouse.row));
    }
    let ctx = self.component_context(false)
        .with_overlay(true)
        .with_screen_area(self.managed_area());

    let (result, is_visible) = {
        let Some(boxed) = self.overlays.get_mut(&OverlayId::CommandPalette) else {
            return EventResult::Ignored;
        };
        let result = boxed.handle_events(event, &ctx);
        let result = match result {
            EventResult::Action(TermWmAction::MenuSelect) => {
                // Unwrap MenuSelect to return the actual selected action
                let mut queue = VecDeque::new();
                boxed.update(TermWmAction::MenuSelect, &ctx, &mut queue);
                if let Some(selected) = boxed.selected_action() {
                    EventResult::Action(selected)
                } else {
                    EventResult::Consumed
                }
            }
            other => other,
        };
        (result, boxed.visible())
    };

    if !is_visible {
        self.overlays.remove(&OverlayId::CommandPalette);
    }
    result
}
```

Same for `menu_overlay_visible`, `close_menu_overlay`, `handle_menu_overlay_event`.

### 3. OverlayId variant (`mod.rs`)

### 4. Strip WmComponent from components
Both `WmCommandPaletteComponent` and `WmMenuOverlay`:
- Remove all `WmComponent` impl blocks
- Remove `hitbox_id`, `process_action`, `consume_area`, `query` methods
- Add `area: Cell<LayoutRect>` field
- Add `show()` / `close()` / `visible()` methods

### 5. Data-injection pipeline (keep items populated)
The palette registry is populated every frame via `draw_plan_renderer.rs`. Keep this pipeline but adapt it to the overlay model:

Add `as_any_mut()` to the `Overlay` trait in `components/mod.rs`:
```rust
fn as_any_mut(&mut self) -> Option<&mut dyn std::any::Any> { None }
```

Implement on `WmCommandPaletteComponent`:
```rust
fn as_any_mut(&mut self) -> Option<&mut dyn std::any::Any> { Some(self) }
```

In `WindowManager`, add:
```rust
pub fn set_command_palette_items(&mut self, items: Vec<MenuItem<TermWmAction>>) {
    if let Some(boxed) = self.overlays.get_mut(&OverlayId::CommandPalette) {
        if let Some(palette) = boxed.as_any_mut()
            .and_then(|a| a.downcast_mut::<WmCommandPaletteComponent>())
        {
            palette.set_items(items);
        }
    }
}
```

In `draw_plan_renderer.rs`, replace `menu.process_action(&ComponentAction::SetMenuItems(items))` with `wm.set_command_palette_items(items)`.

### 6. Remove old command menu infrastructure
- Remove `command_menu.rs` — `handle_wm_menu_event`, `wm_menu_consumes_event`, `fold_menu`, `command_menu_visible`, `open_command_menu`, `close_command_menu`
- Remove `ComponentAction::SetMenuItems`, `ComponentAction::SetManagedArea`
- Remove `ComponentTag::CommandPalette` render path from `draw_plan_renderer.rs`
- Update `term_wm_app.rs`/`main.rs` to register palette as overlay via `wm.open_overlay(OverlayId::CommandPalette, ...)`

### 6. Registration
In the app initialization (`term_wm_app.rs`/`main.rs`), register the command palette as an overlay:
```rust
let palette = Box::new(WmCommandPaletteComponent::new());
wm.open_overlay(OverlayId::CommandPalette, Some(palette));
```

When the user toggles the palette (`Ctrl+G`), call `show()` on it via the overlay system instead of setting `command_menu_visible`.
