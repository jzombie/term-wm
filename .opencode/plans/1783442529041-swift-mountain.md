# Plan: Add Window Title Support to TermWmApp

## Context
The `TermWmApp` wrapper abstracts window registration but forces users to break encapsulation via `app.wm()` to set window titles. This is an API gap — `WindowManager::set_window_title(WindowKey, impl Into<String>)` exists but isn't exposed through the wrapper.

## Changes

### 1. Add pass-through method to `TermWmApp`
**File:** `src/term_wm_app.rs` (line ~132)

Add after `wm()`:
```rust
/// Set the display title for a registered window.
pub fn set_window_title(&mut self, key: WindowKey, title: impl Into<String>) {
    self.wm.set_window_title(key, title);
}
```

### 2. Update `dual_image.rs` example
**File:** `examples/dual_image.rs`

After each `app.register()` call, add title setting:
```rust
let left_key = app.register(left);
app.set_window_title(left_key, "Left Image");

let right_key = app.register(right);
app.set_window_title(right_key, "Right Image");
```
