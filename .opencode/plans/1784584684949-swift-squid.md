# Plan: Centralized Monocle Gating for Chrome Buttons & Command Palette

## Problem

Two independent code paths decide whether minimize/maximize/close actions are visible:
1. `wm_menu_items()` in `window_manager/mod.rs:2302` — builds command palette items
2. `render_window()` + `register_window_chrome_hitboxes()` in `draw_plan_renderer.rs` — draws chrome buttons and registers hitboxes

These are separate `if is_monocle` / no-check branches that can drift.

## Solution

One method on `WindowManager` that both call sites consume.

### Step 1: Add gating method to `WindowManager`

In `crates/term-wm-core/src/window/window_manager/mod.rs`, add near the other monocle helpers (~line 174):

```rust
/// Whether the user can manage window size (minimize, maximize).
/// Returns false in monocle mode where the focused window fills the
/// screen and size management is meaningless.
pub fn can_manage_window_size(&self) -> bool {
    !self.is_monocle()
}
```

### Step 2: Move `wm_menu_items` into `impl WindowManager`

Move the standalone function `wm_menu_items` (line 2220) into the `impl WindowManager` block (before line 2218's closing brace). Convert all parameters to `self` reads:

```rust
// Inside impl WindowManager
pub fn wm_menu_items(&self) -> Vec<MenuItem<crate::actions::TermWmAction>> {
    let mouse_label = if self.mouse_capture_enabled { "Mouse Capture: On" } else { "Mouse Capture: Off" };
    // ... same for clipboard_label, selection_label ...

    let has_focused_window = self.window_count() > 0;
    let focused_window = *self.focus.current();
    let window_titles = self.window_titles();

    let mut items = vec![ /* base items */ ];

    if has_focused_window {
        items.push(MenuItem { label: "Close Window".into(), ... });

        if self.can_manage_window_size() {
            items.extend_from_slice(&[ /* Maximize, Minimize, Toggle Direct Mode */ ]);
        } else {
            for (key, title) in &window_titles {
                items.push(MenuItem { label: format!("Switch to: {}", title).into(), ... });
            }
        }
    }
    items
}
```

The caller in `main.rs:288` simplifies to:
```rust
let items = wm.wm_menu_items();
```
