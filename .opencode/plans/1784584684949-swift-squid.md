# Plan: Centralized Window Management Buttons

## Problem

Two independent code paths define which window management buttons exist:
1. `wm_menu_items()` — builds command palette items (labels, icons, actions)
2. `render_window()` + `register_window_chrome_hitboxes()` — draws chrome buttons and registers hitboxes (symbols, positions)

These lists can drift. The monocle gating check is duplicated.

## Solution

Define window management buttons as a single data structure. Both the palette and chrome renderer consume the same list from one method on `WindowManager`.

### Step 1: Define `WmButton` struct

In `crates/term-wm-core/src/window/window_manager/mod.rs`, add near the top of the file or in a new submodule:

```rust
pub struct WmButton {
    pub action: TermWmAction,
    pub label: &'static str,
    pub symbol: &'static str,  // used in both palette icon and chrome button
}
```

### Step 2: Add `window_management_buttons()` to `WindowManager`

```rust
impl WindowManager {
    /// Returns the window management buttons appropriate for the current mode.
    /// In monocle mode, minimize/maximize are excluded (meaningless when
    /// the focused window fills the screen). Close and DirectMode always appear.
    pub fn window_management_buttons(&self) -> Vec<WmButton> {
        let mut btns = vec![
            WmButton { action: TermWmAction::CloseWindow, label: "Close Window", symbol: "X" },
        ];
        if !self.is_monocle() {
            btns.push(WmButton { action: TermWmAction::MaximizeWindow, label: "Maximize Window", symbol: "▢" });
            btns.push(WmButton { action: TermWmAction::MinimizeWindow, label: "Minimize Window", symbol: "_" });
        }
        btns.push(WmButton { action: TermWmAction::ToggleDirectMode, label: "Toggle Direct Mode", symbol: "D" });
        btns
    }
}
```

### Step 3: Move `wm_menu_items` into `impl WindowManager`

Move the standalone function (line 2220) into the `impl` block. Rewrite to consume `self.window_management_buttons()`:

```rust
pub fn wm_menu_items(&self) -> Vec<MenuItem<crate::actions::TermWmAction>> {
    let mouse_label = if self.mouse_capture_enabled { "Mouse Capture: On" } else { "Mouse Capture: Off" };
    let clipboard_label = if self.clipboard_enabled { "Clipboard Mode: On" } else { "Clipboard Mode: Off" };
    let selection_label = if self.window_selection_enabled { "Window Selection: On" } else { "Window Selection: Off" };

    let mut items = vec![
        MenuItem { label: "Resume".into(), icon: Some("▶"), action: TermWmAction::CloseMenu, disabled: false },
        MenuItem { label: mouse_label.into(), icon: Some("◆"), action: TermWmAction::ToggleMouseCapture, disabled: false },
        MenuItem { label: clipboard_label.into(), icon: Some("■"), action: TermWmAction::ToggleClipboardMode, disabled: false },
        MenuItem { label: selection_label.into(), icon: Some("●"), action: TermWmAction::ToggleWindowSelection, disabled: false },
        MenuItem { label: "New Window".into(), icon: Some("+"), action: TermWmAction::NewWindow, disabled: false },
        MenuItem { label: "Debug Log".into(), icon: Some("≣"), action: TermWmAction::ToggleDebugWindow, disabled: false },
        MenuItem { label: "System Panel".into(), icon: Some("⚙"), action: TermWmAction::ToggleSystemPanel, disabled: false },
        MenuItem { label: "Help".into(), icon: Some("?"), action: TermWmAction::Help, disabled: false },
        MenuItem { label: "Exit UI".into(), icon: Some("⏻"), action: TermWmAction::ExitUi, disabled: false },
    ];

    let has_focused_window = self.window_count() > 0;
    if has_focused_window {
        // Window management buttons from centralized list
        for btn in self.window_management_buttons() {
            items.push(MenuItem {
                label: btn.label.into(),
                icon: Some(btn.symbol),
                action: btn.action,
                disabled: false,
            });
        }
        // Monocle: also add "Switch to" navigation
        if self.is_monocle() {
            let focused = *self.focus.current();
            for (key, title) in self.window_titles() {
                items.push(MenuItem {
                    label: format!("Switch to: {}", title).into(),
                    icon: Some("→"),
                    action: TermWmAction::FocusWindow(key),
                    disabled: key == focused,
                });
            }
        }
    }
    items
}
```

### Step 4: Use `window_management_buttons()` in chrome rendering

In `crates/term-wm-console/src/draw_plan_renderer.rs`:

**4a.** In `register_window_chrome_hitboxes()`, replace the hardcoded button list with iteration over the centralized data. The function receives `buttons: &[WmButton]` (or a slice of `(ChromeTarget, u16)` derived from it).

**4b.** In `render_window()`, replace the hardcoded `Btn` enum and button array with iteration over the same data.

**4c.** `render_window_chrome` passes the buttons down from its caller.

The chrome renderer doesn't need `WindowManager` — it receives the button list via `ChromeCtx` or as a parameter. Add to `ChromeCtx`:

```rust
pub struct ChromeCtx<'a> {
    pub title: &'a str,
    pub focused: bool,
    pub floating: bool,
    pub direct_mode: bool,
    pub hover_pos: Option<(u16, u16)>,
    pub theme: term_wm_core::theme::Theme,
    pub wm_buttons: Vec<crate::WmButton>,  // NEW — from wm.window_management_buttons()
}
```

Then `render_window` and `register_window_chrome_hitboxes` iterate `ctx.wm_buttons` instead of defining their own lists.
