# Plan: Extract WM-specific components into `term-wm-sys-ui-components`

## Goal
Create a new crate `term-wm-sys-ui-components` holding window-manager-specific components currently mixed into `term-wm-ui-components`. This separates reusable UI widgets from WM infrastructure, renames components with a `wm_`/`Wm` prefix, and refactors duplicated patterns into shared abstractions.

## Naming convention
All components in the new crate follow `wm_` filename prefix and `Wm` type prefix:
- `wm_menu_overlay.rs` → `WmMenuOverlay` (already Wm-prefixed, filename changes)
- `sys/wm_debug_log.rs` → `WmDebugLogComponent` (was DebugLogComponent)
- `sys/wm_help_overlay.rs` → `WmHelpOverlayComponent` (was HelpOverlayComponent)
- `sys/wm_keybinding_overlay.rs` → `WmKeybindingOverlayComponent` (was KeybindingOverlayComponent)

## Refactoring: Extract shared "dialog overlay" abstraction (HIGH VALUE)

`WmHelpOverlayComponent` and `WmKeybindingOverlayComponent` are structurally identical — both wrap a `DialogOverlayComponent` + `ScrollViewComponent<C>`. Duplicated across both:

| Aspect | What's duplicated |
|--------|------------------|
| Struct fields | `dialog: DialogOverlayComponent`, `visible: bool`, `content: ScrollViewComponent<C>`, `area: Rect`, `keybindings: KeyBindings` |
| `render()` | ~10 lines of dialog setup (backdrop, rect_for, Clear, Block, inner rect) |
| `handle_event()` | ~15 lines of keybinding-close / outside-click-close / delegate to content |
| `show()` / `close()` / `visible()` | Identical lifecycle |

**Action:** Extract a generic `WmDialogOverlayComponent<C: Component>` in `term-wm-sys-ui-components` that wraps this pattern. Both `WmHelpOverlayComponent` and `WmKeybindingOverlayComponent` will compose it instead of duplicating the pattern. This lives in the new crate (since it's WM-specific: it depends on `Action::CloseHelp` and WM overlay behavior).

## Refactoring: Extend `DialogOverlayComponent::render_backdrop` (MEDIUM VALUE)

`WmMenuOverlay::render_backdrop` does the same cell-dimming loop as `DialogOverlayComponent::render_backdrop` but supports an `exclude: Rect` parameter to skip the dropdown area. 

**Action:** Add an optional `exclude: Option<Rect>` parameter to `DialogOverlayComponent::render_backdrop` in `term-wm-ui-components`. Then `WmMenuOverlay` calls it instead of reimplementing.

## Refactoring: Remove redundant cell-clearing in debug_log (LOW VALUE)

`WmDebugLogComponent::render` has a manual cell-clearing loop (`for y in ... for x in ... cell.reset(); cell.set_symbol(" ")`) that is completely redundant — `ScrollViewComponent` + `TextRendererComponent` will overwrite every cell during their own render.

**Action:** Delete the manual clearing loop.
