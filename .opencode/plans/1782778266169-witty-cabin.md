# Plan: Consolidate Power Profiling into Dedicated Module

## Goal
Move `PowerProfile` from `crates/term-wm-core/src/io/power_profile.rs` to a top-level `crates/term-wm-core/src/power_profile.rs`, making it self-contained and easily linkable. Theme profile colors stay in `theme.rs` as-is.

## Changes

### 1. Create `crates/term-wm-core/src/power_profile.rs`
- Copy `PowerProfile` enum + impl from `io/power_profile.rs`
- Keep `use crate::theme;` dependency (indicator_color still delegates to theme)
- Remove dead `ProfileMid`/`profile_mid_bg` references (already unused)

### 2. Update `crates/term-wm-core/src/lib.rs`
- Add `pub mod power_profile;`

### 3. Update `crates/term-wm-core/src/io/mod.rs`
- Remove `pub mod power_profile;` and `pub use power_profile::PowerProfile;`

### 4. Update imports in all files that reference `PowerProfile`
| File | Old import | New import |
|------|-----------|-----------|
| `io/event_source.rs` | `crate::io::PowerProfile` | `crate::power_profile::PowerProfile` |
| `io/console.rs` | `super::PowerProfile` | `crate::power_profile::PowerProfile` |
| `window/window_manager/mod.rs` | `crate::io::PowerProfile` | `crate::power_profile::PowerProfile` |
| `bottom_panel_trait.rs` | `crate::io::PowerProfile` | `crate::power_profile::PowerProfile` |
| `wm_bottom_panel.rs` | `term_wm_core::io::PowerProfile` | `term_wm_core::power_profile::PowerProfile` |

### 5. `theme.rs` — **no changes**
