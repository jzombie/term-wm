# Plan: Consolidate Power Profiling into Dedicated Module

## Goal
Move the power profiling system from its scattered locations into a single top-level `power_profile` module in `term-wm-core`, making it self-contained and easily linkable. The integration points (EventSource, WindowManager, BottomPanel, runner) will import from the new module but keep their architecture.

## Changes

### 1. Create `crates/term-wm-core/src/power_profile.rs`
New file containing:
- `PowerProfile` enum (moved from `io/power_profile.rs`)
- Inline indicator color constants (`PROFILE_HIGH_COLOR`, `PROFILE_LOW_COLOR`) ‚Äî no longer depends on `theme.rs`
- `poll_interval()`, `name()`, `indicator_color()`, `report_change()` methods
- Remove `ProfileMid` dead code (unused amber variant)

### 2. Update `crates/term-wm-core/src/lib.rs`
- Add `pub mod power_profile;`

### 3. Update `crates/term-wm-core/src/io/mod.rs`
- Remove `pub mod power_profile;` and `pub use power_profile::PowerProfile;`

### 4. Update imports in integration files
- `io/event_source.rs`: `crate::io::PowerProfile` ‚Üí `crate::power_profile::PowerProfile`
- `io/console.rs`: `super::PowerProfile` ‚Üí `crate::power_profile::PowerProfile`
- `window/window_manager/mod.rs`: `crate::io::PowerProfile` ‚Üí `crate::power_profile::PowerProfile`
- `bottom_panel_trait.rs`: `crate::io::PowerProfile` ‚Üí `crate::power_profile::PowerProfile`
- `wm_bottom_panel.rs`: `term_wm_core::io::PowerProfile` ‚Üí `term_wm_core::power_profile::PowerProfile`

### 5. Clean up `crates/term-wm-core/src/theme.rs`
- Remove `BgColor::ProfileHigh`, `ProfileMid`, `ProfileLow` variants
- Remove `profile_high`, `profile_mid`, `profile_low` fields from `Theme` struct
- Remove corresponding `bg()` match arms
- Remove NOIR default values for profile colors
- Remove `profile_high_bg()`, `profile_mid_bg()`, `profile_low_bg()` functions
- Remove contrast test exclusion for these variants (no longer exist)
