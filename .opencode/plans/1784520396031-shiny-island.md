# Plan: Replace Manual z-index Hacks with ZLayer Enum

## Objective

Replace the loose `z_index: usize` field (and its magic-number comparisons) with a strongly-typed `ZLayer` enum, while keeping `engine.rs` region generation and `src/lib.rs` untouched.

## Files to Modify (4 files)

### 1. `crates/term-wm-core/src/draw_plan.rs`

**Add `ZLayer` enum** (before `RenderRegion`):
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
pub enum ZLayer {
    Background = 0,
    TiledWindow = 10,
    FloatingWindow = 20,
    SystemChrome = 30,
    Panel = 40,
    Notification = 50,
    ForegroundLayer = 60,
    Overlay = 70,
}
```

**Update `RenderRegion`**: Replace `pub z_index: usize` with `pub layer: ZLayer`.

**Update `sort_by_z_index`**: Rename body to sort by `r.layer`. Keep `sort_by_z_index` as a backward-compat alias calling `sort_by_layer`.

**Update test helpers** (`make_region`, `make_dimmed_region`, `make_panel_region`): Change `z_index: usize` parameter → `layer: ZLayer`.

**Update `assert_region_z_index`**: Replace with `assert_region_layer(plan, index, expected: ZLayer)`.

**Update `assert_sorted_by_z_index`**: Replace with `assert_sorted_by_layer` checking `window[0].layer <= window[1].layer`.

**Update `test_draw_plan_z_index_sorting`**: Use `ZLayer::*` values and `sort_by_layer()`.

### 2. `crates/term-wm-core/src/engine.rs`

- Replace `use crate::constants::NOTIFICATION_Z_INDEX;` with `use crate::draw_plan::ZLayer;`
- Line 36: `self.draw_plan.sort_by_z_index();` → `self.draw_plan.sort_by_layer();`
- Line 62: `self.draw_plan.sort_by_z_index();` → `self.draw_plan.sort_by_layer();`
- Line 92: `z_index: 0,` → `layer: ZLayer::TiledWindow,`
- Line 183: `z_index: NOTIFICATION_Z_INDEX,` → `layer: ZLayer::Notification,`

### 3. `crates/term-wm-console/src/draw_plan_renderer.rs`

- Add `ZLayer` to imports: `use term_wm_core::draw_plan::{DrawPlan, RegionType, RenderRegion, ZLayer};`
- Line 93 (`render_to_buffer`): `if region.z_index < 10` → `if region.layer <= ZLayer::TiledWindow`
- Line 242 (`render`): `if region.z_index < 10` → `if region.layer <= ZLayer::TiledWindow`

### 4. `crates/term-wm-core/src/constants.rs`

- Remove the `NOTIFICATION_Z_INDEX` constant (line 55-56).

## What is NOT changed

- `src/lib.rs` — untouched (no z_index references exist there).
- Panels, overlays, floating windows — remain rendered via their existing out-of-band paths.
- `RegionType` variants, `DrawPlan`, monocle culling/z-order — untouched.
- `z_depth` (drop-shadow interpolation) — entirely separate, not affected.

## Verification

```bash
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test
```

All existing tests will pass with updated `ZLayer` values replacing the old `usize` z-indices. The sorted-by-layer invariant is equivalent to the old sorted-by-z_index invariant since `ZLayer` derives `Ord` matching the old numeric values.
