# Plan: Spatial-Aware Float-to-Tiled Reattachment

## Problem

When switching from float to tiled mode (via `toggle_tiling` → `tile_window_key`), windows are inserted into the layout tree via `insert_window_balanced`, which picks the largest region and splits it — completely ignoring each window's floating position. Same issue in `reattach_to_tiling_layout` (Iconic→Mapped), which blindly inserts next to `current_focus` with `InsertPosition::Right`.

Result: the tiled layout after float→tiled→float→tiled cycles drifts substantially from the user's original spatial arrangement.

## Approach

Two-phase spatial insertion pipeline, used in both `tile_window_key` and `reattach_to_tiling_layout`:

1. Compute the floating window's center from its stored `floating_rect` (if present).
2. **Phase 1 — exact hit-test**: `.contains()` against each rect from `layout.regions(area)`. If the center falls inside a tiled rect, that's the target.
3. **Phase 2 — Euclidean fallback**: if the center lands in a gap between tiles, compute squared distance (with aspect-ratio-weighted `dy`) to each region's center, pick the closest. Uses `i64` arithmetic to avoid saturation. Returns `Option` — on `None`, falls back to `insert_window_balanced` / `current_focus` (safe).
4. Use `detect_quadrant` on the resolved target rect → `Quadrant` → `InsertPosition` (mapping: North→Top, South→Bottom, West→Left, East→Right).
5. Execute `insert_leaf(target, key, pos)`.

All infrastructure already exists: `LayoutRect::center()`, `detect_quadrant`, `Quadrant`, `InsertPosition`, `TilingLayout::regions(area)`. Aspect ratio constant `CELL_ASPECT_RATIO` already defined in `crates/term-wm-core/src/constants.rs`.

## Changes

### 1. `crates/term-wm-core/src/window/window_manager/layout.rs` — helpers + `reattach_to_tiling_layout()`

Add three helpers at module level (or as methods on `WindowManager`):

```rust
fn quadrant_to_insert_position(q: Quadrant) -> InsertPosition {
    match q {
        Quadrant::North => InsertPosition::Top,
        Quadrant::South => InsertPosition::Bottom,
        Quadrant::West => InsertPosition::Left,
        Quadrant::East => InsertPosition::Right,
    }
}

fn find_closest_region(
    (cx, cy): (i32, i32),
    regions: &[(WindowKey, Rect)],
) -> Option<(WindowKey, Rect)> {
    if regions.is_empty() {
        return None;
    }
    regions.iter()
        .map(|(k, r)| {
            let (rcx, rcy) = r.center();
            let dx = (cx as i64) - (rcx as i64);
            let dy = ((cy as i64) - (rcy as i64)) * 2; // aspect-ratio weighted
            let dist = dx * dx + dy * dy;
            (*k, *r, dist)
        })
        .min_by_key(|(_, _, d)| *d)
        .map(|(k, r, _)| (k, r))
}

fn resolve_target(
    (cx, cy): (i32, i32),
    regions: &[(WindowKey, Rect)],
) -> Option<(WindowKey, Rect)> {
    // Phase 1: exact hit-test
    if let Some(found) = regions.iter().find(|(_, r)| r.contains(cx as u16, cy as u16)) {
        return Some(*found);
    }
    // Phase 2: Euclidean closest
    find_closest_region((cx, cy), regions)
}
```

Rewrite `reattach_to_tiling_layout`:

```rust
pub(super) fn reattach_to_tiling_layout(&mut self, key: WindowKey) {
    // existing early-return checks (layout_contains, managed_layout.is_none) ...

    let current_focus = *self.focus.current();
    let Some(layout) = self.managed_layout.as_mut() else { return; };

    // Try spatial placement via floating_rect
    if let Some(spec) = self.window(key).and_then(|w| w.floating_rect) {
        let rect = spec.resolve(self.managed_area);
        let (cx, cy) = rect.center();
        let regions = layout.regions(self.managed_area);
        if let Some((target_key, _)) = resolve_target((cx, cy), &regions) {
            let quad = detect_quadrant(cx as u16, cy as u16, &rect); // note: detect_quadrant takes target rect
            let pos = quadrant_to_insert_position(quad);
            let inserted = layout.root_mut().insert_leaf(target_key, key, pos);
            if !inserted {
                layout.split_root(key, pos);
            }
            return;
        }
    }

    // Absolute fallback: window was never floating — use current_focus
    if current_focus == key {
        layout.split_root(key, InsertPosition::Right);
        return;
    }
    let inserted = layout.root_mut().insert_leaf(current_focus, key, InsertPosition::Right);
    if !inserted {
        layout.split_root(key, InsertPosition::Right);
    }
}
```

### 2. `crates/term-wm-core/src/window/window_manager/drag.rs` — `tile_window_key()`

```rust
pub(super) fn tile_window_key(&mut self, key: WindowKey) -> bool {
    // Capture floating rect center BEFORE clearing
    let floating_info = self.floating_rect(key)
        .map(|spec| spec.resolve(self.managed_area))
        .map(|r| r.center());

    self.clear_floating_rect(key);

    // ... existing remove_window logic ...

    if let Some(ref mut layout) = self.managed_layout {
        if let Some((cx, cy)) = floating_info {
            let regions = layout.regions(self.managed_area);
            if let Some((target_key, target_rect)) = resolve_target((cx, cy), &regions) {
                let quad = detect_quadrant(cx as u16, cy as u16, &target_rect);
                let pos = quadrant_to_insert_position(quad);
                let inserted = layout.root_mut().insert_leaf(target_key, key, pos);
                if !inserted {
                    layout.split_root(key, pos);
                }
            } else {
                layout.insert_window_balanced(key, self.managed_area);
            }
        } else {
            layout.insert_window_balanced(key, self.managed_area);
        }
    } else {
        self.managed_layout = Some(TilingLayout::new(LayoutNode::leaf(key)));
    }
    // ... focus + draw order ...
}
```
