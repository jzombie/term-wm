# Plan: Spatial-Aware Float-to-Tiled Reattachment

## Problem

When switching from float to tiled mode (via `toggle_tiling` → `tile_window_key`), windows are inserted into the layout tree via `insert_window_balanced`, which picks the largest region and splits it — completely ignoring each window's floating position. The same issue exists in `reattach_to_tiling_layout` (used during Iconic→Mapped transitions), which blindly inserts next to `current_focus` with `InsertPosition::Right`.

Result: The tiled layout after float→tiled→float→tiled cycles can drift substantially from the user's original layout.

## Approach

Replace the spatially-blind insertion logic in both paths with spatial hit-testing + quadrant-based splitting, using existing infrastructure:

- `LayoutRect::center()` — compute window center point
- `detect_quadrant(cursor_col, cursor_row, target)` — returns `Quadrant` (North/South/East/West)
- Quadrant → `InsertPosition` mapping: North→Top, South→Bottom, West→Left, East→Right
- `LayoutNode::regions(area)` or `TilingLayout::regions(area)` — get currently laid-out tiled rects

No new types or tree mutations needed. Pure routing changes.

## Changes

### 1. `crates/term-wm-core/src/window/window_manager/drag.rs` — `tile_window_key()`

**Current**: clears `floating_rect`, then calls `insert_window_balanced` which selects the largest rect and splits it.

**Change**: Before clearing `floating_rect`, capture the window's center. Compute current tiled regions via `layout.regions(self.managed_area)`. Hit-test the center against those regions. If hit:
- Use `detect_quadrant` on the hit target rect → `InsertPosition`
- Insert like: `layout.root_mut().insert_leaf(hit_target, key, pos)`
If miss: fallback to `insert_window_balanced`.

```rust
pub(super) fn tile_window_key(&mut self, key: WindowKey) -> bool {
    // Capture floating rect center BEFORE clearing it
    let center = self.floating_rect(key)
        .map(|spec| spec.resolve(self.managed_area))
        .map(|r| r.center());
    
    self.clear_floating_rect(key);
    // ... existing remove_window logic ...

    if let Some(ref mut layout) = self.managed_layout {
        if let Some((cx, cy)) = center {
            let regions = layout.regions(self.managed_area);
            if let Some((target_key, target_rect)) = regions.iter()
                .find(|(_, r)| r.contains(cx as u16, cy as u16))
            {
                let quad = detect_quadrant(cx as u16, cy as u16, target_rect);
                let pos = quadrant_to_insert_position(quad);
                let _ = layout.root_mut().insert_leaf(*target_key, key, pos);
                // (fallback to split_root if insert_leaf fails)
            } else {
                layout.insert_window_balanced(key, self.managed_area);
            }
        } else {
            layout.insert_window_balanced(key, self.managed_area);
        }
    } else {
        self.managed_layout = Some(TilingLayout::new(LayoutNode::leaf(key)));
    }
    // ...
}
```

### 2. `crates/term-wm-core/src/window/window_manager/layout.rs` — `reattach_to_tiling_layout()`

**Current**: always inserts `key` next to `*self.focus.current()` with `InsertPosition::Right`.

**Change**: Check if the window has a known floating rect (from `Window::floating_rect`). If so, compute its center, hit-test against current tiled regions, and use quadrant-based split. Fallback to `current_focus` + `Right` when no floating rect or hit miss.

```rust
pub(super) fn reattach_to_tiling_layout(&mut self, key: WindowKey) {
    // existing early-return checks...
    
    let center = self.window(key)
        .and_then(|w| w.floating_rect)
        .map(|spec| spec.resolve(self.managed_area))
        .map(|r| r.center());
    
    let (target, pos) = if let Some((cx, cy)) = center {
        let regions = layout.regions(self.managed_area);
        // find containing rect, use quadrant
        // fallback to (current_focus, Right)
    } else {
        (*self.focus.current(), InsertPosition::Right)
    };
    // insert at target with computed pos
}
```

### 3. Add `quadrant_to_insert_position` helper

A small mapping function (in `layout.rs` or `drag.rs`):

```rust
fn quadrant_to_insert_position(q: Quadrant) -> InsertPosition {
    match q {
        Quadrant::North => InsertPosition::Top,
        Quadrant::South => InsertPosition::Bottom,
        Quadrant::West => InsertPosition::Left,
        Quadrant::East => InsertPosition::Right,
    }
}
```
