# Plan: Spatial-Aware Float-to-Tiled Reattachment

## Problem

When switching from float to tiled mode (via `toggle_tiling` → `tile_window_key`), windows are inserted into the layout tree via `insert_window_balanced`, which picks the largest region and splits it — completely ignoring each window's floating position. Same issue in `reattach_to_tiling_layout` (Iconic→Mapped), which blindly inserts next to `current_focus` with `InsertPosition::Right`.

Result: the tiled layout after float→tiled→float→tiled cycles drifts substantially from the user's original spatial arrangement.

## Approach

Two-phase spatial insertion pipeline, used in both `tile_window_key` and `reattach_to_tiling_layout`:

1. Compute the floating window's center from its stored `floating_rect` (if present).
2. **Phase 1 — exact hit-test**: `.contains()` against each rect from `layout.regions(area)`. If the center falls inside a tiled rect, that's the target.
3. **Phase 2 — Euclidean fallback**: if the center lands in a gap, compute squared distance (with aspect-ratio-weighted `dy`, cast to `i64`) to each region's center, pick the closest. Returns `Option`.
4. Use `Quadrant::to_insert_position()` on the resolved target's bounding box.
5. Execute `insert_leaf(target_key, key, pos)`.

The spatial math lives in the layout engine (`hit_test.rs`, `rect.rs`), not in core.

### 2. `crates/term-wm-layout-engine/src/hit_test.rs` — add `resolve_target` and `find_closest_region`

Both are generic over `Id: Copy` and parameterized with `aspect_ratio_weight`:

```rust
/// Find the region whose center is nearest to (cx, cy) using
/// aspect-ratio-weighted Euclidean distance. Returns None for empty input.
pub fn find_closest_region<Id: Copy>(
    cx: i32,
    cy: i32,
    regions: &[(Id, LayoutRect)],
    aspect_ratio_weight: u32,
) -> Option<(Id, LayoutRect)> {
    if regions.is_empty() {
        return None;
    }
    let weight = aspect_ratio_weight as i64;
    regions
        .iter()
        .map(|(id, rect)| {
            let (rcx, rcy) = rect.center();
            let dx = (cx as i64) - (rcx as i64);
            let dy = ((cy as i64) - (rcy as i64)) * weight;
            let dist = dx * dx + dy * dy;
            (*id, *rect, dist)
        })
        .min_by_key(|(_, _, d)| *d)
        .map(|(id, rect, _)| (id, rect))
}

/// Two-phase target resolution:
/// 1. Exact hit-test (contains).
/// 2. Euclidean fallback via `find_closest_region`.
pub fn resolve_target<Id: Copy>(
    cx: i32,
    cy: i32,
    regions: &[(Id, LayoutRect)],
    aspect_ratio_weight: u32,
) -> Option<(Id, LayoutRect)> {
    // Phase 1: exact hit-test
    if let Some(found) = regions
        .iter()
        .find(|(_, r)| r.contains(cx as u16, cy as u16))
    {
        return Some(*found);
    }
    // Phase 2: Euclidean closest
    find_closest_region(cx, cy, regions, aspect_ratio_weight)
}
```

Re-export both from `lib.rs`.

### 3. `crates/term-wm-core/src/window/window_manager/layout.rs` — rewrite `reattach_to_tiling_layout()`

```rust
pub(super) fn reattach_to_tiling_layout(&mut self, key: WindowKey) {
    use crate::layout::LayoutNode;
    if self.layout_contains(key) { return; }
    if self.managed_layout.is_none() {
        self.managed_layout = Some(TilingLayout::new(LayoutNode::leaf(key)));
        return;
    }
    let current_focus = *self.focus.current();
    let Some(layout) = self.managed_layout.as_mut() else { return; };

    // Try spatial placement via floating_rect
    if let Some(spec) = self.window(key).and_then(|w| w.floating_rect) {
        let rect = spec.resolve(self.managed_area);
        let (cx, cy) = rect.center();
        let regions = layout.regions(self.managed_area);
        let weight = crate::constants::CELL_ASPECT_RATIO;
        if let Some((target_key, target_rect)) = resolve_target(cx, cy, &regions, weight) {
            let quad = detect_quadrant(cx as u16, cy as u16, &target_rect);
            let pos = quad.to_insert_position();
            let inserted = layout.root_mut().insert_leaf(target_key, key, pos);
            if !inserted {
                layout.split_root(key, pos);
            }
            return;
        }
    }

    // Absolute fallback: window was never floating — use current_focus
    if current_focus == key {
        layout.split_root(key, InsertPosition::Right);
        return;
    }
    let inserted = layout.root_mut().insert_leaf(current_focus, key, InsertPosition::Right);
    if !inserted {
        layout.split_root(key, InsertPosition::Right);
    }
}
```

## Amendments (Post-Review Fixes)

The following amendments replace the corresponding sections above. The original text is preserved; these amendments override it.

### A1. Dominant-Axis Insert Position

Eliminate naive quadrant-to-`InsertPosition` mapping. The `InsertPosition` is computed from the **target tile's dominant axis**, not from the cursor's diagonal quadrant.

```
fn dominant_axis_insert_position(
    cx: i32,
    cy: i32,
    target: &LayoutRect,
    cell_aspect_ratio: u32,
) -> InsertPosition {
    let visual_w = target.width as u32;
    let visual_h = (target.height as u32) * cell_aspect_ratio;

    if visual_w > visual_h {
        // Dominant axis is Horizontal — force vertical split
        let mid_x = target.x + i32::from(target.width) / 2;
        if cx < mid_x { InsertPosition::Left } else { InsertPosition::Right }
    } else {
        // Dominant axis is Vertical — force horizontal split
        let mid_y = target.y + i32::from(target.height) / 2;
        if cy < mid_y { InsertPosition::Top } else { InsertPosition::Bottom }
    }
}
```

This logic mirrors `insert_window_balanced`'s anti-degeneracy but works with any target rect, not just the largest.

### A2. Sorted Candidate Exhaustion

Replace `resolve_target` (single best) with `sorted_candidates` (all, sorted by distance). The caller iterates candidates in order, trying `insert_leaf` on each. Only the first success wins; `insert_leaf` failure (target not at expected node) advances to the next candidate.

```rust
/// Return all regions sorted by aspect-ratio-weighted Euclidean distance
/// from (cx, cy). Exact hits (contains) are sorted first by priority.
pub fn sorted_candidates<Id: Copy>(
    cx: i32,
    cy: i32,
    regions: &[(Id, LayoutRect)],
    aspect_ratio_weight: u32,
) -> Vec<(Id, LayoutRect)> {
    if regions.is_empty() {
        return Vec::new();
    }
    let weight = aspect_ratio_weight as i64;
    let mut scored: Vec<(Id, LayoutRect, u64)> = regions
        .iter()
        .map(|(id, rect)| {
            let (rcx, rcy) = rect.center();
            let dx = (cx as i64) - (rcx as i64);
            let dy = ((cy as i64) - (rcy as i64)) * weight;
            // distance squared, saturating to u64 range
            let dist = (dx * dx + dy * dy) as u64;
            // Exact hits get priority score 0, others get 1 + distance
            let priority = if rect.contains(cx as u16, cy as u16) {
                0u64
            } else {
                1u64.saturating_add(dist)
            };
            (*id, *rect, priority)
        })
        .collect();
    scored.sort_by_key(|(_, _, p)| *p);
    scored.into_iter().map(|(id, rect, _)| (id, rect)).collect()
}
```

### A3. Rewrite Spatial Insertion in Both Call Sites

Both `reattach_to_tiling_layout` and `tile_window_key` use the same pipeline:

```rust
// Capture floating rect center (before clearing it if needed)
let floating_center = ... ; // Option<(i32, i32)>

if let Some((cx, cy)) = floating_center {
    let regions = layout.regions(self.managed_area);
    let weight = crate::constants::CELL_ASPECT_RATIO;
    let candidates = term_wm_layout_engine::sorted_candidates(cx, cy, &regions, weight);

    let mut inserted = false;
    for (target_key, target_rect) in &candidates {
        let pos = dominant_axis_insert_position(cx, cy, target_rect, weight);
        if layout.root_mut().insert_leaf(*target_key, key, pos) {
            inserted = true;
            break;
        }
    }
    if inserted { /* success path */ }
    else { layout.insert_window_balanced(key, self.managed_area); }
}
```

Key changes from the original code:
- `sorted_candidates` returns ALL regions, not just one. Exact hits come first (priority 0), then Euclidean-sorted.
- `dominant_axis_insert_position` chooses the split direction based on the target tile's dimensions, not the cursor's quadrant.
- The loop iterates candidates in priority order. If `insert_leaf` fails (target moved between region computation and tree mutation), the next candidate is tried.
- `split_root` is **not** called in the spatial path. Only `insert_window_balanced` is used as the safety net when all candidates reject — exactly matching the existing anti-degeneracy in that function.
- The fallback to `current_focus` / `InsertPosition::Right` (for `reattach_to_tiling_layout`) is preserved only when `floating_rect` is `None`.
