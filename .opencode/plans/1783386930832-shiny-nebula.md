# Dynamic Menu Items ‚Äî Only Show Routed Actions

## Problem

The WM command menu hardcodes items like "Debug Log" and "New Window" that are dead (no-op) when using `TermWmApp` because `WindowManagerHost` default trait methods do nothing. Menu items should only appear if they are actually routed to a functional handler.

## Root Cause

`wm_menu_items()` in `window_manager/mod.rs:1965` returns a static list of 9 items. Three are dead in `TermWmApp`:
- **"New Window"** ‚Äî `wm_new_window()` defaults to `Ok(())` (no-op)
- **"Debug Log"** ‚Äî `toggle_debug_window()` defaults to `{}` (no-op)
- **"Help"** ‚Äî opens an empty overlay slot (no component)

The menu is built in `render_overlays()` which has no access to the host.

## Approach

Declarative configuration via `AppBuilder`. The supported actions are static constraints of the host environment, set once at boot.

### 1. Add `supported_menu_actions` to `AppBuilder`

**File:** `crates/term-wm-core/src/config.rs`

Add field and builder method:

```rust
pub struct AppBuilder {
    // ... existing fields ...
    supported_menu_actions: Option<Vec<TermWmAction>>,
}

impl AppBuilder {
    pub fn bare() -> Self {
        Self {
            // ... existing ...
            supported_menu_actions: None, // defaults to full list
        }
    }

    pub fn supported_menu_actions(mut self, actions: Vec<TermWmAction>) -> Self {
        self.supported_menu_actions = Some(actions);
        self
    }

    pub fn build(self) -> Result<WindowManager, ConfigError> {
        let app_ctx = self.app_ctx.ok_or(ConfigError::MissingAppContext)?;
        Ok(WindowManager::with_config(
            self.config,
            app_ctx,
            self.top_panel,
            self.bottom_panel,
            self.command_menu,
            self.supported_menu_actions,
        ))
    }
}
```

### 2. Update `WindowManager::with_config` signature

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

Add parameter and field:

```rust
pub fn with_config(
    config: WmConfig,
    app_ctx: Arc<AppContext>,
    top_component: Option<Box<dyn WmComponent>>,
    bottom_component: Option<Box<dyn WmComponent>>,
    command_menu_component: Option<Box<dyn WmComponent>>,
    supported_menu_actions: Option<Vec<TermWmAction>>,
) -> Self {
    let supported_menu_actions = supported_menu_actions.unwrap_or_else(|| {
        vec![
            TermWmAction::CloseMenu,
            TermWmAction::ToggleMouseCapture,
            TermWmAction::ToggleClipboardMode,
            TermWmAction::ToggleWindowSelection,
            TermWmAction::BringFloatingFront,
            TermWmAction::NewWindow,
            TermWmAction::ToggleDebugWindow,
            TermWmAction::Help,
            TermWmAction::ExitUi,
        ]
    });
    // ... rest of constructor, store supported_menu_actions as field ...
}
```

Add field to struct:
```rust
pub(crate) supported_menu_actions: Vec<TermWmAction>,
```

### 3. Filter menu items in `render_overlays`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

In `render_overlays` (line 1912), filter using the stored field (zero allocation per frame ‚Äî `contains` on a small `Vec`):

```rust
let mut menu_items = wm_menu_items(
    self.mouse_capture_enabled(),
    self.clipboard_enabled(),
    self.window_selection_enabled(),
);
menu_items.retain(|item| self.supported_menu_actions.contains(&item.action));
```

### 4. Update `TermWmApp` to pass constrained list

**File:** `src/term_wm_app.rs`

In `new()` (sys-ui) and `bare()`:

```rust
AppBuilder::bare()
    // ... existing config ...
    .supported_menu_actions(vec![
        TermWmAction::CloseMenu,
        TermWmAction::ToggleMouseCapture,
        TermWmAction::ToggleClipboardMode,
        TermWmAction::ToggleWindowSelection,
        TermWmAction::BringFloatingFront,
        TermWmAction::ExitUi,
    ])
    .build()
```

Excludes: `NewWindow`, `ToggleDebugWindow`, `Help`.

### 5. Update `main.rs` ‚Äî no changes needed

The default (`None`) gives the full 9-item list, which is correct for the full `App`.

### 6. Update all other `WindowManager::with_config` call sites

Pass `None` for the new parameter in all test code and other callers (~55 sites).
