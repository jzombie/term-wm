# Seal Constructor & Migrate External Callers to AppBuilder

## Problem

`WindowManager::with_config` is `pub`, allowing external callers to bypass `AppBuilder` entirely. The 6-argument constructor is an internal implementation detail — the public API should be `AppBuilder` only. 5 external call sites in `tests/` and `crates/term-wm-ui-components/` invoke the raw constructor directly.

## Approach

### 1. Downgrade `with_config` to `pub(crate)`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs:490`

```rust
pub(crate) fn with_config(
```

This keeps all 58 internal callers (in `config.rs`, `runner.rs`, `window_manager/mod.rs` tests) working. External code must use `AppBuilder`.

### 2. Remove `with_config` from public re-exports

**File:** `crates/term-wm-core/src/window/mod.rs`

If `with_config` appears in a `pub use window_manager::*` or explicit re-export, remove it. The `WindowManager` struct itself stays public; only the constructor becomes crate-private.

### 3. Migrate 5 external call sites to `AppBuilder`

All external callers already have `AppBuilder` accessible (via `term_wm::config::AppBuilder` or `term_wm_core::config::AppBuilder`).

#### 3a. `tests/integration_basic.rs:26` — Full chrome

```rust
// Before:
let mut wm = WindowManager::with_config(
    WmConfig::standalone(), ctx,
    Some(top), Some(bottom), Some(menu), None,
);

// After:
let mut wm = AppBuilder::bare()
    .config(WmConfig::standalone())
    .app_ctx(ctx)
    .top_panel(top)
    .bottom_panel(bottom)
    .command_menu(menu)
    .build()
    .expect("test build");
```

#### 3b. `tests/integration_new_window_focus.rs:19` — Full chrome

Same pattern as 3a.

#### 3c. `tests/panic_debug_log.rs:114` — Menu only

```rust
let mut wm = AppBuilder::bare()
    .config(WmConfig::standalone())
    .app_ctx(Arc::new(AppContext::new("test", "0.0.0")))
    .command_menu(menu)
    .build()
    .expect("test build");
```

#### 3d–3e. `crates/term-wm-ui-components/src/terminal.rs:1582,1679` — Bare

```rust
let mut wm = AppBuilder::bare()
    .config(WmConfig::standalone())
    .app_ctx(Arc::new(AppContext::new("test", "0.0.0")))
    .build()
    .expect("test build");
```

### 4. Add missing import to `term-wm-ui-components` tests

`terminal.rs` tests need `use term_wm_core::config::AppBuilder;` added.
