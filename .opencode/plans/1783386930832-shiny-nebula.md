# Eradicate `AppMode` Typestate — Monomorphic Builder

## Problem

The `AppBuilder<M: AppMode>` generic over `Standalone`/`Embedded` markers is dead weight. PTY isolation makes both modes operationally identical — the child process doesn't know it's nested. The typestate adds API friction and compilation overhead for zero benefit. Additionally, `WmConfig::embedded()` and `KeyBindings::embedded()` use "embedded" terminology internally but this word doesn't appear in the public API — naming must be consistent.

## Changes

### 1. `crates/term-wm-core/src/wm_config.rs`

Rename `WmConfig::embedded()` → `WmConfig::minimal()` and update doc comment.

### 2. `crates/term-wm-core/src/keybindings.rs`

Rename `KeyBindings::embedded()` → `KeyBindings::minimal()` and update doc comment.

### 3. `crates/term-wm-core/src/config.rs` — Full rewrite

Delete: `Standalone`, `Embedded`, `sealed` module, `AppMode` trait, `PhantomData`, all typestate impl blocks.

New content:

```rust
use std::sync::Arc;

use crate::app_context::AppContext;
use crate::components::WmComponent;
use crate::keybindings::KeyBindings;
use crate::theme::Theme;
use crate::window::decorator::WindowDecorator;
use crate::window::WindowManager;
use crate::wm_config::{HintVisibility, WmConfig};

#[derive(Debug)]
pub enum ConfigError {
    MissingAppContext,
}

impl std::fmt::Display for ConfigError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            ConfigError::MissingAppContext => write!(f, "missing AppContext in AppBuilder"),
        }
    }
}

impl std::error::Error for ConfigError {}

pub struct AppBuilder {
    config: WmConfig,
    app_ctx: Option<Arc<AppContext>>,
    top_panel: Option<Box<dyn WmComponent>>,
    bottom_panel: Option<Box<dyn WmComponent>>,
    command_menu: Option<Box<dyn WmComponent>>,
}

impl AppBuilder {
    /// Blank canvas — full standalone config, no chrome injected.
    /// Use `.config(WmConfig::minimal())` for a minimal preset.
    pub fn bare() -> Self {
        Self {
            config: WmConfig::default(),
            app_ctx: None,
            top_panel: None,
            bottom_panel: None,
            command_menu: None,
        }
    }

    pub fn app_ctx(mut self, ctx: Arc<AppContext>) -> Self {
        self.app_ctx = Some(ctx);
        self
    }

    pub fn config(mut self, config: WmConfig) -> Self {
        self.config = config;
        self
    }

    pub fn theme(mut self, theme: Theme) -> Self {
        self.config.theme = theme;
        self
    }

    pub fn keybindings(mut self, kb: KeyBindings) -> Self {
        self.config.keybindings = kb;
        self
    }

    pub fn decorator(mut self, decorator: Arc<dyn WindowDecorator>) -> Self {
        self.config.decorator = Some(decorator);
        self
    }

    pub fn hint_visibility(mut self, v: HintVisibility) -> Self {
        self.config.hint_visibility = v;
        self
    }

    pub fn top_panel(mut self, panel: Box<dyn WmComponent>) -> Self {
        self.top_panel = Some(panel);
        self
    }

    pub fn bottom_panel(mut self, panel: Box<dyn WmComponent>) -> Self {
        self.bottom_panel = Some(panel);
        self
    }

    pub fn command_menu(mut self, menu: Box<dyn WmComponent>) -> Self {
        self.command_menu = Some(menu);
        self
    }

    pub fn mouse_capture(mut self, enabled: bool) -> Self {
        self.config.mouse_capture_enabled = enabled;
        self
    }

    pub fn floating_windows(mut self, enabled: bool) -> Self {
        self.config.floating_windows_enabled = enabled;
        self
    }

    pub fn chrome(mut self, enabled: bool) -> Self {
        self.config.chrome_enabled = enabled;
        self
    }

    pub fn panel(mut self, enabled: bool) -> Self {
        self.config.panel_enabled = enabled;
        self
    }

    pub fn build(self) -> Result<WindowManager, ConfigError> {
        let app_ctx = self.app_ctx.ok_or(ConfigError::MissingAppContext)?;
        Ok(WindowManager::with_config(
            self.config,
            app_ctx,
            self.top_panel,
            self.bottom_panel,
            self.command_menu,
        ))
    }
}
```

### 4. `src/lib.rs`

Delete `standalone_builder()` function (was only used by `TermWmApp::new` and `WindowManagerExt::new_standalone`). Delete `WindowManagerExt` trait and impl (no callers in codebase).

Replace with:

```rust
pub use term_wm_core::*;
pub use term_wm_ui_components::*;
pub mod prelude;
pub mod term_wm_app;
pub mod tracing_sub;
pub mod widget_adapter;
pub use widget_adapter::{WidgetAdapter, StatefulWidgetAdapter};
```

### 5. `src/term_wm_app.rs`

| Method | Before | After |
|--------|--------|-------|
| `new()` (sys-ui) | `AppBuilder::bare_standalone()` | `AppBuilder::bare()` + inject components |
| `new()` (no sys-ui) | `Self::bare(app_ctx)` | unchanged |
| `bare()` | `AppBuilder::bare_standalone()` | `AppBuilder::bare()` |
| `embedded()` | `AppBuilder::embedded()` | `AppBuilder::bare().config(WmConfig::minimal())` |

Note: `AppBuilder::bare()` already initializes with `WmConfig::default()` (= `WmConfig::standalone()`), so no `.config()` call needed for the standalone path.

### 6. `src/main.rs`

| Before | After |
|--------|-------|
| `AppBuilder::embedded()` | `AppBuilder::bare().config(WmConfig::minimal())` |
| `AppBuilder::bare_standalone()` | `AppBuilder::bare()` + inject components |
