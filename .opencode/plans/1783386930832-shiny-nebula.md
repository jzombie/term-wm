# Refined Plan: Unified Layout Interface + Typestate Builder

## Core Architectural Change

**Eradicate positional traits.** Delete `TopPanel<Id>`, `BottomPanel`, and `MenuOverlay<Msg>` as independent traits. Replace with a single `WmComponent` trait where layout negotiation, rendering, event handling, and action processing are standard methods. A component's position (top, bottom, overlay) is determined by where it is injected within the layout graph, not by its trait definition.

**Strict dependency inversion.** The core engine (`term-wm-core`) must never know about, depend on, or downcast to concrete component types defined in upper-level crates (`term-wm-sys-ui-components`). All communication between the engine and components flows through the `WmComponent` trait interface.

## Phase 1: Unified `WmComponent` Trait

**File:** `crates/term-wm-core/src/components/mod.rs`

Replace all panel/overlay traits with a single base trait:

```rust
pub trait WmComponent: std::fmt::Debug {
    /// Carve out the component's required space from the available area.
    /// Returns (claimed_rect, remaining_rect).
    /// Default: claims nothing, passes through full area.
    fn consume_area(&mut self, available: Rect) -> (Rect, Rect) {
        (Rect::default(), available)
    }

    /// Render the component into the claimed area.
    /// The `area` parameter is the rect returned by `consume_area`.
    /// The `registry` is used to register clickable regions for mouse hit-testing.
    fn render(
        &mut self,
        frame: &mut UiFrame<'_>,
        area: Rect,
        ctx: &ComponentContext,
        registry: &mut crate::hitbox_registry::HitboxRegistry,
    );

    /// Handle an event before it reaches the window graph.
    /// Returns `EventResult::Ignored` if not consumed.
    fn handle_event(
        &mut self,
        _event: &Event,
        _ctx: &ComponentContext,
    ) -> EventResult<TermWmAction> {
        EventResult::Ignored
    }

    /// Process a system-level action broadcasted by the window manager.
    /// Components handle internally — no downcasting required.
    fn process_action(&mut self, _action: &ComponentAction) {}

    /// Query the component for a result after processing.
    /// Used for actions that need a response (e.g., selected menu item).
    fn query(&self, _query: &ComponentQuery) -> ComponentResponse {
        ComponentResponse::None
    }

    /// Mouse hit-test. Returns true if (x, y) is within this component.
    fn hit_test(&self, _x: u16, _y: u16) -> bool {
        false
    }

    /// Begin a new frame (clear per-frame state).
    fn begin_frame(&mut self) {}

    /// Visible flag (layout engine skips invisible components).
    fn visible(&self) -> bool {
        true
    }

    /// Set visible flag.
    fn set_visible(&mut self, _visible: bool) {}
}
```

**Action and Query enums (core-defined, no concrete types):**

```rust
/// Actions the engine broadcasts to components.
#[derive(Debug, Clone)]
pub enum ComponentAction {
    Restore,
    Outline,
    SetMenuItems(Vec<MenuItem<TermWmAction>>),
    SetMenuAnchor(Option<(u16, u16)>),
    SetManagedArea(Rect),
    SetKeybindingHints(Vec<(TermWmAction, Vec<String>)>),
    SetPowerProfile(PowerProfile),
    SetHintVisibility(HintVisibility),
    ToggleVisibility,
}

/// Queries the engine can ask components.
#[derive(Debug)]
pub enum ComponentQuery {
    SelectedAction,
    KeybindingHints,
    MenuIconRect,
}

/// Responses from component queries.
#[derive(Debug)]
pub enum ComponentResponse {
    None,
    Action(Option<TermWmAction>),
    Hints(Vec<(TermWmAction, Vec<String>)>),
    Rect(Option<Rect>),
}
```

**Key design principle:** The `ComponentAction` and `ComponentQuery` enums are defined in `term-wm-core`. They contain no concrete types. Components in upper-level crates pattern-match on these enums and handle them internally.

**`MenuItem` retained:** `MenuItem` is a pure data struct (not a trait). It stays in `term-wm-core::components::mod` as the payload for `ComponentAction::SetMenuItems`. Only the `Overlay` and `MenuOverlay` traits are deleted.

**No `as_any`/`as_any_mut`:** The `WmComponent` trait deliberately excludes downcast methods. The core engine communicates exclusively through `process_action` and `query`. This enforces strict polymorphism — the caller never knows the concrete type.

**Files to delete:**
- `crates/term-wm-core/src/top_panel_trait.rs`
- `crates/term-wm-core/src/bottom_panel_trait.rs`

**Files to update:**
- `crates/term-wm-core/src/lib.rs` — remove `pub mod top_panel_trait;` and `pub mod bottom_panel_trait;`
- `crates/term-wm-core/src/components/mod.rs` — remove `Overlay` and `MenuOverlay` traits; retain `MenuItem` struct; add `ComponentAction`, `ComponentQuery`, `ComponentResponse`

## Phase 2: Migrate Panel Implementations

### `WmTopPanelComponent`

**File:** `crates/term-wm-sys-ui-components/src/wm_top_panel.rs`

- Remove `impl<R> TopPanelTrait<R> for WmTopPanelComponent<R>` block
- Implement `WmComponent` instead:
  - `consume_area()` calls internal `split_area()`, returns `(claimed_rect, remaining_rect)`
  - `render()` receives the claimed rect from the compositor
  - `process_action()` handles `ToggleVisibility`, `SetHintVisibility`
  - `query()` handles `MenuIconRect` (returns the menu icon rect for overlay positioning)
  - `hit_test()` checks menu icon, window tabs, notification icons
  - `begin_frame()`, `visible()`, `set_visible()` delegate to existing methods

### `WmBottomPanelComponent`

**File:** `crates/term-wm-sys-ui-components/src/wm_bottom_panel.rs`

- Remove `impl BottomPanelTrait for WmBottomPanelComponent` block
- Implement `WmComponent` instead:
  - `consume_area()` reserves 1 row at the bottom for hints, returns `(claimed_rect, remaining_rect)`
  - `render()` receives the claimed rect from the compositor
  - `process_action()` handles `SetKeybindingHints`, `SetPowerProfile`
  - `query()` handles `KeybindingHints`
  - `hit_test()` checks hint rects

### `WmMenuOverlay`

**File:** `crates/term-wm-sys-ui-components/src/wm_menu_overlay.rs`

- Remove `impl MenuOverlay<TermWmAction> for WmMenuOverlay` block
- Remove `impl Overlay<TermWmAction> for WmMenuOverlay` block
- Implement `WmComponent` instead:
  - `consume_area()` — overlays render on top, claim no area: returns `(Rect::default(), available)`
  - `render()` — renders the overlay menu at its anchor position
  - `handle_event()` — intercepts keyboard/mouse events when menu is visible
  - `process_action()` handles:
    - `Restore` -> calls internal `restore()`
    - `Outline` -> calls internal `outline()`
    - `SetMenuItems(items)` -> calls internal `set_items(items)`
    - `SetMenuAnchor(pos)` -> calls internal `set_anchor(pos)`
    - `SetManagedArea(area)` -> calls internal `set_managed_area(area)`
    - `ToggleVisibility` -> toggles visibility
  - `query()` handles `SelectedAction` -> returns the selected action
  - `hit_test()` — checks if click is within the menu bounds
  - `visible()` — returns `self.visible`

**No downcasting.** All menu-specific behavior is handled through `process_action` and `query`.

## Phase 5: `AppBuilder<M>` with Typestates + Feature Flags

**File:** `crates/term-wm-core/src/config.rs`

```rust

    config: AppConfig,

    _mode: PhantomData<M>,
}
```

### Feature Flag Architecture

**`Cargo.toml` (top-level crate only):**
```toml
[features]
default = ["sys-ui"]
sys-ui = ["dep:term-wm-sys-ui-components"]

[dependencies]
term-wm-core = { path = "crates/term-wm-core" }
term-wm-sys-ui-components = { path = "crates/term-wm-sys-ui-components", optional = true }
```

**SemVer Safety Rule:** Feature-gating is restricted to **concrete types and constructors only**. The `WmComponent` trait, `AppConfig` struct fields, and all builder configuration methods are **unconditionally compiled** in `term-wm-core`. This prevents downstream breakage when toggling features.

**`term-wm-core` has NO dependency on `term-wm-sys-ui-components`.** The feature-gated constructors live in the top-level crate where both crates can be legally combined.

### Constructors (in `term-wm-core` — unconditional)

```rust
impl AppBuilder<Standalone> {
```

```rust
pub fn register_managed_layout(&mut self, area: Rect) {
    self.last_frame_area = area;
    // ... hint computation via process_action ...

    let mut remaining = area;
    if let Some(p) = &mut self.top_component {
        if p.visible() {
            let (claimed, rest) = p.consume_area(remaining);
            self.top_claimed = claimed;
            remaining = rest;
        }

    if let Some(p) = &mut self.bottom_component {
        if p.visible() {
            let (claimed, rest) = p.consume_area(remaining);
            self.bottom_claimed = claimed;
            remaining = rest;
        }
    }
    self.managed_area = remaining;
    // ... rest of layout ...
}
```
