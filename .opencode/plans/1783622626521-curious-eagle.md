# Record/Replay & GIF Automation Plan

## Goal
Add CLI flags to record mouse/keyboard interactions, replay them headless/live, and capture rendered output to asciicast `.cast` files for GIF conversion via `agg`.

Key insight from `HitboxRegistry` architecture: recording captures **semantic HitTargets** (not raw coordinates), and playback reverse-lookups current positions from the registry — making macros resilient to layout changes and aspect ratio differences.

## Phase 2: Serializable Macro Format

**New: `src/macro_format.rs`**

```rust
#[derive(Serialize, Deserialize, Clone)]
pub enum RecordedAction {
    /// Keyboard event (inherently semantic — no coordinates)
    KeyPress(KeyCode, KeyModifiers, KeyKind),
    /// Mouse click on a semantic target (buttons, resize handles, etc.)
    Click { target: SerializableHitTarget, offset_dx: i16, offset_dy: i16 },
    /// Drag-and-drop between two semantic targets
    /// Destination can be a target OR None (drop in empty space)
    DragAndDrop {
        source: SerializableHitTarget,
        source_dx: i16,         // offset within source on grab
        source_dy: i16,
        destination: Option<SerializableHitTarget>,
        dest_dx: i16,           // offset within dest on drop (0 if None)
        dest_dy: i16,
        interpolated_moves: u32, // number of intermediate DragMove events to generate
    },
    /// Timing delay before next action
    DelayMs(u64),
}

/// Serializable mirror of HitTarget — uses window indices (not slotmap keys)
#[derive(Serialize, Deserialize, Clone)]
pub enum SerializableHitTarget {
    Window(usize),
    Component(usize, u64),
    Overlay(String),
    TopPanel,
    BottomPanel,
    ChromeResize(usize, ResizeEdge),
    ChromeHeader(usize, HeaderAction),
    LayoutHandle,
}
```

### Recording semantics:

- **Click**: MouseDown + MouseUp within a single target, no significant drag. Record `offset_dx/dy` within target rect.
- **DragAndDrop**: MouseDown on `source`, then MouseUp either on another `destination` target, or at a position that maps to no target (`destination = None`). `interpolated_moves` = number of frames the drag lasted × the frame pacing (e.g., 16ms per move at 60fps → 250 moves for a 4-second drag). `source_dx/dy` and `dest_dx/dy` are the offsets within each target's rect.
- **DragMove intermediate events are NOT stored** — only source and destination. Playback interpolates the path.

## Phase 3: HitboxRegistry Additions + Recording Interceptor

### 3a: find_target_area (reverse lookup)

**File: `crates/term-wm-core/src/hitbox_registry.rs`**

```rust
impl HitboxRegistry {
    /// Find the current screen area of a target (reverse of hit_test).
    /// Scans in reverse (topmost first) for correct z-order.
    pub fn find_target_area(&self, target: &HitTarget) -> Option<LayoutRect> {
        self.entries.iter().rev().find(|e| e.target == *target).map(|e| e.area)
    }
}
```

### 3c: Recording interceptor (in runner.rs)

Hooks inside `run_event_loop`'s event dispatch, at the layer where `HitboxRegistry` is live after render.

**`handle_focused_app_event` (recording mode additions):**

```
OnMouseDown:
  hit_test result = dispatch_mouse returns (HitTarget, rect)
  if hit:
    save drag_source = hit_target, source_dx/offset = mouse_pos - rect.origin
    recording_session.start_drag(hit_target, offset_dx, offset_dy)

OnMouseMove (during drag):
  // Not recorded — intermediate moves are interpolated during playback

OnMouseUp:
  if drag_active:
    hit_test result = dispatch_mouse
    if hit:
      dest_target = Some(convert(hit_target))
      dest_dx = mouse_pos - rect.x, dest_dy = mouse_pos - rect.y
    else:
      dest_target = None, dest_dx = 0, dest_dy = 0
    recording_session.push(DragAndDrop {
      source, source_dx, source_dy,
      destination: dest_target, dest_dx, dest_dy,
      interpolated_moves: elapsed_drag_frames,
    })
    drag_active = false

OnKeyEvent:
  recording_session.push(KeyPress(code, modifiers, kind))
```

## Phase 4: Playback with Reverse Lookup

**New: `crates/term-wm-core/src/io/playback_event_source.rs`**

Implements `EventSource` with semantic-to-coordinate reconstruction:

```rust
pub struct PlaybackEventSource {
    actions: Vec<RecordedAction>,
    index: usize,
    /// Active drag interpolation state
    drag_interp: Option<DragInterpolation>,
    last_event: Instant,
    registry: Rc<RefCell<HitboxRegistry>>,
    window_map: HashMap<usize, WindowKey>,
    done: bool,
}
```

**`read()` processes one action:**

1. **`DelayMs(d)`** → `thread::sleep(d)`, advance, poll again
2. **`KeyPress(...)`** → return `Event::Key(...)`
3. **`Click { target, offset_dx, offset_dy }`**:
   - Resolve `SerializableHitTarget` → `HitTarget` via `window_map`
   - `find_target_area(hit_target)` → get current rect
   - Construct `MouseEvent { column: rect.x + offset_dx, row: rect.y + offset_dy, ... }`
   - Emit MouseDown, then immediately emit MouseUp (both on same position)
4. **`DragAndDrop { source, destination, ... }`**:
   - Resolve source → current rect (position A)
   - Resolve destination → current rect (position B), or use A + (dest_dx, dest_dy) if `None`
   - Compute source_center = A.origin + (source_dx, source_dy)
   - Compute dest_center = (B.origin + (dest_dx, dest_dy)) if B exists, else A.origin + (dest_dx, dest_dy)
   - Set `drag_interp` with (source_center, dest_center, moves_remaining, current_step)
   - Return MouseDown at source_center
5. **When `drag_interp` active** (next read):
   - Interpolate: `t = step / total_moves`
   - `x = lerp(source.x, dest.x, t)`, `y = lerp(source.y, dest.y, t)`
   - Return `Event::Mouse(DragMove)` at interpolated position
   - If last step, clear `drag_interp`, return Drop event

After all actions: `poll()` returns `Ok(false)` forever.

## Phase 6: CLI Integration

**File: `src/main.rs`**

```rust
#[arg(long)]
record: Option<Option<String>>,    // --record [FILE]

#[arg(long)]
playback: Option<String>,          // --playback FILE

#[arg(long)]
capture: Option<String>,           // --capture FILE (.cast)
```

**`main()` logic:**

1. `--playback FILE` → `PlaybackEventSource` with `Rc<RefCell<HitboxRegistry>>` + window map. Optionally wrapped in `CapturedConsoleRenderTarget` if `--capture`.
2. `--record FILE` → `UnifiedEventSource` + `RecordingSession` hooked into runner's dispatch.
3. `--capture FILE` (no playback) → normal `UnifiedEventSource` + `CapturedConsoleRenderTarget`.
