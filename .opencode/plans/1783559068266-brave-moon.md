## SEV-1: BSP Tree Fracture (Phantom Windows)

### Problem
`LayoutNode::remove_leaf` removes the leaf but does not collapse the parent
split. Surviving siblings remain orphaned in dead split containers, creating
void regions that render as "phantom windows".

### Solution
Implement parent contraction on `LayoutNode` in the **core crate**, not
`term-wm-layout-engine::node.rs` (which is a separate, unused type).

**File:** `crates/term-wm-core/src/layout/tiling.rs` — `LayoutNode` impl

Add to `remove_leaf`:

```rust
pub fn remove_leaf(&mut self, target: Id) -> bool {
    match self {
        LayoutNode::Split { children, .. } => {
            let before = children.len();
            children.retain(|child| !child.is_leaf_eq(target));
            if children.len() == before {
                // Target not in this level — recurse
                for child in children.iter_mut() {
                    if child.remove_leaf(target) {
                        // After removal, check for contraction
                        self.contract_children();
                        return true;
                    }
                }
                false
            } else {
                self.contract_children();
                true
            }
        }
        _ => false,
    }
}
```

Add contraction:

```rust
/// After child removal, collapse degenerate splits:
/// - 0 children → replace with Void
/// - 1 child → replace with that child
pub fn contract_children(&mut self) {
    match self {
        LayoutNode::Split { children, .. } => {
            if children.is_empty() {
                *self = LayoutNode::void();
            } else if children.len() == 1 {
                let sole = children.swap_remove(0);
                *self = sole;
            }
        }
        _ => {}
    }
}
```

Run `contract_children` after every `remove_leaf` and after window
transitions to `Unmapped` or floating detachment.

Add void counter cleanup: when a `Void` is removed during contraction,
decrement is not needed (void IDs are unique atomic counters). When a
new `Void` replaces a collapsed split, keep the same void ID generation.

## SEV-1: Scale-Invariant Quadrant Detection (Phase 3)

### Problem
`detect_edge_snap` at `drag.rs:254` uses edge-proximity for tiled pane
insertion. For 80×4 panes, vertical BSP splits are impossible across
95% of the surface.

### Solution
Add `detect_tiled_quadrant` to `crates/term-wm-layout-engine/src/hit_test.rs`:

```rust
/// Scale-invariant cross-product quadrant detection.
/// Both delta and bounding-box dimensions are in cell units —
/// the factor cancels, making the comparison invariant to aspect ratio.
pub fn detect_tiled_quadrant(
    cursor_x: u16,
    cursor_y: u16,
    target_rect: LayoutRect,
) -> InsertPosition
```

Algorithm (verified correct):
1. `cx = target_rect.x + target_rect.width / 2`
2. `cy = target_rect.y + target_rect.height / 2`
3. `dx = cursor_x - cx` (signed i32), `dy = cursor_y - cy`
4. `scaled_dx = |dx| * target_rect.height`
5. `scaled_dy = |dy| * target_rect.width`
6. If `scaled_dx > scaled_dy` → East (`dx >= 0`) / West; else → South (`dy >= 0`) / North
7. Ties resolve to East

**Call site change** — only replace the tiled-insert detection,
NOT the global edge snap:

```rust
// drag.rs:254 ⸺ tiled insert path (REPLACE):
let position = detect_tiled_quadrant(mouse_x, mouse_y, target_layout);

// drag.rs:283 ⸺ global edge snap (KEEP):
let position = detect_edge_snap(mouse_x, mouse_y, managed_layout_rect, 6);
```

Note: The `detect_quadrant` function claimed by the reviewer to already
exist in `hit_test.rs` does NOT exist in the current codebase at HEAD.
The file is 60 lines and only contains `hit_test_leaf`. The code graph
was stale.

## SEV-2: Drag Offset Desync on Kinetic Restore (Phase 1)

### Problem
At `drag.rs:967-977`, `toggle_maximize` updates `initial_x`/`initial_y`
but not `start_x`/`start_y`. Next frame's `move_floating` uses stale
anchor, producing a violent jump.

### Solution
Both drag origin AND mouse anchor must be reset after restore:

```rust
self.toggle_maximize(key);
if let Some(crate::window::FloatRectSpec::Absolute(fr)) = self.floating_rect(key)
    && let Some(&mut MouseCaptureState::DraggingWindow {
        ref mut initial_x,
        ref mut initial_y,
        ref mut start_x,
        ref mut start_y,
        ..
    }) = self.mouse_capture.as_mut()
{
    *initial_x = fr.x;
    *initial_y = fr.y;
    *start_x = col;
    *start_y = row;
}
```

This forces `delta_x = col - start_x = 0` on the immediate next frame.

## SEV-2: Self-Arming Timer with Proliferation Guard (Phase 4)

### Problem
Repeated `move_floating` calls during magnetic zone micro-movements
spawn concurrent 50ms timers, causing CPU starvation.

### Solution
Add `temporal_timer_id: Option<TaskId>` to `WindowManager`. The real
API is `handle.schedule_once(after, task)` where `task` is raw `SystemTask`:

```rust
// mod.rs — WindowManager struct:
pub(crate) temporal_timer_id: Option<TaskId>,

// Arm:
pub fn arm_temporal_dwell_timer(&mut self) {
    use crate::task_scheduler::TaskMessage;
    // Cancel existing timer before creating new one
    if let Some(id) = self.temporal_timer_id.take() {
        if let Some(handle) = &self.system_task_handle {
            let _ = handle.cancel(id);
        }
    }
    if let Some(handle) = &self.system_task_handle {
        self.temporal_timer_id = Some(
            handle.schedule_once(
                Duration::from_millis(50),
                SystemTask::TemporalDwellTick,
            )
        );
    }
}

// On TemporalDwellTick:
// 1. Check self.mouse_capture — if still DraggingWindow + magnetic zone:
//      → re-arm (arm_temporal_dwell_timer again)
//      → schedule render frame
// 2. If not: self.temporal_timer_id = None
```

## SEV-2: Euclidean vs Manhattan Metric Segregation (Phase 1)

### Problem
Deadzone guard uses Manhattan `dx + dy <= 2` — applying aspect-ratio
weighting designed for `dist_sq` to a linear sum is invalid.

### Solution
Two separate metrics. Only Euclidean-squared paths get aspect-weight:

| Path | File:Line | Metric | Weight Y×2? |
|------|-----------|--------|-------------|
| Deadzone guard | `mod.rs:957` | Manhattan `dx + dy <= 2` | **No** |
| Snap suppression | `drag.rs:182` | Euclidean `dx² + dy² < 25` | **Yes** |
| Velocity | `mod.rs:1003` | Euclidean `dist_sq` | **Yes** |
