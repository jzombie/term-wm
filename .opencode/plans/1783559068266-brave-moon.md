## SEV-1: BSP Tree Fracture (Phantom Windows)

### Problem
`remove_leaf` doesn't contract parent splits. `weights` and `constraints`
are parallel arrays to `children` — isolated `retain()` desyncs them.

### Solution: Indexed Parallel Removal

**`remove_leaf`** — pure search, replaces leaf with Void, returns bool:

```rust
pub fn remove_leaf(&mut self, target: Id) -> bool {
    match self {
        LayoutNode::Leaf(id) if *id == target => {
            *self = LayoutNode::Void(VOID_ID_COUNTER.fetch_add(1, Ordering::Relaxed));
            true
        }
        LayoutNode::Split { children, .. } => {
            let mut found = false;
            for child in children.iter_mut() {
                if child.remove_leaf(target) { found = true; }
            }
            found
        }
        _ => false,
    }
}
```

**`contract_tree`** — post-processing with synced parallel vector removal:

```rust
/// Post-order contraction. Maintains parallel arrays: when a child is
/// removed, its weight and constraint are also removed at the same index.
pub fn contract_tree(&mut self) {
    let replacement = match std::mem::replace(
        self,
        LayoutNode::Void(VOID_ID_COUNTER.fetch_add(1, Ordering::Relaxed)),
    ) {
        LayoutNode::Split { direction, mut children, mut weights, mut constraints, resizable } => {
            // Recurse first
            for child in &mut children {
                child.contract_tree();
            }
            // Indexed removal — keeps weights/constraints in sync with children.
            // Use matches! instead of fictional is_void() predicate.
            let mut i = 0;
            while i < children.len() {
                if matches!(children[i], LayoutNode::Void(_)) {
                    children.remove(i);
                    if i < weights.len() { weights.remove(i); }
                    if i < constraints.len() { constraints.remove(i); }
                } else {
                    i += 1;
                }
            }
            // Contract
            match children.len() {
                0 => LayoutNode::Void(VOID_ID_COUNTER.fetch_add(1, Ordering::Relaxed)),
                1 => children.swap_remove(0),
                _ => LayoutNode::Split { direction, children, weights, constraints, resizable },
            }
        }
        other => other,
    };
    *self = replacement;
}
```

**Integration**: Call `contract_tree()` after every `remove_leaf` in
`apply_snap`, `detach_to_floating`, `tile_window_key`, and transition
to `Unmapped`.

## SEV-1: Scale-Invariant Quadrant Detection (Phase 3)

### Problem
`detect_edge_snap` at `drag.rs:254` uses edge-proximity → fails for 80×4.

### Solution

**`detect_tiled_quadrant` in `hit_test.rs`:**

```rust
pub fn detect_tiled_quadrant(
    cursor_x: u16, cursor_y: u16,
    target_rect: LayoutRect,
) -> InsertPosition {
    // x is i32, width is u16 — explicit cast required
    let cx = target_rect.x + i32::from(target_rect.width / 2);
    let cy = target_rect.y + i32::from(target_rect.height / 2);
    let dx = i32::from(cursor_x) - cx;
    let dy = i32::from(cursor_y) - cy;
    let adx = dx.unsigned_abs().saturating_mul(u32::from(target_rect.height));
    let ady = dy.unsigned_abs().saturating_mul(u32::from(target_rect.width));
    if adx > ady || (adx == ady && dx >= 0) {
        if dx >= 0 { InsertPosition::Right } else { InsertPosition::Left }
    } else {
        if dy >= 0 { InsertPosition::Bottom } else { InsertPosition::Top }
    }
}
```

**Call site at `drag.rs:254`:**

```rust
// Replace the edge-proximity + Option guard with direct binding.
```

### Problem
`drag.rs:967-977` updates `initial_x`/`initial_y` but not `start_x`/`start_y`
or `prev_col`/`prev_row`. Stale offsets cause jump + false velocity spike.

### Solution

```rust
if let Some(FloatRectSpec::Absolute(fr)) = self.floating_rect(key) {

        ref mut initial_x, ref mut initial_y,
        ref mut start_x, ref mut start_y,
```

## SEV-2: Self-Arming Timer with Spatial Exit Check (Phase 4)

### Problem
`matches!(DraggingWindow)` fires for ALL dragging, even outside magnetic
zone. Timer cycles forever draining CPU.

### Solution
Check `entered_magnetic_*_at` fields to confirm actual magnetic zone
residence before re-arming:

```rust
pub fn on_temporal_dwell_tick(&mut self) {
    let in_magnetic_zone = matches!(
        &self.mouse_capture,
        Some(MouseCaptureState::DraggingWindow {
            resistance,
            ..
        }) if resistance.entered_magnetic_x_at.is_some()
           || resistance.entered_magnetic_y_at.is_some()
    );
    if in_magnetic_zone {
        self.schedule_render();
        self.arm_temporal_dwell_timer();  // Re-arm for another 50ms
    } else {
        self.temporal_timer_id = None;
    }
}
```

API: `handle.schedule_once(after: Duration, task: SystemTask)` — verified.
