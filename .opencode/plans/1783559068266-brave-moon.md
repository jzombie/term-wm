# Window Physics & BSP Integrity Plan (v4)

## Overview

Resolves all identified SEV-1 and SEV-2 defects. Three key architectural
patterns applied throughout: extract-and-restore for borrow safety,
post-processing for tree contraction, and direct binding for quadrant
replacement.

## SEV-1: EdgeResistance State Annihilation (Phase 2)

### Problem
`move_floating` creates `EdgeResistance` on the stack — dropped each frame.
Temporal threshold and hysteresis never persist.

### Solution: Extract-and-Restore Pattern

Cannot hold `ref mut resistance` while calling `&mut self` methods (E0499).
Instead, extract `MouseCaptureState` via `Option::take()`, mutate locally,
then re-insert.

**1a. Remove `Copy` from `MouseCaptureState`**

`mod.rs:42`: `#[derive(Debug, Clone, Copy, PartialEq)]` → `#[derive(Debug, Clone, PartialEq)]`

**1b. Add `resistance` field to `DraggingWindow`**

```rust
DraggingWindow {
    key: WindowKey,
    resistance: term_wm_layout_engine::EdgeResistance,  // NEW
    // ... existing fields unchanged ...
}
```

Initialize at all capture sites with `EdgeResistance::default_tui()`.

**1c. Update `move_floating` — extract, mutate, re-insert**

```rust
pub(super) fn move_floating(&mut self, /* ... */) {
    // 1. Extract the capture state
    let mut capture = match self.mouse_capture.take() {
        Some(MouseCaptureState::DraggingWindow { .. }) => capture,
        _ => { self.mouse_capture = capture; return; }
    };

    // 2. Get the floating rect
    let Some(FloatRectSpec::Absolute(fr)) = self.floating_rect(key) else {
        self.mouse_capture = Some(capture);
        return;
    };

    // 3. Compute new position (no &mut self needed here)
    let dx = column as i32 - start_mouse_x as i32;
    let dy = row as i32 - start_mouse_y as i32;
    let x = initial_x + dx;
    let y = initial_y + dy;

    // 4. Mutate resistance state on the local copy
    let resistance: &mut EdgeResistance = /* ... */;
    let snapped_x = resistance.apply_x(x, bounds_layout, now_ns);
    let snapped_y = resistance.apply_y(y, bounds_layout, now_ns);

    // 5. Update the floating rect (calls mark_layout_dirty)
    self.set_floating_rect(key, Some(FloatRectSpec::Absolute(FloatRect { x: snapped_x, y: snapped_y, width, height })));

    // 6. Re-insert the updated capture state
    self.mouse_capture = Some(capture);
}
```

Same pattern for `dispatch_mouse` event handler: extract `self.mouse_capture`,
call `self.move_floating(...)`, call `self.update_snap_preview(...)` (which
take `&mut self`), then re-insert.

## SEV-1: BSP Tree Fracture (Phantom Windows)

### Problem
`LayoutNode::remove_leaf` removes leaf but doesn't contract parent splits.
Surviving siblings orphaned in dead split containers → phantom windows.

### Solution: Post-Processing Contraction

Two-phase approach to avoid double-borrow:

**Phase A — `remove_leaf` (returns bool, no mutation of parent)**

```rust
/// Remove the leaf matching `target`. Returns true if found and removed.
/// Does NOT contract parent — that's handled by `contract_tree`.
pub fn remove_leaf(&mut self, target: Id) -> bool {
    match self {
        LayoutNode::Leaf(id) if *id == target => {
            *self = LayoutNode::void();  // Replace leaf with void
            true
        }
        LayoutNode::Split { children, .. } => {
            let mut found = false;
            for child in children.iter_mut() {
                if child.remove_leaf(target) {
                    found = true;
                    // Don't break — continue in case of duplicates
                }
            }
            found
        }
        _ => false,
    }
}
```

Leaf replacement uses direct assignment on a discriminant-less pattern
(no `&children` refs active), so it's safe.

**Phase B — `contract_tree` (post-processing pass)**

```rust
/// Post-order contraction: after all removals, collapse degenerate splits.
/// - 0 children → replace with Void
/// - 1 child → replace with that child
pub fn contract_tree(&mut self) -> bool {
    match self {
        LayoutNode::Split { children, .. } => {
            // First, recurse into children
            for child in children.iter_mut() {
                child.contract_tree();
            }
            // Now it's safe to inspect self — all child refs dropped
            let collapsed = match self {
                LayoutNode::Split { children, .. } if children.is_empty() => {
                    std::mem::replace(self, LayoutNode::void());
                    true
                }
                LayoutNode::Split { children, .. } if children.len() == 1 => {
                    let sole = children.swap_remove(0);
                    // Take ownership of self via replace before mutation
                    // to avoid holding refs to children while writing self
                    std::mem::replace(self, sole);
                    true
                }
                _ => false,
            };
            collapsed
        }
        _ => false,
    }
}
```

**Integration**: Call `contract_tree()` after every `remove_leaf` in
`apply_snap`, `detach_to_floating`, `tile_window_key`, and window
state transitions to `Unmapped`.

## SEV-1: Scale-Invariant Quadrant Detection (Phase 3)

### Problem
`detect_edge_snap` at `drag.rs:254` uses edge-proximity for tiled pane
insertion. Fails for 80×4 panes.

### Solution

Add to `hit_test.rs`:

```rust
/// Scale-invariant cross-product. Always returns a position when cursor
/// is within target_rect (guaranteed by the caller's rect_contains check).
pub fn detect_tiled_quadrant(
    cursor_x: u16, cursor_y: u16,
    target_rect: LayoutRect,
) -> InsertPosition {
    let cx = target_rect.x + target_rect.width / 2;
    let cy = target_rect.y + target_rect.height / 2;
    let dx = i32::from(cursor_x) - i32::from(cx);
    let dy = i32::from(cursor_y) - i32::from(cy);
    let adx = dx.unsigned_abs() * u32::from(target_rect.height);
    let ady = dy.unsigned_abs() * u32::from(target_rect.width);

    if adx > ady || (adx == ady && dx >= 0) {
        if dx >= 0 { InsertPosition::Right } else { InsertPosition::Left }
    } else {
        if dy >= 0 { InsertPosition::Bottom } else { InsertPosition::Top }
    }
}
```

**Call site change** — preserve existing variable bindings:

```rust
// drag.rs:254 — before:
let position = detect_edge_snap(mouse_x, mouse_y, target_layout, 6);
if let Some(pos) = position {
    // ... block with pos, preview, self.drag_snap, self.snap_preview, return

// drag.rs:254 — after:
let pos = detect_tiled_quadrant(mouse_x, mouse_y, target_layout);
{
    let preview = self
        .get_projected_preview(dragging_key, SnapPreviewState::TiledInsert(target_key, pos), area)
        .unwrap_or_else(|| {
            let ep = term_wm_layout_engine::tiled_preview_rect(target_layout, pos);
            Rect { x: ep.x, y: ep.y, width: ep.width, height: ep.height }
        });
    self.drag_snap = Some((Some(target_key), pos, preview));
    self.snap_preview = Some(SnapPreviewState::TiledInsert(target_key, pos));
    return;
}
```

The outer `if let Some(pos)` guard is removed because `detect_tiled_quadrant`
always returns a value when the cursor is inside the target rect (guaranteed
by the `rect.contains` check at line 239).

Line 283 (global edge snap) stays unchanged — it remains `detect_edge_snap`.

## SEV-2: Drag Offset Desync on Kinetic Restore (Phase 1)

### Problem
`drag.rs:967-977` updates `initial_*` but not `start_*` → stale anchor jump.

### Solution
After `toggle_maximize`, reset both origin and anchor. Uses extract-and-restore:

```rust
self.toggle_maximize(key);
if let Some(FloatRectSpec::Absolute(fr)) = self.floating_rect(key) {
    // Extract, mutate, re-insert
    if let Some(mut capture) = self.mouse_capture.take() {
        if let MouseCaptureState::DraggingWindow { ref mut initial_x, ref mut initial_y, ref mut start_x, ref mut start_y, .. } = capture {
            *initial_x = fr.x;
            *initial_y = fr.y;
            *start_x = col;
            *start_y = row;
        }
        self.mouse_capture = Some(capture);
    }
}
```

## SEV-2: Self-Arming Timer with Proliferation Guard (Phase 4)

### Solution
Add `temporal_timer_id: Option<TaskId>` to `WindowManager`.
API: `handle.schedule_once(after: Duration, task: SystemTask)`.

```rust
// Arm with guard:
pub fn arm_temporal_dwell_timer(&mut self) {
    if let Some(id) = self.temporal_timer_id.take() {
        if let Some(handle) = &self.system_task_handle {
            let _ = handle.cancel(id);
        }
    }
    if let Some(handle) = &self.system_task_handle {
        self.temporal_timer_id = Some(
            handle.schedule_once(
                Duration::from_millis(50),
                SystemTask::TemporalDwellTick,
            )
        );
    }
}

// On TemporalDwellTick arrival:
pub fn on_temporal_dwell_tick(&mut self) {
    // Check if still in DraggingWindow + magnetic zone
    let still_active = matches!(
        self.mouse_capture,
        Some(MouseCaptureState::DraggingWindow { .. })
    );
    if still_active {
        // Schedule render frame
        self.schedule_render();
        // Re-arm
        self.arm_temporal_dwell_timer();
    } else {
        self.temporal_timer_id = None;
    }
}
```
