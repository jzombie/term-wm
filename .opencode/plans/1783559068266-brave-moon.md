### Problem
`move_floating` stack-allocates `EdgeResistance` — dropped each frame.
Temporal threshold and hysteresis never persist.

### Solution

**1a. Remove `Copy` from `MouseCaptureState`**

`mod.rs:42`: `#[derive(Debug, Clone, Copy, PartialEq)]` → `#[derive(Debug, Clone, PartialEq)]`

**1c. Extract-operate-restore — no RAII guard, explicit restore at all returns**

No `CaptureGuard` struct (which would hold `&mut self.mouse_capture` and block
`&mut self` methods). Instead, extract at the function entry, pass capture
as parameter to helpers, and explicitly restore before every return:

```rust
// In dispatch_mouse, at the Drag handler entry:
let Some(mut capture) = self.mouse_capture.take() else {
    return true;  // No active drag — safe early return, nothing to restore
};

// Operate: pass capture fields to helpers
match &mut capture {
    MouseCaptureState::DraggingWindow { key, resistance, anchor_x, anchor_y, initial_x, initial_y, start_x, start_y, prev_col, prev_row, prev_time_ns, detach_coordinate } => {
        let velocity_exceeded = /* ... compute from prev_* and prev_time_ns ... */;
        self.move_floating(*key, col, row, *start_x, *start_y,
            *initial_x, *initial_y, velocity_exceeded, resistance);
        // Update prev tracking on capture (not self.mouse_capture)
        *prev_col = col;
        *prev_row = row;
        *prev_time_ns = now_ns;
    }
    MouseCaptureState::ResizingWindow { .. } => { /* existing resize logic */ }
    _ => {}
}
// restore before update_snap_preview call (capture is free — all field borrows dropped)
self.update_snap_preview(key, col, row, &mut capture);

// Restore before every return path:
self.mouse_capture = Some(capture);
return true;
```

Key safety properties:
- Extract at top: `self.mouse_capture = None` throughout the method body
- `&mut self` methods (move_floating, update_snap_preview) compile because
  no outstanding borrow on `self.mouse_capture`
- Every `return` is preceded by `self.mouse_capture = Some(capture)`
- Early returns before extraction are safe (capture was already `None`)

## SEV-1: BSP Tree Fracture (Phantom Windows)

### Problem
`remove_leaf` doesn't contract parent splits → orphaned nodes render as gaps.

### Important: Void preservation
`LayoutNode::Void` is an intentional structural primitive (snap-assist drop
targets, corner-insert placeholders). Tree contraction must NOT purge Void
children from multi-child splits. Only degenerate splits (0 or 1 child)
should be collapsed.

### Solution: `std::mem::replace` for safe enum mutation

**`remove_leaf`** — pure search, void replacement, returns bool:

```rust
pub fn remove_leaf(&mut self, target: Id) -> bool {
    match self {
        LayoutNode::Leaf(id) if *id == target => {
            *self = LayoutNode::Void(VOID_ID_COUNTER.fetch_add(1, Ordering::Relaxed));
            true
        }
        LayoutNode::Split { children, .. } => {
            let mut found = false;
            for child in children.iter_mut() {
                if child.remove_leaf(target) { found = true; }
            }
            found
        }
        _ => false,
    }
}
```

**`contract_tree`** — collapses degenerate splits, preserves Void children:

```rust
/// Post-order: after removal, collapse only degenerate splits.
/// - 0 children → Void
/// - 1 child → replace with that child
/// - 2+ children → leave intact (preserves intentional void placeholders)
pub fn contract_tree(&mut self) {
    let replacement = match std::mem::replace(
        self,
        LayoutNode::Void(VOID_ID_COUNTER.fetch_add(1, Ordering::Relaxed)),
    ) {
        LayoutNode::Split { direction, mut children, weights, constraints, resizable }
            if children.is_empty() =>
        {
            LayoutNode::Void(VOID_ID_COUNTER.fetch_add(1, Ordering::Relaxed))
        }
        LayoutNode::Split { direction, mut children, weights, constraints, resizable }
            if children.len() == 1 =>
        {
            // Recurse into the sole child before replacing
            children[0].contract_tree();
            children.swap_remove(0)
        }
        LayoutNode::Split { direction, mut children, weights, constraints, resizable } => {
            // 2+ children: recurse but don't contract
            for child in &mut children {
                child.contract_tree();
            }
            LayoutNode::Split { direction, children, weights, constraints, resizable }
        }
        other => other,
    };
    *self = replacement;
}
```

**Integration**: Call `contract_tree()` after every `remove_leaf` in
`apply_snap`, `detach_to_floating`, `tile_window_key`, and transition
to `Unmapped`.

## SEV-1: Scale-Invariant Quadrant Detection (Phase 3)

### Problem
`detect_edge_snap` at `drag.rs:254` uses edge-proximity → 80×4 panes make
vertical BSP splits impossible across 95% of surface.

### Note
`detect_quadrant` does NOT exist in `hit_test.rs` at HEAD (60 lines, only
`hit_test_leaf`). The function must be created.

### Solution

Add to `crates/term-wm-layout-engine/src/hit_test.rs`:

```rust
/// Scale-invariant cross-product quadrant detection.
/// Always returns an InsertPosition (caller guarantees cursor inside rect).
pub fn detect_tiled_quadrant(
    cursor_x: u16, cursor_y: u16,
    target_rect: LayoutRect,
) -> InsertPosition {
    let cx = target_rect.x + i32::from(target_rect.width / 2);
    let cy = target_rect.y + i32::from(target_rect.height / 2);
    let dx = i32::from(cursor_x) - cx;
    let dy = i32::from(cursor_y) - cy;
    let adx = dx.unsigned_abs().saturating_mul(u32::from(target_rect.height));
    let ady = dy.unsigned_abs().saturating_mul(u32::from(target_rect.width));
    if adx > ady || (adx == ady && dx >= 0) {
        if dx >= 0 { InsertPosition::Right } else { InsertPosition::Left }
    } else {
        if dy >= 0 { InsertPosition::Bottom } else { InsertPosition::Top }
    }
}
```

Call site at `drag.rs:254`:

```rust
let pos = detect_tiled_quadrant(mouse_x, mouse_y, target_layout);
let preview = self.get_projected_preview(
    dragging_key, SnapPreviewState::TiledInsert(target_key, pos), area,

    let ep = term_wm_layout_engine::tiled_preview_rect(target_layout, pos);
    Rect { x: ep.x, y: ep.y, width: ep.width, height: ep.height }
});
self.drag_snap = Some((Some(target_key), pos, preview));
self.snap_preview = Some(SnapPreviewState::TiledInsert(target_key, pos));

```

## SEV-2: Kinetic Restore on Extracted State (Phase 1)

### Problem
Restore code uses `self.mouse_capture.as_mut()` but field is `None` during
extract. `start_*`/`prev_*` never reset → jump + velocity spike.

### Solution
Operate on capture bindings directly. With extract pattern, the match arm
already binds `initial_x`, `initial_y`, `start_x`, `start_y`, `prev_col`,
`prev_row` as `&mut` references to the local capture:

```rust
// Inside match &mut capture, after toggle_maximize:
if is_maximized && row > anchor_y && row - anchor_y > 2 {
    self.toggle_maximize(key);
    if let Some(FloatRectSpec::Absolute(fr)) = self.floating_rect(key) {
        *initial_x = fr.x;
        *initial_y = fr.y;
        *start_x = col;
        *start_y = row;
        *prev_col = col;
        *prev_row = row;
    }
}
```

All four fields (`initial_*`, `start_*`, `prev_*`) reset atomically against
the extracted capture. No `self.mouse_capture.as_mut()` call.
