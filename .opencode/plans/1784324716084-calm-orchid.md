# Plan: ModalContainer Approach for FAB Menu in Mobile Mode

## Problem
When the viewport is narrow (<80 cols), monocle mode hides the top panel. The command palette overlay's `render_dropdown` bails out because `self.anchor` (derived from top panel's hamburger icon) is `None`. The FAB menu is invisible.

## Solution
Create a reusable `ModalContainer` in `term-wm-ui-components` that uses Ratatui's `Layout` + `Flex::Center` with `Constraint::Length` for centering. Refactor `WmCommandPaletteOverlay::render_dropdown` to use it when `self.anchor` is `None`.

## Architecture

### 1. New: `ModalContainer` in `crates/term-wm-ui-components/src/modal_container.rs`

A free function that:

- **Accepts** `target_width` and `target_height` (exact content dimensions, computed by caller)
- **Computes** a centered rect using `Layout::vertical([Constraint::Length(target_height)]).flex(Flex::Center)` + `Layout::horizontal([Constraint::Length(target_width)]).flex(Flex::Center)`. Ratatui's Layout engine automatically clamps to screen bounds.
- **Renders backdrop** by composing existing `DialogOverlayComponent::render_backdrop` (reuses dimming logic, avoids duplication)
- **Clears** the content area with `ratatui::widgets::Clear` before executing content
- **Calls** provided closure `FnOnce(&mut Buffer, Rect)` to render inner content
- **Returns** the content `LayoutRect` for hit-test caching

```rust
pub fn render_modal(
    backend: &mut dyn term_wm_render::RenderBackend,
    screen_bounds: LayoutRect,
    target_width: u16,
    target_height: u16,
    render_content: impl FnOnce(&mut ratatui::buffer::Buffer, ratatui::layout::Rect),
) -> LayoutRect;
```

No `Component` trait impl needed — rendering primitive only.

### 2. Modify: `WmCommandPaletteOverlay::render_dropdown` in `wm_menu_overlay.rs`

Replace the `let Some(anchor) = self.anchor else { return; }` bail-out with:

- **`self.anchor` is `Some`**: existing anchored dropdown logic (unchanged)
- **`self.anchor` is `None`**: compute `target_width` from max label/icon width, `target_height` from item count; call `render_modal(backend, self.managed_area, target_width, target_height, |buf, rect| { self.menu.render_items(buf, rect, hovered_idx, theme) })`; store returned `LayoutRect` in `self.menu_bounds_cache`

The target dimensions mirror the existing dropdown sizing but without the anchor offset.

### 3. Exports: Update `term-wm-ui-components/src/lib.rs`

Add `pub mod modal_container;` and re-export `render_modal`.
