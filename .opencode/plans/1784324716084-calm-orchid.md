# Plan: Fix FAB Menu Visibility in Mobile Mode

## Problem
When the viewport is narrow (monocle mode, <80 cols), the top panel is hidden. The command palette overlay's `render_dropdown` requires `self.anchor` (derived from the top panel's hamburger icon position) and bails out with `let Some(anchor) = self.anchor else { return; }` — so the menu renders nothing.

## Solution
Modify `render_dropdown` in `crates/term-wm-sys-ui-components/src/wm_menu_overlay.rs` to fall back to a centered modal overlay when `self.anchor` is `None` (mobile mode).

### Behavior
- **Desktop mode** (anchor is `Some`): Same anchored dropdown below hamburger icon, unchanged.
- **Mobile mode** (anchor is `None`): Centered, scaled-up modal overlay covering ~2/3 of the managed area with the command palette menu items. Panel contents (window labels, indicators) can be added to this overlay in a future step.

### Code Change
Replace `render_dropdown` method (lines 112-217 in `wm_menu_overlay.rs`) with the version described in the issue:
- When `anchor` is `Some`: existing logic, unchanged.
- When `anchor` is `None`: compute width/height as `max(width * 2/3, content_width)` and `max(height * 2/3, content_height)`, center in `bounds`.

### Files Modified
1. **`crates/term-wm-sys-ui-components/src/wm_menu_overlay.rs`** — modify `render_dropdown` (lines 112-217)
