# Refactor: Eliminate Dual Hit-Test in Event Routing (v3)

## Problem

Two instances of duplicated spatial math violate the HitboxRegistry data-oriented pattern:

1. **Header buttons**: `dispatch_mouse` Phase 4 calls `registry.hit_test()` → gets `ChromeHeader(key, Drag)` for the full header → **discards the action** → calls `decorator.hit_test(rect, col, row)` to recalculate button positions from scratch.

2. **Component::handle_events default**: Re-does an `is_inside` bounds check that `dispatch_mouse` already verified via the registry before calling `handle_events`.

## Changes

### 1. `crates/term-wm-core/src/components/mod.rs` — Remove bounds check, fix coordinate math (lines 43-65)

Remove the `is_inside` conditional. Compute `LocalMouseEvent` in `i32` space to handle negative absolute coordinates (e.g., floating windows dragged partially off-screen). When `screen_area` is `None`, return `EventResult::Ignored`.

**Before:**
```rust
fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<Msg> {
    if let Event::Mouse(mouse) = event {
        if let Some(screen_area) = ctx.screen_area() {
            let is_inside = i32::from(mouse.column) >= screen_area.x
                && i32::from(mouse.column) < screen_area.x + i32::from(screen_area.width)
                && i32::from(mouse.row) >= screen_area.y
                && i32::from(mouse.row) < screen_area.y + i32::from(screen_area.height);
            if is_inside {
                let local = LocalMouseEvent {
                    col: mouse.column.saturating_sub(screen_area.x.max(0) as u16),
                    row: mouse.row.saturating_sub(screen_area.y.max(0) as u16),
                    kind: mouse.kind,
                    modifiers: mouse.modifiers,
                };
                return self.on_mouse(&local, ctx);
            }
        }
        return EventResult::Ignored;
    }
    self.on_key(event, ctx)
}
```

**After:**
```rust
fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<Msg> {
    if let Event::Mouse(mouse) = event {
        if let Some(screen_area) = ctx.screen_area() {
            let local = LocalMouseEvent {
                col: (i32::from(mouse.column) - screen_area.x).max(0) as u16,
                row: (i32::from(mouse.row) - screen_area.y).max(0) as u16,
                kind: mouse.kind,
                modifiers: mouse.modifiers,
            };
            return self.on_mouse(&local, ctx);
        }
        return EventResult::Ignored;
    }
    self.on_key(event, ctx)
}
```

**Safety**: All Mouse event paths are pre-filtered by `dispatch_mouse` → HitboxRegistry. Keyboard-only callers (`dispatch_focused_event`) are unaffected.

## Scroll Event Routing (Design Decision)

Scroll wheel events (`ScrollUp`/`ScrollDown`/`ScrollLeft`/`ScrollRight`) are the **responsibility of `ScrollViewComponent`**, not individual leaf components. The default trait dispatch routes them to `on_mouse_scroll` which returns `Ignored` — this is correct because:
- `ScrollViewComponent` has its own `handle_events` override that intercepts scroll events before they reach the leaf
- Leaf components (text renderer, terminal, etc.) should never need to implement `on_mouse_scroll` — the scroll container handles it
- `on_mouse_scroll` exists in the trait solely for completeness and for any component that is NOT wrapped in a `ScrollViewComponent` and needs to handle scrolls

### 8. `crates/term-wm-core/src/components/mod.rs` — Exhaustive semantic mouse dispatch

Remove `on_mouse` and `LocalMouseEvent` from the `Component` trait entirely. The old monolithic catch-all is replaced by two new methods covering the full `MouseEventKind` spectrum:

- `on_mouse_scroll` — receives `ScrollUp`/`ScrollDown`/`ScrollLeft`/`ScrollRight` (the `kind` parameter carries which direction)
- `on_mouse_move` — receives `Moved` events

**Exhaustive `handle_events` default** — no wildcard arm, no `LocalMouseEvent` construction:
```rust
fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<Msg> {
    if let Event::Mouse(mouse) = event {
        if let Some(screen_area) = ctx.screen_area() {
            let local_x = (i32::from(mouse.column) - screen_area.x).max(0) as u16;
            let local_y = (i32::from(mouse.row) - screen_area.y).max(0) as u16;
            return match mouse.kind {
                MouseEventKind::Press(btn) =>
                    self.on_mouse_press(local_x, local_y, btn, mouse.modifiers, ctx),
                MouseEventKind::Release(btn) =>
                    self.on_mouse_release(local_x, local_y, btn, mouse.modifiers, ctx),
                MouseEventKind::Drag(btn) =>
                    self.on_mouse_drag(local_x, local_y, btn, mouse.modifiers, ctx),
                MouseEventKind::ScrollUp | MouseEventKind::ScrollDown
                | MouseEventKind::ScrollLeft | MouseEventKind::ScrollRight =>
                    self.on_mouse_scroll(local_x, local_y, mouse.kind, mouse.modifiers, ctx),
                MouseEventKind::Moved =>
                    self.on_mouse_move(local_x, local_y, mouse.modifiers, ctx),
            };
        }
        return EventResult::Ignored;
    }
    self.on_key(event, ctx)
}
```

### 11. `crates/term-wm-ui-components/src/text_renderer.rs` — Split `on_mouse` into semantic methods

The current `on_mouse` handles all event kinds by reconstructing a screen-space `MouseEvent` and delegating to `handle_selection_mouse`. Split into four semantic methods. Do NOT retain `on_mouse`. Replace it with `on_mouse_move`:

fn on_mouse_release(&mut self, local_x: u16, local_y: u16, button: MouseButton, modifiers: KeyModifiers, ctx: &ComponentContext) -> EventResult<TermWmAction> {

Where `handle_local_mouse` is a private helper that reconstructs screen-space coordinates in `i32` space to preserve negative offsets (e.g., when a window is partially off-screen at `x = -5`), avoiding truncation through `layout_rect_to_rect`:

```rust
fn handle_local_mouse(&mut self, local_x: u16, local_y: u16, kind: MouseEventKind, modifiers: KeyModifiers, ctx: &ComponentContext) -> EventResult<TermWmAction> {
    self.viewport_cache.set(ctx.viewport());
    if let Some(handle) = ctx.scroll_handle() {
        self.scroll_handle.replace(Some(handle));
    }
    let raw_area = ctx.screen_area().unwrap_or(LayoutRect { x: 0, y: 0, width: 0, height: 0 });
    let screen_mouse = MouseEvent {
        column: (i32::from(local_x) + raw_area.x).max(0) as u16,
        row: (i32::from(local_y) + raw_area.y).max(0) as u16,
        kind,
        modifiers,
    };
    if handle_selection_mouse(self, self.selection_enabled, &screen_mouse, raw_area) {
        EventResult::Consumed
    } else {
        EventResult::Ignored
    }
}
```
