# Plan: Fix dual_image example and clean up compatibility hacks

## Problem Summary

1. **`render_window` is dead code** — `WindowProvider::render_window()` is still a required trait method but hasn't been called since commit `eafa439` switched the render path to use `window_component()`. Test stubs and example implementations are all empty/no-ops. This is the "compatibility hack" the user wants removed.

2. **Quit doesn't work in dual_image** — `Action::Quit` is `ActionLayer::WmMode`, so `Ctrl+Q` only triggers when the WM overlay is visible. To quit, the user must: Esc → navigate to Exit UI → Enter → confirm. Even then, the default `WindowManagerHost::open_exit_confirm()` calls `open_overlay(id, None)` which does **nothing** (`if let Some(o) = overlay { insert }` — `None` is silently dropped). The confirm dialog is never created, so the user is stuck.

3. **Dead match arms in event handler** — Lines 251-265 of `runner.rs` handle `Action::Quit`, `OpenHelp`, `OpenKeybindings` in the Global-layer block, but these actions are all `WmMode` so those arms can never fire. The real dispatch happens in the WmMode block (lines 371-418) which only runs when the overlay is open.

4. **Images showing** — The coordinate analysis confirms `window.surface.inner` is in local coordinates (relative to the window interior, `x: 0`). The offscreen buffer and `blit_from_signed` handle positioning correctly. SVG loading/rendering code in `ascii_image.rs` is unchanged. The example should work if SVG paths are passed. Any "not showing" issue is likely due to no paths being passed or a load failure, not a code bug.

### B. Fix default `open_exit_confirm()` so it quits correctly

The default `WindowManagerHost::open_exit_confirm()` calls `self.windows().open_overlay(OverlayId::ExitConfirm, None)`, but `open_overlay` silently drops `None` — the overlay is never created. The confirm dialog can never appear, so `handle_exit_confirm_event` always returns `None`, and the user is stuck.

We can't create `ConfirmOverlayComponent` from `term-wm-core` (it lives in `term-wm-ui-components`). Instead, make the default quit directly by setting a flag on `WindowManager`:

**File: `crates/term-wm-core/src/window/window_manager/mod.rs`**
- Add `quit_requested: bool` field (default false) to `WindowManager`
- Add `pub fn request_quit(&mut self) { self.quit_requested = true; }`
- Add `pub fn quit_requested(&self) -> bool { self.quit_requested }`

**File: `crates/term-wm-core/src/runner.rs`**
- Change `WindowManagerHost::open_exit_confirm()` default:
  ```rust
  fn open_exit_confirm(&mut self) {
      self.windows().request_quit();
  }
  ```
- In the idle path (line 474), change `app.quit_requested()` to `app.windows().quit_requested()`

Apps that want a confirm dialog (like `main.rs`) keep their existing `open_exit_confirm()` override — no change needed.
