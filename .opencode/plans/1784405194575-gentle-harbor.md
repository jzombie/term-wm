# Plan: Replace `HitTarget` Enum with Opaque `HitboxId`

## Goal

Remove all domain-specific knowledge from `HitboxRegistry` so it becomes a pure
spatial database (`HitboxId + LayoutRect`). Domain routing moves into the
`WindowManager` as a `HashMap<HitboxId, WindowAction>`; components identify
themselves via their own stored `HitboxId` and check `ctx.active_hitbox`.

## Key Benefits

- **Open-Closed**: Adding new UI elements never requires touching `term-wm-core` hitbox code.
- **Perfect Occlusion**: The registry determines `active_hitbox` once per frame based on render order; components just check their ID.
- **Zero Math**: Components don't need `rect_contains` — they compare IDs.

## Detailed Design

### 1. `HitboxId` (hitbox_registry.rs)

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct HitboxId(pub u64);

impl HitboxId {
    pub fn new() -> Self {
        static NEXT_ID: AtomicU64 = AtomicU64::new(1);
        Self(NEXT_ID.fetch_add(1, Ordering::Relaxed))
    }
}
```

### 5. Rewritten `dispatch_mouse` Phase 4

EventResult::Ignored

The `MouseCaptureState` changes: `ComponentInteraction` adds `hitbox_id: HitboxId`
so the capture continuation session routes correctly even after the registry
has been rebuilt.

### 6. Component changes

Each component that registers a hitbox stores `hitbox_id: HitboxId` set during `new()`:

```rust
// In TerminalComponent

    pub hitbox_id: HitboxId,
    // ... existing fields ...
}

impl TerminalComponent {
    pub fn new() -> Self {
        Self { hitbox_id: HitboxId::new(), ... }
    }
}

impl Component<TermWmAction> for TerminalComponent {
    fn render(...) {
        registry.register(self.hitbox_id, screen_area_lr);
        // no more HitTarget::Component
    }

    fn on_mouse_press(&mut self, local_x, local_y, btn, mods, ctx) -> EventResult<Msg> {
        if ctx.active_hitbox() == Some(self.hitbox_id) {
            // handle the click
        }
        EventResult::Ignored
    }
}
```

## Migration Strategy (Incremental)

**Phase A**: Add `HitboxId` type alongside `HitTarget` (no behavioral changes).

**Phase B**: Add `active_hitbox` to `ComponentContext`, add `WindowAction` and routing maps to `WindowManager`.

**Phase C**: Rewrite `dispatch_mouse` to use the new routing maps. Keep `HitTarget` for backup, then remove it once all paths are covered.

**Phase D**: Update `render_app` and all component `render()` methods to use opaque IDs.

**Phase E**: Remove `HitTarget`, `build_component_map`, `component_area`, `window_area`. Clean up tests.
