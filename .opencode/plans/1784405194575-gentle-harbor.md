# Plan: Replace `HitTarget` with Opaque `HitboxId`

## Goal

Remove all domain-specific knowledge from `HitboxRegistry` — pure spatial
database (`HitboxId + LayoutRect`). Route via two flat transient Vecs built
during render. Occlusion enforced by `SealedComponent<C>` concrete wrapper —
cannot be bypassed by trait override. No singleton broadcast loop.

---

## Critical Fixes

| Flaw | Fix |
|------|-----|
| Per-frame ID generation breaks drag | IDs set once on entity creation, stored persistently |
| Per-frame HashMap thrashing | Flat `Vec<(HitboxId, T)>` rebuilt per render frame; linear scan at dispatch |
| Trait default `handle_events` can be overridden | `SealedComponent<C>` wrapper enforces occlusion in its own `handle_events` — sealed, non-generic, cannot be bypassed |
| Singleton broadcast loop (Phase 5) | Unified `component_routes: Vec<(HitboxId, RouteTarget)>` — ALL route targets pushed during render; single scan at dispatch |
| `hitbox_id()` on base trait | Removed. `HitboxId` lives on entity structs / `SealedComponent` / WM route tables only |

## 1. `HitboxId` Type

```rust
// hitbox_registry.rs
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct HitboxId(pub u64);

impl HitboxId {
    pub fn new() -> Self {
        static NEXT_ID: AtomicU64 = AtomicU64::new(1);
        Self(NEXT_ID.fetch_add(1, Ordering::Relaxed))
    }
}
```

## 4. `SealedComponent<C>` — Non-bypassable Occlusion

A concrete wrapper that enforces the hitbox check before delegating:

```rust
// window_manager/mod.rs (or a new sealed_component.rs)

/// Wraps any `WmComponent` and enforces hitbox-based Z-occlusion.
///
/// The inner component only receives mouse events when
/// `ctx.active_hitbox == self.hitbox_id`. Keyboard events pass
/// through unconditionally.
///
/// # Non-bypassable
/// `SealedComponent::handle_events` is defined on a concrete struct,
/// not as a trait default. Leaf components cannot override it.
pub struct SealedComponent {
    hitbox_id: HitboxId,
    inner: Box<dyn WmComponent>,
}

impl SealedComponent {
    pub fn new(inner: Box<dyn WmComponent>) -> Self {
        Self { hitbox_id: HitboxId::new(), inner }
    }
}

impl Component<TermWmAction> for SealedComponent {
    fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<TermWmAction> {
        // OCCLUSION: hardcoded — cannot be overridden
        if let Event::Mouse(_) = event {
            if ctx.active_hitbox() != Some(self.hitbox_id) {
                return EventResult::Ignored;
            }
        }
        self.inner.handle_events(event, ctx)
    }

    fn render(&mut self, backend: &mut dyn RenderBackend, area: LayoutRect,
              ctx: &ComponentContext, registry: &mut HitboxRegistry) {
        registry.register(self.hitbox_id, area);
        self.inner.render(backend, area, ctx, registry);
    }

    // delegate update(), init(), on_mount(), destroy(), query methods...
    fn update(&mut self, action: TermWmAction, ctx: &ComponentContext, actions: &mut VecDeque<(WindowKey, TermWmAction)>) {
        self.inner.update(action, ctx, actions);
    }
    // ... etc
}
```

The WM stores `SealedComponent` everywhere it previously stored
`Box<dyn WmComponent>`:

| WM field | Before | After |
|----------|--------|-------|
| `Window`'s component | `Box<dyn WmComponent>` | `SealedComponent` |
| `top_component` | `Option<Box<dyn WmComponent>>` | `Option<SealedComponent>` |
| `bottom_component` | `Option<Box<dyn WmComponent>>` | `Option<SealedComponent>` |
| `fab_component` | `Option<Box<dyn WmComponent>>` | `Option<SealedComponent>` |
| `command_menu_component` | `Option<Box<dyn WmComponent>>` | `Option<SealedComponent>` |

The `component_for_key_mut()` method returns `&mut SealedComponent` instead of
`&mut Box<dyn WmComponent>`.

## 5. Persistent IDs on Entity Structs

Structural elements (not wrapped in SealedComponent) also carry persistent IDs:

| Struct | New Field | Set At |
|--------|-----------|--------|
| `Window` | `content_hitbox_id: HitboxId` | `Window::new()` |
| `ResizeHandle` | `hitbox_id: HitboxId` | construction |
| `DragHandle` | `hitbox_id: HitboxId` | construction |
| `SplitHandle` | `hitbox_id: HitboxId` | construction |

These IDs are registered directly with `HitboxRegistry::register()` during
render, and pushed into `structural_routes` Vec.

Notification areas — currently a hitbox with no owning struct. The plan is to
introduce a minimal `NotificationArea` struct (stored in WM) with a persistent
`hitbox_id`.

## 6. Transient Routing Vecs (built each render)

The WM builds two flat Vecs during the render pass:

```rust
// Cleared at render start:
structural_routes: Vec<(HitboxId, WindowAction)>,
component_routes: Vec<(HitboxId, RouteTarget)>,
```

### `structural_routes`

Populated when rendering each window's chrome (resize handles, drag handles,
header buttons) and layout split handles. Each push records the persistent
`HitboxId` + the action to execute.

```rust
enum WindowAction {
    Close(WindowKey),
    Maximize(WindowKey),
    Minimize(WindowKey),
    ToggleDirectMode(WindowKey),
    Drag(WindowKey),
    Resize(WindowKey, ResizeEdge),
    LayoutHandle,
}
```

### `component_routes`

Populated when rendering:
- Each window's content area → `RouteTarget::Window(key)`
- Top panel area → `RouteTarget::TopPanel`
- Bottom panel area → `RouteTarget::BottomPanel`
- FAB area → `RouteTarget::Fab`
- Command menu area → `RouteTarget::CommandMenu`
- Notification areas → `RouteTarget::Notification`
- Overlay areas → `RouteTarget::Overlay(id)`

```rust
enum RouteTarget {
    Window(WindowKey),
    TopPanel,
    BottomPanel,
    Fab,
    CommandMenu,
    Notification,
    Overlay(OverlayId),
}
```

**Why is `RouteTarget` OK but `HitTarget` was not?**
`RouteTarget` lives in the WM module (not `hitbox_registry.rs`), is a private
implementation detail of the WM, and is not part of the public `HitboxRegistry`
API. Adding a new app widget only requires adding a variant to `RouteTarget`
(which is inside the WM — the right place for routing logic).

## 7. Dispatch Algorithm

```
dispatch_mouse(event):
  1. hit_test(position) → Some((hitbox_id, hit_rect))
  2. Set ctx.active_hitbox = Some(hitbox_id), ctx.screen_area = hit_rect

  3. Scan structural_routes Vec:
     Match(hitbox_id) → execute WindowAction immediately
       (resize capture, drag-init, close, max, min, layout handle)

  4. Scan component_routes Vec:
     Match(hitbox_id) → route_to_component(target, event, ctx)
       route_to_component dispatches by RouteTarget variant to the
       correct SealedComponent. SealedComponent's own handle_events
       re-checks occlusion (redundant but correct).

  5. Fall through → Ignored
```

`route_to_component` is a private match on `RouteTarget` that routes to the
correct stored `SealedComponent`:

```rust
fn route_to_component(&mut self, target: RouteTarget, event: &Event,
                       ctx: &ComponentContext) -> EventResult<...> {
    match target {
        RouteTarget::Window(key) =>
            self.component_for_key_mut(key)?.handle_events(event, ctx),
        RouteTarget::Fab =>
            self.fab_component.as_mut()?.handle_events(event, ctx),
        RouteTarget::TopPanel =>
            self.top_component.as_mut()?.handle_events(event, ctx),
        RouteTarget::BottomPanel =>
            self.bottom_component.as_mut()?.handle_events(event, ctx),
        RouteTarget::CommandMenu =>
            self.command_menu_component.as_mut()?.handle_events(event, ctx),
        RouteTarget::Notification =>
            self.notification_component.as_mut()?.handle_events(event, ctx),
        RouteTarget::Overlay(id) =>
            self.overlays.get_mut(&id)?.handle_events(event, ctx),
    }
}
```

## 8. Phase 1 (Active Capture) Integration

`MouseCaptureState::ComponentInteraction` stores the `hitbox_id` so capture
continuation sessions inject the correct `active_hitbox`:

```rust
enum MouseCaptureState {
    DraggingWindow { key, ... },
    ResizingWindow { key, edge, ... },
    ComponentInteraction { key, screen_area, hitbox_id: HitboxId },
    LayoutHandle,
}
```

During capture continuation, the ctx is built with `with_active_hitbox(hitbox_id)`,
ensuring SealedComponent's check passes.

## Migration (5 phases)

### Phase A: Foundation
1. Add `HitboxId` to `hitbox_registry.rs`
2. Add `active_hitbox` field to `ComponentContext`
3. Add `SealedComponent` struct + `WindowAction` enum + `RouteTarget` enum to WM
4. Add persistent `hitbox_id` fields to `Window`, `ResizeHandle`, `DragHandle`, `SplitHandle`

### Phase B: Render
1. Build `structural_routes` and `component_routes` Vecs during render
2. Register all hitboxes via persistent IDs
3. Wrap all `Box<dyn WmComponent>` storage in `SealedComponent`

### Phase C: Dispatch
1. Rewrite `dispatch_mouse` to use Vec scans + `route_to_component`
2. Add `hitbox_id` to `MouseCaptureState::ComponentInteraction`
3. Remove singleton iteration loop

### Phase D: Cleanup
1. Delete `HitTarget`, `build_component_map`, `component_area`, `window_area`
2. Update all tests
3. Remove dead event-routing branches (old `HitTarget` match arms)

### Phase E: Verify
```bash
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test
```
