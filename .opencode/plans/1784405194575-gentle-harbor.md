# Plan: Replace `HitTarget` with Opaque `HitboxId` (Amended)

## Goal

Remove all domain-specific knowledge from `HitboxRegistry` — make it a pure
spatial database (`HitboxId + LayoutRect`). Domain routing uses persistent IDs
stored on entity structs (set once at creation), with lightweight Vec scans at
dispatch time — no per-frame HashMaps, no per-frame ID generation.

## Critical Fixes (per code review)

| Flaw | Fix |
|------|-----|
| Per-frame ID generation breaks drag state | IDs set once on entity creation, stored persistently on the struct |
| Per-frame HashMap reallocation thrashing | No HashMaps rebuilt per frame; use flat Vec or persistent storage scans |
| `WindowAction` contains app widgets (Fab, Panel, etc.) | Purged — only WM primitives remain (Drag, Resize, Close, LayoutHandle, Overlay) |
| Global components without WindowKey unroutable | Unified dispatch: WM routes to all candidate components; each checks `ctx.active_hitbox == self.hitbox_id` |

## Key Design Decisions

### Persistent IDs (set once, never per-frame)

Every entity that registers a hitbox stores its `HitboxId` as a field, set once
during construction:

| Struct | New Field |
|--------|-----------|
| `ResizeHandle` | `hitbox_id: HitboxId` |
| `DragHandle` | `hitbox_id: HitboxId` |
| `SplitHandle` | `hitbox_id: HitboxId` |
| `Window` | `content_hitbox_id: HitboxId` (window content area) |
| `TerminalComponent` | `hitbox_id: HitboxId` |
| `WmFabComponent` | `hitbox_id: HitboxId` |
| `WmCommandPaletteComponent` | `hitbox_id: HitboxId` |
| `WmSessionManagerComponent` | `hitbox_id: HitboxId` (per row or per component) |
| Top/bottom panel components | `hitbox_id: HitboxId` |
| Overlay implementations | `hitbox_id: HitboxId` |
| Notification area | Needs an owning struct with `hitbox_id: HitboxId` |

During render, each entity **re-registers** its pre-existing ID + current rect
into the registry. The ID never changes across frames.

### No per-frame HashMaps

The `WindowManager` builds ONE lightweight Vec per render pass:

```rust
// Cleared at start of render, populated during window loop:
component_routes: Vec<(HitboxId, WindowKey)>,
```

This is a single push per window per frame. No HashMap overhead.

Structural elements (resize handles, drag handles, split handles) carry their
persistent `HitboxId` — the WM scans them linearly at dispatch time. With at
most ~200 structural elements across all windows, this is trivially fast for a
TUI at 60fps.

### Purged `WindowAction` (WM primitives only)

```rust
enum WindowAction {
    Close(WindowKey),
    Maximize(WindowKey),
    Minimize(WindowKey),
    ToggleDirectMode(WindowKey),
    Drag(WindowKey),
    Resize(WindowKey, ResizeEdge),
    LayoutHandle,
    Overlay(OverlayId),  // Overlays are a WM-level concern
}
```

**Removed**: TopPanel, BottomPanel, Notification, Fab, CommandPalette, SessionManager.

These are now handled via the unified component routing mechanism (see below).

### Unified Component Routing (fixes unroutable globals)

The `dispatch_mouse` algorithm:

```
1. hit_test() → (hitbox_id, hit_rect)
2. Build ctx with active_hitbox = Some(hitbox_id), screen_area = hit_rect
3. === Scan persistent structural elements ===
   - ResizeHandle: match → ResizeWindow capture
   - DragHandle: match → DragWindow capture or header button action
   - SplitHandle: match → route to TilingLayout.handle_event()
   - Overlay: match → overlay.handle_events(ctx)
4. === Scan component_routes Vec (window content/components) ===
   - Match → route to that window's component via comp.handle_events(ctx)
5. === Iterate singleton components (let them self-identify) ===
   - top_component, bottom_component, fab_component, command_menu_component
   - notifications component
   - Each: comp.handle_events(ctx)
   - Each checks ctx.active_hitbox == self.hitbox_id → Consumed if matched
6. Fall through → Ignored
```

Components that self-identify:

```rust
fn handle_events(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<TermWmAction> {
    if let Event::Mouse(mouse) = event {
        if matches!(mouse.kind, MouseEventKind::Press(_))
            && ctx.active_hitbox() == Some(self.hitbox_id)
        {
            // This component was clicked — handle it
            return EventResult::Consumed;
        }
    }
    EventResult::Ignored
}
```

## Migration Strategy (5 phases)

### Phase A: Foundation types
1. Add `HitboxId` struct to `hitbox_registry.rs`
2. Add `active_hitbox` to `ComponentContext` (+ accessor, builder)
3. Add `WindowAction` enum (primitives only) to WM
4. Add persistent `hitbox_id` fields to all entity structs (set in constructors)

### Phase B: Render-side changes
1. Rewrite `render_app` (src/lib.rs) to build `component_routes` Vec during window loop
2. Update all `register()` calls to use persistent IDs instead of `HitTarget`
3. Update WM helper methods (`register_panel_hitboxes`, etc.) to use persistent IDs

### Phase C: Dispatch-side changes
1. Rewrite `dispatch_mouse` Phase 2-4
2. Structural element routing via persistent-ID linear scan
3. Window component routing via `component_routes` Vec scan
4. Singleton component routing via iteration + self-identification
5. Update `MouseCaptureState::ComponentInteraction` to include `hitbox_id`

### Phase D: Cleanup
1. Remove `HitTarget` enum
2. Remove `build_component_map`, `component_area`, `window_area`
3. Update all tests

### Phase E: Verify
```bash
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test
```
