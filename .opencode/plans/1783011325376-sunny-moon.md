# Threading Model Implementation Plan

## Current Architecture

| Component | Thread | Role |
|-----------|--------|------|
| `EventLoop::run()` | Main UI | Polls driver via `crossterm::event::poll`, dispatches events, renders on idle ticks |
| crossterm input thread | 1 thread | Polls crossterm every 100ms, sends `UnifiedEvent::Input` via `crossbeam_channel::bounded(256)` |
| PTY reader threads | N per terminal | `read_loop()` blocks on `portable_pty::reader.read()`, writes raw bytes to `Arc<Mutex<Vec<u8>>> pending`, calls wakeup callback |
| `Pty::update()` (called during render) | Main UI | Locks `pending` mutex, feeds bytes to `vt100::Parser`, then reads `screen.cell(row,col)` |
| `Reaper::tick()` | **NEVER CALLED** | Bug: zombies accumulate until app exit's `drain_all()` |

## Changes (in implementation order)

---

### Phase 1 — PowerProfile: 3-State Model

**Files:** `power_profile.rs`, `event_source.rs`, `unified_event_source.rs`, `console.rs`

**Enum change:**
```rust
pub enum PowerProfile {
    Interactive,   // 8ms poll (~125fps) — user actively typing
    Streaming,     // 16ms poll (~60fps) — PTY flowing, no user input
    PowerSaver,    // 500ms poll (~2fps) — idle
}
```

**`profile_from_activity()` update** — new signature: `fn(last_event_at: Option<Instant>, has_dirty_windows: bool) -> PowerProfile`
- `last_event_at < 100ms` → `Interactive`
- `last_event_at < 500ms` → `Streaming`
- `last_event_at >= 500ms` or `None`, but `has_dirty_windows` → `Streaming`
- Otherwise → `PowerSaver`

**`UnifiedEventSource::current_profile()`** — passes `!self.dirty_windows.is_empty()`  
**`ConsoleEventSource::current_profile()`** — needs a `dirty_windows` check or keeps existing logic

**Event loop** — no change to idle render calling (always renders on idle tick; power savings come from longer poll intervals, not skipping renders)

**Theme** — already has `profile_mid: Color` — no changes needed.

---

### Phase 2 — Output Coalescing Timer

**File:** `unified_event_source.rs`

Add `coalesce_deadline: Option<Instant>` field (init `None`).  
Add constant `COALESCE_DELAY = Duration::from_millis(3)`.

**New method `maybe_arm_coalesce(&mut self, key: WindowKey)`:**
- Inserts key into `dirty_windows`
- If `coalesce_deadline` is `None`, sets it to `Instant::now() + COALESCE_DELAY`

**Modified `poll()`:**
- On `PtyWakeup(key)`: call `maybe_arm_coalesce(key)`, then check deadline.
  - If deadline expired: clear it, return `Ok(false)` (triggers idle render with accumulated dirty windows)
  - If deadline active: recalc `remaining = remaining.min(deadline - now)`, continue waiting
- On `Input(event)`: clear `coalesce_deadline` (cancel coalesce on user input), return `Ok(true)`
- At timeout: clear `coalesce_deadline`, return `Ok(false)`

**Test construction** in existing test `drain_pending_preserves_all_input_events` — add `coalesce_deadline: None` field.

---

### Phase 3 — Mechanical Backpressure Documentation

**Files:** `unified_event_source.rs`, `pty.rs`

Add doc comments explaining:
- The bounded channel (256 slots) provides natural backpressure
- When `tx.send()` blocks, the PTY reader thread pauses → OS pipe buffer fills → child process's `write()` blocks
- This prevents memory exhaustion under extreme load (`cat /dev/urandom`)

No behavioral changes.

---

### Phase 4 — Fix Reaper Tick

**File:** `runner.rs` — in the idle handler (`else` branch, before `begin_frame()`):

```rust
app.windows().reaper().tick();
```

This is a bug fix: `Reaper::tick()` was never called from the main loop, so zombie children accumulated until app exit's `drain_all()`.

---

### Phase 5 — Move ANSI Parsing into PTY Reader Thread (Advanced/Optional)

This is a larger refactor. Send the `vt100::Parser` into the reader thread. Replace `pending: Arc<Mutex<Vec<u8>>>` with `screen: Arc<Mutex<vt100::Screen>>` + `dirty: Arc<AtomicBool>`.

**Files:** `pty.rs`, `pane.rs`, `terminal.rs`

**Risk:** `Pane::screen()` returns `&vt100::Screen` — a reference into `self`. With `Arc<Mutex<Screen>>`, we can't return a reference across the lock boundary. Options:
1. Keep current signature but the main thread clones the screen each render (O(rows*cols), acceptable at 60fps)
2. Change `Pane::screen()` to a new pattern (breaks all implementations + `TestPane`)

---

### Phase 6 — DRR Scheduler (Future/Documentation Only)

Add a design doc comment in `runner.rs` describing the future Deficit Round-Robin approach for render budget allocation.

## Implementation Order

```
Phase 1: power_profile.rs → event_source.rs → unified_event_source.rs → console.rs
Phase 2: unified_event_source.rs (coalesce)
Phase 3: unified_event_source.rs + pty.rs (doc comments)
Phase 4: runner.rs (reaper tick)
Phase 5: pty.rs → pane.rs → terminal.rs (advanced)
Phase 6: runner.rs (DRR doc)
```
