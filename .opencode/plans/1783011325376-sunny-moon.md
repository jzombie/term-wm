## Current Architecture (with flaws)

| Component | Thread | Role | Flaw |
|-----------|--------|------|------|
| `EventLoop::run()` | Main UI | Polls driver, dispatches events, renders idle ticks | ANSI parsing blocks render |
| `UnifiedEventSource::new()` | 1 thread | Polls crossterm every 100ms, sends via bounded channel | OK |
| `read_loop()` (per PTY) | N threads | Blocks on PTY read, writes raw bytes to `pending` | Raw bytes only; no parsing |
| `Pty::update()` (called in render) | Main UI | Lock `pending`, feed `vt100::Parser`, render cells | **Chokes under high PTY output** |
| `Reaper::tick()` | **Never called** | Inline zombie management | **Bug: zombies leak until exit** |
| PowerSaver | 500ms poll | ~2fps polling loop | **Wastes CPU; should block** |
| `SlotMap<WindowKey, Window>` | ✅ Already in use | Generational arena for window state | Satisfied mandate |

pub fn profile_from_activity(
    last_event_at: Option<Instant>,
    has_dirty_windows: bool,
) -> PowerProfile {
    match last_event_at {
        Some(t) if (t.elapsed().as_millis() as u64) < INTERACTIVE_THRESHOLD_MS => {
            PowerProfile::Interactive

        Some(t) if (t.elapsed().as_millis() as u64) < STREAMING_THRESHOLD_MS => {
            PowerProfile::Streaming
        }
        _ if has_dirty_windows => PowerProfile::Streaming,
        _ => PowerProfile::PowerSaver,
    }
}

## Phase 4 — Reaper on Background Thread

**File:** `reaper.rs`, `window_manager/mod.rs`, `main.rs`

**Replace `Reaper` with thread-based implementation:**
```rust
pub struct Reaper {
    tx: crossbeam_channel::Sender<ZombieChild>,
    _handle: std::thread::JoinHandle<()>,
    shutdown: Arc<AtomicBool>,
}

impl Reaper {
    pub fn new(shutdown_timeout: Duration) -> Self {
        let (tx, rx) = crossbeam_channel::bounded::<ZombieChild>(64);
        let shutdown = Arc::new(AtomicBool::new(false));
        let s = Arc::clone(&shutdown);
        let _handle = std::thread::Builder::new()
            .name("pty-reaper".into())
            .spawn(move || {
                let mut zombies: Vec<ZombieChild> = Vec::new();
                loop {
                    if s.load(Ordering::Acquire) { break; }

                    // Dynamic timeout: block indefinitely if no zombies,
                    // poll at 50ms if zombies exist (non-blocking)
                    let timeout = if zombies.is_empty() {
                        Duration::from_secs(3600)
                    } else {
                        Duration::from_millis(50)
                    };
                    if let Ok(z) = rx.recv_timeout(timeout) {
                        zombies.push(z);
                        while let Ok(z) = rx.try_recv() { zombies.push(z); }
                    }

                    // Tick — same lifecycle logic as current Reaper::tick()
                    zombies.retain_mut(|z| {
                        if !z.sighup_sent {
                            if let Some(ref mut child) = z.child { let _ = child.kill(); }
                            z.sighup_sent = true;
                            z.killed_at = Some(Instant::now());
                            return true;
                        }
                        if let Some(ref mut child) = z.child
                            && child.try_wait().ok().flatten().is_some()
                        {
                            drop(z.child.take());
                            if let Some(handle) = z.reader_handle.take() { let _ = handle.join(); }
                            return false;
                        }
                        if let Some(killed_at) = z.killed_at
                            && !z.sigkill_sent
                            && killed_at.elapsed() >= shutdown_timeout
                        {
                            if let Some(ref mut child) = z.child { let _ = child.kill(); }
                            z.sigkill_sent = true;
                        }
                        true
                    });
                }
                // drain_all on shutdown
                for mut z in zombies.drain(..) {
                    if let Some(ref mut child) = z.child { let _ = child.kill(); }
                    drop(z.child.take());
                    if let Some(handle) = z.reader_handle.take() { let _ = handle.join(); }
                }
            })
            .expect("reaper thread");
        Self { tx, _handle, shutdown }
    }

    // &self — channel send doesn't require &mut
    pub fn reap(&self, zombie: ZombieChild) {
        let _ = self.tx.send(zombie);
    }
}

impl Drop for Reaper {
    fn drop(&mut self) {
        self.shutdown.store(true, Ordering::Release);
    }
}

impl Default for Reaper {
    fn default() -> Self { Self::new(Duration::from_secs(3)) }
}
```

**`WindowManager`**: `reaper()` returns `&Reaper` (not `&mut`).

**`main.rs::wm_close_window()`**: `app.windows().reaper().reap(zombie)` — no mutability issue.

**Remove `Reaper::tick()` and `Reaper::drain_all()`** — no longer needed (drain happens in Drop).

**`parser_read_loop()` — reader thread owns parser + history:**
```rust
fn parser_read_loop(
    mut reader: Box<dyn Read + Send>,
    shared_screen: Arc<Mutex<vt100::Screen>>,
    dirty: Arc<AtomicBool>,
    bytes_received: Arc<AtomicUsize>,
    last_bytes: Arc<Mutex<Vec<u8>>>,
    dsr_requested: Arc<AtomicBool>,
    pending_title: Arc<Mutex<Option<String>>>,
    pending_resize: Arc<Mutex<Option<PtySize>>>,
    wakeup: WakeupFn,
    scrollback_len: usize,
    rows: u16,
    cols: u16,
) {
    let mut parser = vt100::Parser::new(rows, cols, scrollback_len);
    let mut history: Vec<u8> = Vec::new();  // KEPT for resize replay
    let mut buf = [0u8; 4096];
    let cap = 2 * 1024 * 1024;  // 2MB history cap
    let prune = 1024 * 1024;     // prune to 1MB
    loop {
        // Check for pending resize from main thread
        if let Ok(mut resize_opt) = pending_resize.lock() {
            if let Some(size) = resize_opt.take() {
                let mut new_parser = vt100::Parser::new(size.rows, size.cols, scrollback_len);
                new_parser.process(&history);  // replay history so screen isn't blank
                parser = new_parser;
            }
        }
        match reader.read(&mut buf) {
            Ok(0) => break,
            Ok(n) => {
                bytes_received.fetch_add(n, Ordering::Relaxed);
                // DSR detection (same as current read_loop)
                // OSC 52 / OSC title detection (same as current update())
                // scrollback tracking

                history.extend_from_slice(&buf[..n]);
                if history.len() > cap {
                    let prune_to = prune.min(history.len());
                    history.drain(0..prune_to);
                }

                parser.process(&buf[..n]);  // ← parsing on reader thread

                // Publish to main thread
                *shared_screen.lock().unwrap() = parser.screen().clone();  // Screen: Clone ✓
                dirty.store(true, Ordering::Release);

                // Wakeup
                if let Ok(guard) = wakeup.lock() && let Some(ref cb) = *guard { cb(); }
            }
            Err(_) => break,
        }
    }
}
```
