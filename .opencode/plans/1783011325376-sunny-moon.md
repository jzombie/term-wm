# Threading Model — Final Implementation Plan

## Current Architecture (with flaws)

| Component | Thread | Role | Flaw |
|-----------|--------|------|------|
| `EventLoop::run()` | Main UI | Polls driver, dispatches events, renders idle ticks | ANSI parsing blocks render |
| `UnifiedEventSource::new()` | 1 thread | Polls crossterm every 100ms, sends via bounded channel | OK |
| `read_loop()` (per PTY) | N threads | Blocks on PTY read, writes raw bytes to `pending` | Raw bytes only; no parsing |
| `Pty::update()` (called in render) | Main UI | Lock `pending`, feed `vt100::Parser`, render cells | **Chokes under high PTY output** |
| `Reaper::tick()` | **Never called** | Inline zombie management | **Bug: zombies leak until exit** |
| PowerSaver | 500ms poll | ~2fps polling loop | **Wastes CPU; should block** |
| `SlotMap<WindowKey, Window>` | ✅ Already in use | Generational arena for window state | Satisfied mandate |

## Phase 1 — PowerProfile: 3-State Model

**Files:** `power_profile.rs`, `event_source.rs`, `unified_event_source.rs`, `console.rs`, `runner.rs`

**Enum:**
```rust
pub enum PowerProfile {
    Interactive,   // 8ms poll — user typing (<100ms since last input)
    Streaming,     // 16ms poll — PTY data flowing, no user input
    PowerSaver,    // blocks on channel — no recent activity, no dirty windows
}
```

**`profile_from_activity(last_event_at, has_dirty_windows)` → `PowerProfile`:**
- `<100ms` since last input → `Interactive`
- `<500ms` → `Streaming`
- `has_dirty_windows` → `Streaming`
- otherwise → `PowerSaver`

**`PowerSaver::poll_interval()` returns `Duration::from_secs(3600)`.** Main thread blocks on `recv_timeout()` for the full interval, yielding to the OS. Only PtyWakeup or input events (via the crossbeam channel) wake it.

**Event loop:** idle handler stays (render on every loop iteration). Power savings come from blocking poll, not skipping renders. In PowerSaver mode, a PtyWakeup triggers `poll()` to return `false` after coalesce expiry → idle render → next `poll(3600s)` blocks again.

**`UnifiedEventSource::current_profile()`** — passes `!self.dirty_windows.is_empty()`.

## Phase 2 — Output Coalescing (3ms)

**File:** `unified_event_source.rs`

Add `coalesce_deadline: Option<Instant>` field. Constant `COALESCE_DELAY = 3ms`.

**Modified `poll()` logic on `PtyWakeup(key)`:**
```
dirty_windows.insert(key)
if coalesce_deadline is None:
    coalesce_deadline = now + COALESCE_DELAY
check coalesce:
    if now >= deadline → clear, return Ok(false)  // triggers idle render
    else → remaining = min(remaining, deadline - now); continue
```
On `Input(event)`: clear coalesce, return `Ok(true)` immediately.
At timeout: clear coalesce, return `Ok(false)`.

## Phase 4 — Reaper on Background Thread

**File:** `reaper.rs`, `window_manager/mod.rs`, `main.rs`

**Problem:** Current `Reaper` stores `Vec<ZombieChild>` inline. `tick()` never called. Even if it were, `handle.join()` blocks main thread.

**Solution:** Spawn dedicated reaper thread. The thread uses `recv_timeout()` with **dynamic timeout** to avoid spin-looping:

```rust
pub fn new(shutdown_timeout: Duration) -> Self {
    let (tx, rx) = bounded::<ZombieChild>(64);
    let shutdown = Arc::new(AtomicBool::new(false));
    let s = Arc::clone(&shutdown);
    let _handle = thread::Builder::new()
        .name("pty-reaper".into())
        .spawn(move || {
            let mut zombies: Vec<ZombieChild> = Vec::new();
            loop {
                if s.load(Ordering::Acquire) { break; }

                // Block efficiently: indefinitely if no zombies, 50ms poll if there are
                let timeout = if zombies.is_empty() {
                    Duration::from_secs(3600)
                } else {
                    Duration::from_millis(50)
                };
                if let Ok(z) = rx.recv_timeout(timeout) {
                    zombies.push(z);
                    while let Ok(z) = rx.try_recv() { zombies.push(z); }
                }

                // Tick — same logic as current Reaper::tick()
                zombies.retain_mut(|z| { /* sighup → try_wait → sigkill */ });
            }
            // drain_all: force kill + join all
        })
        .expect("reaper thread");
    Self { tx, _handle, shutdown }
}

// reap(&self) — only needs &self, channel send doesn't require &mut
pub fn reap(&self, zombie: ZombieChild) { let _ = self.tx.send(zombie); }
```

**`WindowManager`** stores `Reaper` as before. `reaper()` returns `&Reaper` (no `&mut` needed).

**`main.rs::wm_close_window()`** — `app.windows().reaper().reap(zombie)` with `&self`.

## Phase 5 — Move ANSI Parsing into PTY Reader Thread (MANDATORY)

**Files:** `pty.rs`, `pane.rs`, `terminal.rs` (TestPane unchanged)

### Design

**Shared state:**
```rust
pub struct Pty {
    shared_screen: Arc<Mutex<vt100::Screen>>,  // reader writes, main reads
    dirty: Arc<AtomicBool>,                     // set by reader after each batch
    cached_screen: vt100::Screen,               // main-thread local copy (avoids per-cell lock)

    // Reader-thread owned (moved into parser_read_loop):
    //   - vt100::Parser
    //   - history: Vec<u8>     ← KEPT: needed for resize replay

    // Removed:
    //   pending: Arc<Mutex<Vec<u8>>>,
    //   parser: vt100::Parser,
    //   history: Vec<u8> (was main-thread owned),
}
```

**`parser_read_loop()`** — reader thread owns parser + history:
```rust
fn parser_read_loop(
    mut reader: Box<dyn Read + Send>,
    shared_screen: Arc<Mutex<vt100::Screen>>,
    dirty: Arc<AtomicBool>,
    dsr_requested: Arc<AtomicBool>,
    pending_title: Arc<Mutex<Option<String>>>,
    pending_resize: Arc<Mutex<Option<PtySize>>>,
    wakeup: WakeupFn,
    scrollback_len: usize,
    rows: u16, cols: u16,
) {
    let mut parser = vt100::Parser::new(rows, cols, scrollback_len);
    let mut history: Vec<u8> = Vec::new();  // KEPT for resize replay
    loop {
        // Check for pending resize from main thread
        if let Ok(mut resize_opt) = pending_resize.lock() {
            if let Some(size) = resize_opt.take() {
                let mut new_parser = vt100::Parser::new(size.rows, size.cols, scrollback_len);
                new_parser.process(&history);  // ← replay history so screen isn't blank
                parser = new_parser;
            }
        }

        match reader.read(&mut buf) {
            Ok(0) => break,
            Ok(n) => {
                history.extend_from_slice(&buf[..n]);
                // Cap history (same as current: 2MB max, prune to 1MB)
                // ...
                parser.process(&buf[..n]);

                // Publish parsed screen to main thread
                *shared_screen.lock().unwrap() = parser.screen().clone();
                dirty.store(true, Ordering::Release);
                // wakeup main thread
                if let Ok(guard) = wakeup.lock() && let Some(ref cb) = *guard { cb(); }
            }
            Err(_) => break,
        }
    }
}
```

**`vt100::Screen` derives `Clone`** (verified: vt100 0.16.2 `Screen` has `#[derive(Clone, Debug)]`).

**Main thread `Pty::screen()`:**
```rust
pub fn screen(&mut self) -> &vt100::Screen {
    if self.dirty.load(Ordering::Acquire) {
        if let Ok(guard) = self.shared_screen.lock() {
            self.cached_screen = guard.clone();  // O(rows*cols) per render frame
        }
        self.dirty.store(false, Ordering::Release);
    }
    &self.cached_screen
}
```

**Clone cost:** O(rows*cols) per main-thread render (not per PTY read). 80×24 = ~1920 cells → microseconds. Acceptable. SoA/SIMD optimization is future work.

**`Pane::screen()` signature** unchanged — `fn screen(&mut self) -> &vt100::Screen`. No API breakage.

**`Pane::screen_mut()` and `Pane::max_scrollback()`** — operate on the cached screen. `set_scrollback()` modifies `cached_screen` directly.

**`TestPane`** — wraps a `vt100::Parser` directly (no reader thread). `screen()` returns `self.parser.screen()` — unchanged.

## SlotMap Verification (Mandate Already Satisfied)

- `WindowManager::windows: SlotMap<WindowKey, Window>` — generational arena in use
- `WindowKey = slotmap::DefaultKey` — lightweight tokens passed throughout
- Access exclusively via `SlotMap::get(key)` / `get_mut(key)` — no `Arc<RwLock>` graphs
- Layout and z-order use `Vec<WindowKey>` only — no heavy objects in ordering data structures
- Heavy components (`ScrollViewComponent<TerminalComponent>`) stored externally in `App::terminals` via `WindowProvider` trait — not inside the SlotMap
