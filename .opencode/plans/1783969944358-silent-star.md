# Mobile-First Terminal Adaptation Plan (Unified Architecture)

## Executive Summary

Adapt `term-wm` for mobile-first terminal environments (Termux, Blink Shell) while preserving desktop experience. The architecture enforces a **single, universal input model** across all environments: default `Passthrough` with FAB-triggered command palette. No split-brain state machine.

---

## Codebase Architecture Context

**Key Crates & Files:**
- `term-wm-core/src/window/window_manager/mod.rs` (5711 lines) — Central WM state
- `term-wm-core/src/runner.rs` (1303 lines) — Event loop orchestration
- `term-wm-core/src/components/mod.rs` (779 lines) — Component trait system
- `term-wm-core/src/actions.rs` (310 lines) — Action definitions
- `term-wm-core/src/wm_config.rs` (183 lines) — Configuration
- `term-wm-core/src/layout/tiling.rs` (1490 lines) — BSP layout engine
- `term-wm-core/src/draw_plan.rs` — DrawPlan intermediate representation
- `term-wm-core/src/hitbox_registry.rs` — Hit-test registry
- `term-wm-console/src/draw_plan_renderer.rs` — Low-level renderer (only paints DrawPlan)
- `term-wm-sys-ui-components/` — Component implementations (chrome, overlays)

**Event Flow:**
1. `UnifiedEventSource` → crossterm input + PTY wakeups + signals
2. `run_event_loop` → layered dispatch: overlays → WM input mode → component
3. `dispatch_mouse` → hit-test registry → capture state machine → component forwarding
4. `render_app` → DrawPlan generation → DrawPlanRenderer paints abstract regions

**Architectural Invariants:**
- `term-wm-console` is a low-level rendering target; it must remain ignorant of business UI
- Components implement `Component<TermWmAction>` and emit into the DrawPlan
- `regions` map must preserve logical geometry of all windows at all times
- `WindowManager.z_order` is the authoritative, persistent stacking order — NEVER mutated by transient viewport constraints
- `WindowManager` must NEVER hold `&mut Buffer` references — all visual feedback is declarative via DrawPlan injection
- Hit-test registry is the single source of truth for coordinate-based interception
- **Single input model**: ONE `WmInputMode` state machine across all environments — no mobile/desktop fork

#### 1.4 Event Routing (Universal Passthrough + Bypass Injection)

**File:** `crates/term-wm-core/src/runner.rs`

The event loop uses a SINGLE input mode check — no mobile/desktop fork:

```rust
// In the event loop closure, after computing `evt`:

match app.wm().input_mode() {
    WmInputMode::Passthrough => {
        // FAB interception BEFORE passthrough (prevents deadlock)
        if let Event::Mouse(mouse) = &evt {
            if matches!(mouse.kind, MouseEventKind::Press(_)) {
                if app.wm().hitbox_registry().hit_test(mouse.column, mouse.row) == Some(HitTarget::Fab) {
                    app.wm().open_command_menu();
                    return flush_state_changes(app, ControlFlow::Continue, false);
                }
            }
        }
        
        // Keyboard shortcut for desktop users (Ctrl+Shift+Space)
        if let Event::Key(key) = &evt {
            if key.kind == KeyKind::Press
                && app.wm().keybindings().matches(TermWmAction::OpenCommandPalette, key)
            {
                app.wm().open_command_menu();
                return flush_state_changes(app, ControlFlow::Continue, false);
            }
        }
        
        // Safe to bypass — neither FAB nor palette shortcut was triggered
        let _ = handle_focused_app_event(&evt, app);
        return flush_state_changes(app, ControlFlow::Continue, false);
    }
    WmInputMode::CommandPalette => {
        // BYPASS INJECTION: If the same universal chord is pressed again
        // while the palette is active, forward the chord to the PTY.
        // This enables nested term-wm instances and app-level chord handling.
        if let Event::Key(key) = &evt {
            if key.kind == KeyKind::Press
                && app.wm().keybindings().matches(TermWmAction::OpenCommandPalette, key)
            {
                // 1. Destroy the Command Palette overlay
                app.wm().close_command_menu();
                
                // 2. Revert to Passthrough
                app.wm().set_input_mode(WmInputMode::Passthrough);
                
                // 3. Synthesize and write the raw chord bytes to the active PTY
                let chord_bytes = app.wm().chord_bytes().to_vec();
                if let Some(focused) = app.wm().focused_window() {
                    app.wm().write_to_pty(focused, &chord_bytes);
                }
                
                return flush_state_changes(app, ControlFlow::Continue, false);
            }
        }
        
        // Normal palette routing
        if let Some(action) = app.wm().handle_wm_menu_event(&evt) {
            app.wm().execute_menu_action(action);
        }
        return flush_state_changes(app, ControlFlow::Continue, false);
    }
    WmInputMode::TapToSwapTargeting => {
        // Route to tap-swap targeting
        if let Event::Mouse(mouse) = &evt {
            if let MouseEventKind::Press(_) = mouse.kind {
                if let Some(target_key) = app.wm().hit_test_pane(mouse.column, mouse.row) {
                    app.wm().execute_tap_swap(target_key);
                    return flush_state_changes(app, ControlFlow::Continue, false);
                }
            }
        }
        app.wm().show_notification("Tap a pane to swap positions");
        return flush_state_changes(app, ControlFlow::Continue, false);
    }
}
```

**Bypass Injection State Machine (for nesting):**

```
User presses Ctrl+Shift+Space (1st time)
    → WM intercepts, opens CommandPalette
    → Child process receives NOTHING

User presses Ctrl+Shift+Space (2nd time, while palette is open)
    → WM detects same chord while in CommandPalette
    → WM closes palette, reverts to Passthrough
    → WM synthesizes raw ANSI bytes of Ctrl+Shift+Space
    → WM writes bytes directly to active PTY
    → Child process (inner term-wm) receives the chord
    → Inner WM opens its own CommandPalette
```

#### 2.2 DrawPlan Culling (NOT Region Destruction)

**File:** `crates/term-wm-core/src/draw_plan.rs`

Add monocle filtering to the DrawPlan projection:
```rust
pub struct DrawPlan {
    regions: Vec<DrawRegion>,
}

impl DrawPlan {
    /// Apply monocle culling: keep focused window at full area,
    /// mark other windows as hidden but preserve their logical geometry.
    pub fn apply_monocle_culling(&mut self, focused_key: WindowKey) {
        for region in &mut self.regions {
            match &region.region_type {
                RegionType::Window(key) if *key == focused_key => {
                    // Focused window gets full area
                }
                RegionType::Window(_) => {
                    region.hidden = true;
                }
                _ => {}
            }
        }
    }
    
    /// Apply monocle Z-order: elevate focused window to top of render stack.
    /// Operates on DrawPlan regions, NOT WindowManager.z_order.
    pub fn apply_monocle_z_order(&mut self, focused_key: WindowKey) {
        self.regions.sort_by(|a, b| {
            let a_focused = matches!(&a.region_type, RegionType::Window(k) if *k == focused_key);
            let b_focused = matches!(&b.region_type, RegionType::Window(k) if *k == focused_key);
            a_focused.cmp(&b_focused)
        });
    }
}
```

```rust
pub struct WmSessionManagerComponent {
    visible: bool,
    sessions: Vec<SessionEntry>,
}

impl Component<TermWmAction> for WmSessionManagerComponent {
    fn render(&mut self, backend: &mut dyn RenderBackend, area: LayoutRect, ctx: &ComponentContext, registry: &mut HitboxRegistry) {
        if !self.visible { return; }
        
        let buffer = backend.buffer_mut();
        let screen_area = ctx.screen_area().unwrap_or(area);
        
        // Clear background
        for y in screen_area.y..screen_area.y + screen_area.height {
            for x in screen_area.x..screen_area.x + screen_area.width {
                if let Some(cell) = buffer.cell_mut((x, y)) {
                    cell.set_symbol(" ").set_style(Style::default().bg(Color::Black));
                }
            }
        }
        
        // Render session list with tap targets
        for (i, entry) in self.sessions.iter().enumerate() {
            let row_rect = LayoutRect {
                x: screen_area.x,
                y: screen_area.y + i as i32,
                width: screen_area.width,
                height: 1,
            };
            if let Some(key) = ctx.window_key() {
                registry.register(HitTarget::Component(key, entry.key), row_rect);
            }
        }
    }
    
    fn on_mouse_press(&mut self, _local_x: u16, local_y: u16, _button: MouseButton, _modifiers: KeyModifiers, _ctx: &ComponentContext) -> EventResult<TermWmAction> {
        let index = local_y as usize;
        if index < self.sessions.len() {
            return EventResult::Action(TermWmAction::FocusWindow(self.sessions[index].key));
        }
        EventResult::Ignored
    }
}
```
