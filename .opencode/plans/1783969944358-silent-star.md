# Mobile-First Terminal Adaptation Plan

## Executive Summary

Adapt `term-wm` for mobile-first terminal environments (Termux, Blink Shell) while preserving desktop experience. The architecture transitions from desktop-heavy assumptions (modifier chords, small tap targets, continuous dragging) to a mobile-friendly paradigm with four core pillars.

---

## Codebase Architecture Context

**Key Crates & Files:**
- `term-wm-core/src/window/window_manager/mod.rs` (5711 lines) — Central WM state
- `term-wm-core/src/runner.rs` (1303 lines) — Event loop orchestration
- `term-wm-core/src/components/mod.rs` (779 lines) — Component trait system

- `term-wm-sys-ui-components/src/wm_top_panel.rs` — Top panel chrome
- `term-wm-sys-ui-components/src/wm_bottom_panel.rs` — Bottom panel chrome
- `term-wm-sys-ui-components/src/wm_menu_overlay.rs` — Menu overlay

**Event Flow:**
1. `UnifiedEventSource` → crossterm input + PTY wakeups + signals
2. `run_event_loop` → layered dispatch: overlays → direct mode → global → WM → component
3. `dispatch_mouse` → hit-test registry → capture state machine → component forwarding
4. `render_app` → DrawPlanRenderer → per-component render calls

## Pillar 2: "Monocle" Screen Size Fallback

### Problem
Tiled splits on screens under 80 columns break UI alignment and readability.

### Solution: Logical vs Rendered Layout Separation

#### 2.1 Layout State

**File:** `crates/term-wm-core/src/layout/tiling.rs`

Add monocle state to `TilingLayout`:
```rust
pub struct TilingLayout<Id: Copy + Eq + Ord> {
    root: LayoutNode<Id>,
    // Existing fields...
    
    // New: Monocle mode state
    monocle_active: bool,
    monocle_width_threshold: u16, // default: 80
}
```

Add methods:
```rust
impl<Id: Copy + Eq + Ord> TilingLayout<Id> {
    pub fn update_monocle_state(&mut self, terminal_width: u16) {
        let should_be_monocle = terminal_width < self.monocle_width_threshold;
        if should_be_monocle != self.monocle_active {
            self.monocle_active = should_be_monocle;
        }
    }
    
    pub fn is_monocle(&self) -> bool {
        self.monocle_active
    }
    
    pub fn focused_leaf(&self) -> Option<&Id> {
        // Find the focused leaf in the BSP tree
        self.find_focused_leaf(&self.root)
    }
}
```

#### 2.2 Render-Time Monocle Injection

**File:** `crates/term-wm-core/src/window/window_manager/layout.rs`

Modify `register_managed_layout` to check screen size:
```rust
pub fn register_managed_layout(&mut self, area: LayoutRect) {
    // Existing layout computation...
    
    // Check for monocle mode
    if let Some(ref mut layout) = self.managed_layout {
        layout.update_monocle_state(area.width);
        
        if layout.is_monocle() {
            // Override regions: only render focused window at full area
            if let Some(focused_key) = self.focused_window() {
                self.regions.clear();
                self.regions.insert(focused_key, area);
                // Other windows are logically present but visually hidden
                return;
            }
        }
    }
    
    // Normal tiling render...
}
```

#### 2.3 Resize Handler

**File:** `crates/term-wm-core/src/runner.rs`

In the event loop, handle `Event::Resize`:
```rust
Event::Resize(w, h) => {
    // Existing resize handling...
    
    // Update monocle state based on new width
    app.wm().update_monocle_mode(w);
    
    // If transitioning from monocle to tiled, restore logical layout
    if !app.wm().is_monocle() && was_monocle {
        app.wm().restore_tiled_layout();
    }
}
```

#### 2.4 Regression Prevention

- `monocle_width_threshold` defaults to 80 columns
- Desktop users with >= 80 columns never see monocle mode
- Logical layout tree is preserved in memory; only rendered regions change
- Resize back to >= 80 columns seamlessly restores tiled view

#### 3.2 Session Manager Overlay

**New File:** `crates/term-wm-sys-ui-components/src/wm_session_manager.rs`

```rust
pub struct WmSessionManagerComponent {
    visible: bool,
    sessions: Vec<SessionEntry>,
    selected_index: usize,
}

struct SessionEntry {
    key: WindowKey,
    title: String,
    working_dir: String,
    is_active: bool,
}

impl Component<TermWmAction> for WmSessionManagerComponent {
    fn render(&mut self, backend: &mut dyn RenderBackend, area: LayoutRect, ctx: &ComponentContext, registry: &mut HitboxRegistry) {
        if !self.visible { return; }
        
        // Clear background with ratatui::widgets::Clear
        let buffer = backend.buffer_mut();
        for y in area.y..area.y + area.height {
            for x in area.x..area.x + area.width {
                if let Some(cell) = buffer.cell_mut((x, y)) {
                    cell.set_style(Style::default().bg(Color::Black));
                }
            }
        }
        
        // Render session list with tap targets
        // Each session is a row with title, working dir, activity indicator
        // Register each row as HitTarget::Component(session_key, entry.key)
    }
    
    fn on_mouse_press(&mut self, local_x: u16, local_y: u16, _button: MouseButton, _modifiers: KeyModifiers, ctx: &ComponentContext) -> EventResult<TermWmAction> {
        // Calculate which session was tapped
        if let Some(entry) = self.hit_test_session(local_x, local_y) {
            return EventResult::Action(TermWmAction::FocusWindow(entry.key));
        }
        EventResult::Ignored
    }
}
```

- Z-ordering ensures overlay appears above all windows

**File:** `crates/term-wm-core/src/runner.rs`

```rust
// After overlay handling, before component dispatch
if app.wm().tap_swap_active() {
    if let Event::Mouse(mouse) = &evt {
        if let MouseEventKind::Press(_) = mouse.kind {
            // Hit-test to find which pane was tapped
            if let Some(target_key) = app.wm().hit_test_pane(mouse.column, mouse.row) {
                app.wm().execute_tap_swap(target_key);
                return flush_state_changes(app, ControlFlow::Continue, false);
            }
        }
    }
    // Show toast: "Select target to swap"
    app.wm().show_notification("Tap a pane to swap positions");
    return flush_state_changes(app, ControlFlow::Continue, false);
}
```
