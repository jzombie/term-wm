# Mobile-First Terminal Adaptation Plan (Unified Architecture)

## Executive Summary

Adapt `term-wm` for mobile-first terminal environments (Termux, Blink Shell) while preserving desktop experience. The architecture enforces a **single, universal input model** across all environments: default `Passthrough` with FAB-triggered command palette. No split-brain state machine.

---

## Codebase Architecture Context

**Key Crates & Files:**
- `term-wm-core/src/window/window_manager/mod.rs` (5711 lines) — Central WM state
- `term-wm-core/src/runner.rs` (1303 lines) — Event loop orchestration
- `term-wm-core/src/components/mod.rs` (779 lines) — Component trait system
- `term-wm-core/src/actions.rs` (310 lines) — Action definitions
- `term-wm-core/src/wm_config.rs` (183 lines) — Configuration
- `term-wm-core/src/layout/tiling.rs` (1490 lines) — BSP layout engine
- `term-wm-core/src/draw_plan.rs` — DrawPlan intermediate representation
- `term-wm-core/src/hitbox_registry.rs` — Hit-test registry
- `term-wm-console/src/draw_plan_renderer.rs` — Low-level renderer (only paints DrawPlan)
- `term-wm-sys-ui-components/` — Component implementations (chrome, overlays)

**Event Flow:**
1. `UnifiedEventSource` → crossterm input + PTY wakeups + signals
2. `run_event_loop` → layered dispatch: overlays → WM input mode → component
3. `dispatch_mouse` → hit-test registry → capture state machine → component forwarding
4. `render_app` → DrawPlan generation → DrawPlanRenderer paints abstract regions

**Architectural Invariants:**
- `term-wm-console` is a low-level rendering target; it must remain ignorant of business UI
- Components implement `Component<TermWmAction>` and emit into the DrawPlan
- `regions` map must preserve logical geometry of all windows at all times
- `WindowManager.z_order` is the authoritative, persistent stacking order — NEVER mutated by transient viewport constraints
- `WindowManager` must NEVER hold `&mut Buffer` references — all visual feedback is declarative via DrawPlan injection
- Hit-test registry is the single source of truth for coordinate-based interception
- **Single input model**: ONE `WmInputMode` state machine across all environments — no mobile/desktop fork

#### 1.4 Event Routing (Universal Passthrough + Bypass Injection)

**File:** `crates/term-wm-core/src/runner.rs`

The event loop uses a SINGLE input mode check — no mobile/desktop fork:

```rust
// In the event loop closure, after computing `evt`:

match app.wm().input_mode() {
    WmInputMode::Passthrough => {
        // Keyboard: FAB chord or palette shortcut → open palette
        if let Event::Key(key) = &evt {
            if key.kind == KeyKind::Press
                && app.wm().keybindings().matches(TermWmAction::OpenCommandPalette, key)
            {
                app.wm().open_command_menu();
                return flush_state_changes(app, ControlFlow::Continue, false);
            }
            // All other keys: pass through to PTY
            let _ = handle_focused_app_event(&evt, app);
            return flush_state_changes(app, ControlFlow::Continue, false);
        }
        
        // Mouse: comprehensive hit-test BEFORE any passthrough
        if let Event::Mouse(mouse) = &evt {
            let hit = app.wm().hitbox_registry().hit_test(mouse.column, mouse.row);
            match hit {
                // FAB tap → open palette
                Some(HitTarget::Fab) => {
                    if matches!(mouse.kind, MouseEventKind::Press(_)) {
                        app.wm().open_command_menu();
                        return flush_state_changes(app, ControlFlow::Continue, false);
                    }
                }
                // Structural WM targets → route to window manager
                Some(HitTarget::WindowBorder(_))
                | Some(HitTarget::ResizeHandle(_))
                | Some(HitTarget::FloatingWindowTitle(_))
                | Some(HitTarget::LayoutHandle) => {
                    // Route to WM's mouse dispatch (drag, resize, snap)
                    app.wm().dispatch_mouse(&evt);
                    return flush_state_changes(app, ControlFlow::Continue, false);
                }
                // Component targets (panels, session manager) → route to component
                Some(HitTarget::Component(_, _))
                | Some(HitTarget::TopPanel)
                | Some(HitTarget::BottomPanel) => {
                    app.wm().dispatch_mouse(&evt);
                    return flush_state_changes(app, ControlFlow::Continue, false);
                }
                // No WM target → safe to pipe to PTY (terminal content area)
                None => {
                    let _ = handle_focused_app_event(&evt, app);
                    return flush_state_changes(app, ControlFlow::Continue, false);
                }
            }
        }
        
        // Other events (resize, focus, paste) → pass through
        let _ = handle_focused_app_event(&evt, app);
        return flush_state_changes(app, ControlFlow::Continue, false);
    }
    WmInputMode::CommandPalette => {
        // BYPASS INJECTION: If the same universal chord is pressed again
        // while the palette is active, forward the chord to the PTY.
        if let Event::Key(key) = &evt {
            if key.kind == KeyKind::Press
                && app.wm().keybindings().matches(TermWmAction::OpenCommandPalette, key)
            {
                app.wm().close_command_menu();
                app.wm().set_input_mode(WmInputMode::Passthrough);
                let chord_bytes = app.wm().chord_bytes().to_vec();
                if let Some(focused) = app.wm().focused_window() {
                    app.wm().write_to_pty(focused, &chord_bytes);
                }
                return flush_state_changes(app, ControlFlow::Continue, false);
            }
        }
        
        if let Some(action) = app.wm().handle_wm_menu_event(&evt) {
            app.wm().execute_menu_action(action);
        }
        return flush_state_changes(app, ControlFlow::Continue, false);
    }
    WmInputMode::TapToSwapTargeting => {
        if let Event::Mouse(mouse) = &evt {
            if let MouseEventKind::Press(_) = mouse.kind {
                if let Some(target_key) = app.wm().hit_test_pane(mouse.column, mouse.row) {
                    app.wm().execute_tap_swap(target_key);
                    return flush_state_changes(app, ControlFlow::Continue, false);
                }
            }
        }
        app.wm().show_notification("Tap a pane to swap positions");
        return flush_state_changes(app, ControlFlow::Continue, false);
    }
}
```

**Key principle:** In Passthrough mode, ALL mouse events hit-test against the `HitboxRegistry` first. Only events that resolve to `None` (pure terminal content area) are piped to the PTY. Structural WM targets (borders, resize handles, floating windows, panels) always route to the WM interaction engine.

**Bypass Injection State Machine (for nesting):**

```
User presses Ctrl+Shift+Space (1st time)
    → WM intercepts, opens CommandPalette
    → Child process receives NOTHING

User presses Ctrl+Shift+Space (2nd time, while palette is open)
    → WM detects same chord while in CommandPalette
    → WM closes palette, reverts to Passthrough
    → WM synthesizes raw ANSI bytes of Ctrl+Shift+Space
    → WM writes bytes directly to active PTY
    → Child process (inner term-wm) receives the chord
    → Inner WM opens its own CommandPalette
```

#### 2.2 DrawPlan Culling (NOT Region Destruction)

**File:** `crates/term-wm-core/src/draw_plan.rs`

Add monocle filtering to the DrawPlan projection:
```rust
pub struct DrawPlan {
    regions: Vec<DrawRegion>,
}

impl DrawPlan {
    /// Apply monocle culling: keep focused window at full area,
    /// mark other windows as hidden but preserve their logical geometry.
    pub fn apply_monocle_culling(&mut self, focused_key: WindowKey) {
        for region in &mut self.regions {
            match &region.region_type {
                RegionType::Window(key) if *key == focused_key => {
                    // Focused window gets full area
                }
                RegionType::Window(_) => {
                    region.hidden = true;
                }
                _ => {}
            }
        }
    }
    
    /// Apply monocle Z-order using stratified layering.
    /// Partitions regions into topological layers, then concatenates
    /// them in correct depth order. The focused window is elevated
    /// within its layer but never above overlays.
    pub fn apply_monocle_z_order(&mut self, focused_key: WindowKey) {
        // Partition into strict topological layers
        let mut hidden_tiled: Vec<DrawRegion> = Vec::new();
        let mut focused_region: Option<DrawRegion> = None;
        let mut other_tiled: Vec<DrawRegion> = Vec::new();
        let mut floating: Vec<DrawRegion> = Vec::new();
        let mut overlays: Vec<DrawRegion> = Vec::new();
        
        for region in self.regions.drain(..) {
            match &region.region_type {
                RegionType::Window(key) if *key == focused_key => {
                    focused_region = Some(region);
                }
                RegionType::Window(_) if region.hidden => {
                    hidden_tiled.push(region);
                }
                RegionType::Window(_) => {
                    other_tiled.push(region);
                }
                RegionType::FloatingWindow(_) => {
                    floating.push(region);
                }
                RegionType::Panel(_) | RegionType::Overlay | RegionType::TargetHighlight(_) => {
                    overlays.push(region);
                }
                _ => {
                    other_tiled.push(region);
                }
            }
        }
        
        // Reassemble in strict depth order:
        // 1. Hidden tiled windows (background, not rendered)
        // 2. Other visible tiled windows
        // 3. Focused window (elevated within tiled layer)
        // 4. Floating windows (above tiled)
        // 5. Overlays (panels, FAB, session manager — always on top)
        self.regions = hidden_tiled;
        self.regions.append(&mut other_tiled);
        if let Some(region) = focused_region {
            self.regions.push(region);
        }
        self.regions.append(&mut floating);
        self.regions.append(&mut overlays);
    }
}
```

```rust
pub struct WmSessionManagerComponent {
    visible: bool,
    sessions: Vec<SessionEntry>,
}

impl Component<TermWmAction> for WmSessionManagerComponent {
    fn render(&mut self, backend: &mut dyn RenderBackend, area: LayoutRect, ctx: &ComponentContext, registry: &mut HitboxRegistry) {
        if !self.visible { return; }
        
        let buffer = backend.buffer_mut();
        let screen_area = ctx.screen_area().unwrap_or(area);
        
        // Clear background
        for y in screen_area.y..screen_area.y + screen_area.height {
            for x in screen_area.x..screen_area.x + screen_area.width {
                if let Some(cell) = buffer.cell_mut((x, y)) {
                    cell.set_symbol(" ").set_style(Style::default().bg(Color::Black));
                }
            }
        }
        
        // Render session list with tap targets
        for (i, entry) in self.sessions.iter().enumerate() {
            let row_rect = LayoutRect {
                x: screen_area.x,
                y: screen_area.y + i as i32,
                width: screen_area.width,
                height: 1,
            };
            if let Some(key) = ctx.window_key() {
                registry.register(HitTarget::Component(key, entry.key), row_rect);
            }
        }
    }
    
    fn on_mouse_press(&mut self, _local_x: u16, local_y: u16, _button: MouseButton, _modifiers: KeyModifiers, _ctx: &ComponentContext) -> EventResult<TermWmAction> {
        let index = local_y as usize;
        if index < self.sessions.len() {
            return EventResult::Action(TermWmAction::FocusWindow(self.sessions[index].key));
        }
        EventResult::Ignored
    }
}
```

```rust
pub fn apply_monocle_culling(&mut self, focused_key: WindowKey, screen: LayoutRect) {
```

```rust
pub fn apply_monocle_z_order(&mut self, focused_key: WindowKey) {
    let mut hidden_tiled: Vec<RenderRegion> = Vec::new();
    let mut focused_region: Option<RenderRegion> = None;
    let mut other_tiled: Vec<RenderRegion> = Vec::new();
    let mut floating: Vec<RenderRegion> = Vec::new();
    let mut overlays: Vec<RenderRegion> = Vec::new();

    for region in self.regions.drain(..) {
        match &region.region_type {
            RegionType::Window(key) if *key == focused_key => {
                focused_region = Some(region);
            }
            RegionType::Window(_) if region.hidden => {
                hidden_tiled.push(region);
            }
            RegionType::Window(_) => {
                other_tiled.push(region);
            }
            RegionType::FloatingWindow(_) => {
                floating.push(region);
            }
            // FAB goes in overlays layer (above everything)
            RegionType::Fab => {
                overlays.push(region);
            }
            RegionType::Panel(_) | RegionType::Overlay | RegionType::TargetHighlight(_) => {
                overlays.push(region);
            }
            _ => {
                other_tiled.push(region);
            }
        }
    }

    // Reassemble in strict depth order:
    // 1. Hidden tiled windows (background, not rendered)
    // 2. Other visible tiled windows
    // 3. Focused window (elevated within tiled layer)
    // 4. Floating windows (above tiled)
    // 5. Overlays (panels, FAB, session manager — always on top)
    self.regions = hidden_tiled;
    self.regions.append(&mut other_tiled);
    if let Some(region) = focused_region {
        self.regions.push(region);
    }
    self.regions.append(&mut floating);
    self.regions.append(&mut overlays);
}
```

Three critical bugs prevent the mobile adaptation from functioning:

1. **FAB Rendering Coordinate Mismatch (SEV-1):** The FAB renders nothing because of coordinate intersection issues. `WmFabComponent::render` uses absolute screen coordinates (`self.fab_rect`) against a locally-sized `RatatuiBackend` buffer (which has origin at 0,0). The intersection of `(77, 23, 3, 1)` with `(0, 0, 3, 1)` is empty, so `bounds.width == 0` and the FAB draws nothing.

2. **Panel Culling Bypass (SEV-1):** Monocle mode culling relies on filtering the `DrawPlan`. However, `engine.rs` explicitly omits panels from the `DrawPlan`, and `src/lib.rs` renders panels manually via `render_panels(backend, wm)`. The `DrawPlan` cannot cull what it does not contain, so the top panel remains visible in monocle mode.

3. **Action Routing Black Hole (SEV-1):** When the FAB is tapped, it returns `TermWmAction::OpenCommandPalette`. The `drain_action_queue` function in `runner.rs` captures this action, queues it, and passes it to the *currently focused window's component* via `comp.update()`. The window component has no capability to open the global command palette, so the action is silently dropped.

### Fix 1: FAB Rendering Coordinates

**File:** `crates/term-wm-sys-ui-components/src/wm_fab.rs`

**Root Cause Analysis:**

The `render_direct_to_buffer` method in `draw_plan_renderer.rs:178` sets up the buffer as follows:
```rust
let mut buffer = std::mem::replace(&mut self.direct_buffer, Buffer::empty(Rect::ZERO));
buffer.resize(area);  // area = Rect { x: 77, y: 23, width: 3, height: 1 }
buffer.reset();
let mut backend = RatatuiBackend::new(buffer, area);
```

After `buffer.resize(area)`, `buffer.area = Rect { x: 77, y: 23, width: 3, height: 1 }`. The buffer's coordinate space is absolute screen-space.

The original `WmFabComponent::render` code:
```rust
let ratatui_area = layout_rect_to_rect(self.fab_rect);  // Rect { x: 77, y: 23, width: 3, height: 1 }
let bounds = ratatui_area.intersection(buffer.area);      // Rect { x: 77, y: 23, width: 3, height: 1 }
// bounds is NON-EMPTY — the intersection should work
for yy in bounds.y..bounds.y.saturating_add(bounds.height) {  // 23..24
    for xx in bounds.x..bounds.x.saturating_add(bounds.width) {  // 77..80
        if let Some(cell) = buffer.cell_mut((xx, yy)) { ... }
    }
}
```

This SHOULD work because `buffer.cell_mut((77, 23))` maps to the buffer's internal storage at offset `(77-77, 23-23) = (0, 0)`. The intersection logic is correct.

**However**, the user reports the FAB renders nothing. The likely cause is that the `RatatuiBackend` created in `render_direct_to_buffer` uses a **different buffer** than the one in `WmFabComponent::render`. Let me trace:

1. `render_direct_to_buffer` creates `backend = RatatuiBackend::new(buffer, area)` where `buffer` is the `direct_buffer`
2. `component.render(&mut backend, ...)` is called — this passes `&mut backend` (which wraps the `direct_buffer`)
3. In `WmFabComponent::render`, `downcast_ratatui(backend)` returns the `RatatuiBackend` which wraps the `direct_buffer`
4. `buffer = &mut ratatui_backend.buffer` — this IS the `direct_buffer`
5. The cells are written to `direct_buffer`
6. Back in `render_direct_to_buffer`, `blit_buffer(&backend.buffer, target_buf, area)` copies from `direct_buffer` to `target_buf`

This should work. **The actual bug may be that `self.fab_rect` is not being set correctly before the render call**, or the hitbox registration is interfering.

**Simpler Fix:** Regardless of the exact root cause, the safest approach is to iterate over `buffer.area` directly, which is guaranteed to be the buffer's actual coordinate space:

**Code Change:**
```rust
// Replace lines 86-103 in wm_fab.rs:
let ratatui_backend = downcast_ratatui(backend);
let buffer = &mut ratatui_backend.buffer;

// Iterate over the buffer's actual allocated area.
// The buffer uses absolute screen-space coordinates (e.g., x: 77, y: 23).
// Using buffer.area.x/y ensures we write to valid cells.
let area = buffer.area;
for yy in area.y..area.y.saturating_add(area.height) {
    for xx in area.x..area.x.saturating_add(area.width) {
        if let Some(cell) = buffer.cell_mut((xx, yy)) {
            cell.set_symbol("≡").set_style(
                Style::default()
                    .fg(Color::White)
                    .bg(Color::DarkGray)
                    .add_modifier(Modifier::BOLD),
            );
        }
    }
}
```

**Verification:** This fix is correct because:
1. `buffer.area` is set by `render_direct_to_buffer` to the FAB's absolute bounds
2. `buffer.area.x..buffer.area.x + buffer.area.width` iterates over valid absolute coordinates
3. `buffer.cell_mut((xx, yy))` with absolute coordinates maps correctly to the buffer's internal storage

### Fix 2: Panel Culling in Monocle Mode


**Verification:** This ensures that when `render_panels()` calls `wm.panel_active()`, it returns `false` in monocle mode, causing the panel to be skipped.

### Fix 3: Global Action Interception

**File:** `crates/term-wm-core/src/runner.rs`

**Root Cause:** The `drain_action_queue` function processes actions from the queue. It handles `SendNotification` specially, but all other actions (including `OpenCommandPalette`) are forwarded to the window component via `comp.update()`. The window component has no capability to open the global command palette, so the action is silently dropped.

**Fix:** Add interception logic for structural WM actions *before* forwarding them to the child component.

**Code Change:**
```rust
// Replace lines 70-89 in runner.rs:
fn drain_action_queue<A: WindowManagerHost>(
    app: &mut A,
    queue: &mut VecDeque<(WindowKey, TermWmAction)>,
) {
    while let Some((key, action)) = queue.pop_front() {
        match action {
            TermWmAction::SendNotification(msg) => {
                tracing::info!("drain_action_queue: SendNotification({})", msg);
                app.wm()
                    .push_notification(msg, std::time::Duration::from_secs(3));
            }
            TermWmAction::OpenCommandPalette => {
                if app.wm().command_menu_visible() {
                    app.wm().close_command_menu();
                } else {
                    app.wm().open_command_menu();
                }
            }
            action => {
                let ctx = app.wm().component_context_for(true, key);
                if let Some(comp) = app.wm().component_for_key_mut(key) {
                    comp.update(action, &ctx, queue);
                }
            }
        }
    }
}
```

**Verification:** When the FAB is tapped and returns `OpenCommandPalette`, `drain_action_queue` now intercepts this action and calls `app.wm().open_command_menu()` directly, bypassing the window component.

### Fix 4: FAB Hitbox Registration (Dead-on-Arrival)

## Amendment 12: FAB Architectural Cleanup

### Issue: FAB Rendering Architecture

**Files to modify:**
1. `crates/term-wm-core/src/draw_plan.rs` - Remove `RegionType::Fab` variant and all references
2. `crates/term-wm-core/src/engine.rs` - Remove FAB region injection
3. `crates/term-wm-sys-ui-components/src/wm_fab.rs` - Anchor geometry to `self.fab_rect`

### Root Cause

1. **Z-Index Illusion and Dead Code (SEV-1):** The `.iter().find()` scan against the `DrawPlan` is an architectural anti-pattern. Because the FAB must render out-of-band to survive panel occlusion, `RegionType::Fab` is functionally dead.
