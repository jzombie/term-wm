# Plan: Gate + Park + 64KB Buffer

## Problem
Under `cat /dev/random`, two windows consume ~200% CPU. The root cause: the reader thread parses 4KB batches and sends PtyWakeups in a tight loop. The coalesce timer's `recv_timeout` loop in `poll()` greedily drains the channel, so it never fills and backpressure never engages. The reader spins at 100%.

## Solution
Three changes working together:

1. **Gate wakeup per consumption cycle** — reader only sends ONE PtyWakeup per dirty→consumed cycle. The `dirty.swap(true)` gate prevents redundant wakeups. When the renderer hasn't consumed, the reader parks via `thread::park()`.

2. **64KB read buffer** — increases throughput per unpark from ~240KB/s (4KB × 60fps) to ~3.8MB/s (64KB × 60fps). More than enough for any terminal workload.

3. **Remove coalesce timer** — with only one wakeup per cycle, the 3ms coalesce window never batches multiple wakeups. Simplify `poll()` to return `Ok(false)` immediately on PtyWakeup.

### 2. `crates/term-wm-pty-engine/src/pty.rs` — `parser_read_loop()`, ~line 517

Replace:
```rust
if let Ok(mut guard) = shared_screen.lock() {
    *guard = parser.screen().clone();
}
dirty.store(true, Ordering::Release);

if let Ok(guard) = wakeup.lock()
    && let Some(ref cb) = *guard
```

With:
```rust
if let Ok(mut guard) = shared_screen.lock() {
    *guard = parser.screen().clone();
}

// Gate: only one wakeup per consumption cycle.
let was_dirty = dirty.swap(true, Ordering::Release);
if !was_dirty {
    if let Ok(guard) = wakeup.lock()
        && let Some(ref cb) = *guard
    {
        cb();
    }
} else {
    // Renderer hasn't consumed yet — park until Pty::screen() unparks.
    thread::park();
}
```

### 4. `crates/term-wm-pty-engine/src/pty.rs` — `Pty::set_wakeup()`, ~line 175

Add unpark at the end — reader might park before wakeup is set during init:
```rust
pub fn set_wakeup(&mut self, cb: Option<Arc<dyn Fn() + Send + Sync>>) {
    if let Ok(mut guard) = self.wakeup.lock() {
        *guard = cb;
    }
    // Reader might park before wakeup is set — wake it up
    if let Some(reader) = &self.reader {
        reader.thread().unpark();
    }
}
```

With the coalesce timer removed, `poll()` returns immediately on PtyWakeup without clearing the set. The key remains in `dirty_windows` for `take_dirty_windows()` to consume.

Update the assertion at line 464:


## Throughput math

- 64KB per read × ~60fps = ~3.8 MB/s
- `grep` on a 100MB log: ~26 seconds
- `cat /dev/random`: child blocked by OS pipe buffer (~8KB macOS, ~64KB Linux), effective rate = pipe buffer size per frame ≤ ~4MB/s
- Normal terminal use (typing, editing): negligible data, instant response
