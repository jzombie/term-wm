# Plan: ArcSwap + Park + 16ms Frame Pacer

## Goal
Eliminate ~44% reader CPU spinning under continuous PTY streaming (`cat /dev/random`) via lock-free ArcSwap double buffer and reader parking with 16ms deadlock safety net.

## Changes

### 1. Add `arc-swap` dependency
**Files:**
- `Cargo.toml` (workspace): Add `arc-swap = "1.7"` to `[workspace.dependencies]`
- `crates/term-wm-pty-engine/Cargo.toml`: Add `arc-swap = { workspace = true }` to `[dependencies]`

### 2. Rewrite `Pty` synchronization in `crates/term-wm-pty-engine/src/pty.rs`

**Field changes:**
- `shared_screen: Arc<Mutex<vt100::Screen>>` → `shared_screen: Arc<ArcSwap<vt100::Screen>>`
- `cached_screen: vt100::Screen` → keep, still needed for mutable paths
- Add `screen_arc: Option<Arc<vt100::Screen>>` — lock-free cached reference for read path
- Update `use` imports: add `arc_swap::ArcSwap`, remove `Mutex` (if no longer needed for other fields)

**Constructor (`spawn_with_scrollback`):**
```rust
let initial = vt100::Parser::new(size.rows, size.cols, scrollback_len);
let shared_screen = Arc::new(ArcSwap::new(Arc::new(initial.screen().clone())));
// ...
Ok(Self {
    // ...
    shared_screen,
    screen_arc: None,
    // cached_screen still initialized (for mutable ops)
    cached_screen: initial.screen().clone(),
    // ...
})
```

**Reader loop (`parser_read_loop`):** Replace Mutex publish + add park:
```rust
// Publish: one clone into Arc, atomic pointer swap (no mutex contention)
let new_screen = Arc::new(parser.screen().clone());
shared_screen.store(new_screen);
dirty.store(true, Ordering::Release);

// Wakeup main thread
if let Ok(guard) = wakeup.lock()
    && let Some(ref cb) = *guard
{
    cb();
}

// Park until main thread consumes this screen
while dirty.load(Ordering::Acquire) && !shutdown.load(Ordering::Acquire) {
    thread::park();
}

if shutdown.load(Ordering::Acquire) {
    break;
}
```

**`Pty::screen()`:** Lock-free load from ArcSwap, unpark reader:
```rust
pub fn screen(&mut self) -> &vt100::Screen {
    self.poll_foreground();
    if self.dirty.swap(false, Ordering::Acquire) {
        // Lock-free load — atomic refcount increment, no clone
        self.screen_arc = Some(self.shared_screen.load());
        // Unpark reader so it can read next batch
        if let Some(reader) = &self.reader {
            reader.thread().unpark();
        }
        // DSR response
        if self.dsr_requested.swap(false, Ordering::Relaxed) {
            if let Some(ref screen) = self.screen_arc {
                let (row, col) = screen.cursor_position();
                let response = format!("\x1b[{};{}R", row.saturating_add(1), col.saturating_add(1));
                let _ = self.write_bytes(response.as_bytes());
            }
        }
    }
    // Return ref to Arc contents — borrows self via screen_arc field
    self.screen_arc.as_ref().map(|arc| &**arc).unwrap_or(&self.cached_screen)
}
```

**`Pty::screen_mut()`:** Clone from Arc on-demand (rare path):
```rust
pub fn screen_mut(&mut self) -> &mut vt100::Screen {
    self.screen();
    if let Some(ref screen_arc) = self.screen_arc {
        self.cached_screen = (**screen_arc).clone();
    }
    &mut self.cached_screen
}
```

**`Pty::max_scrollback()`:** Change to use `screen_mut()` instead of `self.cached_screen` directly:
```rust
pub fn max_scrollback(&mut self) -> usize {
    if self.scrollback_len == 0 { return 0; }
    let screen = self.screen_mut();
    let current = screen.scrollback();
    screen.set_scrollback(self.scrollback_len);
    let max = screen.scrollback();
    screen.set_scrollback(current);
    max
}
```

**Add `Drop` impl** (critical: prevent zombie reader thread when Pty dropped without `into_parts()`):
```rust
impl Drop for Pty {
    fn drop(&mut self) {
        self.shutdown.store(true, Ordering::Release);
        if let Some(reader) = &self.reader {
            reader.thread().unpark();
        }
    }
}
```

**`ParserReadLoopArgs`:** Change `shared_screen` field type to `Arc<ArcSwap<vt100::Screen>>`

**Update tests in `#[cfg(test)]`:**
- `make_parser_test_args()`: Change `shared_screen` initialization to use `ArcSwap::new(Arc::new(...))`
- Existing tests continue to work; the reader loop still sets dirty and processes data correctly
- The park loop in reader requires special handling in tests: either set shutdown before reader starts, or ensure screen() is called to clear dirty

### 3. Leave PowerSaver at 3600s — no change
**File:** `crates/term-wm-core/src/power_profile.rs` — **Do not modify.**

The 16ms deadlock safety net is unnecessary. With the reader parked after each publish, only one PtyWakeup per pane is ever in the bounded channel (capacity 256). The channel can never fill up, so PtyWakeups can never be silently dropped. The 3600s PowerSaver preserves true idle: 0% CPU, CPU deep C-states, no battery drain.
