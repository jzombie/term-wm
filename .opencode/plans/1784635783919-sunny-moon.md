# Plan: Eliminate Enum Delegation Boilerplate with Declarative Macros

## Problem

The facade enums (`CoreWmComponent`, `LayerComponent`, `OverlayComponent`, `TestComponent`) each contain ~250–580 lines of identical `match self` delegation code. Each method simply matches on the enum variant and forwards to the inner type's same-named method. Across 4 enums this is ~1500 lines of boilerplate.

Adding a new variant requires adding a match arm in all 21 `Component` trait methods — easy to miss, tedious to review.

## Solution

Create three `macro_rules!` macros that auto-generate the `Component`, `WmComponent`, and `Overlay` trait implementations:

- `impl_component_delegate!` — generates `Component<TermWmAction>` impl
- `impl_wm_component_delegate!` — generates `WmComponent` impl (for LayerComponent)
- `impl_overlay_delegate!` — generates `Overlay<TermWmAction>` impl (for OverlayComponent)

### Macro Location

Define all three macros in `crates/term-wm-core/src/macros.rs` with `#[macro_export]`. This makes them usable from both `term-wm-core` (for `TestComponent`) and `term-wm-ui-facade` (for facade enums).

### Macro Design

```rust
#[macro_export]
macro_rules! impl_component_delegate {
    ($enum_name:ident { $($variant:ident($inner:ty)),* $(,)? }) => {
        impl $crate::components::Component<$crate::actions::TermWmAction> for $enum_name {
            fn init(&mut self) { match self { $(Self::$variant(c) => c.init(),)* } }
            fn on_mount(&mut self, key: $crate::window::WindowKey, app: &$crate::app_context::AppContext) {
                match self { $(Self::$variant(c) => c.on_mount(key, app),)* }
            }
            fn hitbox_id(&self) -> Option<$crate::hitbox_registry::HitboxId> {
                match self { $(Self::$variant(c) => c.hitbox_id(),)* }
            }
            fn handle_events(&mut self, event: &$crate::events::Event, ctx: &$crate::component_context::ComponentContext) -> $crate::actions::EventResult<$crate::actions::TermWmAction> {
                match self { $(Self::$variant(c) => c.handle_events(event, ctx),)* }
            }
            // ... all 21 methods, each with a single match line
        }
    };
}
```

Key design choice: use fully-qualified paths (`$crate::...`) so the macro works from any crate that depends on `term_wm_core`.

### Steps

1. **Create `crates/term-wm-core/src/macros.rs`** with `impl_component_delegate!`, `impl_wm_component_delegate!`, and `impl_overlay_delegate!`.

2. **Add `pub mod macros;`** to `crates/term-wm-core/src/lib.rs`.

### WmComponent and Overlay Macros

`impl_wm_component_delegate!` is simpler since `WmComponent` has only default methods:
```rust
#[macro_export]
macro_rules! impl_wm_component_delegate {
    ($enum_name:ident { $($variant:ident($inner:ty)),* $(,)? }) => {
        impl std::fmt::Debug for $enum_name {
            fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
                match self { $(Self::$variant(c) => std::fmt::Debug::fmt(c, f),)* }
            }
        }
        impl $crate::components::WmComponent for $enum_name {
            // All methods have default impls — no delegation needed
            // for consume_area, process_action, query, hit_test, begin_frame,
            // visible, set_visible unless overridden.
        }
    };
}
```

`impl_overlay_delegate!` delegates the `Overlay` methods:
```rust
#[macro_export]
macro_rules! impl_overlay_delegate {
    ($enum_name:ident { $($variant:ident($inner:ty)),* $(,)? }) => {
        impl $crate::components::Overlay<$crate::actions::TermWmAction> for $enum_name {
            fn visible(&self) -> bool {
                match self { $(Self::$variant(c) => c.visible(),)* }
            }
            fn shadow_rect(&self, area: $crate::Rect) -> Option<$crate::Rect> {
                match self { $(Self::$variant(c) => c.shadow_rect(area),)* }
            }
            fn handle_confirm_event(&mut self, event: &$crate::events::Event) -> Option<$crate::actions::ConfirmAction> {
                match self { $(Self::$variant(c) => c.handle_confirm_event(event),)* }
            }
            fn mark_dirty(&mut self) { match self { $(Self::$variant(c) => c.mark_dirty(),)* } }
            fn set_menu_items(&mut self, items: Vec<$crate::components::MenuItem<$crate::actions::TermWmAction>>) {
                match self { $(Self::$variant(c) => c.set_menu_items(items),)* }
            }
        }
    };
}
```

Note: `OverlayComponent` also has a `Debug` impl (via `impl_wm_component_delegate!` if needed, or standalone).

3. **Replace `CoreWmComponent`** — the entire file becomes:
```rust
use term_wm_ui_components::scroll_view::ScrollViewComponent;
use term_wm_ui_components::terminal::TerminalComponent;
use term_wm_sys_ui_components::wm_debug_log::WmDebugLogComponent;
use term_wm_sys_ui_components::wm_session_manager::WmSessionManagerComponent;
use term_wm_sys_ui_components::wm_system_panel::WmSystemPanelComponent;

impl_component_delegate!(CoreWmComponent {
    Terminal(ScrollViewComponent<TerminalComponent>),
    DebugLog(WmDebugLogComponent),
    SystemPanel(WmSystemPanelComponent),
    SessionManager(WmSessionManagerComponent),
    Noop(NoopComponent),
});
```

Delete the ~260-line `impl Component for CoreWmComponent` body.

4. **Replace `LayerComponent`** — becomes:
```rust
impl_component_delegate!(LayerComponent {
    TopPanel(WmTopPanelComponent),
    BottomPanel(WmBottomPanelComponent),
    Fab(WmFabComponent),
    NotificationArea(WmNotificationAreaComponent),
    CommandPalette(WmCommandPaletteComponent),
});
impl_wm_component_delegate!(LayerComponent {
    TopPanel(WmTopPanelComponent),
    BottomPanel(WmBottomPanelComponent),
    Fab(WmFabComponent),
    NotificationArea(WmNotificationAreaComponent),
    CommandPalette(WmCommandPaletteComponent),
});
```

Delete the ~260-line `impl Component` and ~70-line `impl WmComponent` bodies. Note that `LayerComponent` process_action, query, hit_test, and begin_frame methods have CUSTOM overrides in the current WmComponent impl — these need to be handled. The `impl_wm_component_delegate!` macro provides default empty impls. Custom overrides can be added as a separate `impl WmComponent for LayerComponent { fn process_action(...) {...} }` block.

5. **Replace `OverlayComponent`** — becomes:
```rust
impl_component_delegate!(OverlayComponent {
    Help(WmHelpOverlayComponent),
    CommandPalette(WmCommandPaletteComponent),
    ExitConfirm(ConfirmOverlayComponent),
});
impl_overlay_delegate!(OverlayComponent {
    Help(WmHelpOverlayComponent),
    CommandPalette(WmCommandPaletteComponent),
    ExitConfirm(ConfirmOverlayComponent),
});
```

Delete ~220-line `impl Component` and ~80-line `impl Overlay` bodies.

6. **Replace `TestComponent`** — the delegation impl in test_component.rs becomes:
```rust
impl_component_delegate!(TestComponent {
    Noop(NoopComponent),
    ActionRecorder(ActionRecorder),
    KeyRecorder(KeyRecorder),
    SelComponent(SelComponent),
    RenderTracker(RenderTracker),
});
```

Delete the ~420-line `impl Component for TestComponent` body (the standalone variant impls for ActionRecorder, KeyRecorder, SelComponent, RenderTracker stay — only the enum delegation impl is replaced).
