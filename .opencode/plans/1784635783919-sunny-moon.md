# Plan: Enum-Backed Components for WindowManager

## Goal

Replace `Box<dyn Component<TermWmAction>>` in `Window::component` with a concrete enum stored in a generational arena, eliminating heap allocation per-window, vtable dispatch, and the `component_downcast_mut` downcast pattern. No backward compat.

---

## Design

### Two-Level SlotMap (Window non-generic, WindowManager<C> generic)

Components live in a separate `SlotMap` inside `WindowManager<C>`. Windows hold only a `ComponentKey`. This keeps `Window` non-generic while containing the generic to `WindowManager<C>` alone.

```rust
pub struct WindowManager<C: Component<TermWmAction>> {
    windows: SlotMap<WindowKey, Window>,
    components: SlotMap<ComponentKey, C>,
    // ... existing fields
}

pub struct Window {
    pub component_key: ComponentKey,  // replaces `component: Box<dyn Component>`
    // ... other fields unchanged
}
```

### Compositional Extensibility (Fixes SEV-1 Extensibility)

Two-layer enum pattern enabling third-party injection without modifying source:

1. **Built-in enum** defined in `term-wm-ui-components` (or similar high-level crate):
```rust
// crates/term-wm-ui-components/src/core_component.rs
pub enum CoreWmComponent {
    Terminal(ScrollViewComponent<TerminalComponent>),
    DebugLog(WmDebugLogComponent),
    SystemPanel(WmSystemPanelComponent),
    SessionManager(WmSessionManagerComponent),
    Noop(term_wm_core::NoopComponent),
}

impl Component<TermWmAction> for CoreWmComponent { ... }
```

2. **Third-party/user enum** wraps the core enum:
```rust
// src/components.rs (binary) or downstream user crate
pub enum AppRootComponent {
    Core(CoreWmComponent),
    CustomAuthWidget(MyAuthWidget),  // third-party extension
}

impl Component<TermWmAction> for AppRootComponent {
    fn render(&mut self, ...) {
        match self {
            Self::Core(c) => c.render(...),       // delegates to CoreWmComponent
            Self::CustomAuthWidget(c) => c.render(...),
        }
    }
}
```

3. **Binary instantiates** via the outer enum:
```rust
type AppWM = WindowManager<AppRootComponent>;
```

### No Vtable Coercion (Fixes SEV-1 Dispatch Leak)

`component_for_key_mut` returns `Option<&mut C>` — never casts to `&dyn Component`. All callers work through `<C: Component<TermWmAction>>` bounds using static dispatch:

```rust
impl<C: Component<TermWmAction>> WindowManager<C> {
    pub fn component_for_key_mut(&mut self, key: WindowKey) -> Option<&mut C> {
        let w = self.windows.get_mut(key)?;
        self.components.get_mut(w.component_key)
    }
}
```

### Generic Propagation to Application Root (Fixes SEV-2)

`App`, `TermWmApp`, `AppBuilder::build()`, `render_app()`, and `DrawPlanRenderer::render()` all become generic over `C: Component<TermWmAction>`. Monomorphization at the binary entry point generates one copy.

## Implementation Steps

### Step 1: Add `ComponentKey` type

**File:** `crates/term-wm-core/src/window/mod.rs`

```rust
slotmap::new_key_type! { pub struct ComponentKey; }
```

### Step 2: Change `Window` struct

**File:** `crates/term-wm-core/src/window/entry.rs`

- Remove `pub component: Box<dyn Component<TermWmAction>>`
- Add `pub component_key: ComponentKey`
- `Window::new()` takes `ComponentKey` instead of boxed component

### Step 3: Make `WindowManager<C>` generic

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

- Add `C: Component<TermWmAction>` generic param to struct and all impl blocks
- Add field `components: SlotMap<ComponentKey, C>`
- Add field `next_component_seq: CounterKey`
- Constructor `with_config` initializes new fields
- `create_window` takes `C`, allocates in arena, stores key in `Window`
- `component_for_key_mut` → **returns `Option<&mut C>`** (NO `&dyn Component` cast)
- `component_for_key` → returns `Option<&C>`
- `spawn` takes `C` directly, calls `on_mount` via `&mut C`
- Remove `spawn_boxed` (no longer needed)
- `set_system_window` takes `C` directly

### Step 4: Update component-access paths

**File:** `crates/term-wm-core/src/window/window_manager/chrome.rs`

`close_window`: `self.components.get_mut(w.component_key).map(|c| c.destroy())` then `self.components.remove(w.component_key)`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

`set_direct_mode`: `self.components.get_mut(w.component_key).map(|c| c.clear_selection())`

### Step 5: Create test enum + update all core tests

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs` (tests section)

Add `#[cfg(test)] pub(crate) enum TestComponent { Noop(NoopComponent), ActionRecorder(ActionRecorder), KeyRecorder(KeyRecorder), SelComponent(SelComponent) }` with full `Component` impl.

Update all ~50 test functions in `mod.rs`, `runner.rs`, and `tests/integration_tiling.rs` to use `WindowManager::<TestComponent>` and `TestComponent::Noop(NoopComponent)` etc.

### Step 6: Create `CoreWmComponent` enum

**New file:** `crates/term-wm-ui-components/src/core_component.rs`

Enum wrapping all built-in window root component types. Full `Component<TermWmAction>` impl delegating via match.

Export from `crates/term-wm-ui-components/src/lib.rs`.

### Step 7: Create `AppRootComponent` enum in binary

**New file or addition:** `src/components.rs`

```rust
pub enum AppRootComponent {
    Core(CoreWmComponent),
}
// No custom variants yet — purely wraps CoreWmComponent.
// Third parties would add variants here.
```

Implement `Component<TermWmAction>` delegating to `CoreWmComponent`.

### Step 8: Make generic propagation chain

**File:** `crates/term-wm-core/src/config.rs`

`AppBuilder::build()` becomes generic:
```rust
impl AppBuilder {
    pub fn build<C: Component<TermWmAction>>(self) -> Result<WindowManager<C>, ConfigError> {
        // ... unchanged body — just returns WindowManager<C>
    }
}
```

**File:** `src/term_wm_app.rs`

`TermWmApp<C: Component<TermWmAction>>` — all methods become generic. Remove `register_boxed`. `register` takes `C` directly.

**File:** `src/lib.rs`

`render_app` becomes `pub fn render_app<C: Component<TermWmAction>>(..., wm: &mut WindowManager<C>, ...)`.

**File:** `crates/term-wm-console/src/draw_plan_renderer.rs`

`DrawPlanRenderer::render()` becomes generic `<C: Component<TermWmAction>>`. `render_window_composite` takes `component: &mut C`. `render_direct` takes `component: &mut C`. `composite_window` closure continues to work (already generic over F).

### Step 9: Update binary creation sites

**File:** `src/main.rs`

Instantiate `App::<AppRootComponent>::new_with(...)`. Replace `Box::new(...)` with `AppRootComponent::Core(CoreWmComponent::Terminal(...))`.

### Step 10: Update examples

**File:** `examples/dual_image.rs` — define a local enum or update to use `CoreWmComponent`.

## Key API Surface Changes

| Current | New |
|---------|-----|
| `WindowManager` (non-generic) | `WindowManager<C: Component<TermWmAction>>` |
| `Window { component: Box<dyn Component> }` | `Window { component_key: ComponentKey }` |
| `create_window(Box::new(...))` | `create_window(AppRootComponent::Core(...))` |
| `component_for_key_mut() -> &mut dyn Component` | `component_for_key_mut() -> &mut C` (static dispatch) |
| `AppBuilder::build() -> WindowManager` | `build::<C>() -> WindowManager<C>` |
| `spawn_boxed` | Removed |
| `register_boxed` | Removed |
| `component_downcast_mut` | Removed (direct match on enum) |
| `DrawPlanRenderer::render(&mut WindowManager)` | `render::<C>(&mut WindowManager<C>)` |
| `render_app(wm: &mut WindowManager)` | `render_app::<C>(wm: &mut WindowManager<C>)` |
