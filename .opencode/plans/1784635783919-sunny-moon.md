# Plan: Enum-Backed Components for WindowManager

## Goal

Replace `Box<dyn Component<TermWmAction>>` in `Window::component` with a concrete enum stored in a generational arena, eliminating heap allocation per-window, vtable dispatch, and the `component_downcast_mut` downcast pattern. No backward compat.

---

## Design

### Two-Level SlotMap (Window non-generic, WindowManager<C> generic)

Components live in a separate `SlotMap` inside `WindowManager<C>`. Windows hold only a `ComponentKey`. This keeps `Window` non-generic while containing the generic to `WindowManager<C>` alone.

```rust
pub struct WindowManager<C: Component<TermWmAction>> {
    windows: SlotMap<WindowKey, Window>,
    components: SlotMap<ComponentKey, C>,
    // ... existing fields
}

pub struct Window {
    pub component_key: ComponentKey,  // replaces `component: Box<dyn Component>`
    // ... other fields unchanged
}
```

### Compositional Extensibility (Fixes SEV-1 Extensibility)

Two-layer enum pattern enabling third-party injection without modifying source:

1. **Built-in enum** defined in `term-wm-ui-components` (or similar high-level crate):
```rust
// crates/term-wm-ui-components/src/core_component.rs
pub enum CoreWmComponent {
    Terminal(ScrollViewComponent<TerminalComponent>),
    DebugLog(WmDebugLogComponent),
    SystemPanel(WmSystemPanelComponent),
    SessionManager(WmSessionManagerComponent),
    Noop(term_wm_core::NoopComponent),
}

impl Component<TermWmAction> for CoreWmComponent { ... }
```

2. **Third-party/user enum** wraps the core enum:
```rust
// src/components.rs (binary) or downstream user crate
pub enum AppRootComponent {
    Core(CoreWmComponent),
    CustomAuthWidget(MyAuthWidget),  // third-party extension
}

impl Component<TermWmAction> for AppRootComponent {
    fn render(&mut self, ...) {
        match self {
            Self::Core(c) => c.render(...),       // delegates to CoreWmComponent
            Self::CustomAuthWidget(c) => c.render(...),
        }
    }
}
```

3. **Binary instantiates** via the outer enum:
```rust
type AppWM = WindowManager<AppRootComponent>;
```

### No Vtable Coercion (Fixes SEV-1 Dispatch Leak)

`component_for_key_mut` returns `Option<&mut C>` â€” never casts to `&dyn Component`. All callers work through `<C: Component<TermWmAction>>` bounds using static dispatch:

```rust
impl<C: Component<TermWmAction>> WindowManager<C> {
    pub fn component_for_key_mut(&mut self, key: WindowKey) -> Option<&mut C> {
        let w = self.windows.get_mut(key)?;
        self.components.get_mut(w.component_key)
    }
}
```

### Generic Propagation to Application Root (Fixes SEV-2)

`App`, `TermWmApp`, `AppBuilder::build()`, `render_app()`, and `DrawPlanRenderer::render()` all become generic over `C: Component<TermWmAction>`. Monomorphization at the binary entry point generates one copy.

## Amendment D: Eradicate Remaining Box<dyn WmComponent / Overlay / VerticalStack Children>

**Problem:** Three remaining `Box<dyn ...>` patterns subvert the static-dispatch architecture:

| Location | Storage | Trait | Call Sites |
|----------|---------|-------|------------|
| `LayerManager::layers` | `HashMap<LayerId, Box<dyn WmComponent>>` | `WmComponent` | `AppBuilder`, `set_notification_component` |
| `WindowManager::overlays` | `SlotMap<OverlayKey, Box<dyn Overlay>>` | `Overlay` | `open_help_overlay`, `open_exit_confirm_overlay`, `open_command_palette_overlay` |
| `VerticalStackComponent::children` | `Vec<Box<dyn Component>>` | `Component` | `WmSystemPanelComponent::new` (5 children: Label, Spacer, Button) |

### Solution: Generic `LayerManager<L>` + Generic Overlays `<O>` + Generic `VerticalStackComponent<C>`

**Critical constraint:** `term-wm-core` must NOT depend on `term-wm-ui-facade` (circular dependency). Therefore `LayerManager` and overlay storage in `WindowManager` must be generic â€” matching the `WindowManager<C>` pattern.

```rust
// term-wm-core â€” fully generic, no concrete UI dependencies
pub struct LayerManager<L: WmComponent> {
    layers: HashMap<LayerId, L>,
}

pub struct WindowManager<
    C: Component<TermWmAction>,
    L: WmComponent = NoopWmComponent,
    O: Overlay<TermWmAction> = NoopOverlay,
> {
    windows: SlotMap<WindowKey, Window>,
    components: SlotMap<ComponentKey, C>,
    layer_manager: LayerManager<L>,
    overlays: SlotMap<OverlayKey, O>,
    // ...
}
```

**Concrete enums live in `term-wm-ui-facade`** â€” the binary instantiates `WindowManager<AppRootComponent, LayerComponent, OverlayComponent>`.

**For `VerticalStack`:** `VerticalStackComponent<C: Component<TermWmAction>>` plus local `PanelChild` enum in `wm_system_panel.rs`. No cross-crate pollution.

### Design Details

**Core changes** (`term-wm-core`):
- `LayerManager<L: WmComponent>` â€” `HashMap<LayerId, L>`, all methods accept/return `&mut L`.
- `WindowManager<C, L, O>` â€” `layer_manager: LayerManager<L>`, `overlays: SlotMap<OverlayKey, O>`.
- Overlay methods (`open_help_overlay`, etc.) take `O` directly.
- `AppBuilder` becomes generic over `<L>` for its `build()` method. Its field types become `Option<L>`.
- `WindowManagerHost` in `runner.rs` gains `<L, O>` generics (or separate bounds). Default overlay methods are removed â€” the trait defines abstract methods, binary/implementors provide the bodies using concrete enum types.
- `set_notification_component` takes `L` instead of `Box<dyn WmComponent>`.
- `NoopWmComponent` and `NoopOverlay` added as default type params (unit structs with empty trait impls for test backward compat). Only needed if existing tests rely on defaults.

**Facade enums** (`term-wm-ui-facade`):

`LayerComponent` â€” wraps all 5 `WmComponent` impls:
```rust

```

1. **Add generics to `LayerManager<L>`** â€” change `HashMap<LayerId, Box<dyn WmComponent>>` to `HashMap<LayerId, L>`. Update `insert`, `get`, `get_mut`, `remove`, `dispatch_foreground`, `dispatch_background`, `render_background`, `render_foreground`.
