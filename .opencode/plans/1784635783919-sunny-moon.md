# Plan: Eliminate Enum Delegation Boilerplate with Declarative Macros

## Problem

The facade enums (`CoreWmComponent`, `LayerComponent`, `OverlayComponent`, `TestComponent`) each contain ~250â€“580 lines of identical `match self` delegation code. Each method simply matches on the enum variant and forwards to the inner type's same-named method. Across 4 enums this is ~1500 lines of boilerplate.

Adding a new variant requires adding a match arm in all 21 `Component` trait methods â€” easy to miss, tedious to review.

## Solution

Create three `macro_rules!` macros that auto-generate the `Component`, `WmComponent`, and `Overlay` trait implementations:

- `impl_component_delegate!` â€” generates `Component<TermWmAction>` impl
- `impl_wm_component_delegate!` â€” generates `WmComponent` impl (for LayerComponent)
- `impl_overlay_delegate!` â€” generates `Overlay<TermWmAction>` impl (for OverlayComponent)

### Macro Location

Define all three macros in `crates/term-wm-core/src/macros.rs` with `#[macro_export]`. This makes them usable from both `term-wm-core` (for `TestComponent`) and `term-wm-ui-facade` (for facade enums).

### Macro Design

All three macros use the same simple matcher â€” only variant names, no type annotations needed since Rust infers the inner type from the enum definition:

```rust
// Pattern: $enum_name:ident { $($variant:ident),* }
```

Key design choice: use fully-qualified paths (`$crate::...`) so the macro works from any crate that depends on `term_wm_core`.

### Steps

1. **Create `crates/term-wm-core/src/macros.rs`** with `impl_component_delegate!`, `impl_wm_component_delegate!`, and `impl_overlay_delegate!`.

2. **Add `pub mod macros;`** to `crates/term-wm-core/src/lib.rs`.

### WmComponent and Overlay Macros

`impl_wm_component_delegate!` delegates all 7 `WmComponent` methods â€” the default trait impls return no-ops, which would silently break layout if the macro left them empty:

```rust
#[macro_export]
macro_rules! impl_wm_component_delegate {
    ($enum_name:ident { $($variant:ident),* $(,)? }) => {
        impl std::fmt::Debug for $enum_name {
            fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
                match self { $(Self::$variant(c) => std::fmt::Debug::fmt(c, f),)* }
            }
        }
        impl $crate::components::WmComponent for $enum_name {
            fn consume_area(&mut self, available: $crate::Rect) -> ($crate::Rect, $crate::Rect) {
                match self { $(Self::$variant(c) => c.consume_area(available),)* }
            }
            fn process_action(&mut self, action: &$crate::components::ComponentAction) {
                match self { $(Self::$variant(c) => c.process_action(action),)* }
            }
            fn query(&self, query: &$crate::components::ComponentQuery) -> $crate::components::ComponentResponse {
                match self { $(Self::$variant(c) => c.query(query),)* }
            }
            fn hit_test(&self, x: u16, y: u16) -> bool {
                match self { $(Self::$variant(c) => c.hit_test(x, y),)* }
            }
            fn begin_frame(&mut self) {
                match self { $(Self::$variant(c) => c.begin_frame(),)* }
            }
            fn visible(&self) -> bool {
                match self { $(Self::$variant(c) => c.visible(),)* }
            }
            fn set_visible(&mut self, visible: bool) {
                match self { $(Self::$variant(c) => c.set_visible(visible),)* }
            }
        }
    };
}
```

`impl_overlay_delegate!` delegates all `Overlay` methods including `as_any_mut`:

```rust
#[macro_export]
macro_rules! impl_overlay_delegate {
    ($enum_name:ident { $($variant:ident),* $(,)? }) => {
        impl $crate::components::Overlay<$crate::actions::TermWmAction> for $enum_name {
            fn visible(&self) -> bool {
                match self { $(Self::$variant(c) => c.visible(),)* }
            }
            fn as_any_mut(&mut self) -> &mut dyn std::any::Any {
                match self { $(Self::$variant(c) => c.as_any_mut(),)* }
            }
            fn shadow_rect(&self, area: $crate::Rect) -> Option<$crate::Rect> {
                match self { $(Self::$variant(c) => c.shadow_rect(area),)* }
            }
            fn handle_confirm_event(&mut self, event: &$crate::events::Event) -> Option<$crate::actions::ConfirmAction> {
                match self { $(Self::$variant(c) => c.handle_confirm_event(event),)* }
            }
            fn mark_dirty(&mut self) { match self { $(Self::$variant(c) => c.mark_dirty(),)* } }
            fn set_menu_items(&mut self, items: Vec<$crate::components::MenuItem<$crate::actions::TermWmAction>>) {
                match self { $(Self::$variant(c) => c.set_menu_items(items),)* }
            }
        }
    };
}
```

3. **Replace `CoreWmComponent`** â€” the entire file becomes:
```rust

Delete the ~260-line `impl Component for CoreWmComponent` body.

4. **Replace `LayerComponent`** â€” becomes:
```rust
impl_component_delegate!(LayerComponent {
    TopPanel, BottomPanel, Fab, NotificationArea, CommandPalette,

    TopPanel, BottomPanel, Fab, NotificationArea, CommandPalette,

```

Delete the ~260-line `impl Component` and ~70-line `impl WmComponent` bodies. The macro delegates all 7 WmComponent methods â€” variants already have their own implementations on the concrete types (WmTopPanelComponent, etc.), so no additional impl block on LayerComponent is needed.

Delete ~220-line `impl Component` and ~80-line `impl Overlay` bodies.

Delete the ~420-line `impl Component for TestComponent` body (the standalone variant impls for ActionRecorder, KeyRecorder, SelComponent, RenderTracker stay â€” only the enum delegation impl is replaced).
