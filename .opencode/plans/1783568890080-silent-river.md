# Fix: Hitbox Registration Pipeline Tests

## Context

The file `tests/integration_tiling.rs` is part of the `term_wm` package (root `Cargo.toml` has `[package] name = "term-wm"`). It tests the public API of the `term-wm` crate, which re-exports `term_wm_core::*`. There is no circular dependency — `term-wm` depends on `term-wm-core`, and the integration test is inside `term-wm`.

## Issue

Pipeline tests use manual `HitTarget::ChromeHeader` injection instead of executing the actual render pipeline. This creates synthetic state.

## Fix

Replace manual hitbox injection with a full `render_app()` call using an in-memory `ratatui::buffer::Buffer` before dispatching mouse events. This forces the production pipeline to naturally populate `wm.hitbox_registry` through its normal traversal of `wm.floating_headers()`.

## Files to Modify

| File | Change |
|------|--------|
| `tests/integration_tiling.rs` | Replace `register_header_hitbox`/`setup_with_hitboxes` with `setup_with_render()`; remove `#[ignore]` from pipeline tests |

## `setup_with_render` Helper

```rust
fn setup_with_render(wm: &mut WindowManager) {
    use ratatui::buffer::Buffer;
    use ratatui::layout::Rect as RatatuiRect;
    use term_wm::render_app;
    use term_wm_console::widget_adapter::RatatuiBackend;
    use term_wm_console::DrawPlanRenderer;
    use term_wm_core::engine::CoreEngine;

    let area = RatatuiRect {
        x: 0,
        y: 0,
        width: AREA.width,
        height: AREA.height,
    };
    let buf = Buffer::empty(area);
    let mut backend = RatatuiBackend::new(buf, area);
    let mut engine = CoreEngine::new();
    let mut renderer = DrawPlanRenderer::new();

    // Executes the production render pipeline — naturally
    // populates wm.hitbox_registry via its normal code path.
    render_app(&mut backend, wm, &mut engine, &mut renderer);
}
```

## Changes to Each Pipeline Test

Replace `register_header_hitbox` and `setup_with_hitboxes` completely. Call `setup_with_render` before EVERY interaction phase.

**Single-action tests** (`drag_to_right_edge_snaps`, `drag_to_left_edge_snaps`, `drag_to_top_maximizes`, `drag_to_corner_quadrant`):
```rust
let (mut wm, keys) = setup();
setup_with_render(&mut wm);
// ... Press/Drag/Release ...
```

**`drag_away_restores_float_geometry`** (two interaction phases):
```rust
let (mut wm, keys) = setup();
setup_with_render(&mut wm);
// Phase 1: Press/Drag/Release (snap right)
// ... layout mutates ...
setup_with_render(&mut wm);  // refresh hitboxes for phase 2
// Phase 2: Press/Drag/Release (drag away)
```

**`double_snap_converges`** (three interaction phases):
```rust
let (mut wm, keys) = setup();
setup_with_render(&mut wm);
// Phase 1: snap right
// ... layout mutates ...
setup_with_render(&mut wm);
// Phase 2: drag away
// ... layout mutates ...
setup_with_render(&mut wm);
// Phase 3: snap left
```
