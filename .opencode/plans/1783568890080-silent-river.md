# Plan: Centralized Tiling Behavior Integration Tests

## Location

New file: `crates/term-wm/tests/integration_tiling.rs`

Uses the same patterns as existing integration tests:
- `term_wm::layout::tiling::{LayoutNode, TilingLayout, InsertPosition, Direction}`
- `term_wm::window::WindowManager` constructed via `term_wm::config::AppBuilder::bare()`
- `term_wm::AppContext::new("test", "0.0.0")`
- `term_wm_layout_engine::LayoutRect` for areas
- `term_wm::layout::Constraint` and `term_wm::layout::Direction`

## Test Specifications

### Module 1: `snap_rule_compliance` (5 tests)

All tests construct a `LayoutNode` tree directly (no WindowManager needed for pure tree ops) or use WindowManager for snap detection.

| Test | What it verifies | Approach |
|------|-----------------|----------|
| `top_edge_triggers_maximize` | Cursor at y=0 sets `SnapPreviewState::Maximize` | Construct WM, set `managed_area`, call `update_snap_preview` at y=0, assert `snap_preview == Some(Maximize)` |
| `left_edge_snap_bisects` | Left edge produces `InsertPosition::Left` with 50% width preview | Call `update_snap_preview` near left edge, assert `drag_snap` position is `Left` and preview width ≈ area.width/2 |
| `right_edge_snap_bisects` | Right edge produces `InsertPosition::Right` with 50% width preview | Same pattern, assert `Right` |
| `corner_snap_produces_quadrant` | Each corner produces correct InsertPosition and quarter-screen preview | Test all 4 corners via `detect_corner_snap` + preview rect verification |
| `snap_preview_priority_corner_over_edge` | Corner detection takes priority over edge when cursor is in corner region | Place cursor in top-left corner (within both corner and edge thresholds), assert corner wins |

### Module 2: `multi_window_tiling` (6 tests)

Pure `LayoutNode` tree tests — no WindowManager needed.

| Test | What it verifies |
|------|-----------------|
| `three_windows_horizontal` | 3 leaves in horizontal split produce 3 non-overlapping regions covering full area |
| `nested_splits_mixed_orientation` | Horizontal split with one child being a vertical split — regions are correct |
| `insert_leaf_next_to_existing` | `insert_leaf(target, new, Right)` splits target and produces correct regions |
| `remove_leaf_collapses_single_child` | Removing one of 2 children collapses the split to a single leaf |
| `remove_last_window_produces_void` | Removing all leaves from a split produces a Void node |
| `normalize_weights_equalizes` | After `normalize_weights`, all leaves get equal area |

### Module 3: `void_node_lifecycle` (4 tests)

| Test | What it verifies |
|------|-----------------|
| `corner_insert_creates_void` | `insert_leaf` with TopLeft position creates a Void placeholder in the tree |
| `replace_void_by_id_fills_placeholder` | `replace_void_by_id(id, leaf)` replaces the Void and returns true |
| `cleanup_removes_all_voids` | `cleanup_after_removal` strips all Void children from splits |
| `void_region_not_rendered` | `layout()` does not produce regions for Void nodes |

### Module 4: `drag_snap_round_trips` (5 tests)

Full WindowManager-level tests using `dispatch_mouse` with synthetic events.

| Test | What it verifies |
|------|-----------------|
| `drag_to_right_edge_snaps_to_half` | Drag header to right edge → Release → window occupies right 50% |
| `drag_away_from_snapped_restores_float` | After snap, drag away → window becomes floating at cursor |
| `drag_to_top_maximizes` | Drag header to y=0 → Release (via Maximize path) → window fills managed area |
| `drag_to_corner_creates_quadrant` | Drag to bottom-right corner → Release → window occupies bottom-right quadrant |
| `multiple_snaps_converge` | Snap right, then snap left, then snap to corner — final state has correct single leaf position |

### Module 5: `property_tests` (4 proptests)

Using `proptest` crate (already available as transitive dev-dependency).

| Test | Invariant |
|------|-----------|
| `insert_never_reduces_leaf_count` | After `insert_leaf(target, new, pos)`, `collect_leaves().len() >= original_len` |
| `layout_regions_never_overlap` | For any tree + area, no two regions in `layout(area)` intersect |
| `layout_regions_cover_full_area` | Union of all regions equals the input area (no gaps, accounting for handle gaps) |
| `weights_always_positive` | After any sequence of insert/remove/normalize, all weights > 0 |
