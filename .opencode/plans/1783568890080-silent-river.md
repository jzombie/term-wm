# Plan: Centralized Tiling Behavior Integration Tests

## Architecture Issue

`apply_snap` (drag.rs:365) calls `layout.root_mut().normalize_weights()`, which recursively walks the **entire tree** resetting ALL weights to 1.0. This destroys user-adjusted split ratios in sibling branches. The `insert_leaf` function (tiling.rs:312-444) is correctly localized — it only modifies the target leaf.

**Fix**: Remove `normalize_weights()` from `apply_snap`. The new Split created by `insert_leaf` already has `weights: vec![1.0, 1.0]`, giving equal share to the two new children. Sibling weights in parent Splits remain untouched.

**Setup helper:**

```rust
fn setup_wm() -> (WindowManager, [WindowKey; 2]) {
    let mut wm = WindowManager::with_config(
        WmConfig::standalone(),
        Arc::new(AppContext::new("test", "0.0.0")),
        None, None, None, None,
    );
    wm.set_panel_visible(false);
    let keys: Vec<_> = (0..2)
        .map(|_| wm.create_window(Box::new(term_wm_core::components::NoopComponent)))
        .collect();

    (wm, [keys[0], keys[1]])
}
```

| Test | Event sequence | Assertion |
|------|---------------|-----------|
| `drag_to_right_edge_snaps` | Press(header) → Drag(right_edge) → Release | `r.x == area.x + area.width / 2`, `r.y == area.y`, `r.width == area.width / 2`, `r.height == area.height` |
| `drag_to_left_edge_snaps` | Press(header) → Drag(left_edge) → Release | `r.x == area.x`, `r.y == area.y`, `r.width == area.width / 2`, `r.height == area.height` |
| `drag_to_top_maximizes` | Press(header) → Drag(y=0) → Release | `r == managed_area` (all four fields) |
| `drag_to_corner_quadrant` | Press(header) → Drag(bottom_right) → Release | `r.x == area.x + area.width / 2`, `r.y == area.y + area.height / 2`, `r.width == area.width / 2`, `r.height == area.height / 2` |
| `drag_away_restores_float_geometry` | Snap → release → drag away → release. Store `pre_snap_rect` and `cursor_offset = (mouse - rect)`. After restore: assert `floating_rect.{x,y,width,height} == pre_snap_rect.{x,y,width,height}` and cursor offset matches stored delta. |
| `double_snap_converges` | Snap right → drag away → snap left → Release | `r.x == area.x`, `r.y == area.y`, `r.width == area.width / 2`, `r.height == area.height` |

**Key detail**: Deadzone guard requires dx+dy > 2. Snap detection is pure geometry (no edge resistance). Drag from center to edge satisfies both.
