# Plan: Fix NSPasteboard stderr Noise in Session Client

## The Bug
When running term-wm inside term-session-client (nested multiplexer), copying a string produces junk characters at the bottom of the screen. The session client's `clipboard.set()` at `term-session-client/src/lib.rs:176` calls `arboard::Clipboard::set_text()`, which invokes macOS `NSPasteboard`. AppKit writes debug messages like `[NSPasteboard _setData:]` to FD 2 (stderr), which is the user's terminal, appearing as visible junk.

## Root Cause
**File:** `crates/term-session-client/src/lib.rs:176` — `clipboard.set(&text)` triggers `arboard` → `NSPasteboard` → stderr noise. FD 2 is the terminal (no redirection), so OS-level stderr output leaks to the screen.

## The Fix

### Add `redirect_fd_to_tracing` helper in `crates/term-session-client/src/lib.rs`
Pipe a given FD through a background thread into `tracing`. Uses `tracing::error!` for stderr and `tracing::info!` for stdout. Uses `read_until` + `String::from_utf8_lossy` so non-UTF-8 bytes from macOS frameworks don't kill the pipe:

```rust
pub fn redirect_fd_to_tracing(target_fd: libc::c_int, is_stderr: bool) -> std::io::Result<()> {
    let mut fds: [libc::c_int; 2] = [0; 2];
    unsafe {
        if libc::pipe(fds.as_mut_ptr()) == -1 {
            return Err(std::io::Error::last_os_error());
        }
        let read_fd = fds[0];
        let write_fd = fds[1];
        if libc::dup2(write_fd, target_fd) == -1 {
            libc::close(read_fd);
            libc::close(write_fd);
            return Err(std::io::Error::last_os_error());
        }
        libc::close(write_fd);
    }
    let name = if is_stderr { "stderr-tracing" } else { "stdout-tracing" };
    std::thread::Builder::new()
        .name(name.into())
        .spawn(move || {
            use std::io::BufRead;
            use std::os::unix::io::FromRawFd;
            let file = unsafe { std::fs::File::from_raw_fd(read_fd) };
            let mut reader = std::io::BufReader::new(file);
            let mut buf = Vec::new();
            while reader.read_until(b'\n', &mut buf).unwrap_or(0) > 0 {
                let text = String::from_utf8_lossy(&buf);
                let trimmed = text.trim();
                if !trimmed.is_empty() {
                    if is_stderr {
                        tracing::error!(target: "c_stderr", "{}", trimmed);
                    } else {
                        tracing::info!(target: "c_stdout", "{}", trimmed);
                    }
                }
                buf.clear();
            }
        })?;
    Ok(())
}
```

### Call it in `run_session()`
At the top of `run_session()` in `lib.rs`, before enabling raw mode, redirect stderr:
```rust
let _ = redirect_fd_to_tracing(libc::STDERR_FILENO, true);
```

Note: No `tracing_subscriber::fmt::init()` needed. When running inside term-wm, the existing subscriber routes events to the Debug Log view. When standalone, events are dropped silently.
