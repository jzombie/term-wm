# Fix: Enable Bracketed Paste Mode

## Root Cause

`term-wm` never sends `\x1b[?2004h` to the host terminal. Without this, pasting floods the child process with individual key events instead of one bundled paste.

## Changes

### 1. `crates/term-wm-console/Cargo.toml`
Add `features = ["bracketed-paste"]` to the crossterm dependency.

### 2. `crates/term-wm-console/src/console_render_target.rs`
- Import `EnableBracketedPaste`, `DisableBracketedPaste` from crossterm
- `enter()`: add `EnableBracketedPaste` to the existing `execute!` call
- `exit()`: add `DisableBracketedPaste` to the existing `execute!` call

### 3. `crates/term-session-client/src/lib.rs`
- Import `EnableBracketedPaste`, `DisableBracketedPaste`
- Startup (line ~199): add `EnableBracketedPaste` to the `queue!` chain
- `TerminalGuard::drop()` (line ~92): add `DisableBracketedPaste` to the `queue!` chain

## Verification

```bash
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test
```

Paste a long test message — arrives as one message.
