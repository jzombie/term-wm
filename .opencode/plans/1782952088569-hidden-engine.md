# Plan: Rename Window ID References to Key

## Problem
The type `WindowKey` already exists (was renamed from `WindowId<Id>`), but field names, function names, parameter names, and local variables still use `id`/`Id` terminology inconsistently.

## Scope

- **Delete** `crates/term-wm-core/src/window/window_manager/sys.rs` — dead code, not declared as a module, references nonexistent types.
- **Rename** struct fields, function names, parameters, and local variables in ~10 files.
- **Rename** generic type parameters `Id` → `K` in layout code.

## Files & Changes

### 0. Delete `sys.rs`
`crates/term-wm-core/src/window/window_manager/sys.rs` — entire file is dead code.

### 1. Struct Field Renames (`id` → `key`)

| Struct | File | Field |
|--------|------|-------|
| `WindowDrawContext` | `window/window_manager/mod.rs:99` | `pub id: WindowKey` → `pub key: WindowKey` |
| `ResizeHandle<R>` → `ResizeHandle<K>` | `layout/floating.rs:13` | `pub id: R` → `pub key: K` |
| `ResizeDrag<R>` → `ResizeDrag<K>` | `layout/floating.rs:20` | `pub id: R` → `pub key: K` |
| `HeaderDrag<R>` → `HeaderDrag<K>` | `layout/floating.rs:33` | `pub id: R` → `pub key: K` |
| `DragHandle<R>` → `DragHandle<K>` | `layout/floating.rs:42` | `pub id: R` → `pub key: K` |
| `FloatingPane<Id>` → `FloatingPane<K>` | `layout/mod.rs:78` | `pub id: Id` → `pub key: K` |

After renaming each field, update all `.id` accesses to `.key` in callers.

### 2. Generic Type Parameter Renames

| Old | New | Files |
|-----|-----|-------|
| `R: Copy + Eq + Ord` | `K: Copy + Eq + Ord` | `layout/floating.rs` (structs + fns) |
| `Id: Copy + Eq + Ord` | `K: Copy + Eq + Ord` | `layout/mod.rs` (FloatingPane) |
| `<Id>` (in LayoutPlan, LayoutNode) | Keep as-is or rename | Keep abstract params generic |

### 3. Function Name Renames (`_id` → `_key`)

Update all call sites in layout.rs, focus.rs, drag.rs, chrome.rs, overlays.rs, runner.rs.

### 4. Parameter & Local Variable Renames

- `layout/floating.rs`: function params `id: R` → `key: K`
- `layout.rs`/`focus.rs`/`drag.rs`: `id: WindowKey` → `key: WindowKey`
- Local vars: `hit_id`/`prev_id`/`dragging_id`/`target_id` → `*_key`
- `r_id`/`float_id`/`floating_ids` → `*_key`
- `target_id` (in `render_resize_outline`) → `target_key`
- `draw.id` → `draw.key`
- `window.id` → `window.key` in `runner.rs`

## Implementation Order

1. **Delete** `sys.rs`
2. **`layout/floating.rs`** — rename generic `R` → `K`, fields `id` → `key`, function params `id` → `key`, local vars
3. **`layout/mod.rs`** — rename `FloatingPane<Id>` → `FloatingPane<K>`, field `id` → `key`
4. **`layout/tiling.rs`** — `floating.id` → `floating.key`
5. **`window/window_manager/layout.rs`** — func renames + internal call sites + local vars
6. **`window/window_manager/focus.rs`** — func rename + call sites
7. **`window/window_manager/drag.rs`** — func rename + `.id` access + local vars
8. **`window/window_manager/chrome.rs`** — call site updates
9. **`window/window_manager/overlays.rs`** — call site update
10. **`window/window_manager/mod.rs`** — `WindowDrawContext.id` → `.key` + test code
11. **`runner.rs`** — `window.id` → `window.key` + `full_region_for_id` call
