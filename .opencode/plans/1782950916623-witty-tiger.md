### Structural root cause

The `arch-tiers/baseline` branch uses a different window identification system:
- **Old (main):** `WindowId<usize>` with `WindowId::App(id)` and `WindowId::System(id)` variants
- **New (baseline):** `WindowKey` (opaque slotmap key), no system window concept

This caused:
1. All tests to be mechanically adapted (type changes only — fine)
2. **5 system-window-specific tests** to be deleted instead of adapted (these tested drag, event rebase, timer behavior for system windows)
3. Test API changes (`wm_focus_app()` → `focused_window()`, `with_config(0, ...)` → `with_config(...)`)
4. `// TODO: Rewrite tests for WindowKey-based API` in runner.rs confirms the migration is acknowledged as incomplete
