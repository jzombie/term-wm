# Plan: PowerProfile + Bottom-Panel Indicator

## Design Principle

No argument-threading. Each component gets `PowerProfile` from its natural owner:
- `WmConfig` is the **single source of truth**
- `ConsoleEventSource::new(profile)` — direct field, owns it for `poll_interval()`
- `WindowManager` owns config + bottom panel → syncs profile via `begin_frame()` (one trait method, one line)
- `WmBottomPanelComponent` stores what it needs internally

Change is logged to `tracing::info!` so it appears in the in-app debug log window.

## Throttling Mechanism (two layers)

### Layer 1: Poll-interval throttling (`PowerProfile::poll_interval`)

Controls how often the event loop checks for input. Higher intervals = fewer wakeups = less CPU.

| Profile | active_ms | idle_ms | idle_threshold_ms |
|---------|-----------|---------|-------------------|
| HighPerformance | 16 | 16 | ∞ (never idle) |
| Balanced | 16 | 200 | 500 |
| PowerSaver | 33 | 500 | 1000 |

### Layer 2: Skip idle renders when nothing changed (`WindowManager.frame_dirty`)

The idle path in `runner.rs::run_app()` (lines 487–516) unconditionally runs `output.draw()` every loop iteration. When the profile is not `HighPerformance`, we skip this draw if no state has changed since the last render.

**Mechanism**:
- `WindowManager` gets a `frame_dirty: bool` field (initialized `true` for first render)
- `frame_dirty` is set to `true` by:
  - `begin_frame()` — called before each render cycle (runner.rs line 500)
  - Any event-processing path that changes state
- After `output.draw()` completes, `frame_dirty` is reset to `false`
- In the idle handler (lines 487–516), if `profile != HighPerformance` and `!frame_dirty`, skip the `output.draw()` call entirely
- `HighPerformance` always renders every idle tick (for smooth animations)

This avoids unnecessary terminal cell rendering, color resolution, scrollback sync, and buffer copies when the display is static.

## Files to Create

### `crates/term-wm-core/src/io/power_profile.rs` (NEW)

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PowerProfile {
    HighPerformance,
    Balanced,
    PowerSaver,
    Custom { active_ms: u64, idle_ms: u64, idle_threshold_ms: u64 },
}
```

Methods:
- `Default` → `Balanced`
- `fn poll_interval(&self, last_event_at: Option<Instant>) -> Duration` — adaptive logic replacing hardcoded consts
- `fn name(&self) -> &'static str`
- `fn skip_idle_render(&self) -> bool` — `true` for `PowerSaver`, `false` for `HighPerformance`/`Balanced`
- `fn indicator_color(&self) -> Color` — returns `theme::profile_high/mid/low_bg()`
- `fn log_change(&self)` — `tracing::info!(target: "power", "profile: {name} ({active_ms}/{idle_ms})")`
