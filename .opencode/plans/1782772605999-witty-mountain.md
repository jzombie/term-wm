# Plan: PowerProfile + Bottom-Panel Indicator

## Throttling Mechanism (two layers)

### Layer 1: Poll-interval throttling (`PowerProfile::poll_interval`)

Controls how often the event loop checks for input. Higher intervals = fewer wakeups.

| Profile | active_ms | idle_ms | idle_threshold_ms |
|---------|-----------|---------|-------------------|
| HighPerformance | 16 | 16 | ∞ |
| Balanced | 16 | 200 | 500 |
| PowerSaver | 33 | 500 | 1000 |

### Layer 2: Skip idle renders when nothing changed (`frame_dirty`)

The idle `handler(None)` in `runner.rs:487-516` unconditionally runs `output.draw()`. With `frame_dirty`, we skip it.

`frame_dirty` is set `true` **only** by:
- Event processing path (keyboard/mouse input handled)
- Window resize
- Explicit invalidation (blinking cursor, animation)

`frame_dirty` is NOT set by `begin_frame()` — that was the cycle-breaking bug. `begin_frame()` still runs on every tick (it handles overlay polling), but the draw is gated on `frame_dirty`.

## Runtime profile changes (existing pattern)

`PowerProfile` follows the same pattern as `mouse_capture_enabled`, `clipboard_enabled`, etc.:

1. Stored as a **direct field** on `WindowManager` (not in `WmConfig`)
2. Initialized from `WmConfig.power_profile` at construction
3. `WindowManager::set_power_profile()` — setter that logs change and sets **both** sync flags
4. `WindowManager::take_power_profile_change()` — consumed by runner for event-source update
5. `WindowManager::needs_profile_sync` — checked/cleared by `begin_frame()` for bottom-panel update
6. `EventSource` trait gets `set_power_profile(&mut self, _profile: PowerProfile)` with no-op default
7. `ConsoleEventSource` overrides it to update its internal `poll_interval()` behavior
8. Runner consumes the change (like mouse capture) and calls `driver.set_power_profile()` — `driver` is `D: EventSource` (input), same as `driver.set_mouse_capture()`

This avoids the mutable-disconnect flaw — `ConsoleEventSource` stays in sync.

**Anti-trap: Single flag, dual consumers.** The two consumers (begin_frame → bottom panel, runner → event source) share a single `needs_profile_sync: bool` flag. Runner doesn't need a separate `take_*` — it stores a local `last_sent_profile` in its closure variable and compares against `app.windows().power_profile()`. No shared mutable state to race on.

## Files to Create

### `crates/term-wm-core/src/io/power_profile.rs` (NEW)

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PowerProfile {
    HighPerformance,
    Balanced,
    PowerSaver,
    Custom { active_ms: u64, idle_ms: u64, idle_threshold_ms: u64 },
}

impl PowerProfile {
    pub fn poll_interval(&self, last_event_at: Option<Instant>) -> Duration;
    pub fn name(&self) -> &'static str;
    pub fn skip_idle_render(&self) -> bool;
    pub fn indicator_color(&self) -> Color;
    fn log_change(&self);
}
```

`skip_idle_render()` returns `true` for all non-`HighPerformance` profiles.

`indicator_color()` maps to `theme::profile_high/mid/low_bg()`.

### `crates/term-wm-core/src/runner.rs`

Before `event_loop.run()`, capture initial profile:
```rust
let mut last_sent_profile = profile;
```

In `run_app()` handler closure:
- After processing events (terminal input, mouse, resize), call `app.windows().invalidate_frame()`
- In idle `handler(None)` path:
  - Still run `begin_frame()`, `update_selection_snapshot()`, `take_expired_super_event()`, etc.
  - Before `output.draw()`: `if !app.windows().frame_dirty() && profile.skip_idle_render() { /* skip draw */ }`
  - After draw: `app.windows().mark_frame_clean()`
- In `flush_state_changes()` closure, compare local against `WindowManager`:
  ```rust
  let current = app.windows().power_profile();
  if current != last_sent_profile {
      let _ = driver.set_power_profile(current);
      last_sent_profile = current;
  }
  ```
  This avoids the dual-flag trap: no `take_*` on WindowManager for the event-source route. The bottom-panel route uses `needs_profile_sync` (consumed by `begin_frame()`). Two consumers, one flag.
