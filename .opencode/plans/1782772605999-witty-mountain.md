# Plan: Add PowerProfile to Fix High CPU on Idle

## Root Cause

The event loop (`event_loop.rs:55-80`) unconditionally calls `handler(None)` → full `output.draw()` on every loop iteration. Combined with the hardcoded active interval (16ms) and idle interval (200ms) in `console.rs:121-148`, this causes continuous rendering even when nothing changed.

## Solution: `PowerProfile` enum

Replace the three hardcoded constants in `ConsoleEventSource::poll_interval()` with a `PowerProfile` enum, plumbed through `WmConfig` and logged on change.

### Files to create/modify

| File | Action | Why |
|------|--------|-----|
| `crates/term-wm-core/src/io/power_profile.rs` | **NEW** | Define `PowerProfile` enum |
| `crates/term-wm-core/src/io/mod.rs` | MODIFY | Export `power_profile` module |
| `crates/term-wm-core/src/io/console.rs` | MODIFY | Use `PowerProfile` in `ConsoleEventSource`, log changes |
| `crates/term-wm-core/src/wm_config.rs` | MODIFY | Add `power_profile` field to `WmConfig` |

### `PowerProfile` enum

Place in `crates/term-wm-core/src/io/power_profile.rs`:

```rust
pub enum PowerProfile {
    HighPerformance,  // 60fps always
    Balanced,         // 60fps active, 5fps idle
    PowerSaver,       // 30fps active, 2fps idle
    Custom { active_ms: u64, idle_ms: u64, idle_threshold_ms: u64 },
}
```

Implement:
- `Default` → `Balanced`
- `fn poll_interval(&self, last_event_at: Option<Instant>) -> Duration` — replicate the current adaptive logic from `console.rs` but parameterized by profile
- `fn name(&self) -> &'static str` — for logging
- `fn log_change(&self)` — calls `tracing::info!` with profile name and parameters (active_ms, idle_ms, threshold_ms)

### `ConsoleEventSource` changes

- Add `power_profile: PowerProfile` field
- Constructor: `pub fn new() -> Self` → `pub fn new(profile: PowerProfile) -> Self` (or keep default and add `setter`)
- `poll_interval()` delegates to `self.power_profile.poll_interval(self.last_event_at)`
- Log profile on construction and on change

### `WmConfig` changes

```rust
pub struct WmConfig {
    // ... existing fields ...
    pub power_profile: PowerProfile,
}
```

Set default in `standalone()` and `embedded()` constructors to `PowerProfile::Balanced`.

### `main.rs` changes

```rust
let config = WmConfig::standalone();
let mut input = ConsoleEventSource::new(config.power_profile);
```

### Profile presets (poll_interval logic)

| Profile | active_ms | idle_ms | idle_threshold_ms | Active fps | Idle fps |
|---------|-----------|---------|-------------------|------------|----------|
| HighPerformance | 16 | 16 | (never idle) | 60 | 60 |
| Balanced | 16 | 200 | 500 | 60 | 5 |
| PowerSaver | 33 | 500 | 1000 | 30 | 2 |
| Custom | user | user | user | varies | varies |

The `poll_interval` method:
1. If `HighPerformance`, always return `active_ms`
2. Otherwise, check `last_event_at` against `idle_threshold_ms` to determine active/idle, return corresponding interval
3. Log when transitioning between idle and active states
