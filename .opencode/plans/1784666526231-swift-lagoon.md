# Plan: Zero-Copy Rendering Pipeline Refactor

## Goal
Eliminate the `Vec<Vec<TerminalCell>>` deep-copy in the render pipeline. Currently `snapshot_from_term` copies every visible cell from the alacritty `Grid` into a `Vec<Vec<TerminalCell>>`, then `render_screen` iterates over that copy to paint the Ratatui buffer. This is O(n) heap allocations per frame (n = cols × rows).

## Design: Inversion of Control (No Crate Boundary Violation)

`terminal.rs` (in `term-wm-ui-components`) must NEVER import `alacritty_terminal` types. The `Pane` trait (in `term-wm-pty-engine`) provides the abstraction boundary. We add a method that accepts a callback closure:

```rust
fn process_visible_cells(
    &mut self,
    columns: u16,
    rows: u16,
    callback: &mut dyn FnMut(u16, u16, &TerminalCell),
    metadata_callback: &mut dyn FnMut(&SnapshotMetadata),
);
```

The backend (`Pty`, `RemotePane`, `TestPane`) locks its internal `Term`, iterates the grid, constructs a transient `TerminalCell` on the stack per cell, and invokes `callback(row, col, &cell)`. After all cells, `metadata_callback` is called with cursor/colors/mode. The lock is dropped once the callbacks return.

This preserves the crate boundary, is zero-copy (the `TerminalCell` lives on the stack), and keeps the trait generic.

## Files to Modify

### 1. `crates/term-wm-pty-engine/src/pane.rs`
- **Add** `SnapshotMetadata` struct (columns, rows, cursor, default_fg, default_bg, alternate_screen, mouse, display_offset)
- **Remove** `cells: Vec<Vec<TerminalCell>>` from `TerminalSnapshot`
- **Add** `process_visible_cells` to `Pane` trait with default impl using old `snapshot()` for backwards compat during migration
- **Add** `is_dirty(&mut self) -> bool` defaulting to `false`

### 2. `crates/term-wm-pty-engine/src/pty.rs`
- **Add** `Pty::process_visible_cells()` — lock `self.term`, iterate grid, call callback per cell, then metadata callback, drop lock
- **Add** `Pty::is_dirty()` — returns `self.dirty.load(Ordering::Acquire)`

### 3. `crates/term-wm-ui-components/src/terminal.rs`
- **Rewrite** `render_screen()` to call `pane.process_visible_cells()` instead of `pane.snapshot()`
- Inside the cell callback: map transient `TerminalCell` to Ratatui `buffer.cell_mut()` directly
- **Skip** `wide_continuation` cells (no buffer mutation for spacers)
- Inside metadata callback: capture cursor, default_fg, default_bg, mouse mode for link overlay / selection rendering
- **Modify** `TestPane::snapshot()` to keep working (or also adopt the callback)

### 4. `crates/term-session-client/src/remote_pane.rs`
- **Add** `RemotePane::process_visible_cells()` — lock `self.term`, iterate grid, call callback per cell

### 5. `crates/term-session-client/src/lib.rs`

- **Fix** wide character handling: skip spacers in the serialization

## Specific Changes (Ordered)

### Phase 1: Trait additions in `pane.rs`

```rust
#[derive(Clone, Debug)]
pub struct SnapshotMetadata {
    pub columns: u16,
    pub rows: u16,
    pub cursor: Option<CursorInfo>,
    pub default_fg: CellColor,
    pub default_bg: CellColor,
    pub alternate_screen: bool,
    pub mouse: MouseProtocol,
    pub display_offset: usize,
}

// Add to Pane trait:
fn process_visible_cells(
    &mut self,
    columns: u16,
    rows: u16,
    callback: &mut dyn FnMut(u16, u16, &TerminalCell),
    metadata_callback: &mut dyn FnMut(&SnapshotMetadata),
);
fn is_dirty(&mut self) -> bool { false }
```

- Remove `cells` from `TerminalSnapshot`
- Update all uses of `TerminalSnapshot` across codebase

### Phase 2: `Pty` impl in `pty.rs`

Implement `process_visible_cells`:
1. Call `self.screen()` (sync dirty/DSR/foreground)
2. Lock `self.term`
3. Compute start_line from display_offset
4. For each row in 0..rows:
   - Get grid line at start_line + i
   - For each col in 0..columns:
     - Resolve cell into transient `TerminalCell` on stack
     - Invoke `callback(i, col, &term_cell)`
5. Build `SnapshotMetadata`, invoke `metadata_callback(&meta)`
6. Lock is dropped after callbacks return

Implement `is_dirty`:
```rust
fn is_dirty(&mut self) -> bool { self.dirty.load(Ordering::Acquire) }
```

### Phase 3: Rewrite `render_screen` in `terminal.rs`

Current flow:
```
pane.snapshot() → clone entire grid → Vec<Vec<TerminalCell>> → nested for loops → buffer.cell_mut()
```

New flow:
```
// sync scroll state (unchanged)
// drag maintenance (unchanged)
// link overlay rebuild (unchanged)
// then:
let mut snapshot_meta: Option<SnapshotMetadata> = None;
pane.process_visible_cells(area.width, area.height,
    &mut |row, col, cell| {
        if cell.wide_continuation { return; }  // SKIP spacer cells
        map cell.fg/bg/style → buffer.cell_mut(x, y)
    },
    &mut |meta| { snapshot_meta = Some(meta.clone()); },
);
let meta = snapshot_meta.unwrap();
// cursor overlay (using meta.cursor, unchanged logic)
```

The `snap` variable disappears entirely — cells are written directly to the buffer.

### Phase 4: `RemotePane` impl

Same pattern as `Pty::process_visible_cells`:
1. Lock `self.term`
2. Iterate grid, construct transient `TerminalCell`, call callback
3. Build `SnapshotMetadata`, call metadata callback

### Phase 5: Fix session client coordinate positioning

In `crates/term-session-client/src/lib.rs`, replace:
```rust
// OLD:
for row_cells in &snap.cells {
    for cell in row_cells { ... lines.push(b'\r'); lines.push(b'\n'); }
}
// NEW (after getting snap):
for (row_idx, row_cells) in snap.cells.iter().enumerate() {
    let _ = out.queue(crossterm::cursor::MoveTo(0, row_idx as u16));
    for cell in row_cells {
        if cell.wide_continuation { continue; }
        let mut buf = [0u8; 4];
        let s = cell.character.encode_utf8(&mut buf);
        out.write_all(s.as_bytes())?;
    }
}
```

## Risk Assessment
- `render_screen` is the riskiest change — scroll sync, drag selection, link overlay, cursor overlay must all still work
- `TestPane::snapshot()` is used in tests — can keep old snapshot impl alongside new `process_visible_cells` for migration
- Crate boundary preserved — `terminal.rs` never imports `alacritty_terminal`
