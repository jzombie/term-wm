# Plan: Zero-Copy Rendering Pipeline Refactor

## Goal
Eliminate `Vec<Vec<TerminalCell>>` deep-copy. Add `process_visible_cells` to `Pane` trait — backend iterates grid, constructs transient `TerminalCell` on stack, calls UI callback. Zero heap allocations per cell, crate boundary preserved.

## Files to Modify

### 1. `crates/term-wm-pty-engine/src/pane.rs`
- **Add** `SnapshotMetadata` struct (columns, rows, cursor, default_fg, default_bg, alternate_screen, mouse, display_offset)
- **Remove** `cells: Vec<Vec<TerminalCell>>` from `TerminalSnapshot`
- **Add** to `Pane` trait:
  ```rust
  fn process_visible_cells(
      &mut self,
      columns: u16,
      rows: u16,
      cell_cb: &mut dyn FnMut(u16, u16, &TerminalCell),
      meta_cb: &mut dyn FnMut(&SnapshotMetadata),
  );
  fn is_dirty(&mut self) -> bool { false }
  ```

### 2. `crates/term-wm-pty-engine/src/pty.rs`
- **Implement** `Pty::process_visible_cells`:
  1. `self.screen()` (sync dirty/DSR)
  2. Lock `self.term`
  3. Iterate grid, construct `TerminalCell` on stack, call `cell_cb(row, col, &cell)`
  4. Build `SnapshotMetadata`, call `meta_cb`
  5. Lock dropped after callbacks return
- **Implement** `Pty::is_dirty()` — returns `self.dirty.load(Ordering::Acquire)`

### 3. `crates/term-wm-ui-components/src/terminal.rs`
- **Rewrite** `render_screen()` to call `pane.process_visible_cells()`:
  - Inside cell callback: skip `wide_continuation`, map `TerminalCell` → Ratatui `buffer.cell_mut()`
  - **Accumulate row strings + byte offsets for linkifier**: track `String` + `Vec<usize>` offsets per row, flush to `row_data` when row index changes
  - Metadata callback: capture cursor, colors, mouse mode for cursor overlay / selection
- Keep scroll sync, drag selection, cursor overlay logic structurally unchanged

### 4. `crates/term-session-client/src/remote_pane.rs`
- **Implement** `RemotePane::process_visible_cells` — same pattern as `Pty`
- **Add** `dirty: bool` field to `RemotePane` struct (not `Cell<bool>` — `is_dirty` takes `&mut self`)
- **Set** `self.dirty = true` inside `drain_pushes` when `processor.advance` is called
- **Implement** `RemotePane::is_dirty()` — returns `std::mem::replace(&mut self.dirty, false)`
- **Add** `osc52: Osc52Extractor` field to `RemotePane` struct
- **Instantiate** `Osc52Extractor::new()` in `RemotePane::new()`
- **Feed** raw `push_rx` chunks into `self.osc52.push()` inside `drain_pushes` **before** `processor.advance` (stateful, handles split Base64 across chunks)

### 5. `crates/term-session-client/src/lib.rs`
- **Rewrite** initialization content-detection loop: use `pane.process_visible_cells()` with a callback that sets `has_content = true` if any `cell.character != ' '`
- **Rewrite** initial full-screen render: use `pane.process_visible_cells()`, for each row emit `crossterm::cursor::MoveTo(0, row)` then write characters, skip `wide_continuation`
- **Remove** all iteration over `snap.cells` (field is deleted, code won't compile otherwise)
- **Remove** dead OSC52 extraction from render loop (moved to `drain_pushes`)

### 6. All test files that construct `TerminalSnapshot` with `cells`
- Update to use `SnapshotMetadata` + `process_visible_cells` pattern, or construct `TerminalSnapshot` without the `cells` field
