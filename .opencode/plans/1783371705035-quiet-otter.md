# Simplify term-wm API for Third-Party Apps

## Problem

Third-party apps must implement two coupled traits (`WindowManagerHost` + `WindowProvider`) totaling ~90 lines of boilerplate. The `dual_image.rs` example is 126 lines with 70% scaffolding. Components needing their `WindowKey` use `Arc<Mutex<Option<WindowKey>>>` hacks. There is no lightweight integration path (term-bench bypasses term-wm entirely).

## Approach: Incremental Additions, No Breaking Changes

Add new types alongside existing ones. Existing `run_window_app` API continues to work unchanged.

---

## Step 1: Add `on_mount` hook to `Component` trait

**File:** `crates/term-wm-core/src/components/mod.rs`

Add a default method after `init()` (line 31):

```rust
/// Called immediately after the component is registered with the WindowManager.
/// Provides the assigned `WindowKey` and app context, resolving lifecycle
/// circularity without `Arc<Mutex<Option<WindowKey>>>` hacks.
fn on_mount(&mut self, _key: WindowKey, _app: &crate::app_context::AppContext) {}
```

**Why:** Solves the lifecycle circularity problem. Components can capture their `WindowKey` directly in `on_mount` instead of using deferred initialization hacks. Default is no-op so all existing components compile unchanged.

**Impact:** Zero breaking change. All existing `impl Component` blocks compile as-is.

## Step 2: Add `spawn` method to `WindowManager`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

Add method after `create_window` (line 218):

```rust
/// Register a component and invoke its `on_mount` hook with the assigned key.
/// Prefer this over `create_window` for components that need their WindowKey.
pub fn spawn<C>(&mut self, mut component: C) -> WindowKey
where
    C: Component<TermWmAction> + 'static,
{
    let app_ctx = self.app_ctx().as_ref().clone();
    let key = self.create_window(Box::new(component));
    // Retrieve the boxed component and call on_mount
    if let Some(comp) = self.component_for_key_mut(key) {
        comp.on_mount(key, &app_ctx);
    }
    key
}
```

**Why:** Single-call registration that automatically resolves the key. Eliminates the need for `Arc<Mutex<Option<WindowKey>>>` in most cases.

**Note:** `on_mount` is called after boxing (not during `insert_with_key`), but before any frame rendering or event polling, which is sufficient to resolve the circularity.

## Step 3: Create `StandaloneApp` builder

**File:** `crates/term-wm-core/src/standalone.rs` (NEW)

```rust
use std::io;
use std::sync::Arc;

use crate::app_context::AppContext;
use crate::components::{Component, component_downcast_mut};
use crate::config::WmBuilder;
use crate::io::{ConsoleRenderTarget, ConsoleEventSource, EventSource, RenderTarget};
use crate::runner::{WindowManagerHost, WindowProvider, run_window_app};
use crate::actions::TermWmAction;
use crate::window::{WindowKey, WindowManager};
use crate::wm_config::WmConfig;

/// A self-contained window manager app that eliminates dual-trait boilerplate.
///
/// # Example
/// ```ignore
/// let mut app = StandaloneApp::new(AppContext::new("myapp", "1.0"));
/// let key = app.register(MyComponent::new());
/// app.run()?;
/// ```
pub struct StandaloneApp {
    wm: WindowManager,
    windows: Vec<WindowKey>,
    should_quit: bool,
    empty_message: String,
}

impl StandaloneApp {
    /// Create a new standalone app with all system chrome (panels, menu).
    pub fn new(app_ctx: AppContext) -> Self {
        Self::from_wm(WindowManager::new_standalone(app_ctx))
    }

    /// Create an embedded app without the command menu, suitable for
    /// embedding in an existing Ratatui application.
    pub fn embedded(app_ctx: AppContext) -> Self {
        Self::from_wm(WindowManager::new_embedded(app_ctx))
    }

    /// Create from an already-constructed WindowManager.
    pub fn from_wm(wm: WindowManager) -> Self {
        Self {
            wm,
            windows: Vec::new(),
            should_quit: false,
            empty_message: "No windows".to_string(),
        }
    }

    /// Set the message shown when no windows are registered.
    pub fn empty_message(mut self, msg: impl Into<String>) -> Self {
        self.empty_message = msg.into();
        self
    }

    /// Register a component as a window. Returns the WindowKey for later access.
    /// Calls `on_mount` on the component after registration.
    pub fn register<C>(&mut self, component: C) -> WindowKey
    where
        C: Component<TermWmAction> + 'static,
    {
        let key = self.wm.spawn(component);
        self.windows.push(key);
        key
    }

    /// Register a pre-boxed component (for dynamic dispatch scenarios).
    pub fn register_boxed(&mut self, component: Box<dyn Component<TermWmAction>>) -> WindowKey {
        let key = self.wm.create_window(component);
        self.windows.push(key);
        key
    }

    /// Borrow the WindowManager for configuration or direct access.
    pub fn wm(&mut self) -> &mut WindowManager {
        &mut self.wm
    }

    /// Get a mutable reference to a registered component by key.
    pub fn component_mut<T: 'static>(&mut self, key: WindowKey) -> Option<&mut T> {
        self.wm.component_for_key_mut(key)
            .and_then(|c| component_downcast_mut::<T>(c))
    }

    /// Request the app to quit after the current event cycle.
    pub fn request_quit(&mut self) {
        self.should_quit = true;
    }

    /// Run the app with default console I/O (enters/exits terminal automatically).
    pub fn run(self) -> io::Result<()> {
        let mut output = ConsoleRenderTarget::new()?;
        output.enter()?;
        let mut input = ConsoleEventSource::new();
        let result = self.run_with(&mut output, &mut input);
        output.exit()?;
        result
    }

    /// Run with custom render target and event source.
    pub fn run_with<O: RenderTarget, D: EventSource>(
        mut self,
        output: &mut O,
        driver: &mut D,
    ) -> io::Result<()> {
        run_window_app(output, driver, &mut self)
    }
}

impl WindowManagerHost for StandaloneApp {
    fn windows(&mut self) -> &mut WindowManager {
        &mut self.wm
    }

    fn quit_requested(&self) -> bool {
        self.should_quit
    }
}

impl WindowProvider for StandaloneApp {
    fn enumerate_windows(&mut self) -> Vec<WindowKey> {
        self.windows.clone()
    }

    fn empty_window_message(&self) -> &str {
        &self.empty_message
    }

    fn window_component(&mut self, key: WindowKey) -> Option<&mut dyn Component<TermWmAction>> {
        self.wm.component_for_key_mut(key)
    }
}
```

**Why:** Replaces ~90 lines of boilerplate with ~30 lines. The `StandaloneApp` internally implements both traits by delegating to `WindowManager`'s built-in methods. Users only call `register()` and `run()`.

## Step 5: Create `WidgetAdapter` and `StatefulWidgetAdapter`

**File:** `crates/term-wm-ui-components/src/widget_adapter.rs` (NEW)

```rust
use ratatui::layout::Rect;
use crate::components::{Component, ComponentContext};
use crate::hitbox_registry::HitboxRegistry;
use crate::ui::UiFrame;
use crate::actions::TermWmAction;

/// Wraps a closure-based Ratatui renderer as a term-wm Component.
/// For stateless widgets (Ratatui's `Widget` trait).
///
/// # Example
/// ```ignore
/// let comp = WidgetAdapter::new(|frame, area, _ctx| {
///     frame.render_widget(Paragraph::new("Hello"), area);
/// });
/// app.register(comp);
/// ```
pub struct WidgetAdapter<F> {
    render_fn: F,
}

impl<F> WidgetAdapter<F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &ComponentContext),
{
    pub fn new(render_fn: F) -> Self {
        Self { render_fn }
    }
}

impl<F> Component<TermWmAction> for WidgetAdapter<F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &ComponentContext),
{
    fn render(
        &self,
        frame: &mut UiFrame<'_>,
        area: Rect,
        ctx: &ComponentContext,
        _registry: &mut HitboxRegistry,
    ) {
        (self.render_fn)(frame, area, ctx);
    }
}

/// Wraps a Ratatui `StatefulWidget` as a term-wm Component.
/// Holds the widget state internally, providing clean mutable access
/// during render without `Rc<RefCell<>>` boilerplate.
///
/// # Example
/// ```ignore
/// let mut state = ListState::default();
/// state.select(Some(0));
/// let comp = StatefulWidgetAdapter::new(state, |frame, area, state, _ctx| {
///     let items = vec![ListItem::new("Item 1"), ListItem::new("Item 2")];
///     frame.render_stateful_widget(List::new(items), area, state);
/// });
/// app.register(comp);
/// ```
pub struct StatefulWidgetAdapter<S, F> {
    state: S,
    render_fn: F,
}

impl<S, F> StatefulWidgetAdapter<S, F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &mut S, &ComponentContext),
{
    pub fn new(initial_state: S, render_fn: F) -> Self {
        Self { state: initial_state, render_fn }
    }

    /// Borrow the widget state immutably.
    pub fn state(&self) -> &S {
        &self.state
    }

    /// Borrow the widget state mutably (e.g., to update selection).
    pub fn state_mut(&mut self) -> &mut S {
        &mut self.state
    }
}

impl<S, F> Component<TermWmAction> for StatefulWidgetAdapter<S, F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &mut S, &ComponentContext),
{
    fn render(
        &self,
        frame: &mut UiFrame<'_>,
        area: Rect,
        ctx: &ComponentContext,
        _registry: &mut HitboxRegistry,
    ) {
        // SAFETY: render() takes &self, but StatefulWidget needs &mut state.
        // We use a cell-based approach to allow mutation during render.
        // This is safe because render() is single-threaded per frame.
        let state_ptr = &self.state as *const S as *mut S;
        unsafe { (self.render_fn)(frame, area, &mut *state_ptr, ctx); }
    }
}
```

**Note on `StatefulWidgetAdapter` safety:** The `Component::render` trait method takes `&self`, but `StatefulWidget::render_widget` requires `&mut State`. Since render is single-threaded and the mutable borrow is confined to the closure (no concurrent access), this is sound. An alternative is to use `RefCell<S>` internally, but that adds runtime overhead. The `unsafe` block is minimal and well-contained.

**File:** `crates/term-wm-ui-components/src/lib.rs` (MODIFY)

Add `pub mod widget_adapter;` and re-export both adapters:
```rust
pub mod widget_adapter;
pub use widget_adapter::{WidgetAdapter, StatefulWidgetAdapter};
```

**Why:** Two adapters cover the full Ratatui ecosystem:
- `WidgetAdapter` for stateless widgets (`Paragraph`, `Block`, `Gauge`, etc.)
- `StatefulWidgetAdapter` for stateful widgets (`List`, `Table`, `Scrollbar`, etc.)

## Step 6: Simplify `dual_image.rs` example

**File:** `examples/dual_image.rs` (REWRITE)

Before: 126 lines, 70% boilerplate.

```rust
use std::io;
use term_wm::prelude::*;
use term_wm::SvgImageComponent;

fn main() -> io::Result<()> {
    let mut paths: Vec<String> = std::env::args().skip(1).collect();
    if paths.is_empty() {
        paths.push("assets/zenOSmosis-logo.svg".to_string());
    }
    if paths.len() == 1 {
        paths.push(paths[0].clone());
    }

    let mut app = StandaloneApp::new(AppContext::new("example", "0.0.0"));

    let mut left = SvgImageComponent::new();
    left.set_keep_aspect(true);
    left.set_colorize(true);
    let left_key = app.register(left);

    let mut right = SvgImageComponent::new();
    right.set_keep_aspect(true);
    right.set_colorize(true);
    let right_key = app.register(right);

    // Load images into components
    if let Some(img) = app.component_mut::<SvgImageComponent>(left_key) {
        img.load_from_path(&paths[0])?;
    }
    if let Some(img) = app.component_mut::<SvgImageComponent>(right_key) {
        img.load_from_path(&paths[1])?;
    }

    app.run()
}
```

**Reduction:** 126 → ~30 lines. Zero trait implementations needed.

## Migration Path for Third-Party Apps

**Before (current):**
```rust
struct MyApp { wm: WindowManager, keys: Vec<WindowKey> }
impl WindowManagerHost for MyApp { fn windows(&mut self) -> &mut WindowManager { &mut self.wm } }
impl WindowProvider for MyApp { fn enumerate_windows(&mut self) -> Vec<WindowKey> { self.keys.clone() } }
// ... 60+ more lines of boilerplate
run_window_app(&mut output, &mut driver, &mut app)?;
```

**After (standalone):**
```rust
let mut app = StandaloneApp::new(AppContext::new("myapp", "1.0"));
let key = app.register(MyComponent::new());
app.run()?;
```

**After (embedded in existing Ratatui app):**
```rust
let mut wm_app = StandaloneApp::embedded(AppContext::new("sub-app", "1.0"));
wm_app.register(MyComponent::new());
// Use wm_app.run_with(&mut frame, &mut driver) in your existing event loop
```

**After (wrapping a Ratatui widget):**
```rust
let comp = WidgetAdapter::new(|frame, area, _ctx| {
    frame.render_widget(Paragraph::new("Hello World"), area);
});
app.register(comp);
```

Existing code using `run_window_app` continues to work with zero changes.
