# Plan: Tab Outline Mode via SystemTask

## Problem
When tabbing through windows (Tab/Shift+Tab), the command palette blocks the view with its item list. In monocle mode, panel overlays also obscure window content.

## Goal
When user presses Tab/Shift+Tab:
- Command palette hides its content → becomes full-screen DIM overlay
- In monocle mode: panels hidden too
- After ~1.5s of no tabbing: palette content + panels restore
- Use `SystemTask::ClearTabOutline` with debounce pattern

## Debounce Pattern

Each Tab press schedules a **new** `ClearTabOutline` task. When a task fires:
- If `now >= tab_outline_until`: user stopped tabbing → clear outline
- If `now < tab_outline_until`: user pressed Tab again (newer task pending) → silently ignore

No task cancellation needed — stale tasks self-cancel via the time check.

## Behavior Matrix

| Scenario | Palette renders | Panels visible |
|----------|----------------|----------------|
| Normal | List + dim backdrop | Normal |
| Tab outline, monocle | Full-screen DIM only | Hidden (skip overlay render) |
| Tab outline, not monocle | Full-screen DIM only | Visible |
| After 1.5s timeout | Restores to normal | Restores to normal |
