# Revised Implementation Plan: Floating/Tiling Window Behavior

## Algorithm: Topology-Aware Balanced Insertion
Instead of `split_root` (which bisects the root), the balanced algorithm:
1. Computes current regions from the tree + workspace bounds
2. Finds the leaf with the largest area (`width * height`)
3. Splits it along its longer axis (`Horizontal` if `width >= height`, else `Vertical`)
4. Inserts the new window adjacent via `insert_leaf`

This ensures each window gets proportional space. The Euclidean-sorted ORDER determines which windows get inserted first (and thus get the largest remaining areas).

## Step 3: Euclidean sort (takes ownership)
**File**: `crates/term-wm-core/src/window/window_manager/drag.rs`

```rust
fn calculate_tiling_order(
    mut floating_rects: Vec<(WindowKey, Rect)>,
    target_rect: Rect,
) -> Vec<(WindowKey, Rect)> {
    let target_center_y = (target_rect.y as i32) + (target_rect.height as i32) / 2;
    let target_center_x = (target_rect.x as i32) + (target_rect.width as i32) / 2;

    floating_rects.sort_by(|a, b| {
        // Scale Y by 2 to account for terminal cell aspect ratio (~2:1 height:width)
        let a_dy = (((a.1.y as i32) + (a.1.height as i32) / 2) - target_center_y) * 2;
        let a_dx = ((a.1.x as i32) + (a.1.width as i32) / 2) - target_center_x;
        let b_dy = (((b.1.y as i32) + (b.1.height as i32) / 2) - target_center_y) * 2;
        let b_dx = ((b.1.x as i32) + (b.1.width as i32) / 2) - target_center_x;
        (a_dx * a_dx + a_dy * a_dy).cmp(&(b_dx * b_dx + b_dy * b_dy))
    });
    floating_rects
}
```

## Step 4: `TilingLayout::insert_window_balanced()`
**File**: `crates/term-wm-core/src/layout/tiling.rs`

```rust
/// Topology-aware insertion: finds the largest leaf by area,
/// splits along its longer axis, inserts `insert` adjacent.
/// Both preview simulation and commit use this exact method.
pub fn insert_window_balanced(&mut self, insert: Id, area: Rect) {
    let regions = self.layout(area);
    if regions.is_empty() {
        self.split_root(insert, InsertPosition::Right);
        return;
    }

    // Find leaf with largest area
    let (largest_id, largest_rect) = regions.iter()
        .max_by_key(|(_, r)| (r.width as u32) * (r.height as u32))
        .copied()
        .unwrap();

    // Split along the longer axis (scale height by 2 for terminal cell aspect ratio)
    let pos = if (largest_rect.width as u32) >= (largest_rect.height as u32) * 2 {
        InsertPosition::Right
    } else {
        InsertPosition::Bottom
    };

    if !self.root.insert_leaf(largest_id, insert, pos) {
        self.split_root(insert, pos);
    }
}
```

fn simulate_position_based_layout(
    windows: Vec<(WindowKey, Rect)>,
    anchor_key: WindowKey,
    snap_position: InsertPosition,
    workspace_bounds: Rect,
) -> Vec<(WindowKey, Rect)> {
    if windows.is_empty() { return Vec::new(); }

    let target_rect = windows.iter()
        .find(|(k, _)| *k == anchor_key)
        .map(|(_, r)| *r)
        .expect("anchor_key must be in windows");

    let others: Vec<_> = windows.iter()
        .filter(|(k, _)| *k != anchor_key)
        .copied()
        .collect();

    if others.is_empty() {
        let mut layout = TilingLayout::new_void();
        layout.split_root(anchor_key, snap_position);
        return layout.regions(workspace_bounds).into_iter()
            .filter(|(k, _)| *k == anchor_key)
            .collect();
    }

    let sorted = calculate_tiling_order(others, target_rect);
    let mut layout = TilingLayout::new(LayoutNode::leaf(sorted[0].0));
    for (key, _) in &sorted[1..] {
        layout.insert_window_balanced(*key, workspace_bounds);
    }
    layout.split_root(anchor_key, snap_position);

    let mut regions = layout.regions(workspace_bounds);
    let mut result = Vec::with_capacity(sorted.len() + 1);
    if let Some(r) = regions.iter().find(|(k, _)| *k == anchor_key).map(|(_, r)| *r) {
        result.push((anchor_key, r));
    }
    for (k, _) in &sorted {
        if let Some(idx) = regions.iter().position(|(rk, _)| rk == k) {
            result.push((*k, regions.swap_remove(idx).1));
        }
    }
    result
}
```

pub(super) fn apply_position_based_layout(
    &mut self,
    anchor_key: WindowKey,
    snap_position: InsertPosition,
    area: Rect,
) {
    let mut all_windows: Vec<_> = self
        .mapped_windows()
        .into_iter()
        .filter(|key| self.is_window_floating(*key))
        .map(|key| (*key, self.region(*key)))
        .collect();

    // Ensure anchor is included even if not in regions (e.g., floating-only window)
    if !all_windows.iter().any(|(k, _)| *k == anchor_key) {
        let rect = self
            .floating_rect(anchor_key)
            .map(|spec| spec.resolve(area))
            .unwrap_or_else(|| self.region(anchor_key));
        all_windows.push((anchor_key, rect));
    }

    if all_windows.is_empty() {
        return;
    }

    let target_rect = all_windows
        .iter()
        .find(|(k, _)| *k == anchor_key)
        .map(|(_, r)| *r)
        .unwrap();

    let others: Vec<_> = all_windows
        .iter()
        .filter(|(k, _)| *k != anchor_key)
        .copied()
        .collect();

    for (key, _) in &all_windows {
        self.clear_floating_rect(*key);
    }

    if others.is_empty() {
        let mut layout = TilingLayout::new_void();
        layout.split_root(anchor_key, snap_position);
        self.managed_layout = Some(layout);
        return;
    }

    let sorted = calculate_tiling_order(others, target_rect);
    let mut layout = TilingLayout::new(LayoutNode::leaf(sorted[0].0));
    for (key, _) in &sorted[1..] {
        layout.insert_window_balanced(*key, area);
    }
    layout.split_root(anchor_key, snap_position);
    self.managed_layout = Some(layout);
}
```

struct SnapPreviewCache {
    positions: Vec<(WindowKey, Rect)>,
    cached_rect: Option<Rect>,
    cached_quadrant: Option<(bool, bool)>,
    dragged_key: Option<WindowKey>,
}

impl SnapPreviewCache {
    fn needs_recalc(&self, mouse_x: u16, mouse_y: u16, dragged: WindowKey) -> bool {
        let Some(rect) = self.cached_rect else { return true; };
        if self.dragged_key != Some(dragged) { return true; }
        let out_of_bounds = mouse_x < rect.x
            || mouse_x >= rect.x + rect.width
            || mouse_y < rect.y
            || mouse_y >= rect.y + rect.height;
        let cur_q = (mouse_x >= rect.x + rect.width / 2, mouse_y >= rect.y + rect.height / 2);
        out_of_bounds || self.cached_quadrant != Some(cur_q)
    }

    fn update(&mut self, mouse_x: u16, mouse_y: u16, rect: Rect, dragged: WindowKey, pos: Vec<(WindowKey, Rect)>) {
        self.cached_quadrant = Some((mouse_x >= rect.x + rect.width / 2, mouse_y >= rect.y + rect.height / 2));
        self.cached_rect = Some(rect);
        self.dragged_key = Some(dragged);
        self.positions = pos;
    }

    fn clear(&mut self) {
        self.cached_rect = None;
        self.cached_quadrant = None;
        self.dragged_key = None;
        self.positions.clear();
    }
}

# Bug Fix Plan

## Bug 1: Tiling windows closes every window except active one
**Root cause**: `apply_position_based_layout` uses `self.regions.ids()` which is a stale rendering artifact. Windows added to the layout tree between renders are invisible to `regions.ids()` and get dropped when the layout is rebuilt.

**Fix** (`crates/term-wm-core/src/window/window_manager/drag.rs`):
Replace `self.regions.ids()` with authoritative SlotMap enumeration, filtering to only active (mapped) windows:

```rust

    .into_iter()
    .filter(|key| self.is_window_floating(*key))

    .collect();
```

## Bug 2: Float-all moves/resizes other windows
**Root cause**: `float_all_windows()` captures `self.region(*key)` which returns the **content** rect (with chrome subtracted), not the full frame rect. This shrinks and shifts windows when they become floating.

**Fix** (`crates/term-wm-core/src/window/window_manager/drag.rs`, line ~470):
Query chrome decorator dimensions and add them back to the captured rect to reconstruct the full frame. Replace:

```rust
let regions: Vec<_> = tiled_keys
    .iter()
    .map(|key| (*key, self.full_region_for_key(*key)))
    .collect();
```

Wait, `self.full_region_for_key` doesn't exist. The correct approach: use `self.regions.get(key)` directly (which returns the full frame before chrome subtraction), or reconstruct the full frame by adding chrome padding:

```rust
let regions: Vec<_> = tiled_keys
    .iter()
    .map(|key| {
        // Use regions.get for the full frame rect; fall back to region(key) if missing
        let full = self.regions.get(*key).copied().unwrap_or_else(|| self.region(*key));
        (*key, full)
    })
    .collect();
```

`self.regions.get(key)` returns the rect as stored in the RegionMap (the full frame before chrome is subtracted by `region_for_key`). This is the correct value for the floating rect.

## Bug 3: New window starts maximized
**Root cause**: `tile_window_key` creates `LayoutNode::leaf(key)` for the first window, which fills the entire managed area.

**Fix**: Do NOT modify `tile_window_key`. Instead, implement the smart-default logic at the app/handler level. In the `NewWindow` action handler (likely `main.rs` or the `TermWmAction` handler), check the terminal width before calling `tile_window`. If the screen is wide (e.g., width > 120), set the window as floating with a sensible default size (80x24) instead of tiling:

```rust

    let default_w = 80u16.min(area.width);
    let default_h = 24u16.min(area.height);
    let x = area.x + (area.width.saturating_sub(default_w) / 2) as i32;
    let y = area.y + (area.height.saturating_sub(default_h) / 2) as i32;

        x, y, width: default_w, height: default_h,
    })));
} else {
    wm.tile_window(key);
}
```

Where `WIDE_SCREEN_THRESHOLD = 120`.

## Bug 7: Tiling algorithm suboptimal (narrow side-by-side windows)
**Root cause**: `insert_window_balanced` uses `width >= height * 2` threshold which is too aggressive for horizontal splits.

**Fix** (`crates/term-wm-core/src/layout/tiling.rs`):
Change the split heuristic to split along the **longer axis** (producing more square-like regions), with a minimum ratio threshold:

```rust
// Account for terminal cell aspect ratio (~2:1 height:width) by scaling height
let h_scaled = (largest_rect.height as u32) * 2;
let w = largest_rect.width as u32;

let pos = if w >= h_scaled * 3 / 2 {
    InsertPosition::Right   // very wide (e.g., 150%+ of scaled height) → columns
} else if h_scaled >= w * 3 / 2 {
    InsertPosition::Bottom  // very tall → rows
} else if w >= h_scaled {
    InsertPosition::Right   // wider after aspect correction → columns
} else {
    InsertPosition::Bottom  // taller after aspect correction → rows
};
```

This prefers splitting along the longer axis, which produces more balanced regions.
