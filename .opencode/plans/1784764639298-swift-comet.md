# Implementation Plan: Floating/Tiling Window Behavior

## Task Summary
Implement automatic float-all behavior and smart tiling positions when snapping.

## Requirements
1. **Automatic float-all**: When any window becomes floating, all other windows automatically become floating too (retain their current positions)
2. **Internal snap preview**: Calculate positions for all windows internally, but only show preview for dragged window
3. **Position-based tiling**: When snapping, place windows in tiled positions closest to their current floating positions

## Current Codebase Understanding

### Key Files
- `crates/term-wm-core/src/window/window_manager/drag.rs` - Drag/snap operations
- `crates/term-wm-core/src/window/window_manager/mod.rs` - Window manager core
- `crates/term-wm-core/src/window/entry.rs` - Window state
- `crates/term-wm-core/src/layout/floating.rs` - Floating window utilities
- `crates/term-wm-core/src/layout/tiling.rs` - Tiling layout engine

### Current Behavior
- `detach_to_floating()` detaches a single window to floating
- `apply_snap()` tiles all floating windows when snapping (lines 351-359)
- `tile_window_key()` tiles individual windows

## Implementation Plan

### Step 1: Add `float_all_windows()` method
**File**: `crates/term-wm-core/src/window/window_manager/drag.rs`

Add a new method that:
1. Iterates through all mapped windows
2. For each tiled window, preserves its current region as a floating rect
3. Detaches from tiling layout
4. Sets `floating_rect` with current position

```rust
pub(super) fn float_all_windows(&mut self) {
    let keys: Vec<_> = self.mapped_windows().collect();
    for key in keys {
        if !self.is_window_floating(key) {
            let rect = self.region(key);
            self.detach_to_floating(key, rect);
        }
    }
}
```

### Step 2: Modify `detach_to_floating()` to trigger float-all
**File**: `crates/term-wm-core/src/window/window_manager/drag.rs`

After the initial detach call, trigger `float_all_windows()` for other tiled windows.

### Step 3: Add `tile_all_windows_with_positions()` method
**File**: `crates/term-wm-core/src/window/window_manager/drag.rs`

New method that:
1. Collects all floating windows with their positions
2. Sorts by spatial proximity
3. Calculates tiled positions that mimic floating layout
4. Uses BSP tree insertion with position-based placement

### Step 4: Modify `apply_snap()` to use position-based tiling
**File**: `crates/term-wm-core/src/window/window_manager/drag.rs`

Update lines 351-359 to use `tile_all_windows_with_positions()` instead of simple `tile_window_key()`.

### Step 5: Update snap preview calculation
**File**: `crates/term-wm-core/src/window/window_manager/drag.rs`

Modify `update_snap_preview()` to calculate positions for all windows internally (for future ghost preview support).

### Step 6: Add actions
**File**: `crates/term-wm-core/src/actions.rs`

Add `FloatAll` and `TileAll` actions if needed for keybindings.
