# Plan: Scroll to Bottom on Keypress in TerminalComponent

## Bug

Typing in a scrolled-up terminal window doesn't scroll to bottom. The pane's internal scrollback resets to 0, but the `ScrollViewComponent`'s viewport offset stays stale. The render sync logic then reverses the scroll-to-bottom.

## Fix

Move the auto-scroll-to-bottom into `TerminalComponent::update`, where `KeyToBytes` / `ClipboardPaste` already reset the PTY scrollback. Add `ctx.scroll_handle().map(|h| h.scroll_vertical_to(usize::MAX))` to notify the parent scroll container.

## Changes

### `crates/term-wm-ui-components/src/terminal.rs`

In `TerminalComponent::update`, add scroll handle notification to both `KeyToBytes` and `ClipboardPaste` handlers:

```rust
TermWmAction::KeyToBytes(bytes) => {
    self.selection.borrow_mut().clear();
    self.pane.borrow_mut().set_scrollback(0);
    if let Some(handle) = ctx.scroll_handle() {
        handle.scroll_vertical_to(usize::MAX);
    }
    if let Err(err) = self.pane.borrow_mut().write_bytes(&bytes) {
        tracing::warn!(?err, "terminal input write failed");
    }
}

TermWmAction::ClipboardPaste(text) => {
    self.selection.borrow_mut().clear();
    self.pane.borrow_mut().set_scrollback(0);
    if let Some(handle) = ctx.scroll_handle() {
        handle.scroll_vertical_to(usize::MAX);
    }
    if let Err(err) = self.pane.borrow_mut().write_bytes(&text.as_bytes()) {
        tracing::warn!(?err, "clipboard paste write failed");
    }
}
```

This keeps `ScrollViewComponent` generic and places the auto-scroll behavior where it belongs — in the terminal component.
