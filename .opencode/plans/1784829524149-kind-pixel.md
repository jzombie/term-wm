# Plan: Add ClosePolicy, Clean Up TermWmApp, Simplify main.rs

## Problem

1. **No closure policy**: `close_window` unconditionally destroys every window. Closing a debug log or system panel via chrome 'X' or keybinding destroys the component, forcing the app to re-create it in toggle functions (a separation-of-concerns violation).
2. **Stale key leak in `TermWmApp`**: `window_keys` vector grows unboundedly — keys are pushed on open but never pruned on close.
3. **Misleading API**: `TermWmApp::register` opens a window immediately (not a registry).

## What's Already Done (skip these)

- `crates/term-wm-core/src/debug_log.rs` exists in core, re-exported by sys-ui.
- `WindowManager::open_window()` exists.
- `TermWmAction::Callback(fn())` exists, handled in `drain_action_queue`, wired to panic button.

## Changes

### 1. Add `ClosePolicy` to Window

**`crates/term-wm-core/src/window/entry.rs`**:
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Default)]
pub enum ClosePolicy {
    #[default]
    Destroy,
    Unmap,
}
```
Add `pub close_policy: ClosePolicy` to `Window` struct, default `Destroy`.

**`crates/term-wm-core/src/window/window_manager/mod.rs`**:
Add `pub fn set_close_policy(&mut self, key: WindowKey, policy: ClosePolicy)`.

**`crates/term-wm-core/src/window/window_manager/chrome.rs`**:
Update `close_window`: if `ClosePolicy::Destroy` → current behavior (destroy + remove); if `ClosePolicy::Unmap` → transition to `Unmapped` only, keep component + SlotMap entry alive.

### 2. Fix `TermWmApp` — remove `window_keys`, rename `register`

**`src/term_wm_app.rs`**:
- Remove `window_keys: Vec<WindowKey>` field and all references to it.
- Rename `register` to `open_window` — just delegates to `self.wm.open_window(component)`.
- Update callers of `register` → `open_window`.

### 3. Simplify `src/main.rs` — pure state transitions

- Set `ClosePolicy::Unmap` on debug log and system panel after creation in `new_with`.
- `toggle_debug_window` / `toggle_system_panel`: read current `window_state(key)`, flip between `Mapped` / `Unmapped`. No `has_window` check, no re-creation.
- `on_panic`: just `transition_window(key, Mapped)`.
- Remove `WmDebugLogComponent::from_handle` usage from main.rs (unused now).
- Keep debug log buffer in core — it continues to capture logs even when unmapped.

### 4. Update tests

**`window_manager/mod.rs`**:
- Update `close_window_cleans_up_layout_and_state` (unchanged, default Destroy).
- Rename test to `close_window_unmap_policy_preserves_slotmap_entry`: set `ClosePolicy::Unmap`, close, assert key survives.
- Add test verifying `ClosePolicy::Destroy` default removes the entry.
