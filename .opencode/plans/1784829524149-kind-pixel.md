# Plan: Add `CanvasScrollView` ŌĆö Offscreen Canvas Adapter (v4)

## Problem

Child components inside `ScrollViewComponent` must manually read `offset_y`, subtract from render positions, and call `set_content_size`. Error-prone.

## Solution

`CanvasScrollView<C>` inside `ScrollViewComponent`. Offers offscreen canvas so children render at natural `(0, 0)`. ScrollViewComponent retains scroll bars, keyboard scrolling, drag handling.

### Hierarchy

```
ScrollViewComponent<CanvasScrollView<VerticalStackComponent<PanelChild>>>
```

## Changes

### A. Add `HitboxRegistry::merge_with_transform` to core

**`crates/term-wm-core/src/hitbox_registry.rs`** ŌĆö New method that consumes a scratch registry, translates all entries by `(tx, ty)`, clips against a viewport, merges into self.

### B. New file: `CanvasScrollView<C>`

**`crates/term-wm-ui-components/src/canvas_scroll_view.rs`**:

```rust
pub struct CanvasScrollView<C> {
    pub inner: C,
    pub content_width: u16,
    pub content_height: u16,
    offscreen_buf: Buffer,
}
```

**render**:
1. `set_content_size` to parent ScrollView
2. Create neutralized child context: `ScrollHandle = None`, viewport offsets zeroed ŌĆö prevents child from double-applying scroll
3. Ensure offscreen buffer sized; reallocate only on dimension change; `reset()` each frame
4. `std::mem::take` ŌåÆ `RatatuiBackend::new(temp_buf, virtual_area)` ŌĆö avoids ownership move error
5. Render child to offscreen backend with scratch `HitboxRegistry`
6. `merge_with_transform`: translate scratch entries by `(area.x - offset_x, area.y - offset_y)`, clip to `area`
7. Re-claim buffer via `mock_backend.into_buffer()`
8. Clamped blit: compute intersection of `(offset..offset+viewport)` and `(0..content)`; check `dst_x >= 0` and `dst_y >= 0` before indexing ŌĆö prevents negative coordinate underflow panic

**handle_events**:
1. Neutralize context identically to render: zeroed viewport, no ScrollHandle, virtual screen area `(0, 0, content_w, content_h)`
2. Translate mouse event: `virtual_x = mouse.x - area.x + offset_x`, `virtual_y = mouse.y - area.y + offset_y` ŌĆö correct formula accounting for non-zero screen origin
3. If translated coordinates are out of virtual bounds, return `Ignored`
4. Delegate to `inner` with translated event and neutralized context

### C. Update WmSystemPanelComponent

**`crates/term-wm-sys-ui-components/src/wm_system_panel.rs`**: Wrap `CanvasScrollView` inside `ScrollViewComponent`.

### D. Revert VerticalStackComponent

**`crates/term-wm-ui-components/src/vertical_stack.rs`**: Remove `set_content_size` and scroll offset changes. Revert to original: children render at `area.y + child_virtual_y` (natural positions).
