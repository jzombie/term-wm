# Plan: Remove System Window Special-Casing + Move Debug Buffer to Core

## Problem

1. **Semantic leak**: `is_system_window` flag makes the WM aware of component types it should never know about, causing divergent `close_window` paths and layout evasion via `set_system_window`.
2. **Fragile buffer placement**: The debug log buffer (`DebugLogHandle`/`GLOBAL_LOG`) lives in `term-wm-sys-ui-components`, tying the log store to the UI crate. When the component is destroyed, the buffer goes with it (unless a global ref is preserved), and re-creating the component requires special plumbing to avoid losing accumulated log history.

## Changes

### Step 1: Move debug log buffer to `term-wm-core`

**New file: `crates/term-wm-core/src/debug_log.rs`**

Move the following from `crates/term-wm-sys-ui-components/src/wm_debug_log.rs` into a new `debug_log` module in core:

- `DEFAULT_MAX_LINES` constant
- `DebugLogBuffer` (struct + impl, make `pub(crate)`)
- `DebugLogHandle` (public struct + `push`, `writer`, `lines`)
- `DebugLogWriter` (public struct + `new`, `flush_pending`, impl `Write`)
- `GLOBAL_LOG: OnceLock<DebugLogHandle>` (private)
- `set_global_debug_log(handle) -> bool` (public)
- `global_debug_log() -> Option<DebugLogHandle>` (public)

Only stdlib deps needed + `strip_ansi_escapes` (already in `term_wm_core::utils::ansi`).

**`crates/term-wm-core/src/lib.rs`**: Add `pub mod debug_log;` export.

**`crates/term-wm-sys-ui-components/src/wm_debug_log.rs`**:
- Remove `DebugLogBuffer`, `DebugLogHandle`, `DebugLogWriter`, `GLOBAL_LOG`, `DEFAULT_MAX_LINES`, the `set_global_debug_log`/`global_debug_log`/`trigger_error` functions, and the `strip_ansi_escapes` import.
- Import `DebugLogHandle` from `term_wm_core::debug_log`.
- The `WmDebugLogComponent` stores `handle: DebugLogHandle` as before, but now the type comes from core.
- Keep `install_panic_hook`, `PANIC_HOOK_INSTALLED`, `WmDebugLogComponent` in this file.
- `install_panic_hook`: change `GLOBAL_LOG.get()` to `global_debug_log()` (from core).

**`src/logging.rs`**:
- Change import: `term_wm_sys_ui_components::wm_debug_log::{DebugLogWriter, global_debug_log}` → `term_wm_core::debug_log::{DebugLogWriter, global_debug_log}`.
- The `#[cfg(feature = "sys-ui")]` guard can stay since `DelegatingWriter` is only needed when UI components exist.

## Key Design Properties

- **Buffer lives in core**: `GLOBAL_LOG` lives in `term-wm-core`, independent of any component lifecycle. Destroying/re-creating `WmDebugLogComponent` never loses accumulated log history.
- **WM is agnostic**: No `is_system_window` flag, no `set_system_window` method, no `closed_windows` queue. `close_window` is uniform for every window.
- **App manages lifecycle**: Toggle functions handle re-creation if the window was closed, using the global buffer handle from core.
