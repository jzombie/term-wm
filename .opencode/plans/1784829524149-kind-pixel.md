# Plan: Fix toggle_tiling — Restore Spatial + Constraint-Guided Insertion

## Problem

My previous fix replaced `toggle_tiling` with an atomic clear-float + `insert_window_balanced` loop, destroying the spatial reconstruction pipeline and bypassing `SizeConstraints::fits_split`. This causes:

1. **Thin columns**: `insert_window_balanced` doesn't guard against splits that produce sub-minimum dimensions.
2. **Wrong snap positions**: `insert_window_balanced` ignores the floating rect coordinates, using area-based partitioning instead of `resolve_target`/`detect_quadrant`.

## Fix

Rewrite `toggle_tiling` using the area-descending bulk insertion pattern specified in the lucky-pixel plan:

```rust
pub fn toggle_tiling(&mut self) {
    if self.managed_layout.is_some() {
        self.float_all_windows();
    } else {
        use crate::layout::LayoutNode;
        let keys: Vec<WindowKey> = self.z_order.iter().copied()
            .filter(|&k| self.window_state(k) == Some(WindowState::Mapped))
            .collect();
        if keys.is_empty() { return; }

        // Partition: windows WITH floating_rect use spatial insertion;
        // windows WITHOUT get appended via insert_window_balanced.
        let mut with_rects: Vec<(WindowKey, Rect)> = Vec::new();
        let mut without_rects: Vec<WindowKey> = Vec::new();
        for &k in &keys {
            if let Some(spec) = self.floating_rect(k) {
                with_rects.push((k, spec.resolve(self.managed_area)));
            } else {
                without_rects.push(k);
            }
        }

        // Sort by area descending — largest windows anchor the macro-geometry
        with_rects.sort_by_key(|(_, r)| u32::MAX - (r.width as u32) * (r.height as u32));

        let constraints = SizeConstraints {
            min_width: self.config.min_tile_width,
            min_height: self.config.min_tile_height,
        };

        // Build tree with spatial + constraint-aware insertion
        for (i, (key, rect)) in with_rects.iter().enumerate() {
            self.clear_floating_rect(*key);
            if i == 0 {
                self.managed_layout = Some(TilingLayout::new(LayoutNode::leaf(*key)));
                continue;
            }
            // Spatial insertion loop (same as tile_window_key but with constraints)
            if let Some(ref mut layout) = self.managed_layout {
                let (cx, cy) = rect.center();
                let regions = layout.regions(self.managed_area);
                let weight = crate::constants::CELL_ASPECT_RATIO;
                if let Some((target_key, target_rect)) =
                    term_wm_layout_engine::resolve_target(cx, cy, &regions, weight)
                {
                    let quad = term_wm_layout_engine::detect_quadrant(
                        cx as u16, cy as u16, &target_rect,
                    );
                    let pos = quad.to_insert_position();
                    if constraints.fits_split(&target_rect, pos.to_orientation()) {
                        let inserted = layout.root_mut().insert_leaf(target_key, *key, pos);
                        if !inserted {
                            layout.split_root(*key, pos);
                        }
                    } else {
                        layout.insert_window_balanced(*key, self.managed_area);
                    }
                } else {
                    layout.insert_window_balanced(*key, self.managed_area);
                }
            }
        }

        // Append windows without floating rects via balanced insertion
        for &key in &without_rects {
            self.clear_floating_rect(key);
            if let Some(ref mut layout) = self.managed_layout {
                layout.insert_window_balanced(key, self.managed_area);
            } else {
                self.managed_layout = Some(TilingLayout::new(LayoutNode::leaf(key)));
            }
        }
    }
}
```

The `SizeConstraints` struct already exists in `term_wm_layout_engine::rect`, with `fits_split` method. The `InsertPosition::to_orientation()` and `Quadrant::to_insert_position()` already exist.

This keeps `z_order`-based key ordering (deterministic) but restores the spatial heuristics and constraint guards.
