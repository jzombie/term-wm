# Plan: Extend `to_local_offset` to Remaining Compatible Locations

## Problem

`MouseEvent::to_local_offset` was added to `events.rs` and adopted by `CanvasScrollView`. Several other locations in the window manager still hand-roll the same coordinate math with inconsistent bounds handling.

## Analysis

Exploration of all 17 locations identified 2 that are compatible with `to_local_offset`:

| Location | File | Status |
|----------|------|--------|
| `localize_event` (test) | `wm/layout.rs:70-86` | **Update** — screen-to-virtual with offset, returns `Option<Event>` |
| `localize_event_content` | `wm/layout.rs:88-107` | **Update** — screen-to-virtual with offset, returns `Option<Event>` |
| Others | multiple | **No change** — see below |

Others intentionally NOT changed (different semantics, input types, or coordinate spaces):
- `component_context.rs:localize_mouse_click` — pure local (no scroll), different return type `(u16, u16)`, event-kind-filtered
- `components/mod.rs:handle_events` — dispatches to `on_mouse_*` callbacks with `(u16, u16)`, no scroll
- `text_renderer.rs` — trait interface uses raw `(u16, u16)`, not `MouseEvent`
- `vertical_stack.rs` — container hit-testing, not coordinate translation
- `scroll_view.rs` — single-axis scrollbar math
- `mouse_coord.rs` — `MousePosition` struct (`i16`), different layer
- `helpers.rs` — operates on raw `u16`, not `MouseEvent`
- `selectable_text.rs` — already-local coords, computes scroll distances

## Changes

### 1. Update `localize_event` (test-only) in layout.rs

Replace the hand-rolled clamping with `to_local_offset(content_width=u16::MAX, content_height=u16::MAX, offset_x=0, offset_y=0)`:

```rust
pub fn localize_event(&self, key: WindowKey, event: &Event) -> Option<Event> {
    let Event::Mouse(mouse) = event else { return None; };
    let dest = self.window_dest(key, self.full_region_for_key(key));
    mouse.to_local_offset(dest, 0, 0, u16::MAX, u16::MAX)
        .map(|m| Event::Mouse(m))
}
```

### 2. Update `localize_event_content` in layout.rs

Replace the hand-rolled clamping with `to_local_offset(content_x+width, content_y+height, offset_x, offset_y)`:

```rust
pub(super) fn localize_event_content(&self, key: WindowKey, event: &Event) -> Option<Event> {
    let Event::Mouse(mouse) = event else { return None; };
    let dest = self.window_dest(key, self.full_region_for_key(key));
    let (offset_x, offset_y) = self.window_content_offset(key);
    // Virtual content bounds: the region beyond the window header/chrome.
    // Content area starts at dest + offset and extends to dest_size - offset.
    let content_w = dest.width.saturating_sub(offset_x);
    let content_h = dest.height.saturating_sub(offset_y);
    mouse.to_local_offset(dest, offset_x as usize, offset_y as usize, content_w, content_h)
        .map(|m| Event::Mouse(m))
}
```
