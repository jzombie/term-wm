# Plan: Remove System Window Special-Casing, Move Debug Buffer to Core, Add Atomic `open_window`

## Problem

1. **Semantic leak**: `is_system_window` flag makes WM aware of component types it should never know about.
2. **Fragile buffer placement**: Debug log buffer lives in sys-ui crate, lost when component is destroyed.
3. **Defective lifecycle encapsulation**: `spawn` leaves the window in `Realized` state, forcing every caller to manually execute a 4-step imperative pipeline (transition, tile/float, focus). This leaks compositor orchestration into application code.

### Step 3: Add atomic `open_window` to `WindowManager`

**New method in `crates/term-wm-core/src/window/window_manager/mod.rs`** (immediately after `spawn`):

```rust
/// Atomic open transaction: spawn, map, place (tile or float), and focus.
pub fn open_window(&mut self, component: C) -> WindowKey {
    let key = self.spawn(component);
    self.transition_window(key, WindowState::Mapped);
    if !self.try_spawn_floating_default(key) {
        self.tile_window_key(key);
    }
    self.focus_window_key(key);
    key
}
```

This replaces the 4-step orchestration duplicated across all call sites.
