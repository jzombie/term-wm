# Plan: Fix toggle_tiling — Spatial Restoration + Constraint Guard + Sync

## Problem

Current `toggle_tiling` atomically clears all floating rects and uses `insert_window_balanced`, which:
1. Destroys spatial coordinates needed by `resolve_target`/`detect_quadrant`
2. Ignores `SizeConstraints::fits_split`, causing thin columns
3. Omits `bifurcate_draw_order()` and `mark_layout_dirty()`, leaving stale draw order

## Changes

### A. Add `fits_split` to `SizeConstraints`

**`crates/term-wm-layout-engine/src/rect.rs`**:

```rust
impl SizeConstraints {
    pub fn fits_split(&self, area: &LayoutRect, orientation: Orientation) -> bool {
        match orientation {
            Orientation::Horizontal =>
                area.width / 2 >= self.min_width && area.height >= self.min_height,
            Orientation::Vertical =>
                area.height / 2 >= self.min_height && area.width >= self.min_width,
        }
    }
}
```

### B. Rewrite `toggle_tiling` in drag.rs

Replace the current implementation with:
1. Collect mapped windows from `z_order` (deterministic visual stack order)
2. Partition into windows WITH `floating_rect` (spatial insertion) and WITHOUT (balanced)
3. Sort `with_rects` by area descending (largest anchor macro-geometry)
4. Build tree: first window → root, subsequent → `resolve_target`/`detect_quadrant` with `fits_split` guard. If constraint fails, fall back to `insert_window_balanced`.
5. Append `without_rects` via `insert_window_balanced`
6. **Mandatory**: `self.bifurcate_draw_order()` + `self.mark_layout_dirty()`

### C. Imports needed in drag.rs

- `use term_wm_layout_engine::SizeConstraints;` (new)
- `WindowState` (already added from previous fix)
- `LayoutNode`, `TilingLayout` (already imported via `crate::layout`)
