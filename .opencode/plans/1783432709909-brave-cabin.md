# Plan: Uniform PgUp/PgDown Scroll in Terminal Shells (Non-Direct Mode)

## Problem

PgUp/PgDown in terminal shells is **not** routed to `ScrollViewComponent` for viewport scrolling. Instead, it's encoded as escape sequences and sent to the PTY. The help overlay (also wrapped in `ScrollViewComponent`) correctly intercepts PgUp/PgDown for scrolling. The behavior should be uniform in non-direct mode.

**Root cause:** Terminal windows are created with `sv.set_keyboard_enabled(false)`, which disables `ScrollViewComponent::on_key`'s keyboard interception. PgUp/PgDown falls through to `TerminalComponent::handle_events`, which encodes them as `\x1b[5~`/`\x1b[6~` and writes to the PTY.

**Why naive `keyboard_enabled=true` fails:** `ScrollViewComponent::on_key` intercepts `Up`, `Down`, `Home`, `End` unconditionally (no modifier check). Enabling it globally would hijack shell history navigation (`Up`) and line editing (`Home`/`End`) in non-direct mode — a split-brain input routing failure.

## Files to Modify

1. **`crates/term-wm-ui-components/src/scroll_view.rs`** — add `ScrollKeyMode` enum, refactor `on_key`

### 1. Add `ScrollKeyMode` enum and update struct

In `crates/term-wm-ui-components/src/scroll_view.rs`, add the enum and replace `keyboard_enabled`:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ScrollKeyMode {
    /// No keyboard scroll interception — all keys fall through.
    None,
    /// Only PageUp/PageDown/Home/End are intercepted for scrolling.
    /// Up/Down/arrow keys fall through to the child (terminal shell).
    PaginationOnly,
    /// Full keyboard scroll: Up/Down/PageUp/PageDown/Home/End.
    /// Used by help overlay, keybindings overlay, debug log.
    Full,
}

pub struct ScrollViewComponent<C> {
    pub content: RefCell<C>,
    shared_state: Rc<RefCell<ViewportSharedState>>,
    v_drag: ScrollbarDrag,
    h_drag: ScrollbarDrag,
    keyboard_mode: ScrollKeyMode,  // replaces keyboard_enabled: bool
}
```

Update `new()` to default to `Full` (preserves existing overlay behavior).

Replace `set_keyboard_enabled(bool)` with `set_keyboard_mode(ScrollKeyMode)`.

### 2. Rewrite `on_key` with `KeyBindings` evaluation and modifier safety

Replace the hardcoded `KeyCode` matching in `on_key` with `KeyBindings::matches()`:

```rust
fn on_key(&mut self, event: &Event, ctx: &ComponentContext) -> EventResult<TermWmAction> {
    let handle = self.viewport_handle();
    let info = handle.info();
    let child_ctx = ctx.with_viewport(info, Some(handle));

    if self.keyboard_mode != ScrollKeyMode::None
        && ctx.focused()
        && !ctx.direct_mode()
        && let Event::Key(key) = event
        && key.kind == crossterm::event::KeyEventKind::Press
    {
        let is_full = self.keyboard_mode == ScrollKeyMode::Full;
        let kb = &ctx.config().keybindings;

        // Pagination-level scrolling (active in PaginationOnly and Full)
        if kb.matches(TermWmAction::ScrollPageUp, key) {
            let height = self.shared_state.borrow().height as isize;
            return EventResult::Action(TermWmAction::ScrollView(-height));
        } else if kb.matches(TermWmAction::ScrollPageDown, key) {
            let height = self.shared_state.borrow().height as isize;
            return EventResult::Action(TermWmAction::ScrollView(height));
        } else if kb.matches(TermWmAction::ScrollHome, key) {
            return EventResult::Action(TermWmAction::ScrollToTop);
        } else if kb.matches(TermWmAction::ScrollEnd, key) {
            return EventResult::Action(TermWmAction::ScrollToBottom);
        }
        // Line-level scrolling (only in Full mode)
        else if is_full && kb.matches(TermWmAction::ScrollUp, key) {
            return EventResult::Action(TermWmAction::ScrollView(-1));
        } else if is_full && kb.matches(TermWmAction::ScrollDown, key) {
            return EventResult::Action(TermWmAction::ScrollView(1));
        }
    }

    self.content.borrow_mut().handle_events(event, &child_ctx)
}
```

Key properties:
- `KeyBindings::matches()` checks **both** `KeyCode` and `KeyModifiers` — `Ctrl+Up`, `Alt+Home`, `Shift+PageUp` are NOT intercepted
- `direct_mode()` guard ensures passthrough when toggled
- `PaginationOnly` mode only intercepts PageUp/PageDown/Home/End — `Up`/`Down` fall through to shell history
- `ctx.keybindings()` gracefully falls through if keybindings unavailable

|------|-----|-----------|--------|
