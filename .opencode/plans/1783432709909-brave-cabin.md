# Plan: Uniform PgUp/PgDown Scroll in Terminal Shells (Non-Direct Mode)

## Problem

PgUp/PgDown in terminal shells is **not** routed to `ScrollViewComponent` for viewport scrolling. Instead, it's encoded as escape sequences and sent to the PTY. The help overlay (also wrapped in `ScrollViewComponent`) correctly intercepts PgUp/PgDown for scrolling. The behavior should be uniform in non-direct mode.

**Root cause:** Terminal windows are created with `sv.set_keyboard_enabled(false)`, which disables `ScrollViewComponent::on_key`'s keyboard interception. PgUp/PgDown falls through to `TerminalComponent::handle_events`, which encodes them as `\x1b[5~`/`\x1b[6~` and writes to the PTY.

## Files to Modify

1. **`/Volumes/2TB Storage Vault/term-wm/src/main.rs`** (~lines 209, 317) — remove `set_keyboard_enabled(false)`
2. **`/Volumes/2TB Storage Vault/term-wm/crates/term-wm-ui-components/src/scroll_view.rs`** (~line 309-338) — add `direct_mode()` guard
3. **`/Volumes/2TB Storage Vault/term-wm/crates/term-wm-ui-components/src/terminal.rs`** — test helpers at lines 1580, 1673 set `keyboard_enabled(false)` — may need updating if tests should reflect new behavior

## Changes

### 1. Enable keyboard on terminal ScrollViews

In `src/main.rs`, remove `sv.set_keyboard_enabled(false)` at lines 209 and 317. This allows `ScrollViewComponent::on_key` to intercept PgUp/PgDown (and Up/Down, Home/End) for viewport scrolling.

### 2. Guard PgUp/PgDown in `ScrollViewComponent::on_key` with `direct_mode()`

In `scroll_view.rs`, modify the `on_key` method to skip viewport scroll handling when `ctx.direct_mode()` is true. When direct mode is active, the inner terminal should receive raw keystrokes for apps like vim/tmux.

```rust
// Existing block (lines 309-338):
if self.keyboard_enabled
    && ctx.focused()
    && let Event::Key(key) = event
    && key.kind == crossterm::event::KeyEventKind::Press
{
    // ADD: skip viewport scroll in direct mode
    if ctx.direct_mode() {
        // In direct mode, forward all keys to the child (terminal)
        // so PgUp/PgDown are encoded to PTY escape sequences.
    } else {
        use crossterm::event::KeyCode;
        match key.code {
            KeyCode::Up => return EventResult::Action(TermWmAction::ScrollView(-1)),
            KeyCode::Down => return EventResult::Action(TermWmAction::ScrollView(1)),
            KeyCode::PageUp => {
                let height = self.shared_state.borrow().height as isize;
                return EventResult::Action(TermWmAction::ScrollView(-height));
            }
            KeyCode::PageDown => {
                let height = self.shared_state.borrow().height as isize;
                return EventResult::Action(TermWmAction::ScrollView(height));
            }
            KeyCode::Home => return EventResult::Action(TermWmAction::ScrollToTop),
            KeyCode::End => return EventResult::Action(TermWmAction::ScrollToBottom),
            _ => {}
        }
    }
}
```

### 3. Remove redundant Shift+PgUp/PgDown handling in `TerminalComponent`

In `terminal.rs` (~lines 69-79), the existing Shift+PgUp/PgDown block is now redundant for non-direct mode (the `ScrollViewComponent` handles it). However, in direct mode, `ScrollViewComponent` passes through, so `TerminalComponent` needs to handle Shift+PgUp/PgDown as before (scroll vt100 scrollback by 10 lines).

**Decision:** Keep the Shift+PgUp/PgDown handling in `TerminalComponent` as-is. It only fires in direct mode (where `ScrollViewComponent` passes through). In non-direct mode, `ScrollViewComponent` handles PgUp/PgDown and Shift+PgUp/PgDown before `TerminalComponent` sees them.

## Behavior Matrix (After Change)

| Mode | Key | Component | Action |
|------|-----|-----------|--------|
| Non-direct | PgUp/PgDown | ScrollViewComponent | Scroll viewport by page |
| Non-direct | Shift+PgUp/PgDown | ScrollViewComponent | Scroll viewport by page (same as PgUp/PgDown) |
| Non-direct | Up/Down | ScrollViewComponent | Scroll viewport by 1 line |
| Direct | PgUp/PgDown | TerminalComponent | Encoded to PTY escape sequences |
| Direct | Shift+PgUp/PgDown | TerminalComponent | Scroll vt100 scrollback by 10 lines |
| Help overlay | PgUp/PgDown | ScrollViewComponent | Scroll viewport by page (unchanged) |
