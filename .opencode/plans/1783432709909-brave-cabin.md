# Plan: Rename ViewportSharedState / ViewportHandle / ViewportContext

## Problem

`ViewportSharedState` is misleading — "Shared" describes the `Rc<RefCell<>>` access mechanism, not the domain responsibility. The types should be named after what they represent.

## Rename Mapping

| Old Name | New Name | Rationale |
|----------|----------|-----------|
| `ViewportSharedState` | `ScrollBounds` | Defines the bounding box and behavioral flags of a scrollable region |
| `ViewportHandle` | `ScrollHandle` | Interface for mutating/inspecting scroll positions |
| `ViewportContext` | `ScrollViewport` | Immutable snapshot of viewport offset and dimensions |

## Files to Modify

1. **`crates/term-wm-core/src/component_context.rs`** — struct defs, impl blocks, field types, method signatures
2. **`crates/term-wm-ui-components/src/scroll_view.rs`** — imports, field types, constructors
3. **`crates/term-wm-ui-components/src/text_renderer.rs`** — imports, field types
4. **`crates/term-wm-ui-components/src/list.rs`** — `ctx.viewport_handle()` → `ctx.scroll_handle()`
5. **`crates/term-wm-ui-components/src/toggle_list.rs`** — `ctx.viewport_handle()` → `ctx.scroll_handle()`
6. **`crates/term-wm-ui-components/src/markdown_viewer.rs`** — test calls to `scroll.viewport_handle()` → `scroll.scroll_handle()`
7. **`crates/term-wm-ui-components/src/terminal.rs`** — test imports, type annotations, struct literals
8. **`AGENTS.md`** — documentation references

## Changes

### 1. `component_context.rs`

- Rename `pub struct ViewportSharedState` → `pub struct ScrollBounds`
- Rename `impl ViewportSharedState` → `impl ScrollBounds`
- Rename `pub struct ViewportHandle` → `pub struct ScrollHandle`
- Rename field `pub shared: Rc<RefCell<ViewportSharedState>>` → `pub scroll: Rc<RefCell<ScrollBounds>>`
- Rename `impl ViewportHandle` → `impl ScrollHandle`
- In `info()`: keep immutable borrow `self.scroll.borrow()`
- In mutation methods (`set_content_size`, `scroll_vertical_to`, etc.): use `self.scroll.borrow_mut()`
- Update all internal `self.shared` references to `self.scroll`
- Rename `pub struct ViewportContext` → `pub struct ScrollViewport`
- Update `ComponentContext` field: `viewport: ViewportContext` → `viewport: ScrollViewport`
- Update `ComponentContext` field: `viewport_handle: Option<ViewportHandle>` → `scroll_handle: Option<ScrollHandle>`
- Rename method `pub fn viewport_handle(&self) -> Option<ViewportHandle>` → `pub fn scroll_handle(&self) -> Option<ScrollHandle>`
- Rename method `pub fn viewport(&self) -> ViewportContext` → `pub fn viewport(&self) -> ScrollViewport`
- Update `with_viewport` parameter: `handle: Option<ViewportHandle>` → `handle: Option<ScrollHandle>`
- Update struct literals in `ComponentContext::new()` and `ViewportHandle::info()`
