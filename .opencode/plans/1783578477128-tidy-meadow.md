# Plan: Fix Critical Flaws in `integration_tiling.rs`

## Target File
`tests/integration_tiling.rs` — Module 5 (`drag_snap_pipeline`)

## Background

The code review identified 4 SEV-1 flaws:
1. **State Persistence:** `setup_with_render()` creates new `CoreEngine` and `DrawPlanRenderer` every frame, wiping compositor state between events.
2. **Evasion:** 3 tests are `#[ignore]`d instead of fixed.
3. **Ghost Validation:** Uses `.is_some()` instead of exact `Rect` assertions.
4. **Final Geometry:** Uses `r.width > 0` instead of exact coordinate assertions.

## Changes

### 1. Fix State Persistence

Replace `setup()` (returns `(WindowManager, [WindowKey; 2])`) and `setup_with_render()` (creates fresh `CoreEngine`/`DrawPlanRenderer` each call).

**New `setup()`:**
```rust
fn setup() -> (WindowManager, CoreEngine, DrawPlanRenderer, [WindowKey; 2]) {
    let (wm, keys) = wm_with_two_windows();
    (wm, CoreEngine::new(), DrawPlanRenderer::new(), keys)
}
```

**New `advance_frame()`:**
```rust
fn advance_frame(wm: &mut WindowManager, engine: &mut CoreEngine, renderer: &mut DrawPlanRenderer) {
    let area = RatatuiRect { x: 0, y: 0, width: AREA.width, height: AREA.height };
    let buf = Buffer::empty(area);
    let mut backend = RatatuiBackend::new(buf, area);
    render_app(&mut backend, wm, engine, renderer);
}
```

Move `use` imports from inside `setup_with_render` to module scope.

### 2. Fix Ghost Geometry Assertions

Replace all `drag_snap_rect().is_some()` checks with exact `Rect` extraction:

- `drag_to_right_edge_snaps` — ghost = `{x:40, y:0, w:40, h:24}` (right half)
- `drag_to_corner_quadrant` — ghost = `{x:40, y:12, w:40, h:12}` (bottom-right quadrant)
- `drag_to_top_maximizes` — ghost = `{x:0, y:0, w:80, h:24}` (full area)
- `drag_away_restores_float_geometry` — ghost for right-edge drag = `{x:40, y:0, w:40, h:24}`
- `double_snap_converges` Phase 1 — ghost = `{x:40, y:0, w:40, h:24}` (right half)

### 3. Remove `#[ignore]` + Fix `release_leaves_valid_state` → `drag_to_top_maximizes`

**Root cause:** Drags to `(header.x, 0)`. When `header.x` is near 0, corner snap (TopLeft) fires at priority 1 instead of top-edge maximize.

**Fix:** Drag to center column `(AREA.width / 2, 0)` to trigger pure maximize.

### 4. Remove `#[ignore]` from `drag_to_corner_quadrant`

**Root cause:** Engine state reset between frames. **Fix:** Persistent `CoreEngine`/`DrawPlanRenderer` from #1 resolves this.

### 5. Remove `#[ignore]` from `drag_away_restores_float_geometry`

**Root cause:** Engine state reset. **Fix:** Persistent engine+renderer from #1 resolves this.

### 6. Delete `drag_moves_floating_window`

Entirely remove this test — its edge-snap pathway is fully superseded by `drag_to_right_edge_snaps`, which already verifies the same drag→snap→ghost→release→region sequence.

### 7. Fix Exact Geometry in `double_snap_converges`

**Phase 1:** Already has exact `r1` coordinate assertions (correct).

**Phase 2:** Replace `r2.width > 0` with exact floating-rect extraction matching the `drag_away_restores_float_geometry` pattern:
```rust
let float_panes = wm.floating_panes();
let (_, float_spec) = float_panes.iter()
    .find(|(k, _)| *k == keys[0])
    .expect("window should be floating after drag-away");
if let FloatRectSpec::Absolute(fr) = float_spec {
    assert_eq!(fr.width, AREA.width / 2, "phase 2: floating width must match snapped width");
    assert_eq!(fr.height, AREA.height, "phase 2: floating height must match snapped height");
} else {
    panic!("phase 2: expected absolute float rect");
}
```

**Phase 3:** Replace `r3.width > 0 && r3.height > 0` with exact coordinates:
```rust
assert_eq!(r3.x, AREA.x, "phase 3: left-snapped x");
assert_eq!(r3.y, AREA.y, "phase 3: left-snapped y");
assert_eq!(r3.width, AREA.width / 2, "phase 3: left-snapped width");

```

Update all remaining tests in Module 5 (`drag_snap_pipeline`) to use new 4-tuple `setup()` + `advance_frame()` signature.
