# Plan: Fix Critical Flaws in `integration_tiling.rs`

## Target File
`tests/integration_tiling.rs` ŌĆö Module 5 (`drag_snap_pipeline`)

## Background

The code review identified 4 SEV-1 flaws:
1. **State Persistence:** `setup_with_render()` creates new `CoreEngine` and `DrawPlanRenderer` every frame, wiping compositor state between events.
2. **Evasion:** 3 tests are `#[ignore]`d instead of fixed.
3. **Ghost Validation:** Uses `.is_some()` instead of exact `Rect` assertions.
4. **Final Geometry:** Uses `r.width > 0` instead of exact coordinate assertions.

## Changes

### 1. Fix State Persistence

Replace `setup()` (returns `(WindowManager, [WindowKey; 2])`) and `setup_with_render()` (creates fresh `CoreEngine`/`DrawPlanRenderer` each call).

**New `setup()`:**
```rust
fn setup() -> (WindowManager, CoreEngine, DrawPlanRenderer, [WindowKey; 2]) {
    let (wm, keys) = wm_with_two_windows();
    (wm, CoreEngine::new(), DrawPlanRenderer::new(), keys)
}
```

**New `advance_frame()`:**
```rust
fn advance_frame(wm: &mut WindowManager, engine: &mut CoreEngine, renderer: &mut DrawPlanRenderer) {
    let area = RatatuiRect { x: 0, y: 0, width: AREA.width, height: AREA.height };
    let buf = Buffer::empty(area);
    let mut backend = RatatuiBackend::new(buf, area);
    render_app(&mut backend, wm, engine, renderer);
}
```

Move `use` imports from inside `setup_with_render` to module scope.

### 7. Replace `press_makes_window_floating` with `press_preserves_tiled_state`

The legacy test codifies a premature state decoupling bug ŌĆö a Press event on a tiled header must NOT immediately decouple the window, only focus and capture the pointer.

**New test:**
```rust
#[test]
fn press_preserves_tiled_state() {
    let (mut wm, mut engine, mut renderer, keys) = setup();
    advance_frame(&mut wm, &mut engine, &mut renderer);
    assert!(!wm.is_window_floating(keys[0]), "starts tiled");

    let header = header_rect(&wm, keys[0]);
    let down = make_mouse(MouseEventKind::Press(MouseButton::Left), header.x as u16, header.y as u16);
    wm.dispatch_mouse(&down);
    advance_frame(&mut wm, &mut engine, &mut renderer);

    // Press alone must NOT decouple ŌĆö deadzone drag requirement
    assert!(!wm.is_window_floating(keys[0]), "press must not make window floating");

    // Drag that breaches the kinetic deadzone (dx+dy > 2 cells)
    let drag_x = (header.x + 5) as u16;
    let drag = make_mouse(MouseEventKind::Drag(MouseButton::Left), drag_x, header.y as u16);
    wm.dispatch_mouse(&drag);
    advance_frame(&mut wm, &mut engine, &mut renderer);

    assert!(wm.is_window_floating(keys[0]), "must be floating after drag breaches deadzone");
}
```

### 10. Engine Refactor: Defer Floating Decoupling to Drag Deadzone Breach

The `press_preserves_tiled_state` test will fail against the current `dispatch_mouse` implementation because `HeaderAction::Drag` (line 1332) calls `set_floating_rect` immediately on Press.

**Required engine change** in `crates/term-wm-core/src/window/window_manager/mod.rs`:

1. **Press handler (line 1312-1374):** Remove the `set_floating_rect` call in the `else` branch (line 1332) for tiled windows. Keep `self.mouse_capture = Some(MouseCaptureState::DraggingWindow { ... })` so the pointer is captured.

2. **Drag handler (line 1002):** When the deadzone is breached (`dx + dy > 2`) and the window is NOT yet floating, call `set_floating_rect` with the window's current tiled position (from `visible_region_for_key`) before proceeding to `move_floating`:

```rust
// Before move_floating, ensure window has a floating rect
if !self.is_window_floating(*key) {
    let rect = self.visible_region_for_key(*key);
    let width = rect.width.max(1);
    let height = rect.height.max(1);
    self.set_floating_rect(
        *key,
        Some(FloatRectSpec::Absolute(FloatRect {
            x: rect.x, y: rect.y, width, height,
        })),
    );
    // Update initial position to match
    *initial_x = rect.x;
    *initial_y = rect.y;
}
```

This ensures the cursor offset math in `move_floating` works correctly ŌĆö `initial_x/initial_y` now match the window's tiled position, and the mouse delta is applied relative to `start_x/start_y` (the cursor position at Press time).
