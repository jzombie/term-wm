# Plan: Unified Deferred-Esc Handler

## Problem

Two independent double-Esc systems (normal passthrough via `wm_overlay_opened_at`, K-mode via `k_mode_esc_at`) and the first Esc in K mode is forwarded to the terminal immediately without any panel notification.

## Solution

A unified `esc_pending` state machine that **defers** the first Esc press until either a second Esc arrives (open overlay) or the timeout expires (forward to terminal). The panel shows the countdown immediately from the pending state, regardless of whether the overlay is visible.

## Key Design

```
First Esc (K mode) → esc_pending = Some((Instant, KeyEvent))
                    → event NOT forwarded
                    → panel shows "Esc pending: XXXms · press Esc for menu"

Second Esc within window → esc_pending = None
                         → open WM overlay (both events consumed)

Timeout (checked in idle handler) → esc_pending = None
                                  → forward stored KeyEvent to terminal

Non-matching key → esc_pending = None
                 → forward key to terminal immediately
```

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum EscPressResult {
    /// Second press of WmToggleOverlay within window → open overlay
    DoubleEsc,
    /// First press of WmToggleOverlay — deferred until timeout or second press
    Pending,
    /// Not a WmToggleOverlay key (or outside K mode) — forward immediately
    Forward,
}
```

## Event Flow Trace

1. K mode active, user presses Esc
2. `Some(event)` → K mode check → `handle_esc_press(key, true)` returns `Pending`
3. Handler returns Continue (Esc NOT forwarded)
4. Event loop calls `None` handler → `take_expired_esc_event()` → None (timer hasn't expired) → `begin_frame()` → panel shows countdown
5a. **Second Esc within window**: `handle_esc_press(key, true)` → esc_pending cleared → `DoubleEsc` → `open_wm_overlay()`
5b. **Non-Esc key**: `handle_esc_press(key, false)` → esc_pending cleared → `Forward` → forwarded to terminal
5c. **Timeout**: Next `None` handler → `take_expired_esc_event()` → returns stored Esc → forwarded to terminal
