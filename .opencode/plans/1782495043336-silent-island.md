# Plan: Strict No-Conflict Keybinding Architecture

## Problem

The README and help.md document a no-conflict philosophy where **only Esc** is intercepted by the WM. Other WM actions (Tab for focus switching, scrolling, etc.) should only be available after entering WM Mode via Esc. But the actual implementation intercepts Tab globally for `FocusNext`/`FocusPrev`, violating this contract.

## Solution

Introduce a **layered keybinding architecture** with two scopes:

- **`ActionLayer::Global`** — Only `WmToggleOverlay` (Esc). Always active.
- **`ActionLayer::WmMode`** — Everything else (FocusNext, FocusPrev, scrolling, etc.). Only active when the WM overlay is visible.

The bottom panel dynamically updates to show only the currently active layer's keybindings.

## Files to Modify

### 1. `crates/term-wm-core/src/actions.rs`
- Add `ActionLayer` enum (`Global`, `WmMode`)
- Add `Action::layer(&self) -> ActionLayer` method:
  - `WmToggleOverlay` → `Global`
  - All others → `WmMode`

### 2. `crates/term-wm-core/src/keybindings.rs`
- Add `action_for_key_in_layer(&self, key, layer) -> Option<Action>` — filters action lookup by layer
- Add `bottom_hints_for_layer(&self, layer, max) -> Vec<(Action, Vec<String>)>` — filters bottom hints by layer

### 3. `crates/term-wm-core/src/runner.rs` — **main change**

Restructure event dispatch:

- **`mapped_action`** (line 179): Use `action_for_key_in_layer(Global)` instead of `action_for_key()`
- **Layer 2b**: Only dispatch Global actions (Quit, OpenHelp, etc. won't match here since only WmToggleOverlay is Global)
- **WM overlay section** (lines 311-375):
  - Add FocusNext/FocusPrev handling via `handle_focus_event()` BEFORE the catch-all key consumption (line 371)
  - This lets Tab work for focus switching in WM mode
- **Focus routing** (lines 405-413): 
  - Guard with `!wm_mode` (embedded only — standalone non-overlay mode passes Tab through to terminal)
- **Keep mouse click focus switching unchanged**

Event flow summary:
```
Normal mode (standalone):  Esc → WM overlay | Tab → terminal
WM mode (overlay visible): Tab → focus switch | arrows → menu | other → consumed
Embedded mode:             Tab → focus switch | all other → terminal
```

### 4. `crates/term-wm-core/src/window/window_manager/layout.rs`
- In `register_managed_layout()` (line 178-186):
  - Determine current layer: `Global` if overlay not visible, `WmMode` if visible
  - Use `bottom_hints_for_layer(layer, 6)` instead of `bottom_hints(6)`
  - In embedded mode, always use `WmMode` layer (since overlay never opens but hints should show available actions)

### 5. Documentation
- No changes needed — README and help.md already correctly describe the Esc-only philosophy. The implementation will now match.
