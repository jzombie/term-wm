# Plan: Cursor Hover Overlay via Background Color Inversion

## Problem

Add a visual mouse cursor indicator using `Modifier::REVERSED` (background color inversion) on the cell under the hardware mouse pointer. No dedicated cursor glyph — just invert the cell's style to preserve underlying text.

## Design Decisions (from review)

- **No new state field** on `WindowManager`. Derive interaction mode from existing fields:
  - `hover.is_none()` → Idle
  - `hover.is_some() && mouse_capture.is_none()` → Hovering
  - `mouse_capture.is_some()` → Active (grab/drag/resize)
- **No new Component**. Simple function called last in `render_app()`.
- **No raw screen coords in ComponentContext**. Keep hover logic centralized.
- **Boundary-safe badge rendering**. Clamp to buffer edges.


### 2. Add Cursor Overlay Render Function (`term-wm-console/src/draw_plan_renderer.rs`)

New public function `render_cursor_overlay()`:

```rust
pub fn render_cursor_overlay(
    buf: &mut Buffer,
    wm: &WindowManager,
    theme: &Theme,
) {
    let Some((hx, hy)) = wm.hover_pos() else { return };
    let hx = hx as u16;
    let hy = hy as u16;

    // Clamp to buffer bounds
    if hx >= buf.area.width || hy >= buf.area.height {
        return;
    }

    // Derive interaction state from existing WindowManager fields
    let is_active = wm.mouse_capture_active();

    // Apply REVERSED modifier to cell under cursor
    if let Some(cell) = buf.cell_mut((hx, hy)) {
        cell.set_style(cell.style().add_modifier(Modifier::REVERSED));
    }

    // Active state: render a 1x1 badge adjacent to cursor
    if is_active {
        // Determine badge position: prefer (X+1, Y+1), fall back to (X-1, Y+1) if at right edge
        let badge_col = if hx + 1 < buf.area.width { hx + 1 } else { hx.saturating_sub(1) };
        let badge_row = if hy + 1 < buf.area.height { hy + 1 } else { hy.saturating_sub(1) };
        // Only render badge if within bounds and different from hover cell
        if (badge_col, badge_row) != (hx, hy) {
            if let Some(cell) = buf.cell_mut((badge_col, badge_row)) {
                cell.set_char('⊞'); // or some appropriate symbol
                cell.set_style(Style::default().add_modifier(Modifier::REVERSED));
            }
        }
    }
}
```

Also add a public accessor for `mouse_capture_active` on `WindowManager` if not already present (check — it likely exists as `capture_active()`).

### 3. Invoke in `render_app()` (`src/lib.rs`)

Call `render_cursor_overlay()` as the **absolute last rendering step** — after `render_overlays()` at the end of `render_app()`:

```rust
// Render overlays (command menu, help, exit confirm)
render_overlays(backend, wm);

// Cursor overlay — MUST be last (highest Z)
if let Some(rb) = backend.as_any_mut().downcast_mut::<RatatuiBackend>() {
    render_cursor_overlay(&mut rb.buffer, wm, &wm.config().theme);
}
```

### 4. Export the function

Add `render_cursor_overlay` to the `use` imports at the top of `src/lib.rs`.

### 5. Add tests (`term-wm-console/src/draw_plan_renderer.rs`)

Add `#[cfg(test)]` tests:
- Cursor at (0,0) applies REVERSED
- Cursor outside buffer bounds is no-op
- Active state badge renders adjacent
- Active state badge clamped at right/bottom edges
- Idle (no hover) is no-op
