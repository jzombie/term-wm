# Plan: Cursor Hover Overlay via Background Color Inversion

## Problem

Add a visual mouse cursor indicator using `Modifier::REVERSED` (background color inversion) on the cell under the hardware mouse pointer. No dedicated cursor glyph — invert the cell's style to preserve underlying text.

## Design Decisions (from review)

- **No new state field** on `WindowManager`. Derive interaction mode from existing fields.
- **No new Component** — simple function called last in `render_app()`.
- **No raw screen coords in ComponentContext** — keep hover logic centralized in render step.
- **Boundary-safe**: clamp all cell access to buffer dimensions.
- **Active badge preserves text**: use `Modifier::REVERSED` only, no character overwriting.
- **Read-only accessor**: add `is_mouse_captured(&self) -> bool` to WindowManager (no mutation).

### 2. Add Read-Only Capture Accessor (`term-wm-core/src/window/window_manager/mod.rs`)

Add a non-mutating query method so `term-wm-console` can read capture state without triggering pruning ticks:

```rust
/// Returns true if the window manager is currently tracking an active
/// mouse capture (drag, resize, component interaction, or layout handle).
/// Read-only — no side effects, no deadline pruning.
pub fn is_mouse_captured(&self) -> bool {
    self.mouse_capture.is_some()
}
```

### 3. Add Cursor Overlay Render Function (`term-wm-console/src/draw_plan_renderer.rs`)

New public function:

```rust
use ratatui::style::Modifier;

/// Apply background-color inversion to the cell under the mouse cursor.
/// Must be called as the absolute last render step (highest Z-order).
/// Uses `Modifier::REVERSED` on existing cell content — no text is overwritten.
pub fn render_cursor_overlay(buf: &mut Buffer, wm: &WindowManager, _theme: &Theme) {
    let Some((hx, hy)) = wm.hover_pos() else { return };

    // Clamp to buffer bounds
    if hx >= buf.area.width || hy >= buf.area.height {
        return;
    }

    // Derive interaction state from existing WindowManager fields (read-only)
    let is_active = wm.is_mouse_captured();

    // Apply REVERSED to cell under cursor (preserves text)
    if let Some(cell) = buf.cell_mut((hx, hy)) {
        cell.set_style(cell.style().add_modifier(Modifier::REVERSED));
    }

    // Active state: also invert the adjacent cell as a "badge"
    // Uses REVERSED only — no character overwriting, preserves layout.
    if is_active {
        let badge_col = if hx + 1 < buf.area.width { hx + 1 } else { hx.saturating_sub(1) };
        let badge_row = if hy + 1 < buf.area.height { hy + 1 } else { hy.saturating_sub(1) };
        if (badge_col, badge_row) != (hx, hy) {
            if let Some(cell) = buf.cell_mut((badge_col, badge_row)) {
                cell.set_style(cell.style().add_modifier(Modifier::REVERSED));
            }
        }
    }
}
```

### 4. Invoke in `render_app()` (`src/lib.rs`)

Call as the **absolute last step** — after `render_overlays()`:

```rust
// Render overlays (command menu, help, exit confirm)
render_overlays(backend, wm);

// Cursor overlay — MUST be last (highest Z)
if let Some(rb) = backend.as_any_mut().downcast_mut::<RatatuiBackend>() {
    render_cursor_overlay(&mut rb.buffer, wm, &wm.config().theme);
}
```

### 5. Export the function in imports

Add `render_cursor_overlay` to the existing `use` block in `src/lib.rs`.

### 6. Add tests (`term-wm-console/src/draw_plan_renderer.rs`)

Add `#[cfg(test)]` tests using the in-memory rendering pattern (`Buffer::empty` + mock `WindowManager`):
- Hover at (0,0) applies REVERSED
- Hover outside buffer → no-op
- Active state inverts adjacent cell
- Adjacent cell clamped at right/bottom edges
- No hover → no-op
