# Plan: Simplify term-wm API for Third-Party Apps

## Goal

Make `term-bench`, `examples/dual_image.rs`, and third-party apps easier to use with the window manager by eliminating the dual-trait boilerplate (`WindowManagerHost` + `WindowProvider`) and providing a fluent builder API.

## Current Pain Points

1. Every app must implement **both** `WindowManagerHost` and `WindowProvider` traits (coupled via supertrait)
2. `window_component()` requires manual key→component routing (the "manual dispatcher anti-pattern")
3. Components needing their `WindowKey` must use `Arc<Mutex<Option<WindowKey>>>` hacks
4. Terminal setup/teardown (raw mode, alternate screen, mouse capture) is manual boilerplate
5. No `prelude` module — imports are verbose

## Approach: Incremental Additions, No Breaking Changes

Add new types alongside existing ones. Existing `run_window_app` API continues to work unchanged.

## Step 2: Add `spawn` method to `WindowManager`

**File:** `crates/term-wm-core/src/window/window_manager/mod.rs`

Add method after `create_window` (line 218):

```rust
/// Register a component and invoke its `on_mount` hook with the assigned key.
/// Prefer this over `create_window` for components that need their WindowKey.
pub fn spawn<C>(&mut self, mut component: C) -> WindowKey
where
    C: Component<TermWmAction> + 'static,
{
    let app_ctx = self.app_ctx().clone(); // shallow Arc clone, not deep structural clone
    let key = self.create_window(Box::new(component));
    if let Some(comp) = self.component_for_key_mut(key) {
        comp.on_mount(key, &app_ctx);
    }
    key
}
```

**Also add `has_window`** for O(1) liveness checks without dynamic dispatch:

```rust
/// Returns true if the key references a live window in the SlotMap.
/// O(1) — no component extraction or vtable resolution.
pub fn has_window(&self, key: WindowKey) -> bool {
    self.windows.contains_key(key)
}
```

Single-call registration that automatically resolves the key. Eliminates `Arc<Mutex<Option<WindowKey>>>` in most cases.

**Also add `spawn_boxed`** for pre-boxed components:

```rust
pub fn spawn_boxed(&mut self, component: Box<dyn Component<TermWmAction>>) -> WindowKey {
    let app_ctx = self.app_ctx().clone(); // shallow Arc clone
    let key = self.create_window(component);
    if let Some(comp) = self.component_for_key_mut(key) {
        comp.on_mount(key, &app_ctx);
    }
    key
}
```

## Step 3: Create `TermWmApp` builder

**New file:** `src/term_wm_app.rs`

```rust
pub struct TermWmApp {
    wm: WindowManager,
    windows: Vec<WindowKey>,
    should_quit: bool,
    empty_message: String,
}

impl TermWmApp {
    /// Create a new app with all system chrome (panels, menu).
    pub fn new(app_ctx: AppContext) -> Self { ... }

    /// Create an embedded app without command menu, for use inside
    /// an existing Ratatui application.
    pub fn embedded(app_ctx: AppContext) -> Self { ... }

    /// Create from an already-constructed WindowManager.
    pub fn from_wm(wm: WindowManager) -> Self { ... }

    /// Set the message shown when no windows are registered.
    pub fn empty_message(mut self, msg: impl Into<String>) -> Self { ... }

    /// Register a component as a window. Returns the WindowKey.
    /// Calls `on_mount` on the component after registration.
    pub fn register<C: Component<TermWmAction> + 'static>(&mut self, component: C) -> WindowKey { ... }

    /// Register a pre-boxed component (for dynamic dispatch).
    /// Calls `on_mount` on the component after registration, matching `register`.
    pub fn register_boxed(&mut self, component: Box<dyn Component<TermWmAction>>) -> WindowKey {
        let key = self.wm.spawn_boxed(component);
        self.windows.push(key);
        key
    }

    /// Borrow the WindowManager for configuration or direct access.
    pub fn wm(&mut self) -> &mut WindowManager { ... }

    /// Get a mutable reference to a registered component by key.
    pub fn component_mut<T: 'static>(&mut self, key: WindowKey) -> Option<&mut T> { ... }

    /// Request the app to quit after the current event cycle.
    pub fn request_quit(&mut self) { ... }

    /// Run with default console I/O (enters/exits terminal automatically).
    pub fn run(self) -> io::Result<()> { ... }

    /// Run with custom render target and event source.
    pub fn run_with<O: RenderTarget, D: EventSource>(self, output: &mut O, driver: &mut D) -> io::Result<()> { ... }
}

impl WindowManagerHost for TermWmApp {
    fn windows(&mut self) -> &mut WindowManager {
        &mut self.wm
    }

    fn quit_requested(&self) -> bool {
        self.should_quit
    }
}

impl WindowProvider for TermWmApp {
    fn enumerate_windows(&mut self) -> Vec<WindowKey> {
        // Prune dead keys — WindowManager is the authoritative owner.
        // Uses O(1) has_window check, no dynamic dispatch.
        self.windows.retain(|&key| self.wm.has_window(key));
        self.windows.clone()
    }

    fn empty_window_message(&self) -> &str {
        &self.empty_message
    }

    fn window_component(&mut self, key: WindowKey) -> Option<&mut dyn Component<TermWmAction>> {
        self.wm.component_for_key_mut(key)
    }
}
```

**Register in:** `src/lib.rs` — add `pub mod term_wm_app;`

## Step 5: Create `WidgetAdapter` and `StatefulWidgetAdapter`

**New file:** `crates/term-wm-ui-components/src/widget_adapter.rs`

```rust
/// Wraps a closure-based renderer as a term-wm Component.
/// For stateless widgets (Paragraph, Block, Gauge, etc.)
pub struct WidgetAdapter<F> {
    render_fn: F,
}

impl<F> Component<TermWmAction> for WidgetAdapter<F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &ComponentContext),
{
    fn render(&self, frame: &mut UiFrame<'_>, area: Rect, ctx: &ComponentContext, _reg: &mut HitboxRegistry) {
        (self.render_fn)(frame, area, ctx);
    }
}

/// Wraps a Ratatui StatefulWidget + its State as a term-wm Component.
/// Uses RefCell for interior mutability so render(&self) can pass &mut State
/// to the render closure without unsafe.
pub struct StatefulWidgetAdapter<S, F> {
    state: std::cell::RefCell<S>,
    render_fn: F,
}

impl<S, F> StatefulWidgetAdapter<S, F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &mut S, &ComponentContext),
{
    pub fn new(initial_state: S, render_fn: F) -> Self {
        Self {
            state: std::cell::RefCell::new(initial_state),
            render_fn,
        }
    }

    /// Mutate the widget state within a strictly bounded scope.
    /// The RefMut guard is dropped when the closure returns, preventing
    /// leaked borrows across render ticks.
    pub fn modify_state<R>(&self, f: impl FnOnce(&mut S) -> R) -> R {
        let mut state = self.state.borrow_mut();
        f(&mut *state)
    }

    /// Inspect the widget state within a strictly bounded scope.
    pub fn inspect_state<R>(&self, f: impl FnOnce(&S) -> R) -> R {
        let state = self.state.borrow();
        f(&*state)
    }
}

impl<S, F> Component<TermWmAction> for StatefulWidgetAdapter<S, F>
where
    F: Fn(&mut UiFrame<'_>, Rect, &mut S, &ComponentContext),
{
    fn render(&self, frame: &mut UiFrame<'_>, area: Rect, ctx: &ComponentContext, _reg: &mut HitboxRegistry) {
        let mut state = self.state.borrow_mut();
        (self.render_fn)(frame, area, &mut *state, ctx);
    }
}
```

**Note:** `StatefulWidgetAdapter` uses `RefCell<S>` for interior mutability. `Component::render` takes `&self`, but `StatefulWidget::render_widget` needs `&mut State`. `RefCell` provides safe runtime borrow checking — `borrow_mut()` panics only if already borrowed (which cannot happen during single-threaded render). State accessors use bounded closures (`modify_state`, `inspect_state`) instead of leaking `Ref`/`RefMut` guards, preventing accidental borrow holding across render ticks.

**File:** `crates/term-wm-ui-components/src/lib.rs` — add `pub mod widget_adapter; pub use widget_adapter::*;`

```rust
use std::io;
use term_wm::prelude::*;
use term_wm::SvgImageComponent;

fn main() -> io::Result<()> {
    let mut paths: Vec<String> = std::env::args().skip(1).collect();
    if paths.is_empty() {
        paths.push("assets/zenOSmosis-logo.svg".to_string());
    }
    if paths.len() == 1 {
        paths.push(paths[0].clone());
    }

    let mut app = TermWmApp::new(AppContext::new("example", "0.0.0"));

    let mut left = SvgImageComponent::new();
    left.set_keep_aspect(true);
    left.set_colorize(true);
    let left_key = app.register(left);

    let mut right = SvgImageComponent::new();
    right.set_keep_aspect(true);
    right.set_colorize(true);
    let right_key = app.register(right);

    if let Some(img) = app.component_mut::<SvgImageComponent>(left_key) {
        img.load_from_path(&paths[0])?;
    }
    if let Some(img) = app.component_mut::<SvgImageComponent>(right_key) {
        img.load_from_path(&paths[1])?;
    }

    app.run()
}
```

## Step 7: Migrate existing components to use `on_mount`

**File:** `crates/term-wm-ui-components/src/terminal.rs` (MODIFY)

The PTY background thread needs the `WindowKey` to route events, but the key is only available after registration. Defer thread spawning from `new()` to `on_mount` so the key is guaranteed to exist:

```rust
pub struct TerminalComponent {
    // ... existing fields (pty, parser, etc.) ...
    // Store PTY spawn config, not the thread itself
    spawn_config: Option<PtySpawnConfig>,
    window_key: Option<WindowKey>,
}

impl TerminalComponent {
    pub fn new(...) -> Self {
        // Build PTY, parser, etc. but do NOT spawn the background thread yet.
        // Store the config needed to spawn it later.
        Self {
            spawn_config: Some(PtySpawnConfig { /* pty, command, etc. */ }),
            window_key: None,
            // ...
        }
    }
}

impl Component<TermWmAction> for TerminalComponent {
    fn on_mount(&mut self, key: WindowKey, _app: &AppContext) {
        self.window_key = Some(key);
        // Spawn the PTY background thread NOW, with the key available by value.
        if let Some(config) = self.spawn_config.take() {
            self.spawn_pty_thread(key, config);
        }
    }

    fn init(&mut self) {
        // No thread spawning here — deferred to on_mount.
    }
}
```

This structurally guarantees the thread cannot execute without its routing identity — no race condition possible.

**File:** `crates/term-wm/src/main.rs` (MODIFY)
