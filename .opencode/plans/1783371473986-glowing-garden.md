# Plan: I/O Burst Budget via Condvar

## Problem

The reader thread loops on `read()` without throttling. For infinite streams, it pegs a CPU core. Need an I/O burst budget: allow multiple chunks to coalesce, only wait when the budget is exhausted and the UI hasn't rendered yet.

## Fix

**File:** `crates/term-wm-pty-engine/src/pty.rs`

Add to `Pty`:
```rust
dirty_cond: Arc<(Mutex<bool>, Condvar)>,
```

Reader thread — track bytes, wait only when budget exceeded:
```rust
let mut bytes_since_render = 0usize;
const IO_BURST_BUDGET: usize = 256 * 1024; // 256 KB

loop {
    // ... read + process bytes ...
    bytes_since_render += n;

    // Edge-triggered wakeup
    if !dirty.swap(true, Ordering::AcqRel) {
        cb(PtyStatus::Wakeup);
    }

    // Budgeted backpressure: only wait when burst budget exceeded
    if bytes_since_render >= IO_BURST_BUDGET {
        let (lock, cvar) = &*dirty_cond;
        let mut is_dirty = lock.lock().unwrap();
        *is_dirty = true;
        while *is_dirty {
            is_dirty = cvar.wait(is_dirty).unwrap();
        }
        bytes_since_render = 0;
    }
}
```

**File:** `crates/term-wm-pty-engine/src/pane.rs`

Add to `Pane` trait:
```rust
fn notify_rendered(&self) {}
```

Implement on `Pty`:
```rust
fn notify_rendered(&self) {
    let (lock, cvar) = &*self.dirty_cond;
    let mut is_dirty = lock.lock().unwrap();
    *is_dirty = false;
    cvar.notify_one();
}
```

**File:** `crates/term-wm-ui-components/src/terminal.rs`

In `render_screen`, after `pane.take_dirty()`:
```rust
pane.notify_rendered();
```
