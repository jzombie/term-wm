# Plan: I/O Burst Budget via Condvar

## Problem

Reader thread loops on `read()` without throttling. For infinite streams, pegs a CPU core. Need I/O burst budget: allow multiple chunks to coalesce, only wait when budget exceeded and UI hasn't rendered.

## Fix

**File:** `crates/term-wm-pty-engine/src/pty.rs`

Add to `Pty`:
```rust
dirty_cond: Arc<(Mutex<()>, Condvar)>,
```

The `AtomicBool` `dirty` is the single source of truth. The `Mutex<()>` exists solely for Condvar synchronization.

Reader thread:
```rust
let mut bytes_since_render = 0usize;
const IO_BURST_BUDGET: usize = 256 * 1024;

loop {
    // ... read + process bytes ...
    bytes_since_render += n;

    // Edge-triggered wakeup
    if !dirty.swap(true, Ordering::AcqRel) {
        cb(PtyStatus::Wakeup);
    }

    // Budgeted backpressure
    if bytes_since_render >= IO_BURST_BUDGET {
        let (lock, cvar) = &*dirty_cond;
        let mut guard = lock.lock().unwrap();
        while dirty.load(Ordering::Acquire) {
            guard = cvar.wait(guard).unwrap();
        }
        bytes_since_render = 0;
    }
}
```

**File:** `crates/term-wm-pty-engine/src/pane.rs`

Add to `Pane` trait:
```rust
fn clear_dirty_and_notify(&self) {}
```

Implement on `Pty`:
```rust
fn clear_dirty_and_notify(&self) {
    let (lock, cvar) = &*self.dirty_cond;
    let _guard = lock.lock().unwrap();
    self.dirty.store(false, Ordering::Release);
    cvar.notify_one();
}
```

**File:** `crates/term-wm-ui-components/src/terminal.rs`

In `render_screen`, replace `pane.take_dirty()` with:
```rust
pane.clear_dirty_and_notify();
```
