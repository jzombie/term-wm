# Plan: I/O Burst Budget via Condvar

## Problem

The reader thread loops on `read()` without throttling. For infinite streams like `cat /dev/random`, it pegs a CPU core at 100%. The main thread only holds the Mutex for ~2ms per frame, leaving the reader 14ms to ingest megabytes unbounded.

## Fix

Add a `Condvar` so the reader waits when dirty is still true (UI hasn't rendered yet). The main thread notifies the Condvar after clearing dirty in `take_dirty()`.

**File:** `crates/term-wm-pty-engine/src/pty.rs`

Add to `Pty`:
```rust
dirty_cond: Arc<(Mutex<bool>, Condvar)>,
```

In reader thread, after setting dirty:
```rust
if !dirty.swap(true, Ordering::AcqRel) {
    // Wake the main thread
    cb(PtyStatus::Wakeup);
}
// If the UI hasn't consumed the previous frame yet, wait
{
    let (lock, cvar) = &*dirty_cond;
    let mut is_dirty = lock.lock().unwrap();
    while *is_dirty {
        is_dirty = cvar.wait(is_dirty).unwrap();
    }
}
```

**File:** `crates/term-wm-pty-engine/src/pane.rs`

Add to `Pane` trait:
```rust
fn notify_rendered(&self) {}
```

Implement on `Pty`:
```rust
fn notify_rendered(&self) {
    let (lock, cvar) = &*self.dirty_cond;
    let mut is_dirty = lock.lock().unwrap();
    *is_dirty = false;
    cvar.notify_one();
}
```

**File:** `crates/term-wm-ui-components/src/terminal.rs`

In `render_screen`, after `pane.take_dirty()`:
```rust
pane.notify_rendered();
```
