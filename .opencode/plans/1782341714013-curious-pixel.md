# Plan: Refined Configurable Keybindings and Action Dispatch Integration

## Goal
Implement a robust, configurable keybinding system with context-aware hints and a full keybinding reference overlay, prioritizing system integrity, collision avoidance, and configuration reliability.

## Design

### 1. Robust Event Propagation (Priority Chain)
To avoid hijacking keystrokes intended for nested applications, we will implement a strictly ordered propagation chain:
- **Layer 1 (Overlays)**: Active modal overlays (help, keybindings, exit confirm) receive events first. If they are visible, they consume events; propagation halts.
- **Layer 2 (WM Global Bindings)**: The centralized dispatcher matches against `KeyBindings`. If a match is found, it consumes the event and mutates WM state.
- **Layer 3 (Pass-through)**: If no match is found, the system assumes the event is for the active pseudo-terminal child process. It is converted to its raw byte sequence and forwarded down the PTY pipeline.

### 2. Configurable KeyBindings in WmConfig
- Add `keybindings: KeyBindings` field to `WmConfig` (defaults to `KeyBindings::default()`).
- **Hint Visibility Configuration**: Add `hint_visibility: HintVisibility` to `WmConfig`:
    ```rust
    pub enum HintVisibility { Always, OnDemand, Never }
    ```
- Implement a `Configuration Validator` that runs on application startup:
    - Detects keybinding collisions and ensures mandatory actions exist.
    - Fallback to "safe mode" hardcoded bindings and log a warning if core mappings are missing.

### 3. State-Driven Context-Aware Hint System
- Implement `Action::bottom_hint_priority(&self) -> Option<u8>`.
- **HintProvider Trait**: Components expose their own keybinding hints dynamically via `HintProvider`.
- **Trigger Logic**:
    - **Mode-Driven**: Hints activate automatically when context changes (e.g., entering "Command Mode").
    - **Config-Driven**: `HintVisibility` (`Always`, `OnDemand`, `Never`) determines if hints are shown.
    - **Embedded/Constraint-Driven**: 
        - **Automatic Suppression**: Hint rendering is automatically suppressed if available vertical space is insufficient (e.g., < 5 rows), regardless of `HintVisibility` setting.
        - **Override**: Users may force `Always` visibility if absolute space permits, but the system prioritizes screen integrity over hint visibility in constrained embedded environments.
        - **Fallback**: In embedded modes where hints are suppressed, the `KeybindingOverlayComponent` remains the primary authoritative discoverability tool, ensuring functionality is never obscured.

### 4. Keybinding Reference Overlay
- Define `Category` enum for logical grouping.
- `KeybindingOverlayComponent` actively iterates over the live `&KeyBindings` reference at render time to ensure 100% fidelity.
- Overlay intercepts navigation/dismissal keys (`Esc`, `q`) before passing events to the WM dispatcher.
