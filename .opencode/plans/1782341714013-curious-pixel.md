# Plan: Refined Configurable Keybindings and Action Dispatch Integration

## Goal
Implement a robust, configurable keybinding system with bottom-panel hints and a full keybinding reference overlay, addressing the current lack of a general action dispatch mechanism.

## Design

### 1. Unified Action Dispatch Mechanism
Instead of scattered, manual event handling (like `WmToggleOverlay` at line 188), we will introduce a centralized action dispatch loop in `runner.rs`.
- **Infrastructure**: Implement a `dispatch_action(app: &mut A, action: Action)` function in `runner.rs` or `WindowManager`.
- **Integration**: The event loop in `runner.rs` will call `app.windows().keybindings().action_for_key(key)` and, if a match is found, call `dispatch_action`.
- **Special Cases**: Keep existing special-case logic for things that bypass global keybindings (like `SelectionPreview`, `ExitConfirm`) but route all general WM and application actions through the new central dispatcher.

### 2. Configurable Keybindings in WmConfig
- Add `keybindings: KeyBindings` field to `WmConfig` (defaults to `KeyBindings::default()`).
- Update `WindowManager` to store `KeyBindings`.
- Resolve the double-borrow issue in `runner.rs:159` by cloning the `KeyBindings` map at the start of the event loop or using a scoped immutable reference after the mutable borrow of the `app` struct is released (e.g., by separating the event-matching phase from the state-mutation phase).

### 3. Bottom Panel Hints
- Add `keybinding_hints` field to `Panel`.
- Add `fn bottom_hint_priority(&self) -> Option<u8>` to `Action` (with existing `Display` impl for labeling).
- `KeyBindings::bottom_hints(max: usize)` to compute hints.
- Update `Panel::render_bottom()` to render hints on the left.

### 4. Keybinding Reference Overlay
- Define `Category` enum for grouping actions in the overlay.
- Implement `KeybindingOverlayComponent` that iterates over `&KeyBindings` to display all actions, grouped by `Category`.
- Add `OpenKeybindings` variant to `Action`.
- Register the new overlay in the WM event propagation chain, ensuring it intercepts navigation/close events (like `Esc`) before they pass to the `WindowManager` or child processes.
