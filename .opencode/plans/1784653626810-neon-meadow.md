# Migration: vt100 → alacritty_terminal

## Why

Replace `vt100` (which lacks text reflow on resize, causing cascading prompt) with `alacritty_terminal` (which has native text reflow via `Term::resize()`). The `vt100` crate is removed entirely.

---

## Dependencies

**Add to workspace `Cargo.toml`:**
```
alacritty_terminal = "0.26"
```

**Add to crates depending on it:**
- `term-wm-pty-engine`: `alacritty_terminal = { workspace = true }`
- `term-wm-ui-components`: already has path to `term-wm-pty-engine` — no new direct dep
- `term-session-client`: `alacritty_terminal = { workspace = true }`
- `term-session-server`: remove unused `vt100`

**Remove from all crates:** `vt100 = { workspace = true }`

## Phase 2a: Define PtyListener (pty.rs or new file)

**File: `crates/term-wm-pty-engine/src/pty.rs`** (or `crates/term-wm-pty-engine/src/listener.rs`)

Create a custom `EventListener` that captures title changes. The existing `take_pending_title()` mechanism and `foreground_title` polling stay unchanged.

```rust
use alacritty_terminal::event::{Event, EventListener};

pub(crate) struct PtyListener {
    pending_title: Arc<Mutex<Option<String>>>,
}

impl EventListener for PtyListener {
    fn send_event(&self, event: Event) {
        match event {
            Event::Title(title) => {
                if let Ok(mut guard) = self.pending_title.lock() {
                    *guard = Some(title);
                }
            }
            // Clipboard, color requests, etc. can be added later
            _ => {}
        }
    }
}
```

The `PtyListener` is stored inside `Term<PtyListener>` (as `Term`'s event_proxy field). When the processor parses an OSC title sequence, `Term::set_title()` calls `self.event_proxy.send_event(Event::Title(...))`, writing into the same `Arc<Mutex<Option<String>>>` that `Pty::take_pending_title()` reads.

**Thread safety**: The processor runs in the reader thread; `take_pending_title()` is called from the main UI thread. `Arc<Mutex<>>` handles this correctly.

## Phase 3: Rewrite Pty internals (pty.rs)

**File: `crates/term-wm-pty-engine/src/pty.rs`**

### Struct fields
```
- shared_parser: Arc<Mutex<vt100::Parser>>   → REMOVE
- dirty: Arc<AtomicBool>                     → probably KEEP (for backpressure)
- dirty_cond: Arc<(Mutex<()>, Condvar)>      → probably KEEP (for backpressure)

+ term: Arc<Mutex<alacritty_terminal::term::Term<VoidListener>>>
```

### `spawn_with_scrollback()` constructor
Replace:
```rust
let initial_parser = vt100::Parser::new(size.rows, size.cols, scrollback_len);
let shared_parser = Arc::new(Mutex::new(initial_parser));
```
With:
```rust
use alacritty_terminal::term::{Term, Config};
use alacritty_terminal::term::test::TermSize;  // or own Dimensions impl
// Actually: implement Dimensions ourselves since TermSize is #[cfg(test)]

struct PtyTermSize { columns: usize, screen_lines: usize }
impl Dimensions for PtyTermSize { ... }

let config = Config { scrolling_history: scrollback_len, ..Default::default() };
let pty_size = PtyTermSize { columns: size.cols as usize, screen_lines: size.rows as usize };
let term = Term::new(config, &pty_size, VoidListener);
let term = Arc::new(Mutex::new(term));
```

### `parser_read_loop()` function
Replace `vt100::Parser` with `vte::ansi::Processor + Term`.

Current pattern:
```rust
let shared_parser: Arc<Mutex<vt100::Parser>> = ...;
// reader loop:
loop {
    // check pending_resize, create new parser with history replay
    // read bytes: shared_parser.lock().unwrap().process(&buf[..n]);
}
```

New pattern:
```rust
let term: Arc<Mutex<Term<VoidListener>>> = ...;
let mut processor = vte::ansi::Processor::new();
// reader loop:
loop {
    // check pending_resize → term.lock().unwrap().resize(new_size)
    // read bytes: processor.advance(&mut *term.lock().unwrap(), &buf[..n]);
}
```

### History replay on resize
**Remove entirely!** `Term::resize()` handles text reflow natively. No need to create a new parser and replay history.

### Replace all methods that use shared_parser

| Current (vt100) | Replacement (alacritty) |
|---|---|
| `parser = shared_parser.lock().unwrap(); parser.screen().cursor_position()` | `term.lock().unwrap().cursor().point` |
| `parser.screen().scrollback()` | `term.lock().unwrap().grid().display_offset()` |
| `parser.screen_mut().set_scrollback(n)` | `term.lock().unwrap().scroll_display(Scroll::Delta(delta))` |
| `parser.screen().alternate_screen()` | `term.lock().unwrap().mode().contains(TermMode::ALT_SCREEN)` |
| `parser.screen().state_formatted()` | Full-frame snapshot (no vt100 state needed) |
| `parser.screen().contents()` | Collect from grid cells |

### `screen()` / `sync_screen()` method
Keep for DSR handling and backpressure. The DSR response can read cursor position from `term.cursor()`. The dirty flag mechanism stays.

### `screen_lines()` method
Read from `term.grid()` cells directly.

### Colors
Replace `parser.screen().fgcolor()` / `bgcolor()` with reading from `term.colors()`:
```rust
let colors = term.lock().unwrap().colors();
// colors[NamedColor::Foreground as usize] for default fg
```

## Phase 4: Rewrite TerminalComponent (terminal.rs)

**File: `crates/term-wm-ui-components/src/terminal.rs`**

### Remove or update imports
```
- use vt100::MouseProtocolEncoding;    → REMOVE
+ use term_wm_pty_engine::pane::{TerminalSnapshot, TerminalCell, RgbColor, CursorInfo, MouseProtocol, ...};
```

### `render_screen()` method
Replace the parser-based rendering with snapshot-based rendering:

```rust
// BEFORE:
let parser_arc = pane.shared_parser();
let parser = parser_arc.lock().unwrap();
let screen = parser.screen();
for row in start_row.. {
    for col in start_col.. {
        let cell = screen.cell(row, col);
        // read cell properties
    }
}
let (row, col) = screen.cursor_position();
```

```rust
// AFTER:
let snap = pane.snapshot(area.width, area.height);
for row in start_row.. {
    for col in start_col.. {
        let cell = &snap.cells[row as usize][col as usize];
        // read cell.character, cell.fg, cell.bg, cell.bold, etc.
    }
}
if let Some(cursor) = &snap.cursor {
    // render cursor at (cursor.row, cursor.column)
}
snap.default_fg, snap.default_bg, snap.mouse, snap.alternate_screen...
```

### `handle_events()` method
Replace:
```rust
let screen = pane.shared_parser().lock().unwrap().screen();
let encoding = screen.mouse_protocol_encoding();
let mode = screen.mouse_protocol_mode();
```
With: store mouse protocol info in a snapshot or add a dedicated Pane method.

**Simplest:** call `pane.snapshot(0, 0)` just to get the mouse info, or add a dedicated `fn mouse_protocol(&mut self) -> MouseProtocol` to Pane trait.

### Color conversion functions
Replace `vt100::Color` → ratatui mapping with `RgbColor` → ratatui mapping:

```rust
// Remove: fn vt_color_to_ratatui(color: vt100::Color)
// Add:    fn rgb_to_ratatui(color: &RgbColor) -> ratatui::style::Color
fn rgb_to_ratatui(c: &RgbColor) -> TColor {
    TColor::Rgb(c.r, c.g, c.b)
}
```

Default colors: `snap.default_fg` / `snap.default_bg` instead of `screen.fgcolor()` / `screen.bgcolor()`.

### Cell property mapping

| vt100 Cell method | alacritty Cell field | TerminalSnapshot field |
|---|---|---|
| `cell.contents()` | `cell.c` | `cell.character` |
| `cell.fgcolor()` / `cell.bgcolor()` | `cell.fg` / `cell.bg` → convert Color to RgbColor | `cell.fg` / `cell.bg` |
| `cell.bold()` | `cell.flags.contains(Flags::BOLD)` | `cell.bold` |
| `cell.dim()` | `cell.flags.contains(Flags::DIM)` | `cell.dim` |
| `cell.italic()` | `cell.flags.contains(Flags::ITALIC)` | `cell.italic` |
| `cell.underline()` | `cell.flags.intersects(Flags::ALL_UNDERLINES)` | `cell.underline` |
| `cell.inverse()` | `cell.flags.contains(Flags::INVERSE)` | `cell.inverse` |
| `cell.is_wide_continuation()` | `cell.flags.contains(Flags::WIDE_CHAR_SPACER)` | `cell.wide_continuation` |

### `ScrollbackGuard` struct
Replace `&'a mut vt100::Screen` with `&'a mut TerminalSnapshot` or use the Pane's scrollback methods directly.

### `selection_text_for_range()` method
Replace `screen.contents_between()` with reading text from the grid via scrollback access. Use `Pane::scrollback()` / `Pane::set_scrollback()` for navigation and read individual cells from snapshots.
