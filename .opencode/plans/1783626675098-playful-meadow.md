# Fix: PowerSaver Profile Never Activates

## Bug Description

The profile indicator stays at "Streaming" (yellow) and never drops to PowerSaver. Three interacting root causes prevent the power calibration loop from working.

### Root Cause A: Dirty-windows never consumed (`runner.rs` + `unified_event_source.rs`)

`dirty_windows` in `UnifiedEventSource` are never consumed in production code — `take_dirty_windows()` is defined but only called in tests. This means the set grows without bound and `current_profile()` always sees `has_dirty_windows == true`, preventing PowerSaver.

### Root Cause B: Logging feedback loop / Heisenbug (`power_profile.rs`)

Even after fixing Root Cause A, the debug log creates a circular dependency:

1. Profile transitions to PowerSaver
2. `report_change()` logs `tracing::info!("profile: PowerSaver")`
3. Log subscriber writes to debug log component, dirtying its `WindowKey`
4. Next `poll()` finds the dirty log window → profile back to Streaming
5. Step 1 fires again → infinite 60fps loop

The subscriber ceiling is `Level::DEBUG` (`tracing_sub.rs:94`), so `info!` always passes through. Downgrading to `trace!` breaks the cycle.

### Root Cause C: `pending_work` hijacked by scheduler heap occupancy (`runner.rs` + `task_scheduler.rs`)

Even after fixing Root Causes A and B, the profile stays at Streaming because:

1. `driver.set_pending_work(system_handle.has_pending())` is called every cycle (`runner.rs:558`)
2. `has_pending()` returns `true` if **any** task exists in the scheduler heap (`task_scheduler.rs:143`)
3. `current_profile()` factors `pending_work` into the `has_dirty_windows` check (`unified_event_source.rs:476`)
4. So ANY scheduled task (e.g. `SuperPassthrough` timeout, `DragSnap` timer) forces `Streaming` permanently

## Fix

### Critical insight: must bridge scheduler deadline into EventSource

The `EventLoop` is generic over `D: EventSource` and has NO knowledge of the `TaskScheduler`. If we just remove `has_pending()` from `set_pending_work()`, PowerSaver's 3600s poll interval will block all scheduled tasks. We need a **new trait method** on `EventSource` that the runner uses to push the next scheduler deadline into the driver, and the driver clamps its `poll_interval()` against it.

### Files to Modify

1. **`crates/term-wm-core/src/io/event_source.rs`** — Root Cause C + A
   - Add `set_max_sleep_duration(&mut self, duration: Option<Duration>)` to the `EventSource` trait with a default no-op impl.
   - **CRITICAL (SEV-1):** Forward in blanket `impl<T: EventSource + ?Sized> EventSource for &mut T`:
     ```rust
     fn set_max_sleep_duration(&mut self, duration: Option<std::time::Duration>) {
         (**self).set_max_sleep_duration(duration)
     }
     ```
   - Add `take_dirty_windows()` to the `EventSource` trait with default empty return (Root Cause A).
   - **Must use fully qualified paths** (`std::collections::HashSet`, `crate::window::WindowKey`).
   - Forward in blanket impl:
     ```rust
     fn take_dirty_windows(&mut self) -> std::collections::HashSet<crate::window::WindowKey> {
         (**self).take_dirty_windows()
     }
     ```

2. **`src/unified_event_source.rs`** — Root Cause C + A
   - **New field:**
     ```rust
     max_sleep_duration: Option<Duration>,
     ```
     Initialize to `None` in `UnifiedEventSource::new()`.
   - **Implement trait override:**
     ```rust
     fn set_max_sleep_duration(&mut self, duration: Option<Duration>) {
         self.max_sleep_duration = duration;
     }
     ```
   - **Clamp `poll_interval()`:**
     ```rust
     fn poll_interval(&self) -> Duration {
         let base = self.current_profile().poll_interval();
         match self.max_sleep_duration {
             Some(max_sleep) => base.min(max_sleep),
             None => base,
         }
     }
     ```
   - Move the inherent `take_dirty_windows()` into the `impl EventSource for UnifiedEventSource` block (Root Cause A).
   - Remove `self.dirty_windows.clear()` from the fallthrough path (Root Cause A).

3. **`crates/term-wm-core/src/runner.rs`** — Root Cause C + A
   - **Add this to `flush_state_changes`, BEFORE the profile check:**
     ```rust
     // Clamp driver sleep to the next scheduler deadline so PowerSaver
     // doesn't block for 3600s when a task is pending.
     driver.set_max_sleep_duration(system_handle.time_until_next());
     ```
   - **Change line 558 from:**
     ```rust
     driver.set_pending_work(system_handle.has_pending());
     ```
     to:
     ```rust
     driver.set_pending_work(system_handle.is_keep_awake_active());
     ```
   - **Swap order in `flush_state_changes`** — consume dirty windows BEFORE profile check (Root Cause A).
   - **Only consume dirty in idle path** — use `!did_panic` flag (Root Cause A).

4. **`crates/term-wm-core/src/task_scheduler.rs`** — Root Cause C
   - Add `is_keep_awake_active()` method:
     ```rust
     pub fn is_keep_awake_active(&self) -> bool {
         self.inner.borrow().keep_awake
     }
     ```

5. **`crates/term-wm-core/src/power_profile.rs`** — Root Cause B
   - Change `tracing::info!` to `tracing::trace!` in `report_change()` (line 131).

### Design Rationale

- **`set_max_sleep_duration`** bridges the architectural gap between the generic `EventLoop` (no scheduler knowledge) and the `TaskScheduler` (deadline-aware). The runner, which owns both, passes `time_until_next()` into the driver. The driver clamps `poll_interval()` so the event loop never blocks past the next task deadline.
- **`is_keep_awake_active()`** replaces `has_pending()` to signal only transient UI state (overlays, animations) that requires tight frame pacing. Background tasks don't need 60fps polling.
- Together, these let PowerSaver's 3600s interval apply only when truly idle (no overlays, no pending tasks). When a task is scheduled, the 3600s drops to exactly the task deadline, and the event loop wakes naturally without busy-waiting.

```rust
#[test]
fn poll_interval_clamped_by_max_sleep_duration() {
    let (_tx, rx) = bounded(EVENT_CHANNEL_CAPACITY);
    let mut source = UnifiedEventSource { rx, tx: _tx, _input_handle: std::thread::spawn(|| {}),
        shutdown: Arc::new(AtomicBool::new(false)), dirty_windows: HashSet::new(),
        exited_windows: Vec::new(), pending_event: None, input_buffer: VecDeque::new(),
        signal_received: false, normalizer: KeyboardNormalizer::new(), last_event_at: None,
        frame_pacer: FramePacer::new(), pending_work: false, max_sleep_duration: None };
    let dummy = std::thread::spawn(|| {}); source._input_handle = dummy;

    // PowerSaver default interval is 3600s
    assert_eq!(source.poll_interval(), Duration::from_secs(3600));

    // Clamp to 100ms
    source.set_max_sleep_duration(Some(Duration::from_millis(100)));
    assert_eq!(source.poll_interval(), Duration::from_millis(100));

    // Remove clamp — back to 3600s
    source.set_max_sleep_duration(None);
    assert_eq!(source.poll_interval(), Duration::from_secs(3600));
}
```

**6-8. `runner.rs`** — new `power_calibration_tests` module:
```rust
#[cfg(test)]
mod power_calibration_tests {
    use super::*;
    use std::collections::HashSet;
    use std::time::Duration;
    use crate::actions::SystemTask;
    use crate::task_scheduler::TaskScheduler;
    use crate::window::WindowKey;

    struct SpyEventSource {
        captured_max_sleep: Option<Option<Duration>>,
        mock_dirty_windows: HashSet<WindowKey>,
    }

    impl EventSource for SpyEventSource {
        fn poll(&mut self, _: Duration) -> io::Result<bool> { Ok(false) }
        fn read(&mut self) -> io::Result<Event> { unreachable!() }
        fn next_key(&mut self) -> io::Result<crate::events::KeyEvent> { unreachable!() }
        fn next_mouse(&mut self) -> io::Result<crate::events::MouseEvent> { unreachable!() }
        fn set_max_sleep_duration(&mut self, d: Option<Duration>) { self.captured_max_sleep = Some(d); }
        fn take_dirty_windows(&mut self) -> HashSet<WindowKey> { std::mem::take(&mut self.mock_dirty_windows) }
    }

    #[test]
    fn scheduler_deadline_propagates_to_driver() {
        let mut driver = SpyEventSource { captured_max_sleep: None, mock_dirty_windows: HashSet::new() };
        let sched = TaskScheduler::<SystemTask>::new();
        let handle = sched.handle();
        handle.schedule_once(Duration::from_millis(250), SystemTask::DragSnap);

        let time_left = handle.time_until_next();
        driver.set_max_sleep_duration(time_left);

        let captured = driver.captured_max_sleep.unwrap().unwrap();
        assert!(captured <= Duration::from_millis(250));
        assert!(captured > Duration::from_millis(200));
    }

    #[test]
    fn dirty_keys_are_drained_cleanly_on_frame_completion() {
        let mut key_set = HashSet::new();
        key_set.insert(WindowKey::default());
        let mut driver = SpyEventSource { captured_max_sleep: None, mock_dirty_windows: key_set };

        let first = EventSource::take_dirty_windows(&mut driver);
        assert_eq!(first.len(), 1);

        let second = EventSource::take_dirty_windows(&mut driver);
        assert!(second.is_empty(), "dirty keys must drain between cycles");
    }
}
```
